# xAI, Grok 4.5 공개

원문: <https://x.ai/news/grok-4-5>

HN 토론: <https://news.ycombinator.com/item?id=48835111>

## 요약

xAI가 코딩, 에이전트 작업, 지식 노동을 겨냥한 플래그십 모델
Grok 4.5를 공개했다.
수만 개의 NVIDIA GB300 GPU로 학습했으며, 원시 토큰량보다
데이터 품질을 우선하는 방향을 택했다고 밝혔다.

주요 벤치마크 성적은 Terminal Bench 2.1에서 83.3%,
SWE Bench Pro에서 64.7%, DeepSWE 1.0에서 62.0%다.
초당 80토큰의 속도를 낸다.
SWE 과제당 평균 출력 토큰 수는 약 15,954개로, Claude Opus의
67,020개 대비 약 4.2배 더 효율적이라고 주장한다.
가격은 입력 100만 토큰당 2달러, 출력 100만 토큰당 6달러다.

학습 방식은 중복 제거, 품질 점수화, 도메인 중심 데이터 선별을
통해 "규모보다 신호 품질"을 강조한다.
강화학습은 수십만 개의 과제를 대상으로 진행됐으며, 다단계
소프트웨어 엔지니어링 작업에 초점을 맞췄다.
Grok Build, Cursor(전 요금제), xAI 콘솔을 통해 이용할 수 있고,
EU 지역 출시는 7월 중순으로 예정돼 있다.
Grok Build와 Cursor에서는 제한적인 무료 이용을 제공한다.

## 분석

가장 눈에 띄는 수치는 벤치마크 점수 자체보다 토큰 효율이다.
같은 SWE 과제를 Claude Opus 대비 4분의 1이 채 안 되는
출력 토큰으로 처리한다는 주장은, 절대 성능보다 실사용
비용 구조에서 차별화를 노리는 전략으로 읽힌다.
"규모보다 데이터 품질"이라는 프레이밍은 스케일링 법칙의
한계가 논의되는 시점에서, 데이터 큐레이션이 다음 경쟁
축이 되고 있음을 보여준다.

HN에서 Tiberium은 이 효율성을 다른 모델들과 나란히 놓고
비교했다.
Opus 대비 4배 나은 추론 효율을 내면서도 가격은 100만
토큰당 입력 2달러·출력 6달러라, GPT-5.6(5달러·30달러)이나
Opus 4.8(5달러·25달러)과 비교하면 확실히 경제적이라는
평가다.[^Tiberium]
wxw는 xAI가 발표 자료에 토큰 효율성 섹션을 별도로 넣은
점을 긍정적으로 평가하며, 앞으로 다른 모델 발표에서도
이런 지표가 더 눈에 띄게 다뤄지길 바란다고 남겼다.[^wxw]

## 비평

xAI가 공개한 효율성 수치(4.2배)는 자체 측정 기준이며,
Claude Opus와의 비교가 동일한 과제 집합과 동일한 채점
기준으로 이뤄졌는지는 발표 자료만으로 확인하기 어렵다.
토큰 효율과 실제 작업 완료 품질은 별개의 지표이므로,
더 적은 토큰을 쓴다는 사실이 곧 더 나은 결과물을 뜻하지는
않는다.
공식 발표 페이지(x.ai)는 접근 제한으로 원문 전체를 확인할
수 없었으며, 이 요약은 GeekNews에 정리된 내용에 주로
의존했다.

pveierland는 벤치마크에서 1위를 주장하지 않는 점은
신선하다면서도, 246쪽 분량의 시스템 카드를 함께 낸 Opus
4.8과 비교하면 문서화 수준이 아직 미흡하다고 지적했다.[^pveierland]
DCKing도 비슷한 맥락에서, 실제로 뭔가를 말해 주는
벤치마크 몇 개를 골라 제시한 점은 평가할 만하지만, 이는
동시에 다른 다수의(왜곡되기 쉬운) 벤치마크에서는 굳이
1위를 주장하지 않는 선택적 노출일 수 있다고 짚었다.[^DCKing]
실제 사용 경험에서도 평가가 엇갈렸다.
vessenes는 openclaw에서 Grok 4를 써 봤을 때 언어적으로는
유창했지만 에이전틱 툴 호출에서는 어려움을 겪었다고
전했다.[^vessenes_grok]
반면 rayiner는 법률 업무에 시험 삼아 적용해 Opus급
품질에 더 빠른 속도를 냈다고 평가했고,[^rayiner] mholt는
직접 테스트한 세 모델 중 특정 iOS 앱을 만드는 과제에서
Grok이 가장 나은 결과를 냈다고 밝혔다.[^mholt]

경쟁 구도에 대한 논의도 오갔다.
minraws는 OpenAI와 Anthropic이 한동안 주춤한 사이 GLM,
Kimi, Grok, DeepSeek 등 후발 주자들이 빠르게 따라붙었다며
"모트(moat)가 어디 있느냐"는 질문을 던졌다.[^minraws]
vb-8448은 이번이 처음으로 선두 모델이 신규 경쟁 모델에
곧바로 추월당하지 않은 사례처럼 보인다며, Anthropic이
해자를 찾은 것인지 업계 전체가 한계에 부딪힌 것인지
물었다.[^vb-8448]

## 인사이트

Grok 4.5는 벤치마크 우위보다 비용·효율 우위를 전면에 내세운
출시라는 점에서, 모델 성능이 상향 평준화되는 국면에서 경쟁이
어디로 이동하고 있는지를 보여준다.
에이전트가 장시간 반복적으로 모델을 호출하는 워크플로가
늘어날수록, 절대 성능보다 "같은 결과를 얼마나 싸게 내는가"가
채택 결정에 더 큰 영향을 미칠 가능성이 크다.
다만 HN 반응이 보여주듯, 가격·벤치마크 우위와 실제 에이전틱
작업에서의 신뢰성은 서로 다른 축이라 둘 다 따로 검증해야
한다는 점은 여전히 유효하다.
