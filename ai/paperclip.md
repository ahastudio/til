# Paperclip

<https://paperclip.ing/>

<https://github.com/paperclipai/paperclip>

오픈소스 AI 에이전트 오케스트레이션 플랫폼. 슬로건은 **"zero-human
company를 위한 오케스트레이션"**이다. 에이전트 한 명이 아니라, 에이전트로
구성된 조직 전체를 운영하는 것이 목표다.

"OpenClaw이 직원이라면, Paperclip은 회사다."

## 핵심 개념

단일 에이전트를 잘 다루는 프레임워크가 아니다. 에이전트 팀에 조직도,
예산, 목표, 거버넌스를 부여해 실제 회사처럼 운영하는 인프라다. 에이전트
런타임에 무관하다 — 하트비트 신호를 보낼 수 있으면 무엇이든 "채용"된다.
Claude Code 세션, OpenClaw 봇, Python 스크립트, 쉘 커맨드, HTTP 웹훅 모두
연결 가능하다.

채팅 창 기반의 일회성 대화가 아니라, 에이전트가 리부트를 넘어서 세션을
유지하며 작업을 계속 수행하는 **지속 운영(persistent operation)** 모델이다.

## 기술 스택

- **백엔드**: Node.js 20+, TypeScript (코드베이스의 97.5%)
- **프런트엔드**: React
- **데이터베이스**: PostgreSQL (로컬은 임베디드, 프로덕션은 외부 연결)
- **배포**: 자체 호스팅(self-hosted), Paperclip 계정 불필요
- **라이선스**: MIT

## 주요 기능

### 오케스트레이션

- 조직도 기반 에이전트 계층 구조 관리
- 스케줄 하트비트 및 이벤트 기반 실행
- 태스크 체크아웃으로 중복 작업 방지(원자적 연산)
- 세션 재시작 후에도 에이전트 상태 유지

### 목표 정렬(Goal Alignment)

태스크에 목표 계보(goal ancestry) 전체를 포함시켜, 에이전트가 제목만
보는 게 아니라 "왜 이 일을 하는지"까지 일관되게 파악하게 한다.

### 비용 관리

- 에이전트별 월 예산 한도 설정
- 80% 도달 시 경고, 100% 도달 시 에이전트 자동 일시정지
- 에이전트·태스크·프로젝트·목표 단위 비용 추적

### 거버넌스

- 승인 게이트(approval gate) 강제
- 설정 변경 버전 관리 및 롤백
- 불변 감사 로그(immutable audit log)

### 멀티 컴퍼니 격리

단일 배포로 여러 회사를 운영할 수 있다. 모든 엔티티가 company-scoped로
분리되어 각각 독립적인 데이터와 감사 추적을 갖는다.

## 시작하기

```bash
npx paperclipai onboard --yes
```

또는:

```bash
git clone https://github.com/paperclipai/paperclip.git
cd paperclip && pnpm install && pnpm dev
```

`http://localhost:3100`에서 바로 실행된다.

## Paperclip이 아닌 것

공식 문서가 명시적으로 선을 긋는다: **챗봇, 에이전트 프레임워크, 워크플로우
빌더, 프롬프트 매니저, 단일 에이전트 도구, 코드 리뷰 시스템**이 아니다.

## 분석과 인사이트

### "도구 사용"과 "조직 운영"의 경계

ChatGPT에 프롬프트 한 줄 날리는 것은 도구를 쓰는 일이다. 공유 목표,
예산 한도, 감사 추적을 갖춘 에이전트 15개가 협업하는 것은 조직을
운영하는 일이다. Paperclip은 이 경계를 정의하고 인프라로 구현한다.

### Zero-Human Company의 전제 조건

에이전트 하나가 잘 작동한다고 회사가 돌아가지 않는다. 회사에는 조직도,
예산 통제, 책임 소재, 감사 추적이 필요하다. Paperclip의 베팅은 이 관리
인프라가 LLM 성능만큼이나 중요하다는 것이다. 성능은 빠르게 따라잡히지만,
신뢰할 수 있는 운영 인프라는 다른 문제다.

### 런타임 무관성의 전략적 함의

특정 에이전트 런타임에 종속되지 않는 것은 단순한 기술적 유연성이 아니다.
Claude Code, OpenClaw, 커스텀 스크립트를 동등하게 다루면 에이전트 생태계
전쟁에서 중립 지대를 유지할 수 있다. 특정 LLM 벤더가 에이전트 런타임
시장을 독식해도 Paperclip의 오케스트레이션 레이어는 살아남는다.

### 오픈소스 전략

MIT 라이선스로 완전 공개한 것은 경쟁 전략이기도 하다. 인프라 레이어는
오픈소스로 신뢰를 쌓고, Clipmart(원클릭 company 템플릿 마켓플레이스)로
수익화를 노린다. HashiCorp나 Elastic의 초기 경로와 유사하다.

### 2026년 맥락

2026년 초, 복수의 창업자가 직원 없이 AI 에이전트만으로 회사를 운영하는
실험을 진행 중이다. Paperclip은 이 흐름의 인프라 계층을 노린다. 4,400개
이상의 GitHub 스타는 이 방향에 대한 시장의 초기 신호다.

### 위험 요소

- 에이전트 자율 운영의 감사 가능성과 책임 소재는 여전히 미성숙한 영역이다
- 예산 한도와 승인 게이트가 있어도, 에이전트 판단 오류의 연쇄 효과는
  예측하기 어렵다
- "Zero-human"은 마케팅 언어다. 실제로는 설계·감독·개입하는 사람이 반드시
  필요하다
