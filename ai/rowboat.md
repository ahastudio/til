# Rowboat - 업무 맥락을 잊지 않는 AI 코워커

<https://www.rowboatlabs.com/>

<https://github.com/rowboatlabs/rowboat>

## 문제: AI는 매번 기억을 잃는다

ChatGPT에 "지난주 회의에서 나온 결론이 뭐였지?"
라고 물으면 대답 못 한다.
내가 직접 회의록을 복붙해야 한다.

Notion AI에 "김 대리가 담당하는 프로젝트 현황"을
물으면 Notion 안의 문서만 뒤진다.
이메일, 캘린더, 회의록은 못 본다.

**업무 맥락은 흩어져 있고,
기존 AI 도구는 단일 세션에서 단일 소스만 본다.**

Rowboat는 이 문제를 정면으로 공략한다.
이메일, 회의록, 캘린더를 연결해
Knowledge Graph를 만들고,
세션이 끝나도 맥락이 사라지지 않는다.

Y Combinator S24 배치 출신.

## 동작 원리: 업무가 곧 기억이 된다

Rowboat는 별도로 "기억해줘"라고 할 필요가 없다.
**평소 하던 업무 자체가 Knowledge Graph를 만든다.**

### 1단계: 수집 — 커넥터가 업무를 빨아들인다

Gmail, Google Calendar, Granola, Fireflies 등
커넥터가 이메일과 회의록을 자동으로 가져온다.

사용자가 따로 정리하거나 복붙할 필요 없다.
일하면 데이터가 쌓인다.

### 2단계: 추출 — AI가 맥락을 구조화한다

수집된 원본에서 AI가 핵심 엔티티를 뽑아낸다:

- **사람**: 누가 관여했는가
- **프로젝트**: 어떤 프로젝트에 대한 것인가
- **의사결정**: 무엇이 결정되었는가
- **약속**: 누가 뭘 하기로 했는가

이걸 날것 그대로 덤프하는 게 아니라
**관계를 식별하고 연결한다.**
"김 대리 → 프로젝트 X → 결정 사항 Y"
같은 링크가 자동으로 생긴다.

### 3단계: 저장 — 마크다운 + 백링크

추출된 지식은 Obsidian 호환 Vault에
**플레인 마크다운 파일**로 저장된다.

```
vault/
├── people/
│   ├── 김대리.md        ← [[프로젝트 X]] 백링크
│   └── 박팀장.md
├── projects/
│   └── 프로젝트-X.md    ← [[김대리]], [[결정사항]]
├── decisions/
│   └── 2025-01-15-기술스택-변경.md
└── meetings/
    └── 2025-01-15-클라이언트-미팅.md
```

파일 간 `[[백링크]]`로 연결된다.
Obsidian에서 열면 그래프 뷰로 시각화된다.

핵심은 **독점 포맷이 아니라 텍스트 파일**이라는 것.
`cat`으로 읽고, `grep`으로 검색하고,
`git`으로 버전 관리할 수 있다.

### 4단계: 활용 — 맥락을 가진 AI가 일한다

사용자가 질문하거나 작업을 요청하면
AI는 Vault에서 관련 맥락을 찾아서 응답한다.

"클라이언트 A 미팅 준비해줘"라고 하면
Knowledge Graph에서 해당 클라이언트와 연결된
이메일, 회의록, 의사결정, 미결 사항을 모아
브리핑을 생성한다.

**매번 맥락을 처음부터 설명할 필요가 없다.
그래프에 이미 축적되어 있기 때문이다.**

### 5단계: 자동화 — Background Agent

Background Agent는 이 파이프라인 위에서
**사람 개입 없이 반복 작업을 실행**한다.

실행 조건, 주기, 결과를 Vault에 다시 쓰는 방식까지
사용자가 제어한다.

```
트리거: 매일 오전 8시
→ Google Calendar에서 오늘 일정 확인
→ 각 미팅 관련 Knowledge Graph 탐색
→ 브리핑 마크다운 생성
→ vault/daily/2025-01-16-브리핑.md 에 저장
```

사람은 아침에 브리핑 파일만 열어보면 된다.

## 왜 이 구조가 중요한가

이게 왜 중요한가:

- **서비스 종료 = 데이터 소멸** 공식이 깨진다.
  Rowboat가 망해도 마크다운 파일은 남는다.
- **AI가 뭘 기억하는지 직접 열어본다.**
  모델 파라미터 안에 숨겨진 기억이 아니라
  편집 가능한 텍스트 파일이다.
  AI가 엉뚱한 걸 기억하면 직접 고친다.
- **Git으로 버전 관리 가능.**
  지식 베이스의 변경 이력을 추적할 수 있다.
- **모델을 바꿔도 지식은 그대로.**
  Ollama, LM Studio 같은 로컬 모델이든
  호스팅 API든 갈아끼울 수 있다.
  지식이 모델에 종속되지 않는다.

## 극단적 사용 시나리오

### 1. 회의 10분 전, 아무것도 모른다

내일 클라이언트 미팅인데
지난 3개월간 이 클라이언트와 주고받은
이메일, 회의록, 슬랙 내용을 기억 못 한다.

기존 방식: Gmail 검색, Notion 검색,
슬랙 검색을 각각 돌리고 정리하는 데 30분.

Rowboat: Knowledge Graph에 이미 연결되어 있다.
"클라이언트 A와의 미해결 이슈 정리해줘"
→ 과거 의사결정, 미결 사항, 담당자까지
하나의 브리핑으로 나온다.

**동작 원리**: 커넥터가 이미 이메일과 회의록을
수집해 `[[클라이언트 A]]` 노드에 연결해 뒀다.
AI는 이 노드에서 백링크를 따라가며
관련 정보를 모은다.

### 2. 퇴사자가 생겼다

핵심 인력이 퇴사하면 그 사람 머릿속 맥락이
통째로 증발한다.

기존 방식: 인수인계 문서를 쓰라고 하지만
절반도 안 남기고 떠난다.

Rowboat: 그 사람이 참여한 회의록, 이메일,
의사결정이 Knowledge Graph에 축적되어 있다.
퇴사자의 맥락이 조직에 남는다.

**동작 원리**: `people/김대리.md`를 열면
이 사람이 관여한 모든 프로젝트, 의사결정,
약속이 백링크로 연결되어 있다.
별도 인수인계 없이도 맥락이 추적 가능하다.

### 3. "그때 왜 그렇게 결정했지?"

6개월 전 기술 스택을 바꾼 이유를 아무도 모른다.
의사결정 배경이 사라졌다.

기존 방식: 당시 참석자에게 물어보지만
기억이 다 다르다.

Rowboat: 해당 시점의 회의록, 이메일 스레드,
관련 문서가 그래프로 연결되어 있다.
결정의 맥락을 시간순으로 추적 가능.

**동작 원리**: `decisions/` 디렉터리에
결정 시점, 참석자, 근거 문서가 마크다운으로 남아 있다.
`git log`로 언제 어떤 맥락이 추가됐는지까지 본다.

### 4. 반복 업무를 에이전트에 넘긴다

Background Agent가 매일 아침:

- 오늘 일정 기반 브리핑 생성
- 어제 받은 이메일 중 답장 필요한 것 초안 작성
- 진행 중인 프로젝트의 상태 업데이트 정리

사람은 검토만 하고 보낸다.

**동작 원리**: Background Agent는
트리거 조건에 따라 자동 실행되고,
Knowledge Graph를 읽어 작업한 뒤
결과를 다시 Vault에 마크다운으로 쓴다.
무엇이 실행됐고 뭘 썼는지 파일로 확인 가능.

## 공격적 질문: 왜 Notion AI가 아닌가

| 기준             | Notion AI        | Rowboat              |
|------------------|------------------|----------------------|
| 데이터 소유권    | Notion 서버      | 로컬 마크다운        |
| 소스 범위        | Notion 문서만    | 이메일+회의록+캘린더 |
| 서비스 종료 시   | 데이터 이전 필요 | 파일 그대로 존재     |
| 맥락 지속성      | 세션 단위        | 누적                 |
| 오프라인 접근    | 제한적           | 완전 지원            |
| AI 기억 수정     | 불가             | 마크다운 직접 편집   |
| 기억의 형태      | 블랙박스         | 텍스트 파일          |
| 버전 관리        | 없음             | Git 가능             |

Notion AI는 Notion 안에서만 작동하는
검색 엔진에 가깝다.
Rowboat는 여러 소스를 관통하는
**맥락 축적 엔진**이다.
