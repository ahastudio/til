<https://www.rowboatlabs.com/>

## 문제

ChatGPT에 "지난주 회의 결론이 뭐였지?" → 대답 못 한다. 직접 복붙해야 한다.

Notion AI에 "김 대리 프로젝트 현황" → Notion 문서만 뒤진다. 이메일, 캘린더,
회의록은 못 본다.

**업무 맥락은 흩어져 있고, 기존 AI는 단일 세션에서 단일 소스만 본다.**

## 동작 원리

별도로 "기억해줘"라고 할 필요 없다. **일하면 Knowledge Graph가 쌓인다.**

### 수집 → 추출 → 저장 → 활용 → 자동화

**수집**: Gmail, Calendar, Granola, Fireflies 등 커넥터가 이메일과 회의록을
자동으로 가져온다.

**추출**: AI가 사람, 프로젝트, 의사결정, 약속을 뽑아내고 관계를 연결한다. "김
대리 → 프로젝트 X → 결정 사항 Y".

**저장**: Obsidian 호환 Vault에 플레인 마크다운 + `[[백링크]]`로 저장.

```
vault/
├── people/
│   ├── 김대리.md        ← [[프로젝트 X]]
│   └── 박팀장.md
├── projects/
│   └── 프로젝트-X.md    ← [[김대리]], [[결정사항]]
├── decisions/
│   └── 2025-01-15-기술스택-변경.md
└── meetings/
    └── 2025-01-15-클라이언트-미팅.md
```

독점 포맷이 아니라 텍스트 파일이다. `cat`으로 읽고, `grep`으로 검색하고,
`git`으로 버전 관리한다.

**활용**: "클라이언트 A 미팅 준비해줘" → 그래프에서 관련 이메일, 회의록, 미결
사항을 모아 브리핑을 생성한다. 매번 맥락을 처음부터 설명할 필요 없다.

**자동화**: Background Agent가 트리거 조건에 따라 반복 작업을 실행하고 결과를
Vault에 다시 쓴다.

```
트리거: 매일 오전 8시
→ 오늘 일정 확인
→ 관련 Knowledge Graph 탐색
→ vault/daily/2025-01-16-브리핑.md 생성
```

## 왜 이 구조인가

- **서비스 종료해도 데이터는 남는다.** 마크다운 파일이니까.
- **AI 기억을 직접 열어보고 고친다.** 블랙박스가 아니라 텍스트 파일이니까.
- **모델을 갈아끼워도 지식은 그대로.** Ollama든 호스팅 API든 Knowledge Graph는
  영향 없다.

## 시나리오

### 회의 10분 전, 아무것도 모른다

기존: Gmail, Notion, 슬랙 각각 검색. 30분.

Rowboat: `[[클라이언트 A]]` 노드에 이메일과 회의록이 이미 연결되어 있다.
백링크를 따라가며 브리핑 자동 생성.

### 핵심 인력이 퇴사했다

기존: 인수인계 문서 쓰라고 하지만 절반도 안 남기고 떠난다.

Rowboat: `people/김대리.md`에 이 사람이 관여한 프로젝트, 의사결정, 약속이
백링크로 연결되어 있다. 별도 인수인계 없이 맥락 추적 가능.

### "그때 왜 그렇게 결정했지?"

기존: 6개월 전 결정. 참석자에게 물어보지만 기억이 다 다르다.

Rowboat: `decisions/` 디렉터리에 결정 시점, 참석자, 근거 문서가 남아 있다.
`git log`로 맥락 변경 이력까지 추적.

### 반복 업무를 에이전트에 넘긴다

Background Agent가 매일 아침: 일정 브리핑, 이메일 초안, 프로젝트 업데이트.
사람은 검토만 하고 보낸다. 뭘 실행했고 뭘 썼는지 파일로 확인 가능.

## vs Notion AI

| 기준           | Notion AI        | Rowboat              |
| -------------- | ---------------- | -------------------- |
| 데이터 소유권  | Notion 서버      | 로컬 마크다운        |
| 소스 범위      | Notion 문서만    | 이메일+회의록+캘린더 |
| 서비스 종료 시 | 데이터 이전 필요 | 파일 그대로 존재     |
| 맥락 지속성    | 세션 단위        | 누적                 |
| AI 기억 수정   | 불가             | 마크다운 직접 편집   |

Notion AI는 Notion 안에서만 작동하는 검색 엔진에 가깝다. Rowboat는 여러 소스를
관통하는 **맥락 축적 엔진**이다.
