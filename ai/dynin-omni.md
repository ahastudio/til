# Dynin-Omni: 마스크드 디퓨전 기반 옴니모달 파운데이션 모델

원문: <https://dynin.ai/omni/>

## 요약

서울대학교 AIDAS 연구실이 Dynin-Omni를 공개했다.
텍스트, 이미지, 비디오, 음성을 하나의 아키텍처에서
이해(understanding)와 생성(generation) 모두 처리하는
마스크드 디퓨전(masked diffusion) 기반 옴니모달 파운데이션 모델이다.

핵심 혁신은 자기회귀(autoregression) 대신
마스크드 토큰 디노이징(masked token denoising)을 사용한 것이다.
토큰을 순차적으로 생성하는 대신
마스크된 토큰을 한꺼번에 복원하므로
이미지, 비디오 같은 비순차적 데이터를 더 자연스럽게 처리한다.

아키텍처의 주요 특징은 네 가지다.
모든 모달리티를 공유 이산 토큰 공간에 매핑하는 통합 토큰 스페이스,
모든 모달리티를 균일하게 처리하는 단일 트랜스포머 백본,
모달리티별 디코딩 스케줄,
신뢰도 기반 반복 개선(confidence-based refinement)이다.

6가지 기능을 지원한다.
텍스트 추론, 이미지 이해, 비디오 이해,
이미지 생성, 이미지 편집, 음성 처리(ASR과 TTS)다.
성능 면에서는 텍스트 생성이 유사 옴니모달 모델 대비 4-5배 빠르고,
이미지 생성은 전문 모델 대비 절반의 스텝으로 유사 품질을 달성한다.

논문은 arXiv(2604.00007)에,
모델과 데모는 Hugging Face에 공개되어 있다.

## 분석

### 자기회귀 vs 마스크드 디퓨전의 구조적 차이

GPT, Claude, Gemini 등 주류 LLM은 자기회귀 방식으로
토큰을 하나씩 순차 생성한다.
이 방식은 텍스트에는 자연스럽지만
이미지나 비디오처럼 공간적 구조가 있는 데이터에는 비효율적이다.
Dynin-Omni의 마스크드 디퓨전은
"순서가 없는 데이터는 순서 없이 생성한다"는 원칙으로
이 비효율을 구조적으로 해결한다.

### 옴니모달의 의미: 이해와 생성의 통합

기존 멀티모달 모델 대부분은 "이해"(입력으로 받기)에 집중하고
"생성"(출력으로 만들기)은 별도 모델에 위임한다.
Dynin-Omni는 이해와 생성을 하나의 모델에서 처리한다.
이미지를 보고 설명할 수 있으면서 동시에 이미지를 생성할 수 있고,
음성을 인식하면서 음성을 합성할 수도 있다.
이 양방향성이 "옴니모달"의 핵심이다.

### 국내 연구실의 파운데이션 모델 공개

서울대 AIDAS 연구실이라는 출처가 주목할 만하다.
파운데이션 모델 연구가 빅테크와 대규모 스타트업에 집중된 상황에서
대학 연구실이 경쟁력 있는 옴니모달 모델을 공개한 것은
연구 접근성이 높아졌다는 신호다.
Hugging Face, arXiv, GitHub을 통한 공개 전략도
학술 커뮤니티의 표준적 배포 경로를 따르고 있다.

## 비평

### 벤치마크의 구체성 부족

"텍스트 생성 4-5배 빠름", "이미지 생성 절반 스텝"이라는 수치가
어떤 모델과 어떤 조건에서 비교한 것인지 명확하지 않다.
"유사 옴니모달 모델"과 "전문 모델"이 구체적으로 무엇인지,
동일 하드웨어에서의 비교인지 알 수 없다.
arXiv 논문에서 상세 벤치마크를 확인해야 하지만,
발표 페이지만으로는 성능 주장을 검증하기 어렵다.

### 모델 크기와 학습 데이터의 미공개

모델의 파라미터 수, 학습 데이터 규모와 구성,
학습에 소요된 컴퓨트 자원에 대한 정보가 부족하다.
옴니모달 모델의 실용성을 판단하려면
추론에 필요한 최소 하드웨어 사양을 알아야 하는데,
이 정보가 발표 페이지에 명시되지 않았다.

### 각 모달리티의 개별 성능 검증

6가지 기능을 나열했지만
각 모달리티에서 전문 모델과의 정밀 비교가 부족하다.
특히 음성 처리(ASR/TTS)의 품질이
Whisper 같은 전문 모델과 어느 수준에서 비교되는지,
비디오 이해가 실용적 수준인지 알기 어렵다.

## 인사이트

### 마스크드 디퓨전은 자기회귀의 근본적 한계를 정면 공략한다

자기회귀 모델의 "왼쪽에서 오른쪽으로" 생성 방식은
텍스트라는 특정 모달리티에 최적화된 귀납적 편향이다.
이미지의 픽셀은 왼쪽 위에서 오른쪽 아래로 "읽히는" 것이 아니고,
비디오 프레임은 선형 순서로만 이해되지 않는다.
마스크드 디퓨전은 이 편향을 제거하여
모달리티에 구애받지 않는 생성 방식을 제안한다.

이 접근이 의미 있는 이유는 확장성에 있다.
새로운 모달리티(3D, 촉각, 화학 구조 등)를 추가할 때
자기회귀 방식은 "이것을 어떤 순서로 토큰화할 것인가"라는
설계 질문이 매번 발생한다.
마스크드 디퓨전에서는 이산 토큰 공간에만 매핑하면 되므로
모달리티 확장이 구조적으로 더 용이하다.

다만 텍스트 생성에서는 자기회귀가 여전히
인간의 "생각하면서 쓰기" 패턴과 자연스럽게 일치하므로,
두 방식의 하이브리드가 최적 해일 가능성이 높다.

### 옴니모달 통합은 "AI 에이전트의 감각 기관"을 만드는 일이다

별개의 전문 모델을 파이프라인으로 연결하는 것과
하나의 모델이 여러 모달리티를 네이티브로 처리하는 것은
질적으로 다른 경험을 만든다.
전자에서는 모달리티 간 전환 시 컨텍스트가 손실되지만,
후자에서는 "이 이미지에서 이 부분을 수정하고,
결과를 음성으로 설명해줘"가 하나의 추론 과정이 된다.

이것이 AI 에이전트에 중요한 이유는
에이전트가 세계와 상호작용하려면
여러 감각을 동시에 처리해야 하기 때문이다.
사람이 화면을 보면서 음성을 듣고 텍스트를 읽는 것처럼,
에이전트도 멀티모달 입력을 통합적으로 이해해야
복잡한 태스크를 수행할 수 있다.
Dynin-Omni의 이해+생성 양방향 통합은
이 방향의 한 걸음이다.

### 대학 연구실의 파운데이션 모델은 "특화 영역"에서 승부해야 한다

서울대 AIDAS 연구실이 범용 파운데이션 모델에서
Google, OpenAI, Meta와 정면 경쟁하기는 현실적으로 어렵다.
컴퓨트 자원, 데이터 규모, 엔지니어링 인력에서 격차가 크기 때문이다.

그러나 "마스크드 디퓨전 기반 옴니모달"이라는 특정 아키텍처에서
선도적 연구를 공개함으로써 다른 종류의 가치를 만들 수 있다.
학술 커뮤니티에서의 인용과 영향력,
이 아키텍처를 기반으로 한 후속 연구의 허브 역할,
그리고 이 접근이 유효하다면
빅테크가 채택하거나 인수하는 시나리오도 가능하다.

INSAIT이 Gemma 기반으로 불가리아어 모델을 만든 것처럼,
대학 연구실의 경쟁력은 범용이 아니라
아키텍처 혁신이나 특정 도메인 특화에서 나온다.
Dynin-Omni가 마스크드 디퓨전이라는 차별화된 축을 선택한 것은
이 관점에서 올바른 포지셔닝이다.
