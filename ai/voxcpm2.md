# VoxCPM2: 토크나이저 없는 다국어 음성 합성 시스템

원문: <https://github.com/OpenBMB/VoxCPM>

## 요약

VoxCPM2는 OpenBMB에서 공개한 오픈소스 다국어 텍스트-투-스피치(TTS) 시스템이다.
기존 TTS 시스템이 음성을 이산 토큰으로 변환하는 과정을 거치는 것과 달리,
VoxCPM2는 "토크나이저 프리(tokenizer-free)" 아키텍처를 채택하여 확산 기반
자기회귀(diffusion autoregressive) 방식으로 연속적인 음성을 직접 생성한다.

모델 규모는 2B 파라미터이며, 200만 시간 이상의 다국어 데이터로 학습되었다.
한국어를 포함한 30개 언어와 9개 중국어 방언을 지원한다.
48kHz 스튜디오 품질의 오디오를 출력하며, 벤치마크 성능은 영어 기준
Seed-TTS-eval에서 1.84% WER, 중국어 만다린 기준 0.97% CER을 기록했다.

음성 생성 기능은 세 가지 모드를 제공한다.
텍스트 설명으로 새로운 음성을 디자인하는 Voice Design,
목표 화자의 특성을 제어하며 복제하는 Controllable Cloning,
그리고 극도로 정밀한 복제를 수행하는 Ultimate Cloning이다.
5~10분의 오디오만으로 화자 적응(speaker adaptation) 파인튜닝이 가능하다.

실용적 접근성도 주목할 만하다.
Apache-2.0 라이선스로 공개되어 상업적 사용이 자유롭고,
GGML/GGUF 포맷을 통한 CPU 추론을 지원하며,
8GB VRAM GPU에서도 동작한다.
고성능 GPU에서는 RTF(Real-Time Factor) 0.13까지 달성한다.

## 분석

### 토크나이저 프리 아키텍처의 기술적 의미

기존 TTS 시스템 대부분은 음성을 이산 토큰으로 변환한 뒤 언어 모델로 처리하는
파이프라인을 따른다.
이 과정에서 양자화 손실이 발생하고, 토크나이저의 코드북 크기가 음성 품질의
상한을 결정한다.
VoxCPM2가 채택한 확산 기반 자기회귀 방식은 이 병목을 우회한다.
연속 공간에서 직접 음성을 생성하므로 이산화에 따른 정보 손실이 구조적으로
제거된다.

이 접근법은 이미지 생성 분야에서 확산 모델이 GAN을 대체한 궤적과 유사하다.
이미지에서도 이산 토큰 기반 생성(VQVAE 등)보다 연속 확산 모델이 품질에서
우위를 점했고, 음성 도메인에서도 같은 패턴이 반복되고 있다.

### 다국어 지원의 규모와 전략

30개 언어와 9개 중국어 방언이라는 지원 범위는 단순한 숫자 이상의 의미를 갖는다.
중국어 방언을 별도로 구분하여 지원한다는 것은 음성 합성의 난제인 음운 체계
다양성을 정면으로 다루겠다는 의지를 보여준다.
OpenBMB가 중국 기반 연구 그룹이라는 점을 고려하면, 이는 자국 시장의
방언 수요를 충족하면서 동시에 글로벌 언어 커버리지를 확보하려는 이중 전략이다.

한국어 지원이 포함된 점도 주목할 만하다.
한국어는 음절 구조가 비교적 규칙적이지만 경음, 격음, 연음 등 음운 변동이
복잡하여 TTS 시스템에서 자연스러운 발화를 달성하기 어려운 언어 중 하나다.

### 하드웨어 접근성과 오픈소스 전략

8GB VRAM에서 동작하고 CPU 추론까지 지원한다는 것은 2B 파라미터 모델의
효율적 설계를 보여준다.
GGML/GGUF 포맷 지원은 llama.cpp 생태계와의 연동을 의미하며,
이는 LLM 로컬 추론 커뮤니티가 축적해온 양자화 및 최적화 기법을
음성 합성 모델에도 적용할 수 있게 한다.

GeekNews에서 cr543l이 RTX 3060에서 테스트하여 우수한 품질을 확인했다는
보고는 이 접근성이 실제로 작동함을 증명하는 사례다.

## 비평

### 벤치마크 수치의 맥락

영어 WER 1.84%, 만다린 CER 0.97%는 인상적인 수치이지만,
Seed-TTS-eval이라는 특정 벤치마크에서의 결과라는 점을 유의해야 한다.
벤치마크 데이터셋의 난이도 분포, 화자 다양성, 발화 길이 분포에 따라
실사용 환경에서의 성능은 달라질 수 있다.
특히 30개 언어 중 영어와 만다린 외 언어에 대한 정량적 평가가 공개되지
않은 점은 아쉽다.
한국어 성능에 대한 구체적 벤치마크가 없으면, 한국어 지원이 실용적
수준인지 판단하기 어렵다.

### 200만 시간 학습 데이터의 구성 문제

200만 시간 이상의 다국어 데이터로 학습되었다는 설명은 규모를 강조하지만,
언어별 데이터 비율이 공개되지 않았다.
30개 언어를 균등하게 분배하면 언어당 약 6.7만 시간이지만, 실제로는
영어와 중국어에 데이터가 편중되어 있을 가능성이 높다.
소수 언어의 데이터 양이 부족하면 해당 언어의 음성 품질이 저하되는 것은
다국어 모델의 고질적 문제이며, VoxCPM2도 이 한계에서 자유롭지 않을 것이다.

### 음성 복제의 윤리적 측면

Voice Design과 Ultimate Cloning 기능은 기술적으로 인상적이지만,
음성 딥페이크의 위험을 내포한다.
5~10분의 오디오로 화자 적응이 가능하다는 것은 공개된 인터뷰나 영상만으로
타인의 음성을 복제할 수 있다는 뜻이다.
Apache-2.0 라이선스는 사용에 제한을 두지 않으므로, 악의적 사용에 대한
기술적 안전장치가 모델 수준에서 어떻게 구현되어 있는지에 대한 설명이
필요하다.

## 인사이트

### 토크나이저의 퇴장이 의미하는 모달리티 통합의 방향

VoxCPM2의 "토크나이저 프리" 아키텍처는 TTS 기술의 개선을 넘어서 멀티모달
AI의 구조적 전환을 예고한다.
지금까지 대규모 언어 모델(LLM)이 텍스트 외 모달리티를 다루는 방식은
이산 토큰화에 의존해왔다.
이미지는 VQVAE 토큰으로, 음성은 코덱 토큰으로 변환한 뒤 텍스트 토큰과
동일한 시퀀스 위에서 처리하는 것이 주류 패러다임이었다.
이 접근의 근본 한계는 연속적인 신호를 이산적인 기호로 강제 변환하면서
필연적으로 정보가 손실된다는 점이다.

VoxCPM2가 확산 기반 자기회귀로 이 문제를 우회한 것은 음성 도메인에서의
개별 사건이 아니라 더 넓은 흐름의 일부다.
이미지 생성에서도 Stable Diffusion이 토큰 기반 접근을 압도했고,
동영상 생성에서도 연속 잠재 공간(latent space)에서 직접 생성하는 모델이
우위를 점하고 있다.
이 패턴이 음성으로까지 확장되면, 결국 텍스트만이 토큰이라는 이산 표현에
남게 되고, 나머지 모달리티는 모두 연속 공간에서 처리되는 비대칭 구조가
형성된다.

이 비대칭 구조의 2차 효과는 모달리티 간 변환 비용의 비대칭화다.
텍스트에서 음성으로, 텍스트에서 이미지로의 변환은 이산에서 연속으로의
전환이므로 확산 모델이 자연스럽게 처리한다.
반대로 음성에서 텍스트로, 이미지에서 텍스트로의 변환은 연속에서 이산으로의
전환이므로 여전히 토크나이저(또는 인코더)가 필요하다.
이는 생성(generation)과 이해(understanding)의 아키텍처가 근본적으로 다른
방향으로 분화할 가능성을 시사한다.

OpenBMB가 MiniCPM-o에서 멀티모달 이해 모델을 만들고 VoxCPM2에서 음성 생성
모델을 만드는 것은 우연이 아닐 수 있다.
이해와 생성이 서로 다른 아키텍처를 요구한다면, 이를 별도 모델로 분리하되
통합 인터페이스로 묶는 전략이 합리적이다.

### 5분 파인튜닝이 열어젖히는 음성 인터페이스의 개인화 시대

5~10분의 오디오로 화자 적응이 가능하다는 것은 기술 명세 이상의 의미를 갖는다.
이 정도의 데이터 요구량은 일반 사용자가 스마트폰으로 녹음할 수 있는 수준이다.
이전 세대의 음성 복제 기술이 수십 시간의 스튜디오 녹음을 요구했던 것과
비교하면, 이는 음성 복제의 민주화라 부를 수 있는 변화다.

이 기술이 실용화되면 1차적으로 콘텐츠 제작의 생태계가 변한다.
팟캐스트 호스트가 자신의 음성으로 다국어 버전을 자동 생성하고,
교육 콘텐츠 제작자가 자신의 목소리로 30개 언어의 강의를 제공하는 것이
가능해진다.
더 나아가 고인의 음성으로 오디오북을 만들거나, 성우의 음성을 라이선싱하여
게임 캐릭터에 적용하는 등 음성 자체가 거래 가능한 디지털 자산이 되는
시나리오가 열린다.

그러나 이 민주화에는 어두운 이면이 있다.
5분의 오디오는 공인 인터뷰, 유튜브 영상, SNS 음성 메시지 등에서 쉽게
확보할 수 있는 양이다.
기술적으로 누구든 타인의 음성을 복제할 수 있는 상황에서 "진짜 음성"의
개념이 근본적으로 흔들린다.
이는 딥페이크 이미지가 "진짜 사진"의 증거 능력을 약화시킨 것과 동일한
패턴이며, 음성 도메인으로의 확장은 보이스 피싱, 신원 사칭 등 더 직접적인
위협을 수반한다.

역사적으로 복제 기술의 민주화는 항상 인증 기술의 수요를 폭발시켰다.
인쇄술이 위조 문서를 낳았고 워터마크가 뒤따랐으며,
디지털 복제가 해적판을 낳았고 DRM이 뒤따랐다.
음성 복제의 민주화는 음성 워터마킹, 음성 출처 인증, 실시간 음성 진위 판별
기술에 대한 수요를 급증시킬 것이다.

### 8GB VRAM의 의미: TTS 모델의 엣지 컴퓨팅 진입

VoxCPM2가 8GB VRAM GPU에서 동작하고 CPU 추론까지 지원한다는 것은
TTS 기술이 클라우드에서 엣지로 이동하는 전환점을 나타낸다.
8GB VRAM은 현재 가장 저가형 게이밍 GPU(GTX 1650 Super, RTX 3060 등)의
사양이며, 사실상 GPU가 있는 모든 데스크톱에서 실행 가능하다는 뜻이다.
cr543l이 RTX 3060에서 우수한 품질을 확인한 것은 이 접근성이 이론이 아닌
현실임을 증명한다.

이 엣지 이동의 1차 효과는 지연시간(latency)의 극적 감소다.
클라우드 TTS는 네트워크 왕복 시간이 불가피하지만, 로컬 추론은 이를
제거한다.
RTF 0.13이라는 수치는 고성능 GPU 기준이지만, 8GB VRAM 환경에서도
실시간에 가까운 합성이 가능하다면, 이는 실시간 대화형 음성 인터페이스를
개인 하드웨어에서 구동할 수 있음을 의미한다.
실시간 통역, 실시간 음성 변환, 음성 기반 게임 NPC 등 저지연이 필수인
응용 분야가 클라우드 의존 없이 실현 가능해진다.

2차 효과는 프라이버시 모델의 근본적 변화다.
음성 데이터는 생체 정보에 해당하며, 클라우드 TTS를 사용하면 사용자의
음성 특성이 서버에 전송된다.
로컬 추론은 이 데이터 전송 자체를 제거하여 프라이버시 우려를 구조적으로
해소한다.
GDPR이나 한국의 개인정보보호법 같은 규제 프레임워크에서 음성 데이터의
처리가 점점 엄격해지는 추세를 고려하면, 로컬 추론은 규제 준수 비용을
대폭 낮추는 실질적 이점이 된다.

GGML/GGUF 포맷 지원이 갖는 전략적 의미도 크다.
이 포맷은 llama.cpp를 중심으로 형성된 로컬 LLM 생태계의 사실상 표준이다.
VoxCPM2가 이 포맷을 지원함으로써, LLM 로컬 추론을 위해 이미 구축된
도구와 인프라(Ollama, LM Studio 등)를 그대로 활용할 수 있게 된다.
이는 기존 생태계의 네트워크 효과에 편승하는 전략이며,
채택 속도를 가속화하는 데 결정적 역할을 할 것이다.
crawler가 GeekNews에서 MiniCPM-o 개발팀의 음성 합성 모델이라며 기대감을
표현한 것은 OpenBMB의 기존 커뮤니티 신뢰가 새 모델의 채택을 촉진하는
메커니즘을 보여준다.
