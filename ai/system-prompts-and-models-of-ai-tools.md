# System Prompts and Models of AI Tools

<https://github.com/x1xhlol/system-prompts-and-models-of-ai-tools>

30개 이상의 AI 코딩 도구에서 사용하는 시스템 프롬프트, 도구 정의,
모델 설정을 모아놓은 저장소. 2025년 3월 공개 이후 GitHub 스타
127,000개 이상을 기록했다.

## 요약

AI 코딩 어시스턴트들이 내부적으로 사용하는 시스템 프롬프트를
수집·정리한 아카이브다. 각 도구별로 디렉터리가 분리되어 있고,
프롬프트 텍스트 파일과 도구 정의 JSON이 포함되어 있다.
30,000줄 이상의 문서가 축적되어 있다.

## 분석

### 프롬프트 구조의 공통 패턴

각 도구의 시스템 프롬프트를 비교하면 반복되는 설계 패턴이
보인다.

1. **역할 정의**: "You are an AI assistant that helps with
   software engineering tasks" 같은 페르소나 설정이 가장 먼저
   온다.
2. **도구 명세**: 파일 읽기, 편집, 검색, 터미널 실행 등 사용
   가능한 도구를 구조화된 형식으로 나열한다. JSON Schema로
   정의하는 경우가 많다.
3. **행동 규칙**: 안전성 가이드라인, 코드 스타일 규칙, 금지
   행위 목록이 명시적으로 포함된다.
4. **출력 형식**: 마크다운 포매팅, 코드 블록 사용법, 응답 길이
   제한 등 출력 형태를 제어한다.

### 버전 관리의 의미

Cursor의 경우 Agent Prompt v1.0부터 v1.2, 2.0, 그리고 날짜별
버전(2025-08-07, 2025-09-03)까지 이력이 남아 있다. 프롬프트가
어떻게 진화했는지 추적할 수 있다는 점에서 단순 유출을 넘어
시스템 프롬프트 엔지니어링의 변천사를 보여준다.

### 도구 정의의 차이

- **Cursor**: `Agent Tools v1.0.json`으로 도구를 별도 JSON
  파일로 분리한다.
- **Replit**: `Tools.json`으로 비슷한 접근을 취한다.
- **Windsurf**: 프롬프트와 도구 정의를 별도 텍스트 파일로
  나눈다(`Prompt Wave 11.txt`, `Tools Wave 11.txt`).
- **Devin AI**: 프롬프트 파일만 존재하고 도구 정의가 별도로
  분리되지 않았다.

도구 정의를 프롬프트와 분리하는 경향이 뚜렷해지고 있다.
시스템이 복잡해질수록 관심사 분리(Separation of Concerns)
원칙이 프롬프트 설계에도 적용된다.

## 인사이트

### 1. 프롬프트는 제품의 핵심 차별화 요소다

같은 기반 모델(Claude, GPT-4)을 사용하면서도 각 도구의 동작이
크게 다른 이유는 시스템 프롬프트와 도구 정의에 있다. 모델
자체보다 프롬프트 엔지니어링이 사용자 경험을 결정한다.

### 2. 시스템 프롬프트 보안은 환상이다

127,000개 스타가 증명하듯, 시스템 프롬프트는 유출된다.
프롬프트 비공개에 의존하는 방어 전략은 성립하지 않는다.
진짜 해자(moat)는 프롬프트가 아니라 실행 인프라, 데이터
파이프라인, 사용자 경험의 총체에 있다.

### 3. 프롬프트 설계는 소프트웨어 아키텍처와 닮았다

역할 분리, 관심사 분리, 버전 관리, 인터페이스 정의 등
소프트웨어 설계 원칙이 프롬프트 설계에도 그대로 적용된다.
프롬프트 엔지니어링이 성숙해질수록 소프트웨어 엔지니어링의
모범 사례를 더 많이 차용하게 될 것이다.

### 4. AI 도구 생태계의 수렴 진화

30개 이상의 도구가 독립적으로 발전했음에도 프롬프트 구조가
유사하다. 파일 시스템 접근, 터미널 실행, 검색이라는 기본
도구 세트가 거의 동일하다. 이는 AI 코딩 어시스턴트의
최소 기능 집합(minimum viable toolset)이 이미 수렴했음을
의미한다. 차별화는 기본 기능이 아니라 워크플로우 통합과
사용자 경험에서 나올 수밖에 없다.

### 5. 학습 자료로서의 가치

AI 도구를 만들거나 커스터마이징하려는 개발자에게 실전
레퍼런스가 된다. CLAUDE.md, `.cursorrules` 같은 프로젝트별
설정 파일을 작성할 때, 해당 도구의 시스템 프롬프트가 어떤
구조를 기대하는지 알면 더 효과적인 지시를 내릴 수 있다.
