# MiroFish: 다중 에이전트 기반 예측 시뮬레이션 엔진

원문: <https://github.com/666ghj/MiroFish>

## 요약

MiroFish는 다중 에이전트 군집 지능(swarm intelligence)을 활용하여
미래 시나리오를 시뮬레이션하는 예측 엔진이다.
실세계 데이터(뉴스, 정책 문서, 금융 데이터)를 시드로 투입하면
수천 개의 자율 에이전트가 각자의 성격, 기억, 행동 논리를 가지고
상호작용하며 미래 궤적을 예측한다.

아키텍처는 5단계 워크플로로 구성된다.
시드에서 엔티티와 관계를 추출하는 그래프 구축(GraphRAG),
에이전트 생태계 환경 설정,
다중 라운드 시뮬레이션 실행,
리포트 생성,
결과에 대한 심층 탐색이다.

듀얼 플랫폼 시뮬레이션으로 여러 예측 경로를 병렬 탐색하며,
사용자가 "신의 시점(God's-eye view)"으로
시뮬레이션 중 동적으로 변수를 주입할 수 있다.
시뮬레이션 후에는 개별 에이전트와 대화할 수도 있다.

기술 스택은 Python 3.11-3.12 백엔드, Vue 프론트엔드,
Zep Cloud 메모리 레이어, OpenAI SDK 호환 LLM이다.
OASIS 엔진 기반이며 Shanda Group이 인큐베이팅하고 있다.
AGPL-3.0 라이선스로 공개되었으며
GitHub 스타 48,000개 이상을 기록했다.

## 분석

### GraphRAG + 에이전트 시뮬레이션의 결합

MiroFish의 차별점은 단순 LLM 생성이 아니라
구조화된 지식 그래프 위에서 에이전트를 실행한다는 것이다.
시드 데이터에서 엔티티와 관계를 추출하여 그래프를 구축하고,
이 그래프의 노드가 에이전트의 행동 맥락이 된다.
GraphRAG가 "세계의 구조"를, 에이전트가 "세계의 동학"을 담당하는
이중 레이어 구조다.

### 시뮬레이션 vs 예측의 구분

MiroFish는 "예측"을 표방하지만
실제로는 "시뮬레이션"에 더 가깝다.
특정 조건 하에서 에이전트들이 어떻게 상호작용하는지를
탐색하는 것이지, 확률적 예측값을 산출하는 것이 아니다.
이 구분이 중요한 이유는 사용자의 기대를 관리하기 때문이다.
"미래를 예측한다"와 "가능한 시나리오를 탐색한다"는
전혀 다른 가치 제안이다.

### OASIS 엔진과 Shanda Group의 맥락

OASIS(Open Agent Social Interaction Simulations) 엔진 기반이라는 점은
사회 시뮬레이션 연구의 맥락에서 이해해야 한다.
게임 회사인 Shanda Group의 인큐베이팅도
게임 내 NPC 시뮬레이션 기술의 실세계 응용이라는 관점에서 볼 수 있다.

## 비평

### "수천 개 에이전트"의 실질적 가치

수천 개의 에이전트가 상호작용한다는 것은 인상적이지만,
각 에이전트의 행동이 LLM 호출에 의존한다면
시뮬레이션 1회의 비용과 시간이 상당할 수 있다.
에이전트 수가 늘어날수록 비용은 선형 이상으로 증가하는데,
이에 대한 비용 가이드나 최적화 전략이 보이지 않는다.

### 시뮬레이션 결과의 검증 가능성

시뮬레이션이 "재미있는 시나리오"를 생성하는 것과
"유의미한 예측"을 제공하는 것은 다르다.
에이전트의 행동 논리가 LLM의 확률적 출력에 기반하므로,
같은 시드를 넣어도 매번 다른 결과가 나올 수 있다.
시뮬레이션 결과의 재현성과 검증 방법론에 대한
설명이 부족하다.

### 정책 시뮬레이션 적용의 위험성

정책 테스트를 유스케이스로 내세우지만,
LLM 기반 에이전트가 실제 시민의 행동을 얼마나
정확하게 모사하는지는 근본적으로 불확실하다.
에이전트의 "성격"과 "행동 논리"가
LLM의 학습 데이터에 내재된 편향을 반영할 수 있으며,
이를 기반으로 정책 결정을 내리는 것은
편향된 시뮬레이션 위의 편향된 결정이 될 수 있다.

### 48K 스타의 맥락

GitHub 스타 48,000개는 인상적인 수치이지만,
이 규모의 스타가 실제 사용량과 비례하는지는 별개다.
특히 중국 기반 프로젝트에서
스타 수와 실제 프로덕션 사용 사이의 괴리가 크게 나타나는 사례가 있다.

## 인사이트

### "디지털 트윈 사회"는 사회과학의 실험실을 만든다

물리학에는 실험실이 있고 생물학에는 페트리 접시가 있지만,
사회과학에는 통제된 실험 환경이 없었다.
MiroFish 같은 에이전트 시뮬레이션은
사회적 현상을 "실험실에서 재현"할 수 있는 가능성을 열었다.

정책 변경의 2차, 3차 효과를 사전에 탐색하거나,
여론 형성 과정을 관찰하거나,
경제적 충격의 전파 경로를 추적하는 것이
시뮬레이션으로 가능해진다.
이는 사회과학 연구 방법론의 근본적 변화다.

다만 이 "실험실"의 한계는 분명하다.
LLM 에이전트가 실제 인간의 비합리성, 감정, 문화적 맥락을
얼마나 충실하게 반영하는지에 따라
시뮬레이션의 외적 타당성(external validity)이 결정된다.
현재 수준에서는 "정밀한 예측"보다
"가능한 시나리오의 발견"에 더 적합하다.

### 게임 NPC 기술과 사회 시뮬레이션의 수렴

Shanda Group(게임 회사)이 사회 시뮬레이션 엔진을 인큐베이팅하는 것은
두 영역의 기술적 수렴을 보여준다.
게임 NPC는 "그럴듯한 행동을 하는 자율 캐릭터"이고,
사회 시뮬레이션 에이전트도 같은 기술적 요구사항을 갖는다.
성격, 기억, 행동 논리, 다른 에이전트와의 상호작용이라는
구성 요소가 동일하다.

Stanford의 "Generative Agents" 논문이
Smallville이라는 게임 환경에서 사회 시뮬레이션을 시연한 것도
이 수렴의 증거다.
게임 산업에서 축적된 NPC AI 기술이
사회 시뮬레이션, 도시 계획, 재난 시뮬레이션 등으로
응용 영역을 넓히는 흐름이 가속될 것이다.

### "신의 시점" 인터페이스는 시뮬레이션의 핵심 UX다

MiroFish의 "God's-eye view" 동적 변수 주입은
단순한 기능이 아니라 시뮬레이션 도구의 핵심 UX 패턴이다.
시뮬레이션이 유용하려면 사용자가
"만약 이 변수가 바뀌면?"이라는 질문을
실시간으로 탐색할 수 있어야 한다.

전통적 시뮬레이션 도구(MATLAB, AnyLogic 등)에서는
파라미터를 바꾸고 다시 실행하는 사이클이 길었다.
LLM 기반 시뮬레이션에서는 자연어로 변수를 주입하고
즉시 결과를 관찰하는 것이 가능해졌다.
이 인터랙션 패턴은 시뮬레이션을
전문가 도구에서 의사결정 보조 도구로 전환시킨다.

시뮬레이션 후 개별 에이전트와 대화할 수 있다는 기능도 같은 맥락이다.
"왜 이 에이전트는 이렇게 행동했는가"를 직접 질문함으로써
시뮬레이션 결과의 인과적 이해를 돕는다.
블랙박스 시뮬레이션의 설명 가능성 문제를
대화 인터페이스로 해결하려는 시도다.
