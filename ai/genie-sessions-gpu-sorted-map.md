# Genie Sessions: GPU Sorted Map

Kent Beck의 Genie Sessions 라이브 코딩에서 배운 AI 협업 인사이트.

영상: <https://www.youtube.com/watch?v=58n6ludV-iw>

## Genie Sessions

Kent Beck은 Genie Sessions을 통해 Augmented Coding(증강 코딩)을 시연한다. **대본
없이, 연습 없이** 진행되는 라이브 코딩으로, 높은 불안감 속에서 학습 기회를
극대화한다.

이번 에피소드에서는 GPU를 활용한 Sorted Map을 Rust와 WGPU로 구현한다.

## AI와 협업하는 자세

### AI는 친구이자 학습 파트너

AI를 단순한 코드 생성기가 아닌, "리소스" 또는 "친구"로 활용한다.

> AI는 프로그래머에게 많은 노력을 요구하지만, 동시에 엄청난 가치를 제공하는
> 도구다.

### 열린 자세

> AI가 코딩하는 방식처럼 코딩하도록 만드는 것은 잘못된 접근 방식이다.

AI가 인간 프로그래머보다 **더 나은 프로그래밍 방식을 제시할 가능성**을
열어두어야 한다.

## AI 코딩 에이전트 활용 전략

### 1. 학습 촉매제로 활용

AI를 코드 생산 도구가 아닌 **학습 기회를 창출하는 촉매제**로 본다.

- 모르는 개념은 AI에게 질문하여 이해
- "동화처럼 설명해 달라" 등 다양한 방식으로 정보 요청
- AI가 생성한 코드를 통해 새로운 기술 습득

### 2. 실험과 탐색

초기에는 정답을 찾으려 하지 말고 **실험**한다.

- 다양한 도구와 접근 방식을 시도
- 자신에게 맞는 워크플로우를 발견
- 구체적인 구현은 AI와 함께 학습하며 결정

### 3. 컨텍스트 관리

AI가 "이상해지기" 시작하면 **작업을 분할**한다.

- 컨텍스트 창의 30~40%를 초과하면 품질 저하
- 큰 작업을 작은 조각으로 나누기
- **수천 시간의 경험**을 통해 AI의 한계를 감지하는 감각 습득

### 4. TDD 유지

복잡한 환경에서도 **테스트 주도 개발**을 고수한다.

- 작은 단위의 테스트로 정확성 확인
- 점진적으로 기능 추가
- 불안감을 느끼더라도 단계별 검증

### 5. 유연한 롤백

어려우면 **이전 단계로 돌아갈 수 있는 유연성** 유지.

- 복잡한 접근이 안 되면 단순한 접근으로 롤백
- 완벽한 해결책보다 동작하는 해결책 우선
