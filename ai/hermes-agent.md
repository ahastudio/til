# Hermes Agent — 경험으로부터 스킬을 생성하는 자기 학습형 에이전트

> 원문: <https://hermes-agent.nousresearch.com>

## 요약

Nous Research가 개발한 Hermes Agent는
서버에 상주하며 경험으로부터 스킬을 생성하고 개선하는 자율 AI 에이전트다.
코딩 코파일럿이나 챗봇 래퍼가 아니라,
실행할수록 더 유능해지는 지속형 시스템을 지향한다.

주요 특징:

- **자기 학습 루프:** 문제 해결 과정에서 스킬을 자동 생성하고 개선
- **영속 메모리:** Honcho 기반, FTS5 + LLM 요약으로 세션 간 기억 유지
- **멀티 플랫폼 게이트웨이:** CLI, Telegram, Discord, Slack, WhatsApp, Signal, Email
- **40개 이상 내장 도구:** 웹 검색, 터미널, 브라우저, 비전, 이미지 생성, TTS 등
- **실행 환경:** 로컬, Docker, SSH, Modal, Singularity
- **200개 이상 LLM 모델 호환:** Nous Portal, OpenRouter, OpenAI 호환 API
- **MIT 라이선스 오픈소스**

## 분석

Hermes Agent의 핵심 차별점은 "스킬 생성"이라는 메타 학습 능력이다.
일반적인 에이전트가 주어진 도구를 사용하는 데 그치는 반면,
Hermes Agent는 반복되는 작업 패턴을 스킬로 추상화하여
다음 실행에서 더 효율적으로 동작한다.

이는 Claude Code의 스킬 시스템과 유사하지만 방향이 다르다.
Claude Code에서 스킬은 사용자가 작성하거나 커뮤니티에서 가져오는 정적 자산인 반면,
Hermes Agent에서 스킬은 에이전트가 경험으로부터 동적으로 생성하는 산출물이다.

멀티 플랫폼 게이트웨이는 에이전트의 접근성을 극적으로 넓힌다.
Telegram에서 시작한 대화를 Slack에서 이어갈 수 있다는 것은
에이전트가 특정 인터페이스에 종속되지 않는다는 의미다.

다만 GeekNews 댓글(theogn)이 지적했듯,
OpenClaw, Ductor, CC-Connect 등 유사 에이전트 플랫폼과의
실질적 차별점이 사용자에게 명확히 전달되지 않고 있다.
기능 목록이 아닌 사용 시나리오로 포지셔닝할 필요가 있다.

## 비평

"실행할수록 똑똑해진다"는 비전은 매력적이지만,
자동 생성된 스킬의 품질 보증 메커니즘이 불분명하다.
잘못된 패턴이 스킬로 고착화되면 오히려 성능이 저하될 수 있다.
인간의 검증 없는 자기 학습은 편향 증폭의 위험을 내포한다.

$5 VPS부터 GPU 클러스터까지 지원한다는 것은 유연성이지만,
$5 VPS에서의 실제 성능은 의문이다.
40개 이상의 도구와 영속 메모리 시스템을 저사양에서 운영하면
응답 지연이 실용성을 해칠 수 있다.

MIT 라이선스는 생태계 확장에 유리하지만,
OpenClaw로부터의 자동 마이그레이션을 제공한다는 점에서
후발주자가 선발주자의 사용자 기반을 흡수하려는
에이전트 플랫폼 간 경쟁의 격화를 보여준다.

## 인사이트

- **에이전트의 가치는 "학습 곡선의 방향"으로 결정된다.**
  대부분의 도구는 사용자가 학습 곡선을 올라가야 한다.
  Hermes Agent의 자기 학습 루프는 이 방향을 뒤집어,
  도구가 사용자에게 적응하는 모델을 제안한다.
  이것이 실현되면 "설정"이라는 개념 자체가 사라진다.
  에이전트가 사용 패턴으로부터 최적의 설정을 스스로 도출하기 때문이다.

- **에이전트 플랫폼의 "안드로이드 순간".**
  OpenClaw, Ductor, CC-Connect, Hermes Agent 등
  유사한 기능을 가진 에이전트 플랫폼이 난립하는 현재 상황은
  2008-2010년 안드로이드 초기와 닮았다.
  기능 목록으로는 차별화가 불가능해지고,
  결국 생태계(스킬 마켓플레이스, 커뮤니티, 통합)가 승자를 결정할 것이다.
  agentskills.io라는 오픈 포맷은 이 생태계 전쟁의 첫 수를 놓은 것이다.

- **"경험으로부터의 스킬 생성"은 조직 학습의 메타포다.**
  개인이 반복 작업을 매뉴얼로 문서화하듯,
  Hermes Agent는 실행 이력을 스킬로 문서화한다.
  이것이 팀 단위로 확장되면 "조직의 암묵지"가
  에이전트의 스킬 라이브러리로 명시화되는 효과가 생긴다.
  에이전트가 지식 관리 도구가 되는 경로다.
