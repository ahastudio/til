# CS336: Language Modeling from Scratch

<https://cs336.stanford.edu/>

HN 토론: <https://news.ycombinator.com/item?id=48357075> (552점, 51개 댓글)

## 소개

Stanford CS336은 대형 언어 모델(LLM, Large Language Model)을 처음부터 직접 구축하는 5학점 구현 중심 과목이다.
Tatsunori Hashimoto와 Percy Liang이 공동 강의하며, 2026년 봄학기 기준 총 19개 강의로 구성된다.
강의실 수업과 함께 YouTube 녹화본이 공개되어 외부 학습자도 접근할 수 있다.

이 과목이 다른 AI/ML 과목과 근본적으로 다른 점은 “전체 파이프라인을 직접 구현한다”는 것이다.
토크나이저(tokenizer)부터 트랜스포머(Transformer) 아키텍처, GPU 커널(kernel), 분산 학습(distributed training), 데이터 수집과 전처리, 그리고 정렬(alignment)까지 전 과정이 과제로 주어진다.
“어떻게 쓰는가”가 아닌 “어떻게 작동하는가”를 가르친다.

## 커리큘럼 구조

5개 과제가 LLM 개발 파이프라인의 각 단계를 담당한다.

| 과제      | 마감     | 주제                                              |
| --------- | -------- | ------------------------------------------------- |
| 과제 1    | 4/15     | 토크나이저, 트랜스포머 구현, 옵티마이저, 학습    |
| 과제 2    | 4/29     | FlashAttention2(Triton), 분산 학습                |
| 과제 3    | 5/6      | 트랜스포머 구성요소 분석, 스케일링 법칙 적용      |
| 과제 4    | 5/20     | Common Crawl 전처리, 필터링, 중복 제거            |
| 과제 5    | 6/3      | 지도 미세조정(SFT), 강화학습(RL), 안전 정렬       |

주차별 주제도 체계적으로 설계되어 있다.
1~2주는 토크나이제이션(tokenization)과 PyTorch 기초, 3~4주는 아키텍처와 어텐션(attention) 대안, 5~6주는 GPU·TPU와 Triton 커널, 7~8주는 병렬 처리, 9~11주는 스케일링 법칙과 추론, 12~14주는 평가와 데이터, 15~17주는 사후 훈련(post-training)과 정렬·멀티모달리티(multimodality)를 다룬다.

사전 요건도 높다. Python 숙련도, PyTorch 경험, GPU 메모리 계층 이해, 대학 수준의 미적분학과 선형대수, 확률통계, 기계 학습(ML) 기초를 모두 요구한다.
“모델을 쓸 줄 아는 사람”이 아니라 “모델이 어떻게 작동하는지 아는 사람”을 대상으로 한다.

## 분석

### 전체 파이프라인 교육의 희소성

대부분의 LLM 관련 강의는 특정 레이어에 집중한다. 프롬프트 엔지니어링(prompt engineering), 파인튜닝(fine-tuning), 또는 시스템 설계.
CS336은 다르다. 토크나이저부터 안전 정렬까지 전체 스택을 단일 학기 안에 다룬다.

이 접근이 가능한 이유는 구현 과목이기 때문이다. 이론을 강의하는 것이 아니라 코드로 만드는 것을 요구한다.
과제 2에서 FlashAttention2를 Triton으로 직접 구현하게 하는 과목은 극히 드물다.
이것은 단순한 알고리즘 이해가 아니라 하드웨어 수준의 최적화를 직접 다루는 경험이다.

### 데이터를 핵심 과목으로 편입시킨 설계

Common Crawl 데이터 처리를 별도 과제(과제 4)로 분리한 것은 의미 있는 설계 결정이다.
데이터 수집과 전처리는 LLM 품질을 결정하는 핵심 요소이지만, 대부분의 교육 과정에서 주변부로 취급된다.
강의 주제에서도 13~14주는 각각 “데이터 소스·데이터셋”과 “필터링·중복 제거”를 독립적으로 다룬다.
모델 아키텍처와 동등한 위상으로 데이터 공학을 다루는 것은 산업 현장의 현실을 제대로 반영한 것이다.

### 강화학습의 위치

과제 5에서 RLVR(Reinforcement Learning from Verifiable Rewards)을 다루는 것은 2026년 기준의 최신 동향을 빠르게 흡수한 증거다.
DeepSeek-R1 이후 추론 능력을 위한 강화학습이 LLM 후처리의 핵심 기법으로 부상했으며, 이를 학부·대학원 교육 과정에 포함시킨 것은 빠른 커리큘럼 갱신이다.
게스트 강연으로 Daniel Selsam(형식 검증 연구자)과 Dan Fu(시스템 ML 연구자)를 초청한 것도 강화학습과 시스템 최적화라는 맥락 안에서 읽힌다.

## 비평

### 강점: 구현을 통한 깊이

이 과목의 가장 큰 강점은 “구현이 곧 이해”라는 철학을 일관되게 유지한다는 것이다.
FlashAttention2를 Triton으로 구현하고, 분산 학습 코드를 직접 짜고, Common Crawl을 처리해본 사람과 논문만 읽은 사람의 차이는 결정적이다.
이론이 실제 하드웨어 제약, 수치 안정성 문제, 데이터 품질 노이즈와 부딪히는 지점을 직접 경험한다.

Stanford라는 자원도 중요하다. GPU 사용료가 시간당 $4.99~$7.49인 환경에서 실질적인 계산 실험이 가능하며, Modal의 무료 크레딧 스폰서십이 이를 보완한다.
유사한 구현 과목을 다른 기관에서 운영하려면 상당한 컴퓨팅 예산이 필요하다.

### 약점: 높은 진입 장벽과 재현 가능성의 한계

이 과목의 깊이는 동시에 진입 장벽이다.
GPU 시스템 이해, PyTorch 경험, 수학 기초를 모두 갖추고 Triton 커널 수준까지 다룰 수 있는 학생은 전 세계적으로도 소수다.
강의 녹화본이 공개되더라도 과제의 컴퓨팅 비용과 전제 지식이 실질적 접근을 제한한다.

HN에서 2025년 버전을 독학으로 이수한 한 학습자는 Linux/NVIDIA GPU 환경 설정의 구체적 어려움을 제기했다.[^fg137]
하네스가 특정 CUDA 버전과 아키텍처를 요구하며, Mac OS 환경에서는 메모리 분석 중 Python 코드가 멈추며 강제 재부팅이 발생할 수 있다고 보고했다.
추천 GPU의 시간당 비용이 $4.99~$7.49 수준이어서 장시간 실험을 개인이 감당하기 어렵다는 지적도 있었다.[^skerit]
한편 30명으로 시작한 온라인 그룹 스터디에서 마지막 세션까지 완주한 인원이 8명이었다는 경험담은[^sonabinu] 실질적 완주율이 낮다는 점을 보여준다.

과목이 다루는 범위도 선택의 문제를 낳는다. 한 학기 안에 전체 파이프라인을 다루다 보면 각 주제의 깊이에 제약이 생긴다.
분산 학습 전문가 과목, 데이터 전처리 전문 과목과 비교하면 각각의 주제에서 타협이 있을 수밖에 없다.

### 보완할 시각: 평가 기준의 부재

이 과목에서 생략된 것이 있다. “좋은 LLM이란 무엇인가”에 대한 명시적 기준이다.
12주에 평가(evaluation)를 다루지만, 벤치마크 포화 문제, 평가 데이터 오염, 인간 선호도 평가의 한계 같은 현재 진행형 논쟁은 커리큘럼에서 주변부에 머문다.
구현의 완성도를 검증하는 방법을 함께 가르치지 않으면, 작동하지만 측정할 수 없는 모델을 만드는 법을 배우는 데 그칠 수 있다.

## 인사이트

### LLM 스택의 어느 레이어에서 일할 것인가

CS336이 암묵적으로 던지는 질문이 있다. LLM 시대의 엔지니어는 스택의 어디에 서야 하는가.

현재 LLM 개발 스택은 크게 세 레이어로 나뉜다. 하드웨어·커널 수준(CUDA, Triton), 모델 아키텍처·학습 수준(PyTorch, 분산 학습), 응용·파인튜닝·에이전트 수준(API, 프롬프팅, RAG).
대부분의 엔지니어와 교육 과정은 세 번째 레이어에 집중한다. 진입 장벽이 낮고 수요가 즉각적이기 때문이다.
CS336은 첫 번째와 두 번째 레이어를 정면으로 다룬다.

이 선택의 함의는 시간이 지날수록 커진다. 세 번째 레이어는 추상화가 빠르게 발전하고 진입 장벽이 낮아져 경쟁이 심화된다.
첫 번째와 두 번째 레이어는 물리적 제약(하드웨어 특성, 수치 정밀도, 메모리 대역폭)에 기반하기 때문에 추상화의 속도가 상대적으로 느리다.
CS336이 가르치는 것은 현재의 기술이 아니라, 빠르게 변하는 위쪽 레이어를 이해하기 위한 불변의 기반이다.

### 데이터 공학이 모델 공학과 동등한 지위를 갖는 시대

CS336 커리큘럼에서 데이터 처리가 별도 과제 한 개를 차지한다는 것은 작은 사실처럼 보이지만 큰 방향 전환을 반영한다.

GPT-3 이전 시대의 ML 엔지니어링은 모델 아키텍처 설계가 핵심이었다. 어떤 레이어를 어떻게 쌓을 것인가.
GPT-3 이후 스케일링 법칙이 확립되면서 아키텍처보다 데이터 규모와 품질이 결과를 결정한다는 것이 명확해졌다.
Chinchilla 법칙, 그 이후의 데이터 반복 논쟁, 합성 데이터 생성까지 모든 중요한 발전이 데이터 중심으로 이루어졌다.

Common Crawl을 직접 처리해본 사람은 “데이터 품질”이 추상적 개념이 아님을 안다.
중복, 노이즈, 독성 콘텐츠, 언어 불균형, 품질 필터의 민감도와 recall 사이의 트레이드오프.
이 경험 없이 모델 평가 결과를 해석하면 숫자는 있지만 원인을 모른다.
CS336이 데이터를 핵심 과목으로 편입시킨 것은 LLM 엔지니어링의 무게중심이 이미 이동했음을 인정하는 것이다.

### 강의가 연구 방향을 드러내는 방식

CS336의 강사인 Hashimoto와 Liang이 어떤 주제에 얼마나 많은 시간을 할당하는지를 보면 그들이 중요하다고 생각하는 것이 보인다.

스케일링 법칙에 세 개 강의를 할당하고(9, 11주, 과제 3), 추론 전용 강의를 별도로 편성했다(10주).
사후 훈련(post-training)과 RLVR에 두 개 강의를 배정한 것(15, 16주)은 정렬이 아키텍처만큼 중요한 연구 영역이 됐음을 보여준다.
게스트 강연자로 선택한 Daniel Selsam은 형식 검증과 수학적 추론 분야의 연구자다.
이 선택들을 종합하면 강사들이 주목하는 방향이 보인다. 스케일 이후의 효율, 검증 가능한 추론 능력, 데이터 품질.

연구 대학의 강의 커리큘럼은 교과서가 아니라 현재 진행 중인 연구의 지도다.
CS336이 무엇을 가르치는지보다 무엇을 중요하게 여기는지를 읽으면, 2026년 LLM 연구의 무게중심을 엿볼 수 있다.
