# Superpowers

코딩 에이전트를 위한 구조화된 소프트웨어 개발 워크플로우 프레임워크.

<https://github.com/obra/superpowers>

## 핵심 철학

코딩 에이전트에게 "코드 짜줘"라고 바로 던지지 않는다.
**설계 → 계획 → 구현 → 검증**의 단계를 강제한다.
즉흥적 개발 대신 체계적 프로세스를, 가정 대신 증거 기반
검증을 추구한다.

## 7단계 워크플로우

| 단계                      | 하는 일                          |
|---------------------------|----------------------------------|
| 1. Brainstorming          | 요구사항을 명세로 정제           |
| 2. Git Worktree Setup     | 격리된 개발 브랜치 구성          |
| 3. Planning               | 2~5분 단위 작업으로 분해         |
| 4. Subagent Development   | 작업별 전용 서브에이전트 투입    |
| 5. Test-Driven Development| RED-GREEN-REFACTOR 강제          |
| 6. Code Review            | 스펙 준수 + 코드 품질 2단계 검토 |
| 7. Branch Completion      | 머지 및 정리                     |

## 활용 인사이트

### 브레인스토밍의 "하드 게이트"

Brainstorming 스킬에는 하드 게이트가 있다.
설계를 제출하고 사용자 승인을 받기 전까지
**어떤 구현 행위도 차단**된다.
"간단한 프로젝트"라는 예외도 없다.

이 제약이 왜 중요한가: 에이전트는 본능적으로 코드부터
작성하려 한다. 설계 없이 구현에 돌입하면 방향을
잃거나 불필요한 복잡도가 생긴다.
하드 게이트는 이 충동을 구조적으로 막는다.

### 서브에이전트 격리 패턴

작업마다 새로운 서브에이전트를 생성한다.
이전 작업의 컨텍스트가 오염되지 않는다.

서브에이전트의 수명은 다음과 같다:

1. 작업 컨텍스트를 받고 질문이 있으면 먼저 묻는다
2. 구현 → 테스트 → 커밋 → 셀프 리뷰
3. 스펙 준수 리뷰어가 검토한다
4. 코드 품질 리뷰어가 검토한다
5. 두 리뷰를 모두 통과해야 완료 처리된다

핵심은 **질문을 먼저 하게 한다**는 점이다.
구현 전에 모호한 부분을 표면화시켜서
잘못된 가정으로 코드를 작성하는 것을 방지한다.

### TDD를 "강제"하는 방식

테스트를 나중에 쓰는 게 아니라,
테스트가 먼저 실패하는 것을 **관찰**해야 한다.
실패를 목격하지 않은 테스트는 올바른 동작을
검증한다고 보장할 수 없다.

가장 강력한 규칙: **대응하는 실패 테스트 없이
작성된 프로덕션 코드는 삭제해야 한다.**
예외 없이. 이미 작성한 코드에 대한 매몰 비용
오류를 원천 차단한다.

### 2단계 코드 리뷰

하나의 리뷰가 아니라 두 번 본다:

1. **스펙 준수 리뷰**: 요구사항을 충족하는가?
2. **코드 품질 리뷰**: 구현이 깨끗한가?

"동작하지만 더러운 코드"와 "깨끗하지만 스펙에서
벗어난 코드"를 모두 걸러낸다.

### 배치 실행과 체크포인트

계획의 모든 작업을 한 번에 실행하지 않는다.
3개씩 배치로 실행하고, 배치마다 사람이 확인한다.
블로커, 테스트 실패, 불명확한 지시를 만나면
추측하지 않고 즉시 멈춘다.

## 기존 접근법과의 비교

**Spec-Driven Development(SDD)**는 "무엇을" 만들지
명세로 정의한다. Superpowers는 "어떻게" 만들지의
**프로세스 전체**를 구조화한다.

**File-based Planning**은 파일로 진행 상황을
추적한다. Superpowers는 추적을 넘어서 각 단계의
**진입/퇴장 조건**을 강제한다.

Superpowers를 SDD, File-based Planning과 함께
사용하면 명세 → 프로세스 → 추적이 모두 갖춰진다.

## 설치

Claude Code, Cursor, Codex, OpenCode를 지원한다.
Claude Code의 경우 마켓플레이스에서 설치 가능하다.
