# Superpowers

코딩 에이전트를 위한 구조화된 소프트웨어 개발 워크플로우 프레임워크.
설계 → 계획 → 구현 → 검증의 7단계를 조합 가능한 스킬로 제공한다.

<https://github.com/obra/superpowers>

## 에이전트는 코드부터 짜려 한다

에이전트에게 기능을 요청하면 즉시 구현에 돌입한다.
설계를 건너뛰고, 질문 없이 가정하고, 확인 없이 진행한다.
결과물은 겉보기엔 동작하지만 방향이 틀려 있거나
불필요하게 복잡한 경우가 많다.

Superpowers는 이 문제를 **하드 게이트**로 해결한다.
브레인스토밍 스킬은 설계를 제출하고 사용자 승인을
받기 전까지 어떤 구현 행위도 차단한다.
"간단한 프로젝트"라는 예외도 없다.
에이전트의 충동을 프로세스로 막는 것이다.

## 컨텍스트 오염을 격리로 풀다

하나의 에이전트가 여러 작업을 연속 처리하면
이전 작업의 컨텍스트가 다음 작업에 스며든다.
Superpowers는 작업마다 새 서브에이전트를 생성한다.

서브에이전트는 구현 전에 **질문부터** 한다.
모호한 부분을 먼저 표면화시켜서,
잘못된 가정으로 코드를 작성하는 것을 방지한다.
구현 후에는 스펙 준수 리뷰와 코드 품질 리뷰를
별도로 거친다. 두 리뷰를 모두 통과해야 완료다.

## 실패를 먼저 목격해야 한다

테스트가 먼저 실패하는 것을 관찰해야 한다.
실패를 목격하지 않은 테스트는 올바른 동작을
검증한다고 보장할 수 없다.

가장 강력한 규칙이 있다.
**대응하는 실패 테스트 없이 작성된 프로덕션 코드는
삭제한다.** 예외 없이.
이미 작성한 코드에 대한 매몰 비용 오류를
원천 차단하는 장치다.

## 리뷰를 두 번 하는 이유

코드 리뷰를 한 번에 하면 두 가지가 섞인다.
"요구사항을 충족하는가"와 "구현이 깨끗한가"는
서로 다른 질문이다.

Superpowers는 이를 분리한다.
스펙 준수 리뷰가 먼저, 코드 품질 리뷰가 다음이다.
"동작하지만 더러운 코드"와 "깨끗하지만 스펙에서
벗어난 코드"를 모두 걸러낸다.

## 사람은 3개마다 확인한다

계획의 모든 작업을 한 번에 실행하지 않는다.
3개씩 배치로 실행하고, 배치마다 사람이 확인한다.
블로커, 테스트 실패, 불명확한 지시를 만나면
에이전트는 추측하지 않고 즉시 멈춘다.

자율성과 통제의 균형점이 여기에 있다.
에이전트가 충분히 일하되, 방향이 틀어지기 전에
사람이 개입할 수 있는 간격이다.
