# 거대 코딩 AI 시대의 소프트웨어 제품

원문: Ramin (@ramin_m_h) -
[Software Products in The Age of Big Coding AIs](https://twitter.com/ramin_m_h/status/2021843654709506072)

## 가치 사슬의 역전

코드 작성이 값싸지면 **무엇을 만들지 아는 것** 과 **그것이 작동함을 증명하는
것** 이 희소 가치가 된다. 프로그래머의 역할이 "코드를 짜는 사람"에서 **"제약을
정의하고 결과를 검증하는 사람"** 으로 전환된다.

## 세 가지 역설

**1. 속도의 역설 — 빠를수록 위험하다.** AI가 코드를 100배 빨리 쓰지만, 인간의
리뷰 대역폭은 그대로다. 생성 속도와 검증 속도의 격차에서 기술 부채가 폭발한다.
이것이 BIG SLOP이다. [Vibe Coding](./vibe-coding.md)의 편리함에 취하면 부채 축적
속도도 100배가 된다.

**2. 규모의 역설 — 에이전트를 늘려도 나아지지 않는다.**
[1인간 + 1에이전트](./one-human-one-agent.md)가 수백 개 서브에이전트 스웜을
이긴다. 브룩스의 "맨먼스 미신"이 AI에도 적용된다. 병목은 지능이 아니라 **조정
비용** 이다.

**3. 도구의 역설 — AI 도구가 AI에게 먼저 잡아먹힌다.** 프롬프트 템플릿, 범용
오케스트레이터, 도구 호출 래퍼, 에이전트 실행 UI — 이 "AI 도구" 계층이 가장 먼저
대체된다. 거대 AI가 이런 기능을 내재화하기 때문이다.

## 테스트는 새로운 프로그래밍 언어다

테스트의 역할이 뒤집힌다. 과거에는 코드가 맞는지 **사후 검증** 했다. 지금은
에이전트에게 "맞다"의 의미를 **사전 정의** 한다. "이 테스트가 깨지면 안 된다"는
제약이 에이전트에게 가장 명확한 지침이 된다. 테스트 스위트가 인간의 의도와
기계의 실행을 잇는 **인터페이스** 가 된다.

[SDD](./spec-driven-development.md)의 "명세가 단일 정보원(single source of
truth)"이라는 사상과 정확히 일치한다.

## 조직 문화의 전환

**기술 명세를 실행 가능한 코드로 전환하라.** "응답시간 p95가 100ms 미만이어야
한다"는 문서 대신 벤치마크 하네스와 CI 게이트를 만들어라. 이건 개발 방식의
변화가 아니라 **조직 문화의 근본적 전환** 이다. "문서를 쓰고 회의하는" 조직에서
"코드로 계약하고 CI로 검증하는" 조직으로.

**모든 출력을 기계 파싱 가능하게 구조화하라.** 에이전트는 시간 개념이
없고(time-blind) 컨텍스트가 제한적이다. 빌드/테스트 출력, 로그, 문서를
에이전트가 바로 파싱할 수 있는 형태로 바꿔라.

## 살아남는 제품의 조건

SaaS는 이제 **에이전트**(API), **오퍼레이터**(대시보드), **이해관계자**(리포트)
세 사용자를 동시에 설계해야 한다.

해자(moat)가 있는 영역:

1. **AI 검증 도구** — 에이전트가 코드를 많이 쓸수록 검증 수요가 폭발한다. AI가
   발전할수록 오히려 가치가 커지는 유일한 영역.
2. **레거시 인프라 전환** — PDF, 스프레드시트, 사내 위키에 갇힌 지식을
   에이전트가 파싱할 수 있는 형태로 전환.
3. **물리적 제약이 해자인 영역** — 저지연, 에어갭, 디바이스 AI. 클라우드 API
   호출로는 물리 법칙을 우회할 수 없다.
4. **과학 특화 모델** — 예측 의학, 신약 발견, 재료 과학.
5. **Human-in-the-Loop** — 의사결정이 중요한 곳에서 인간 전문가를 루프에
   유지하는 제품.
