# X-OmniClaw: 엣지 네이티브 멀티모달 Android 에이전트

<https://github.com/OPPO-Mente-Lab/X-OmniClaw>

<https://arxiv.org/abs/2605.05765>

<https://eggplant95.github.io/X-OmniClaw-Page/>

## 소개

X-OmniClaw는 OPPO Mente Lab이 개발한 엣지 네이티브(edge-native) 멀티모달
Android 에이전트다. 가상 환경 없이 물리적 Android 기기 위에서 직접 구동되며,
실시간 시각 텔레메트리를 캡처하고 네이티브 터치 인터랙션을 실행해 앱 간 작업을
수행한다.

“Omni”는 온스크린 UI 상태, 실세계 시각 맥락, 오디오 입력이라는 세 가지 감지
도메인의 통합을 의미하고, “X-”는 교차 모달(cross-modal) 특성을 강조한다. 화면
UI에만 의존하는 기존 접근과 달리, 카메라와 음성까지 하나의 인식-행동 프레임워크로
통합한다.

## 아키텍처

### 핵심 실행 방법론

모든 인터페이스 상호작용을 **관찰(Observation) → 추론(Reasoning) → 실행(Execution)**
루프로 구성한다. 현재 화면과 이전 액션의 결과를 인식하고, 다음 최적 동작을
추론한 뒤, Android 원자적 액션으로 실제 실행한다. 태스크 완료 또는 종료까지
반복된다.

### 4계층 시스템 아키텍처

| 계층      | 컴포넌트             | 역할                                                              |
| --------- | -------------------- | ----------------------------------------------------------------- |
| Perception | Multi-modal Input   | ASR, 스크린샷/녹화 프레임, 접근성 트리 → 통합 컨텍스트           |
| Planning  | Agent Loop           | 메인 에이전트 루프, 태스크 분해, Kotlin 브릿지 디스패치           |
| Execution | Device Scheduler     | 스냅샷, 시뮬레이션 UI 액션, 앱 라이프사이클 관리                 |
| Verification | Success Monitor   | 액션 후 체크 및 루프 감지(드리프트/완료 판단)                     |

### 세 가지 핵심 역량

**Omni Perception**: UI 상태, 실세계 시각 컨텍스트, 음성 입력, 예약 트리거,
플로팅 위젯, 외부 채널을 통합한다. 시간적 정렬(temporal alignment)과
장면 기반 VLM 이해를 사용해 원시 스트림을 구조화된 인텐트로 변환한다.

**Omni Memory**: 태스크 연속성을 위한 워킹 메모리와 로컬 멀티모달 데이터에서
추출된 장기 개인 메모리를 결합한다. 개인화된 멀티턴 상호작용과
메모리 기반 자동화를 가능하게 한다.

**Omni Action**: 하이브리드 UI 증거에 대한 관찰-추론-실행 루프를 실행하고,
사용자 네비게이션을 재사용 가능한 딥링크/인텐트 기반 스킬로 변환한다.

## 주요 기능

- **온디바이스 실행 에이전트 루프**: 멀티턴 태스크에 버지팅과 루프 감지 적용
- **스트리밍 관찰 및 비용 추적**: 단계, 사고, 툴 호출, 결과를 스트림으로 제공하고
  LLM 사용량 누적
- **통합 디바이스 툴**: UI 이해, 탭/타이핑, 앱 실행, 스크린샷, 클립보드를
  단일 인터페이스로 제공
- **비전 폴백 및 이중 트랙 결정**: 구조화 이해 우선, 복잡한 페이지에서는
  비전으로 폴백
- **음성-액션 파이프라인**: ASR + 화면 이해로 PTT 시점 프레임과 결정을 정렬
- **딥링크 및 재현 가능 플로우**: “한 번 기록하면 다음엔 한 문장으로” 패턴
- **병렬 세션 및 제어된 중단**: 세션 격리, 세션 단위 인터럽트

## 분석

X-OmniClaw는 모바일 AI 에이전트 연구의 주요 과제인 “실기기 동작”을 정면으로
다룬다. 대부분의 GUI 에이전트 연구가 에뮬레이터나 웹 환경을 대상으로 하는 데
반해, 이 프로젝트는 실제 Android 기기에서 접근성 서비스(Accessibility Service),
오버레이, 스크린 캡처 권한 등 시스템 레벨 인프라를 활용한다.

아키텍처 측면에서는 Perceive → Plan → Act → Verify의 4단계 폐쇄 루프가
특징적이다. Verification 계층이 명시적으로 독립된 것은 단순 실행 루프에서
흔히 발생하는 “성공으로 잘못 판단하는” 문제를 다루기 위한 설계 선택이다.
루프 감지(loop detection) 기능도 포함되어 에이전트가 무한 루프에 빠지는
상황을 방지한다.

멀티모달 입력 통합도 주목할 만하다. 음성-비전 스파인이라 불리는 구조에서
PTT(Push-to-Talk) 발화 시점의 화면 프레임과 음성 인식 결과를 시간적으로
정렬하여 하나의 결정 인풋으로 만든다. 이는 “지금 이 물건 타오바오에서 얼마야?”
같은 문맥 의존 질의를 가능하게 한다.

스킬 시스템은 “행동 클로닝(behavior cloning)” 아이디어를 실용적으로 구현한다.
사용자가 한 번 수행한 네비게이션 경로를 딥링크/인텐트 기반 스킬로 압축하여,
이후에는 자연어 한 문장으로 재현할 수 있다. `skill-creator` 스킬이 이 과정
자체를 자동화한다는 점도 흥미롭다.

한계도 있다. 7개의 시스템 권한(접근성, 오버레이, 스크린 캡처, 사진, 모든 파일
접근, 카메라, 마이크)을 요구한다는 점은 일반 사용자에게 높은 진입 장벽이다.
또한 현재 데모와 스킬 예시가 중국 앱(타오바오, 샤오홍수, 메이투안, 캡컷) 중심이어서
국제 사용성은 아직 제한적이다.

## 비평

프로젝트의 투명성 문제가 눈에 띈다. README의 Acknowledgments 섹션에서 Operit
커뮤니티로부터 지적을 받았으며 코드 자기검토를 진행했다고 명시되어 있다.
이는 코드 출처와 라이선스 준수에 대한 논란이 있었음을 시사한다. Apache 2.0
라이선스로 배포되지만, 상류(upstream) 프로젝트들과의 관계가 완전히 정리되지
않은 상태로 보인다.

기술 보고서(arXiv:2605.05765)의 저자 소속이 2026년 날짜를 포함하고 있다.
README 업데이트 로그도 2026년 날짜를 사용한다. 이 문서를 작성하는 시점(2026년
5월 22일)을 감안하면 최신 연구지만, 논문의 동료 심사(peer review)를 거치지
않은 기술 보고서 수준임을 감안해야 한다.

에이전트 루프의 신뢰성 문제는 여전히 열린 과제다. 복잡한 앱 환경에서 접근성
트리가 불완전하거나 동적으로 변하는 경우, 비전 폴백이 얼마나 안정적으로
작동하는지에 대한 정량적 평가가 README에는 없다. 병렬 세션 지원은 인상적이지만,
동시에 여러 세션이 기기 상태를 조작할 때 충돌을 어떻게 처리하는지는 불명확하다.

## 인사이트

### 에이전트의 “기억”이 모바일 자동화의 핵심 차별화 요소가 된다

기존 모바일 자동화 도구(Tasker, Shortcuts 등)는 명시적으로 프로그래밍된
트리거-액션 쌍으로 작동한다. 사용자가 정확한 조건과 단계를 사전에 정의해야
한다. X-OmniClaw의 Omni Memory 구조는 이 패러다임을 근본적으로 바꾼다.

워킹 메모리는 현재 태스크의 컨텍스트를 유지하여 멀티턴 대화와 실행을 가능하게
한다. 예를 들어 “아까 그 사진 찾아서 편집해줘”처럼 이전 인터랙션을 참조하는
요청을 처리할 수 있다. 장기 개인 메모리는 사용자의 사진, 연락처, 앱 사용 패턴을
인덱싱하여 “내 사진 중 파리 여행 것들”이라는 모호한 쿼리에도 응답할 수 있게 한다.

이 메모리 시스템이 성숙해지면 AI 에이전트의 경쟁력은 “얼마나 스마트한 LLM을
쓰는가”보다 “얼마나 개인화된 맥락을 보유하는가”로 이동할 수 있다. 기기에
장기간 축적된 개인 데이터가 온디바이스 에이전트의 핵심 자산이 되는 것이다.
이는 클라우드 기반 AI 서비스가 가질 수 없는 구조적 우위다.

### 스킬 시스템과 행동 클로닝이 만드는 새로운 자동화 레이어

`skill-creator` 스킬은 단순한 편의 기능이 아니라 에이전트 자체가 자신의
역량을 확장하는 메타-러닝(meta-learning) 메커니즘이다. 사용자가 에이전트와
함께 복잡한 작업을 한 번 수행하면, 에이전트가 그 과정을 `SKILL.md` 파일로
추상화하여 저장한다. 이후에는 자연어 한 문장으로 동일한 작업을 재현한다.

이 패턴은 전통적인 RPA(Robotic Process Automation)의 “레코드 앤 플레이” 개념과
유사하지만 결정적인 차이가 있다. RPA는 정확한 좌표와 엘리먼트 ID를 기록하여
앱 업데이트 시 쉽게 깨진다. X-OmniClaw의 스킬은 딥링크와 인텐트 기반이며,
에이전트가 화면 상태를 이해하여 동적으로 경로를 조정한다.

앱 간 자동화의 미래는 이처럼 “사용자가 한 번 보여주면 에이전트가 추상화하는”
방향으로 수렴할 가능성이 높다. 전문 개발자가 없어도 사용자 자신이 개인화된
자동화 스킬을 생성하고 공유하는 에코시스템이 형성될 수 있다. `skills/`
디렉토리가 일종의 사용자 정의 플러그인 저장소 역할을 한다는 점에서 이미
그 방향성을 엿볼 수 있다.

### 온디바이스 에이전트의 권한 구조가 새로운 신뢰 모델을 요구한다

X-OmniClaw는 7가지 고위험 권한을 요구한다. 접근성 서비스는 화면의 모든 텍스트를
읽을 수 있고, 오버레이는 다른 앱 위에 UI를 그릴 수 있으며, 모든 파일 접근은
기기의 전체 저장소에 접근할 수 있다. 이 권한들의 조합은 이론상 패스워드, 금융
정보, 개인 메시지 등 모든 민감한 데이터에 접근 가능함을 의미한다.

이 권한 묶음은 에이전트가 실제로 유용한 작업을 수행하기 위해 불가피하다는
역설이 있다. 타오바오에서 가격을 검색하고 결과를 갤러리 사진과 연관 짓는
작업을 하려면 화면 읽기와 파일 접근이 모두 필요하다. 그러나 동시에 이는
악의적이거나 버그가 있는 에이전트가 초래할 수 있는 피해의 범위도 극대화한다.

기존의 앱 권한 모델은 “이 앱이 카메라에 접근할 수 있습니까?”라는 정적 질문에
답하도록 설계되어 있다. 에이전트 시대에는 “이 에이전트가 지금 이 순간 카메라를
사용하는 행동이 내가 요청한 태스크에 합당한가?”라는 동적 질문이 필요하다.
Android 생태계와 규제 기관 모두 이 새로운 신뢰 모델에 대한 준비가 아직
충분하지 않은 상태다.
