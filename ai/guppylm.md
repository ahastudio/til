# GuppyLM: 900만 파라미터로 배우는 언어 모델의 작동 원리

<https://github.com/arman-bd/guppylm>

## 요약

GuppyLM은 약 870만 파라미터의 초소형 언어 모델로, 교육 목적으로 설계된
오픈소스 프로젝트다. 저자의 핵심 메시지는 “언어 모델 훈련은 마법이 아니다”이며,
Colab 노트북 하나로 5분 안에 직접 훈련할 수 있도록 만들었다.

모델 아키텍처는 의도적으로 단순하다. 6개의 트랜스포머 블록, 히든 차원 384,
어텐션 헤드 6개, FFN 차원 768, 어휘 크기 4,096 BPE 토큰, 컨텍스트 창 128토큰.
GQA, RoPE, SwiGLU 같은 현대적 최적화 기법은 일부러 제외했다. 핵심 원리를
명확히 보여주는 것이 목적이기 때문이다.

훈련 데이터는 6만 개의 합성 대화다. 물고기 캐릭터 'Guppy' 관점에서 생성된
60개 주제(수조 환경, 먹이, 감각 경험, 철학적 질문)로 구성된 JSON 형식이다.
템플릿 조합(수조 오브젝트 30종, 먹이 유형 17종, 활동 25종)으로 생성했다.

배포 방식도 다양하다. WebAssembly로 브라우저에서 직접 실행 가능한 10MB
ONNX 모델, Google Colab 노트북, 로컬 PyTorch 설치 모두 지원한다.
MIT 라이선스 오픈소스다.

## 분석

### 교육적 설계 원칙

GuppyLM의 가장 중요한 설계 결정은 단순성 유지다. 현대 LLM은 GQA, Flash
Attention, RoPE, SwiGLU, MoE 등 수십 가지 기법이 겹겹이 쌓여 있어 초보자가
“왜 이렇게 작동하는가”를 이해하기 극히 어렵다. GuppyLM은 2017년 “Attention Is
All You Need” 논문의 바닐라 트랜스포머를 거의 그대로 구현해 이 진입 장벽을
제거했다.

물고기 캐릭터 선택도 교육적으로 영리하다. 컨텍스트 창 128토큰의 자연스러운
망각이 물고기의 “3초 기억력” 특성과 연결되고, 소문자·짧은 문장만 쓰는 제약이
모델의 어휘 처리 방식을 학습자가 직접 관찰할 수 있게 한다.

### 멀티턴 포기의 의미

저자는 멀티턴 대화를 시도했으나 3-4 교환 후 품질이 급격히 저하되어 싱글턴
방식으로 결정했다. 이것은 버그가 아니라 작은 모델의 근본적 한계다. 컨텍스트
유지를 위한 어텐션 패턴이 128토큰 창 안에서 이전 대화를 추적하기에는 파라미터가
절대적으로 부족하다.

### GeekNews 커뮤니티 반응

댓글에서 Karpathy의 microgpt/minGPT와 비교 논의가 이루어졌다. 차이점은
GuppyLM이 완성된 캐릭터 경험(물고기 페르소나)까지 제공한다는 것이다.
bbycroft.net의 LLM 3D 시각화를 함께 활용하면 학습 효과가 높다는 제안도 있었다.
“Minix처럼 학생들이 기능을 추가하며 배우는 도구”로 활용할 수 있다는 평가도 인상적이다.

## 비평

### 강점

직접 실행 가능한 브라우저 데모와 Colab 노트북의 조합은 학습 진입 장벽을 최소화한다.
“5분 안에 훈련”이라는 약속을 지키기 위한 합성 데이터 설계가 실용적이다.
모델 성격을 시스템 프롬프트가 아닌 가중치에 직접 임베드한 것은 파인튜닝과 프롬프팅의
차이를 직관적으로 보여주는 좋은 교육 사례다.

### 약점

소문자 훈련 데이터로 인한 대문자 토큰 처리 이슈가 발견되었다(커뮤니티 보고). 이는
토크나이저와 훈련 데이터 분포의 관계를 설명하는 좋은 사례지만, 문서화가 필요하다.

4,096개의 BPE 어휘는 현대 모델(32K~128K)에 비해 극단적으로 작다. 이 선택이
교육 목적으로 정당하지만, 실제 텍스트 처리에서 어떤 영향을 미치는지 명시적인
설명이 없다.

## 인사이트

### 이해 가능한 AI의 역설

GuppyLM이 보여주는 역설이 있다. 이 모델은 완전히 이해 가능하지만, 동시에 완전히
쓸모가 없다. 870만 파라미터의 모델이 할 수 있는 것은 물고기 말투로 짧은 대화를
나누는 것뿐이다. 반면 GPT-4나 Claude 같은 대형 모델은 거의 모든 것을 할 수 있지만,
왜 그렇게 작동하는지 아무도 완전히 이해하지 못한다.

이 역설은 AI 교육의 근본 문제를 드러낸다. 우리가 가르칠 수 있는 것(작은 모델)과
실제로 사용하는 것(큰 모델) 사이의 간극이 너무 크다. 의대생이 해부학을 배울 때
실제 인간을 해부하듯, AI 학습자도 실제 규모의 모델을 해부할 수 있어야 한다. 하지만
GPT-4의 “해부”는 수억 달러의 컴퓨팅 자원을 필요로 한다. GuppyLM은 이 간극에 대한
최선의 현실적 대안이다.

### 합성 데이터의 교육적 투명성

GuppyLM의 훈련 데이터가 완전히 합성 데이터라는 점은 교육 도구로서 독특한 강점이다.
실제 LLM 훈련에서 데이터의 출처와 구성은 영업 비밀에 가깝다. 어떤 데이터가
모델의 어떤 행동을 만드는지 추적하기 어렵다. GuppyLM은 이 과정을 완전히 투명하게
공개한다. “물고기는 왜 항상 먹이를 언급하는가?” → “훈련 데이터에 17종의 먹이 유형이
있고 60개 주제 카테고리 중 상당수가 먹이와 연결되어 있기 때문”이라고 직접 확인할 수 있다.

이것은 LLM의 “행동 = 데이터 분포의 반영”이라는 원리를 가장 직관적으로 이해할 수 있는
방법이다. 대형 모델에서 관찰되는 편향, 스테레오타입, 특이한 선호의 근원이 바로
훈련 데이터라는 것을 직접 실험으로 검증할 수 있다.

### 파라미터 스케일링과 능력의 비선형성

870만 파라미터로 할 수 있는 것과 없는 것의 경계가 명확하다. 기본 패턴 매칭과
단문 생성은 가능하지만, 멀티턴 맥락 추적, 추론, 지식 검색은 불가능하다. 이 경계가
GPT-3(1750억 파라미터)에서는 어떻게 달라지는가? 단순 비례가 아니다. 20배의 파라미터가
20배 좋은 성능을 만드는 것이 아니라, 특정 임계점을 넘으면 질적으로 다른 능력이 등장한다.

이것이 스케일링 법칙(Scaling Law)의 핵심이다. GuppyLM은 스케일링 법칙의 하한을
직접 체험하게 해준다. “왜 870만 파라미터로는 추론이 안 되는가?”를 이해하면,
“왜 1000억 파라미터에서는 추론이 되는가?”에 대한 직관이 생긴다. 이 직관은 API 문서를
읽는 것으로는 절대 얻을 수 없다.
