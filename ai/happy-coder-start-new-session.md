# Happy Coder - Start New Session

<https://happy.engineering/docs/features/>

<https://happy.engineering/docs/guides/happy-coder-best-practices/>

Happy Coder v1.5.0(2025-09-18)에서 추가된 기능. 모바일 기기에서 원격으로
새로운 Claude Code 또는 Codex 세션을 직접 생성하고 시작할 수 있다.

## 이전 버전과의 차이

| 항목   | 이전            | 이후                |
| ------ | --------------- | ------------------- |
| 시작   | 데스크톱에서만  | 모바일에서도 가능   |
| 모바일 | 모니터링만      | 생성 + 모니터링     |
| 지원   | Claude Code     | Claude Code + Codex |
| 흐름   | 단방향 핸드오프 | 양방향 전환         |

## 사전 준비

원격 세션 시작 기능을 사용하려면 `happy connect`로 벤더 인증을 미리
완료해야 한다. API 키가 Happy 클라우드에 암호화되어 저장되며, 원격
세션 생성 시 이 키를 사용해 에이전트를 실행한다.

```bash
# Happy 계정 로그인
happy auth login

# 벤더 인증
happy connect claude
happy connect codex

# 연결 상태 확인
happy connect status
```

## 작동 방식

1. 모바일 앱에서 "Start New Session" 버튼을 탭한다.
2. Claude Code 또는 Codex 중 에이전트를 선택한다.
3. 작업 디렉토리와 초기 프롬프트를 입력한다.
4. Happy Agent가 데스크톱 머신에서 에이전트 프로세스를 원격 기동한다.
5. WebSocket을 통해 모바일과 실시간 양방향 통신이 시작된다.

핵심은 샌드박스나 별도 서비스에 접속하는 것이 아니라, 실제 머신에서
실행되는 실제 Claude Code 세션에 직접 연결된다는 점이다. 로컬
코드베이스에 대한 완전한 접근 권한을 가진다.

## 디바이스 전환

세션이 시작되면 양쪽 디바이스에서 동일한 세션에 접근할 수 있다.
"주(primary)" 디바이스와 "보조(secondary)" 디바이스의 구분이 없다.

- 모바일에서 시작한 세션을 데스크톱 키보드의 아무 키나 눌러 즉시
  데스크톱으로 전환할 수 있다.
- 데스크톱에서 벗어나면 모바일에서 이어서 작업한다.
- 터미널이 오프라인일 때도 대화 기록에 접근할 수 있다.

## 병렬 세션 실행

여러 AI 에이전트를 동시에 실행할 수 있다. 한 세션에서 프론트엔드를
리팩토링하는 동안 다른 세션에서 백엔드 테스트를 작성하고, 모바일에서
세션 간을 즉시 전환할 수 있다.

```bash
# 머신 A: 프론트엔드 작업
happy

# 머신 B: 백엔드 작업
happy

# 같은 머신에서 병렬 실행도 가능
# 별도 터미널에서 각각 happy 실행
```

## 베스트 프랙티스

### 모든 디바이스 연결

가장 큰 워크플로우 개선은 모든 디바이스와 브라우저를 같은 계정에
연결하는 것이다. 주력 노트북뿐 아니라 코드를 실행할 수 있는 모든
컴퓨터를 연결한다. 컴퓨트를 많이 연결할수록 활용도가 높아진다.

### 아이디어에서 구현까지의 마찰 제거

"아이디어 → 구현"의 경로를 메시지 확인만큼 간단하게 만드는 것이
목표다. 산책 중에 더 나은 변수명이 떠오르면 폰을 꺼내서 바로
적용한다. 작은 변경이라도 반복해서 근육 기억(muscle memory)을
만든다.

### 기존 워크플로우의 확장

Happy Coder는 기존 워크플로우를 대체하는 것이 아니라 확장한다.
터미널에서 `claude`나 `codex`를 직접 사용하는 것도 여전히 가능하다.
모바일은 의무가 아니라 선택지를 제공한다. 책상에 있을 때는
터미널에서, 자리를 비울 때는 폰에서. 세션은 멈춘 곳에서 정확히
이어진다.

### Git Worktree 활용

Happy Coder가 Git Worktree를 직접 관리하지는 않지만, 약간의 설정으로
실험을 원활하게 만들 수 있다. 병렬 세션마다 별도 worktree를
할당하면 서로 간섭 없이 독립적으로 작업할 수 있다.

### 모바일은 모니터링 중심으로

모바일에서 복잡한 코드를 직접 편집하는 것은 비현실적이다.
모바일의 역할은 다음에 집중하는 것이 효과적이다:

- 장시간 작업의 진행 상황 확인
- 권한 요청 승인/거부
- 간단한 프롬프트로 새 작업 지시
- PR 리뷰 시작

## 인사이트

### 컴퓨팅의 편재화

"Start New Session"은 단순한 원격 시작 버튼이 아니다. AI 코딩
에이전트를 데스크톱이라는 물리적 제약에서 해방시키는 전환점이다.
커피를 마시다가, 출퇴근 중에, 소파에 앉아서 코딩 세션을 시작할 수
있다는 것은 개발자의 "작업 가능 시간"을 근본적으로 재정의한다.

### 감독 패러다임의 변화

이전 모델은 "데스크톱에서 시작 → 모바일로 감독"이었다. 이제는
"어디서든 시작 → 어디서든 감독 → 어디서든 전환"이 된다. 시작
지점에 대한 제약이 사라지면서, 에이전트 감독이 장소에 독립적인
활동이 된다.

### 비동기 개발의 현실화

아이디어가 떠오르는 순간과 그것을 구현할 수 있는 순간 사이의 간극이
줄어든다. 이 간극은 단순한 불편이 아니라, 아이디어가 사라지거나
동기가 감소하는 실질적 비용이다. 모바일에서 바로 세션을 시작해
에이전트에게 작업을 맡기면, "나중에 해야지"가 "지금 시작"으로
바뀐다.

### 멀티 에이전트 오케스트레이션

병렬 세션 실행과 결합하면 개발자 한 명이 여러 에이전트를 동시에
운용하는 "오케스트레이터" 역할을 하게 된다. 이는 개인 개발자가
소규모 팀의 생산성을 낼 수 있는 가능성을 열어준다. 다만 여러
에이전트의 작업이 서로 충돌하지 않도록 Git Worktree 같은 격리
전략이 필수적이다.
