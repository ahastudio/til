# 법망 — PostgreSQL 기반 한국 법령 JSON API

> 원문: <https://api.beopmang.org>

## 요약

법망(Beopmang)은 대한민국 법령, 조문, 판례, 의안을 JSON으로 제공하는 무료 REST API다.
AI 에이전트가 한국 법률 정보를 쉽게 소비할 수 있도록 설계되었으며,
회원가입, 인증키, HWP 파싱 없이 바로 사용할 수 있다.

주요 특징:

- 법제처 데이터 기반, 매주 토요일 동기화, 커버리지 99.9% 이상
- PostgreSQL + pgvector로 20만 조문 임베딩 (768차원 벡터)
- 키워드, 시맨틱, 하이브리드 검색 지원
- MCP 엔드포인트(`/mcp`) 제공
- 분당 100건 제한, 500 동시접속 스트레스 테스트 통과
- IP, 검색어 등 개인정보 일체 수집하지 않음

## 분석

법률 정보 접근의 핵심 병목은 데이터 포맷이었다.
국가법령정보센터의 XML/HWP/PDF를 에이전트가 직접 파싱하려면
토큰 소모가 막대하고 테이블 구조가 손실된다.
법망은 이 전처리를 서버 측에서 해결하여
공식 API 대비 토큰 사용량을 1/10로 줄였다고 주장한다.

API 설계는 `law.search → law.get → law.diff → law.history`라는
4단계 워크플로우로 정리되어 있어,
에이전트가 탐색적으로 법령을 좁혀가는 패턴에 최적화되어 있다.

pgvector를 활용한 시맨틱 검색은 법률 용어의 동의어 문제를 완화한다.
"해고"를 검색할 때 "면직", "파면", "해임"까지 잡아내는 것이 가능하다.

MCP 지원은 Claude, ChatGPT 등 에이전트 플랫폼과의 통합을 한 단계 쉽게 만든다.
다만 Gemini는 아직 미지원이라고 명시하고 있다.

GeekNews 댓글에서 ninebow는 자신의 프로젝트(legalize-kr)로
5,575개 법률의 81,542개 개정 이력을 추적하고 있다고 밝혔다.
법률 데이터 인프라에 대한 커뮤니티의 수요가 분산적으로 표출되고 있다.

## 비평

법대생(miceonvenus)이 개인 프로젝트로 운영하면서
"무료, 수익화 계획 없음"을 선언한 점은 인상적이지만 동시에 리스크다.
임베딩 인프라 비용을 어떻게 지속할 것인지에 대한 의문은
GeekNews 댓글(qyahzn2004)에서도 제기되었다.

공개 24시간 만에 2만 건 이상의 API 호출이 발생했다는 것은
수요가 충분하다는 증거지만, 트래픽이 증가하면 분당 100건 제한과
단일 운영자 체제가 병목이 될 수 있다.

법적 면책 조항("참고용, 법적 효력 없음")은 필수적이나,
에이전트가 이 API를 사용해 법률 조언을 생성할 경우
최종 사용자가 면책 조항을 인지하지 못할 위험이 있다.

## 인사이트

- **전처리가 곧 인프라다.**
  LLM 시대에 "데이터를 제공한다"는 것의 의미가 바뀌고 있다.
  원본 데이터를 그대로 노출하는 것은 더 이상 서비스가 아니다.
  토큰 효율적 포맷으로 전처리하는 것 자체가 핵심 가치가 되었다.
  법망이 공식 API 대비 토큰 1/10을 달성했다는 것은
  같은 정보라도 "에이전트가 소비하기 좋은 형태"로 가공하는 것이
  별도의 인프라 계층이 되었음을 보여준다.

- **수직 도메인 MCP의 출현.**
  Korean Law MCP에 이어 법망까지,
  한국 법률이라는 좁은 도메인에서 두 개의 독립 프로젝트가 거의 동시에 등장했다.
  이는 MCP가 범용 도구가 아니라
  도메인 특화 데이터 브릿지로서 더 빠르게 확산되고 있음을 시사한다.
  의료, 세무, 부동산 등 규제가 복잡한 영역에서 유사한 프로젝트가 연쇄적으로 나올 것이다.

- **법률 정보의 민주화와 그 역설.**
  법률 정보가 API 하나로 접근 가능해지면
  "법을 모르는 것이 면책 사유가 될 수 없다"는 법언이 실질적 의미를 갖게 된다.
  동시에, 에이전트가 법률을 해석하기 시작하면
  "에이전트의 법률 해석이 틀렸을 때 누가 책임지는가"라는
  새로운 법적 공백이 생긴다.
  법률 정보 접근성이 높아질수록 법률 리터러시의 중요성은 오히려 증가한다.
