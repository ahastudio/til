# Vultr Object Storage

<https://www.vultr.com/products/object-storage/>

## 소개

Vultr Object Storage는 S3 API 호환 인터페이스를 통해 제공되는 온디맨드
오브젝트 스토리지 서비스다.
물리 인프라 없이 용량을 동적으로 조정할 수 있으며,
14개 리전·33개 클라우드 데이터센터에서 제공된다.
데이터 내구성은 11 나인(99.999999999%)을 보장하며,
미디어 에셋 관리, 백업, CDN 오리진, 로그 아카이빙 등 다양한 용도에 쓰인다.

## 명세

### 스토리지 티어

5개 성능 등급을 제공하며 용도에 따라 선택할 수 있다.

| 티어        | 월 요금       | 주요 용도                        |
| ----------- | ------------- | -------------------------------- |
| Standard    | $18/TB        | 대용량 저비용 스토리지           |
| Premium     | $36/TB        | 일반 애플리케이션 데이터         |
| Performance | $50/TB        | 고성능 인터넷 워크로드           |
| Accelerated | $100/TB       | 쓰기 집중적인 고부하 애플리케이션 |
| Archival    | $6/TB         | 비정기 접근 데이터 장기 보관     |

이그레스(egress) 대역폭은 초기 할당량 초과분에 대해 $0.01/TB/월이 적용된다.
API 요청에 대한 추가 비용은 없다.

### 주요 기능

- **내구성**: 다중 디바이스 이중화로 11 나인(99.999999999%) 보장
- **확장성**: 물리 인프라 관리 없이 오브젝트 추가·삭제로 용량 조정
- **S3 API 호환**: 기존 S3 호환 애플리케이션 및 서드파티 도구와 즉시 연동
- **글로벌 리전**: 14개 리전, 33개 데이터센터

### 주요 사용 사례

- 이미지·오디오·영상 미디어 에셋 관리
- cPanel, WordPress 호환 자동 백업
- CDN 오리진 및 재해 복구
- 클라우드 네이티브 애플리케이션 데이터 및 로그 아카이빙

## 분석

### 5단계 티어 구조가 드러내는 스토리지 시장 세분화

Vultr의 5단계 티어($6~$100/TB)는 단순한 가격 나열이 아니라 시장 세분화 전략이다.
Archival($6/TB)은 AWS S3 Glacier와 경쟁하고,
Standard/Premium은 일반 애플리케이션 데이터를 노리며,
Accelerated($100/TB)는 쓰기 집중 워크로드라는 틈새를 공략한다.
하나의 제품으로 서로 다른 구매 결정자(비용 최적화 팀, 개발자, 인프라 엔지니어)를
동시에 대상으로 삼는 구조다.

### S3 API 호환성이 핵심 전략인 이유

“S3 API 호환”은 기능 설명이 아니라 락인(lock-in) 해소 전략이다.
AWS S3를 쓰는 기존 코드베이스는 엔드포인트 URL만 바꾸면 Vultr로 전환할 수 있다.
마이그레이션 비용이 낮다는 것은 고객이 Vultr를 시도할 때의 진입 장벽이 낮고,
동시에 경쟁사로 다시 돌아가는 것도 쉽다는 의미다.
S3 API 호환은 양날의 검이다.

### 이그레스 요금 구조의 의미

“API 요청 무료, 이그레스 $0.01/TB”라는 가격 구조는 중요하다.
AWS S3와 달리 많은 대안 스토리지 제공자(Cloudflare R2, Backblaze B2)는
이그레스를 무료로 제공하거나 매우 저렴하게 책정한다.
Vultr가 이그레스 비용을 유지한다는 것은 CDN 없이 직접 트래픽을 처리하는
워크로드에서 경쟁력이 제한됨을 의미한다.

## 비평

### 강점: 직관적인 티어 구성과 투명한 가격

5개 티어를 용도 기준으로 설명하고 가격을 명확히 제시한다.
“엔터프라이즈 가격은 문의”가 아니라 실제 금액을 나열한 것은 신뢰를 준다.
개발자와 스타트업이 예산 계획을 세우기 쉬운 구조다.

### 약점: 티어 간 기술적 차별화 미설명

Standard와 Premium의 기술적 차이, Accelerated가 쓰기 집중 워크로드에서
왜 유리한지에 대한 설명이 없다.
디스크 유형(HDD vs NVMe SSD), 복제 방식, IOPS 수치 등의 스펙이 빠져 있어
가격 차이의 근거를 판단하기 어렵다.
“고성능 워크로드”라는 표현만으로는 구체적인 선택 기준이 되지 않는다.

### 약점: SLA 및 가용성 보장 조건 미제시

11 나인 내구성을 주장하지만, 이것이 어떤 조건하의 수치인지 설명이 없다.
단일 리전인지 멀티 리전인지, 장애 발생 시 복구 목표(RTO/RPO)는 무엇인지,
SLA 위반 시 크레딧 정책은 어떻게 되는지가 빠져 있다.
내구성 수치는 마케팅 문구로 쓰일 때 구체적인 조건이 없으면 검증이 불가능하다.

### 약점: Cloudflare R2, Backblaze B2와의 비교 부재

현재 오브젝트 스토리지 시장에서 Cloudflare R2(이그레스 무료)와
Backblaze B2(저가 정책)는 강력한 대안이다.
제품 페이지 특성상 경쟁사 비교를 하지 않는 것은 자연스럽지만,
구매를 고려하는 사람이 이 페이지에서 얻을 수 있는 정보는 제한적이다.

## 인사이트

### S3 API 표준화가 만든 스토리지 시장의 구조 변화

AWS가 S3를 출시한 2006년 이후, S3 API는 오브젝트 스토리지의 사실상 표준이
됐다.
Vultr, Cloudflare R2, Backblaze B2, MinIO 모두 S3 API를 지원한다.
이것은 AWS가 의도하지 않았을 결과다.
자사 서비스의 API를 공개 표준처럼 확산시키면 생태계를 지배할 수 있다고
생각했겠지만, 동시에 경쟁자들이 그 표준 위에서 더 저렴하게 구현할 수 있는
기반도 제공한 셈이다.

이 구조는 소프트웨어 역사에서 반복되는 패턴과 다르다.
일반적으로 표준을 만든 기업이 가장 큰 이익을 얻는다.
그러나 클라우드 스토리지에서는 인프라 비용이 결정적이기 때문에,
AWS보다 저렴한 데이터센터 구조를 가진 후발 주자가 같은 API로 더 낮은 가격을
제시할 수 있다.
S3 API 호환성은 AWS의 경쟁 우위를 줄이는 무기가 됐다.

장기적으로 이 추세는 오브젝트 스토리지를 범용 상품(commodity)으로 만든다.
가격과 지역 커버리지, 이그레스 정책이 선택 기준이 되고,
기능 차별화는 점점 어려워진다.
Vultr의 5단계 티어는 이 상품화 압력 속에서 세분화로 살아남으려는 시도다.

### 이그레스 요금은 클라우드의 “탈출 세금”이다

클라우드 스토리지 경쟁에서 가장 중요한 변수 중 하나는 이그레스 비용이다.
AWS S3는 데이터를 인터넷으로 내보낼 때 GB당 요금을 부과한다.
이 구조는 고객이 데이터를 클라우드에 올리면 꺼내기 어렵게 만드는 장치다.
Cloudflare R2가 이그레스 무료를 내세운 것은 이 장치를 정면으로 공격한 전략이다.

Vultr가 $0.01/TB의 이그레스 요금을 유지하는 것은 AWS보다 훨씬 저렴하지만,
무료는 아니다.
대용량 트래픽이 발생하는 CDN 오리진 용도로는 이그레스 무료 경쟁자와 비교할 때
불리해진다.
반면 내부 데이터 이동이 많고 외부 트래픽이 적은 백업·아카이빙 용도에서는
이 비용이 큰 문제가 되지 않는다.

이그레스 요금 정책은 단순한 가격 항목이 아니라,
서비스가 어떤 워크로드를 주요 고객으로 보는지를 드러내는 신호다.
Vultr의 이그레스 정책은 “CDN 트래픽보다는 데이터 보관”에 더 적합한
서비스임을 암시한다.

### 멀티 티어 스토리지 전략이 요구하는 아키텍처 판단력

5단계 티어를 제대로 활용하려면 워크로드별 접근 패턴을 정확히 이해해야 한다.
“자주 읽는 데이터는 Premium, 거의 안 읽는 데이터는 Archival”이라는 단순한
분류는 쉽지만, 실제로는 더 복잡하다.
접근 빈도뿐 아니라 레이턴시 요구사항, 읽기/쓰기 비율, 오브젝트 크기 분포,
라이프사이클 정책까지 고려해야 올바른 티어를 선택할 수 있다.

많은 팀이 스토리지 티어를 처음 설계할 때 과도하게 고성능 티어를 선택한다.
이는 두 가지 이유 때문이다.
첫째, 접근 패턴 데이터가 없어서 최악의 경우를 기준으로 고른다.
둘째, 나중에 티어를 낮추는 과정(데이터 마이그레이션)이 번거롭다.
결과적으로 Archival($6/TB)에 보관해도 되는 데이터가 Premium($36/TB)에 저장된다.

이 문제를 해결하는 현대적인 접근은 스토리지 라이프사이클 정책이다.
데이터가 일정 기간 접근되지 않으면 자동으로 더 저렴한 티어로 이동하도록 설정한다.
Vultr가 이 라이프사이클 자동화를 얼마나 지원하는지는 이 페이지에서 확인할 수 없는데,
이 기능의 유무가 실제 운영 비용에 큰 차이를 만든다.
