# 시크릿 관리 도구 비교

## 왜 필요한가

API 키, DB 비밀번호, 토큰 같은 시크릿을 `.env` 파일이나 코드에 그대로
커밋하면 유출 위험이 크다. 시크릿 관리 도구는 이런 값들을 중앙에서
암호화해 저장하고, 접근 권한을 통제하며, 필요할 때만 애플리케이션이나
CI/CD 파이프라인에 주입하는 역할을 한다.

핵심 관심사는 크게 세 가지다.

- **저장(storage)**: 시크릿을 어디에, 어떻게 암호화해서 보관하는가
- **접근 제어(access control)**: 누가/무엇이 어떤 시크릿을 읽을 수 있는가
- **배포(distribution)**: 애플리케이션 실행 시점에 시크릿을 어떻게 주입하는가

## AWS Secrets Manager

AWS가 제공하는 완전관리형 시크릿 저장소.

- **형태**: SaaS (AWS 관리형)
- **강점**: RDS/Redshift 등 DB 자격증명 자동 로테이션을 기본 지원. IAM과
  긴밀하게 통합되어 있어 AWS 리소스 접근 제어를 그대로 재사용할 수 있음.
  Lambda, ECS 등 AWS 서비스와의 통합이 쉬움.
- **약점**: 시크릿당 과금(개당 월 $0.40 + API 호출당 과금)이라 시크릿
  개수가 많아지면 비용이 늘어남. AWS 생태계에 강하게 종속됨.
- **적합한 경우**: 이미 AWS 위주로 인프라를 구성한 팀, DB 자격증명 자동
  로테이션이 필요한 경우.

## HashiCorp Vault

가장 오래되고 기능이 풍부한 오픈소스 시크릿 관리 플랫폼.

- **형태**: 셀프호스팅 오픈소스 (HashiCorp Cloud Platform으로 매니지드
  버전도 제공)
- **강점**: 정적 시크릿뿐 아니라 동적 시크릿(요청 시점에 단기 자격증명을
  발급하고 자동 만료)까지 지원하는 것이 가장 큰 차별점. PKI 인증서 발급,
  암호화 서비스(Transit), Kubernetes/AWS/GCP 등 폭넓은 인증 백엔드를
  지원. 세밀한 정책 기반 접근 제어(ACL).
- **약점**: 학습 곡선이 가파르고 직접 운영 시 고가용성 클러스터 구성,
  unsealing, 백업 등 운영 부담이 크다. 2023년 라이선스를 BSL(Business
  Source License)로 변경하면서 상용 서비스 제공에 제약이 생겼고, 이
  결정에 반발해 OpenBao라는 오픈소스 포크가 만들어짐.
- **적합한 경우**: 대규모 조직, 동적 시크릿·PKI 등 고급 기능이 필요한
  경우, 멀티클라우드/온프레미스를 아우르는 통합 시크릿 계층이 필요한 경우.

## Infisical

개발자 경험(DX)에 초점을 맞춘 오픈소스 시크릿 관리 도구.

- **형태**: 오픈소스 (셀프호스팅 또는 클라우드 SaaS)
- **강점**: 웹 대시보드와 CLI가 직관적이고, `.env` 파일 기반 워크플로우와
  자연스럽게 연결됨. Git과 유사한 시크릿 버전 관리, 브랜치별 환경 분리,
  PR 형태의 시크릿 변경 검토(secret change requests) 등 개발팀 협업에
  맞춘 기능이 많음. 무료 티어가 넉넉해 소규모 팀이 부담 없이 시작하기 좋음.
- **약점**: Vault만큼 오래되지 않아 엔터프라이즈 레퍼런스나 생태계 통합이
  상대적으로 적음. 동적 시크릿 등 고급 기능은 Vault 대비 제한적.
- **적합한 경우**: 스타트업·중소 규모 팀, 개발자 친화적인 UI/워크플로우를
  중시하는 경우.

## Doppler

시크릿을 "동기화"하는 데 초점을 맞춘 SaaS.

- **형태**: SaaS (셀프호스팅 옵션 없음)
- **강점**: 하나의 중앙 저장소에서 여러 클라우드(AWS, GCP, Vercel,
  Heroku 등)와 CI 도구로 시크릿을 자동 동기화. 설정이 매우 간단하고
  CLI 하나로 로컬 개발 환경에도 바로 주입 가능(`doppler run -- <command>`).
  여러 환경/프로젝트를 다루는 팀에 우호적인 UI.
- **약점**: 셀프호스팅 불가 — 반드시 Doppler의 클라우드를 신뢰해야 함.
  Vault처럼 동적 시크릿이나 PKI 같은 고급 보안 기능은 없음.
- **적합한 경우**: 여러 클라우드·플랫폼에 흩어진 배포 파이프라인을
  하나의 시크릿 소스로 동기화하고 싶은 팀.

## 선택 기준 요약

| 기준 | 추천 |
| --- | --- |
| 이미 AWS 중심 인프라 | AWS Secrets Manager |
| 동적 시크릿, PKI, 대규모 엔터프라이즈 | HashiCorp Vault (또는 OpenBao) |
| 개발자 경험, 소규모 팀, 빠른 시작 | Infisical |
| 멀티 플랫폼 동기화, 셀프호스팅 불필요 | Doppler |

공통적으로 주의할 점은, 어떤 도구를 쓰든 시크릿을 코드/이미지에 굽지
않고 런타임에 주입하는 원칙, 접근 로그를 남기는 감사(audit) 기능,
그리고 유출 시 신속한 로테이션 절차를 갖추는 것이 도구 선택 자체보다
중요하다는 것이다.
