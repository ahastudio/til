# PyTorch로 LLM 처음부터 만들기

원문: <https://jkf87.github.io/llm-from-scratch-2026-05-05>

## 소개

이 튜토리얼은 GPT 계열 언어 모델을 PyTorch로 처음부터 직접 구현하는 과정을
6개 파트로 나눠 설명하는 한국어 실습 가이드다.
셰익스피어 텍스트 데이터셋을 활용해 문자 수준(character-level) 토크나이저부터
트랜스포머 블록, 학습 파이프라인, 텍스트 생성 전략까지 전 과정을 다룬다.
완성된 모델은 약 1,080만 개의 파라미터를 가지며, 시(poetry) 데이터셋 학습
경진대회로 마무리된다.

## 명세

### Part 1: 토크나이저

셰익스피어 텍스트에서 등장하는 65개의 고유 문자를 정수 인덱스에 매핑하는
문자 수준 토크나이저를 구현한다.
바이트 페어 인코딩(Byte-Pair Encoding, BPE)보다 시퀀스 길이가 길어지지만,
소규모 데이터셋에서 통계적 밀도를 높이는 데 유리하다.

### Part 2: 트랜스포머 아키텍처

트랜스포머 블록 6개를 쌓은 구조를 구현한다.
각 블록의 구성은 아래와 같다.

```text
LayerNorm → Self-Attention → Residual → LayerNorm → MLP → Residual
```

멀티헤드 어텐션(Multi-Head Attention)이 서로 다른 표현 공간에서 패턴을
병렬로 추출한다.
전체 파라미터 수는 약 1,080만 개다.

### Part 3: 학습 파이프라인

다음 토큰 예측(next-token prediction) 방식으로 크로스 엔트로피(cross-entropy)
손실을 최소화한다.
학습 안정성을 위해 아래 기법들을 적용한다.

- 학습률 웜업(learning rate warmup)
- 코사인 감쇠(cosine decay) 스케줄링
- 그래디언트 클리핑(gradient clipping)
- 검증 손실 모니터링 및 체크포인트 저장

약 100만 자 규모의 데이터셋에서 1,080만 파라미터를 학습하면 검증 손실이
학습 손실과 벌어지는 과적합(overfitting)이 나타난다.

### Part 4: 텍스트 생성 전략

세 가지 생성 전략을 비교 구현한다.

- **탐욕적 디코딩(greedy decoding)**: 매 스텝에서 가장 높은 확률 토큰 선택.
  결정론적이지만 반복이 심하다.
- **온도 조절(temperature scaling)**: 확률 분포의 날카로움을 조절해 다양성을
  제어한다.
- **top-k 필터링**: 상위 k개 토큰 외의 확률을 0으로 만들어 엉뚱한 토큰 생성을
  억제한다.

### Part 5: 아키텍처 변형과 실험

tiny / small / medium 세 가지 설정으로 모델 용량과 성능 사이의 관계를 탐구한다.
손실 곡선 시각화로 학습 상태를 진단한다.

### Part 6: 경진대회 과제

시(poetry) 데이터셋으로 처음부터 학습해 독창적인 시를 생성하는 것이 목표다.
평가 기준은 일관성, 창의성, 구조, 감성적 공명이며, 재현성을 위해 시드(seed)를
고정한다.

## 분석

### Karpathy 계보를 잇는 한국어 실습서

이 튜토리얼의 설계는 Andrej Karpathy의 nanoGPT / makemore 프로젝트와 구조적으로
매우 유사하다.
문자 수준 토크나이저, 셰익스피어 데이터셋, 소규모 트랜스포머 스택이라는 조합은
nanoGPT 교육 버전의 전형적인 패턴이다.
Karpathy의 접근법이 탁월한 이유는 “최소한의 코드로 핵심 개념을 명확히 드러낸다”는
것인데, 이 튜토리얼은 그 철학을 한국어 컨텍스트에 이식했다.

### 학습 안정성 기법의 교육적 배치

워밍업 + 코사인 감쇠 + 그래디언트 클리핑을 처음부터 포함한 것은 좋은 선택이다.
실전에서는 이 세 기법이 거의 항상 함께 쓰이는데, 초보자용 튜토리얼에서는 흔히
생략된다.
검증 손실 모니터링을 통해 과적합을 직접 관찰하게 하는 구성도 인상적이다.
“데이터 100만 자, 파라미터 1,080만 개”라는 불균형을 학습자가 직접 경험하도록
설계한 것이다.

### 경진대회 형식의 교육적 가치

6번째 파트를 경진대회로 구성한 것은 전략적이다.
학습자가 배운 내용을 새로운 데이터셋에 적용해 보도록 강제하면서도,
성공 기준(일관성, 창의성, 구조, 감성)이 정량 지표와 정성 평가를 모두 포함한다.
AI 생성 텍스트의 품질 평가가 얼마나 다차원적인지를 체험하게 한다.

## 비평

### 강점: 전체 파이프라인을 하나의 흐름으로

토크나이저부터 생성, 경진대회까지 6개 파트가 끊김 없이 이어진다.
각 파트가 독립적으로도 읽히지만 전체로도 완결된다.
특히 텍스트 생성 전략을 3가지로 비교한 구성은, 단순히 “이렇게 구현한다”를 넘어
“왜 이 전략이 다른가”를 이해하게 만든다.

### 약점: 문자 수준 토크나이저의 한계 미설명

현대 LLM은 거의 예외 없이 BPE나 SentencePiece를 쓴다.
문자 수준 토크나이저가 소규모 실습에 적합하다는 설명은 있지만,
실제 프로덕션 모델로 확장할 때 왜 이 방식이 한계를 갖는지에 대한 설명이 없다.
“양보다 질이 더 중요하다”는 데이터 철학도 구체적인 근거 없이 주장으로 끝난다.

### 약점: 멀티헤드 어텐션 내부 미설명

어텐션 메커니즘의 수학적 구조(Q, K, V 행렬, scaled dot-product)에 대한
설명이 생략된 것으로 보인다.
트랜스포머의 핵심 구성요소를 블랙박스로 남겨두면, 학습자는 코드를 실행할 수 있어도
왜 작동하는지 이해하기 어렵다.

### 약점: 경진대회 평가 기준의 모호성

“감성적 공명”을 평가 기준으로 삼는 것은 매력적이지만 측정하기 어렵다.
인간 평가자가 있는지, 자동 지표(perplexity, BLEU)를 보조로 쓰는지,
평가 과정이 재현 가능한지에 대한 설명이 부족하다.

## 인사이트

### “처음부터 만들기”가 가르치는 것은 코드가 아니다

LLM을 처음부터 구현하는 튜토리얼의 진짜 가치는 완성된 모델이 아니다.
1,080만 파라미터짜리 모델을 직접 만들어보면, GPT-4나 Claude 같은 수백억
파라미터 모델이 어떻게 다른지를 구체적인 감각으로 이해하게 된다.
“크기가 n배면 성능이 어떻게 달라지는가”를 추상적으로 설명하는 것과,
직접 tiny/small/medium 모델을 비교 실험해보는 것의 차이는 크다.

이 감각은 AI 도구를 사용하는 사람에게도 중요하다.
프롬프트를 쓰는 사람이 토크나이저가 어떻게 작동하는지 알면,
왜 특정 표현이 더 잘 처리되는지, 왜 긴 컨텍스트에서 성능이 떨어지는지를
직관적으로 이해할 수 있다.
“처음부터 만들기”는 사용자를 위한 교육이기도 하다.

역사적으로 비슷한 패턴이 있다.
운영체제를 “MINIX로 배운” 세대는 Linux를 더 깊이 이해했고,
컴파일러를 직접 구현해본 프로그래머는 최적화 컴파일러의 한계를 더 잘 안다.
LLM을 직접 만들어본 사람이 앞으로 AI 시스템을 설계하고 평가하는 역할을 맡을
가능성이 높다.

### 소규모 모델 실습이 드러내는 스케일링의 역설

1,080만 파라미터 모델에서 나타나는 과적합은 의도된 실패다.
데이터 100만 자에 파라미터 1,080만 개를 얹으면 모델은 필연적으로 훈련 데이터를
암기하려 한다.
이 실패 경험이 “왜 현대 LLM은 수조 토큰의 데이터가 필요한가”를 체감하게 만든다.

스케일링 법칙(scaling laws)에 따르면 모델 크기, 데이터 크기, 연산량은 거의
선형적인 관계를 가진다.
Chinchilla 논문이 보여준 것처럼, 같은 연산 예산에서 모델 크기를 줄이고
데이터를 늘리는 것이 성능에 더 유리한 경우가 많다.
이 튜토리얼의 실패 실험은 그 직관을 손으로 느끼게 해주는 장치다.

작은 모델에서 실패를 경험한 사람은 큰 모델의 성공을 더 잘 설명할 수 있다.
GPT-3가 왜 갑자기 등장한 것이 아니라, 작은 모델들의 실패에서 축적된 교훈의
결과인지를 이해하게 된다.
“왜 이 크기에서 이 문제가 생기는가”를 아는 것이 엔지니어의 실력이다.

### 한국어 AI 교육 콘텐츠의 구조적 공백을 채우는 방식

LLM 관련 고품질 튜토리얼은 대부분 영어로 작성되어 있다.
Karpathy의 유튜브 강의, Hugging Face의 NLP 코스, fast.ai의 실습서 모두 영어다.
한국어 학습자는 이 콘텐츠를 소화하면서 언어와 기술 개념을 동시에 처리해야 한다.

이 튜토리얼이 한국어로 작성된 것은 단순한 번역 이상의 의미를 가진다.
개념 설명의 맥락, 예시의 선택, 오류 메시지에 대한 해석 방식이 한국어 학습자의
사고 패턴에 맞게 설계될 수 있다.
“양보다 질이 더 중요하다”는 문장이 한국어로 쓰였을 때와 영어로 쓰였을 때
학습자에게 전달되는 감각은 다르다.

그러나 이 공백을 채우는 시도가 아직 소수에 그친다는 점은 구조적 문제다.
AI 역량이 점점 더 중요해지는 시대에, 고품질 한국어 AI 교육 자료의 부족은
한국 개발자 생태계 전체의 역량 격차로 이어질 수 있다.
오픈소스 커뮤니티가 이 역할을 일부 담당하고 있지만,
체계적인 한국어 AI 교육 커리큘럼의 필요성은 여전히 크다.
