# Supermemory — AI를 위한 메모리 & 컨텍스트 엔진

<https://supermemory.ai/>

<https://github.com/supermemoryai/supermemory>

## 소개

Supermemory는 AI 시스템을 위한 메모리와 컨텍스트 레이어다.
AI는 기본적으로 대화 사이에 모든 것을 잊는다.
Supermemory는 이 한계를 해소하기 위해 대화에서 사실을 자동 추출하고, 사용자 프로필을 유지하며, 필요한 순간에 관련 컨텍스트를 전달한다.

세 가지 주요 사용 대상이 있다.
AI 도구 사용자는 MCP 서버나 브라우저 확장을 설치하면 모든 호환 AI 어시스턴트에 지속적 메모리를 추가할 수 있다.
개발자는 API를 통해 Vercel AI SDK, LangChain, OpenAI Agents SDK 등 주요 프레임워크와 통합한다.
조직은 시스템 전체를 로컬에 배포하여 완전한 데이터 제어권을 유지한다.

## 주요 기능

**메모리 시스템**: 대화에서 사실을 자동 추출한다.
시간적 변화, 모순, 만료된 정보를 자동으로 처리한다.

**사용자 프로필**: 두 종류의 컨텍스트를 유지한다.
장기 안정적 사실과 최근 활동 내역으로 구분하며 쿼리당 약 50밀리초 내에 갱신된다.

**하이브리드 검색**: RAG(Retrieval-Augmented Generation)와 개인화된 메모리를 단일 쿼리에서 결합한다.
지식 베이스 문서와 사용자별 컨텍스트를 함께 제공한다.

**멀티모달 처리**: PDF, OCR이 적용된 이미지, 동영상 트랜스크립션, AST 기반 청킹이 적용된 코드를 처리한다.

**데이터 커넥터**: Google Drive, Gmail, Notion, OneDrive, GitHub와 실시간 웹훅으로 연동한다.

## 성능 벤치마크

세 가지 주요 벤치마크에서 1위를 기록했다.

- LongMemEval: 81.6% 정확도
- LoCoMo: 장기 대화에서의 사실 회상
- ConvoMem: 개인화 및 선호 학습

## 설치 및 접근

**호스팅 플랫폼**: `app.supermemory.ai`에서 코드 없이 바로 사용 가능하다.

**로컬 배포**: 단일 바이너리로 설치한다.

```bash
curl -fsSL https://supermemory.ai/install | bash
```

완전 오프라인 운영이나 커스텀 모델 통합이 가능하다.

**패키지 매니저**: npm과 PyPI 패키지를 통해 API 통합을 지원한다.

## 기술 스택

TypeScript(65.8%), MDX(27.6%), Python(5.8%)으로 작성됐다.
Postgres, Remix, Tailwind CSS, Vite, Cloudflare 인프라, Drizzle ORM을 사용한다.

## 분석

### AI 메모리 문제의 두 가지 층위

AI 메모리 부재 문제는 두 층위에서 작동한다.
첫 번째는 세션 간 기억 부재다.
오늘 이야기한 것을 내일 다시 설명해야 한다.
두 번째는 더 미묘한 문제로, 사용자가 AI와 오래 작업할수록 AI가 사용자를 더 잘 이해해야 하는데 그렇지 못한 것이다.
Supermemory는 두 번째 문제를 더 깊이 다룬다.

“사용자 프로필”과 “장기 안정적 사실 vs. 최근 활동”의 구분은 이 두 번째 문제의 해법이다.
사람은 변하지 않는 특성(직업, 전문 도메인, 선호하는 스타일)과 변화하는 컨텍스트(현재 작업, 최근 논의한 문제)를 모두 가진다.
이 두 종류를 구분하지 않으면 메모리는 노이즈가 된다.

### 하이브리드 검색이 의미하는 아키텍처적 선택

기존 RAG 시스템은 외부 지식 베이스를 검색한다.
Supermemory의 하이브리드 검색은 지식 베이스 검색과 개인화된 메모리 검색을 동시에 수행한다.
이것은 아키텍처적으로 중요한 선택이다.
“이 문서에서 무엇이 관련 있는가”와 “이 사용자에게 무엇이 관련 있는가”를 동시에 묻는 것이다.

이 접근이 효과적이려면 두 검색 결과가 올바른 비율로 결합되어야 한다.
일반 정보와 개인화 정보 사이의 가중치를 잘못 설정하면 응답이 지나치게 개인화되거나 반대로 개인화가 없는 것처럼 느껴진다.
Supermemory가 이 균형을 어떻게 조정하는지는 실제 사용해보기 전에는 평가하기 어렵다.

### 로컬 배포 옵션의 전략적 의미

단일 바이너리 로컬 배포는 단순한 기능 추가가 아니다.
조직의 데이터 거버넌스 요구를 충족하기 위한 전략적 선택이다.
의료, 금융, 법률 같은 규제 산업에서는 사용자 데이터와 대화 내용이 외부 서버에 전송되는 것 자체가 규정 위반이다.
로컬 배포 옵션은 이 시장을 타깃한다.

그러나 로컬 배포는 유지보수 부담을 사용자에게 전가한다.
새 기능과 보안 패치를 직접 업데이트해야 하고, 인프라 문제를 직접 해결해야 한다.
진정한 엔터프라이즈 친화적 솔루션이 되려면 로컬 배포의 관리형 버전이나 전용 지원이 필요하다.

## 비평

### 벤치마크 1위의 실제 의미를 검증해야 한다

세 개 벤치마크에서 1위라는 주장은 강력하게 들리지만, 벤치마크 자체의 한계를 확인해야 한다.
LongMemEval, LoCoMo, ConvoMem은 모두 합성된 테스트 데이터를 기반으로 한다.
실제 사용자의 다양하고 비구조화된 대화에서 메모리를 어떻게 추출하고 관리하는지는 벤치마크가 포착하지 못한다.

특히 “모순 처리”와 “만료된 정보 처리”는 실제로 매우 어렵다.
사용자가 직업을 바꾸거나, 이전에 좋아한다고 했던 것을 이제 싫어하게 된 경우를 어떻게 감지하는가.
점진적으로 변하는 선호나 컨텍스트의 미묘한 이동은 기존 NLP 패러다임으로 포착하기 어렵다.
벤치마크 점수는 이 어려운 케이스들을 대표하지 못한다.

### 메모리 추출의 프라이버시 리스크

Supermemory의 핵심 기능인 자동 사실 추출은 근본적인 프라이버시 질문을 제기한다.
사용자가 명시적으로 기억하도록 요청하지 않은 정보도 추출될 수 있다.
사용자가 어느 날 실수로 언급한 민감한 정보, 맥락 없이는 무해하지만 누적되면 문제가 되는 정보들이 자동으로 프로필에 기록될 수 있다.

이것은 단순한 보안 취약점이 아니라 설계 철학의 문제다.
“AI가 더 많이 알수록 더 잘 돕는다”는 전제와 “사용자가 무엇이 기록되는지 알고 통제해야 한다”는 원칙 사이의 긴장이다.
Supermemory가 이 긴장을 어떻게 해소하는지는 제품 설명에서 충분히 다루어지지 않는다.

### 오픈소스 지속 가능성의 불확실성

GitHub 저장소가 공개되어 있고 무료 사용이 가능하지만, 비즈니스 모델이 명확하지 않다.
MCP 서버나 API 통합을 통한 수익화 구조가 보이지 않는다.
오픈소스 AI 인프라 프로젝트는 기업 후원이나 명확한 수익 모델 없이는 지속 가능성이 낮다.

핵심 인프라로 Supermemory에 의존하는 애플리케이션을 구축하기 전에, 프로젝트의 장기 유지 보수 가능성을 평가해야 한다.
벤더 종속이나 프로젝트 중단 리스크는 특히 사용자 데이터가 저장되는 메모리 인프라에서 치명적이다.

## 인사이트

### AI 메모리의 표준화가 새로운 플랫폼 경쟁을 만든다

Supermemory가 MCP 표준을 채택한 것은 단순한 기술 선택이 아니다.
MCP가 에이전트 통합의 표준이 된다면, 그 위에서 작동하는 메모리 레이어를 제공하는 것은 플랫폼 위치를 선점하는 전략이다.
어떤 AI 어시스턴트도 Supermemory가 있으면 메모리를 갖게 된다는 것은 에이전트 생태계에서 수평적 인프라 역할을 겨냥한다.

이 전략이 성공하면 메모리는 AI 어시스턴트들이 경쟁하는 영역이 아니라 공유 인프라가 된다.
그러나 주요 AI 회사들도 독자적인 메모리 레이어를 구축하고 있다.
ChatGPT의 Memory 기능, Claude의 프로젝트 기능 등 경쟁자들이 이미 자체 솔루션을 제공한다.
Supermemory의 포지셔닝이 이 경쟁에서 어떻게 살아남을지는 MCP 생태계의 성장 속도에 달려 있다.

### 개인화와 망각 사이의 균형이 AI 신뢰의 핵심이다

AI가 더 많은 것을 기억할수록 더 유용해지지만, 동시에 사용자가 과거의 맥락에 묶이는 문제가 발생한다.
몇 년 전에 기록된 선호가 현재의 응답을 잘못 유도할 수 있다.
인간 관계에서도 오래된 친구가 예전의 자신에 대한 고정관념을 가지면 불편하듯, AI가 오래된 메모리를 과도하게 활용하면 오히려 방해가 된다.

이것은 기술 문제이기 전에 설계 철학 문제다.
AI 메모리의 이상적인 상태는 완전한 기억도 완전한 망각도 아니다.
관련성이 높은 것은 오래 유지하고, 낮아진 것은 자연스럽게 가중치가 줄어드는 동적 시스템이 필요하다.
Supermemory의 “만료된 정보 처리” 기능이 이 철학을 어떻게 구현하는가가 실제 사용 경험의 품질을 결정할 것이다.

### 메모리 레이어가 AI 에이전트의 정체성을 만든다

현재 AI 에이전트들은 사용자마다 동일하게 시작한다.
메모리 레이어가 보편화되면 에이전트는 사용자별로 다른 “경험”을 가지게 된다.
같은 Claude나 GPT-4라도 Supermemory 같은 레이어를 통해 각 사용자의 역사를 누적하면, 에이전트는 점점 그 사용자에게 특화된 존재가 된다.

이것은 AI 에이전트의 정체성에 관한 흥미로운 질문을 제기한다.
에이전트의 정체성이 베이스 모델에 있는가, 아니면 누적된 메모리에 있는가.
사용자가 에이전트를 바꾸면 그 에이전트와 함께 쌓인 컨텍스트를 잃는다.
이것은 AI 제공자에 대한 전환 비용(switching cost)을 만들며, 사용자 락인(lock-in)의 새로운 형태가 된다.
