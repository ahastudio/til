# awesome-llm-apps

> A curated collection of awesome LLM apps built with RAG and
> AI agents.

<https://github.com/Shubhamsaboo/awesome-llm-apps>

## 개요

LLM 기반 애플리케이션의 실전 구현 사례를 모은 오픈소스
저장소다. RAG, AI 에이전트, 멀티 에이전트 팀, 음성 에이전트,
MCP 에이전트 등 다양한 패턴을 다룬다. OpenAI, Anthropic,
Google, xAI 및 Llama, Qwen 같은 오픈소스 모델을 지원한다.

GitHub 스타 98.4k, 포크 14.3k로 LLM 앱 분야에서 가장 큰
커뮤니티 저장소 중 하나다. Apache-2.0 라이선스.

Knowledge Graph RAG, Vision RAG 등 18개 이상의 RAG

있어 프라이버시가 중요한 경우에도 활용할 수 있다.

## 분석

### 구조적 특징

각 프로젝트는 독립 디렉토리로 구성되어 있어 clone 후
원하는 프로젝트만 골라서 바로 실행할 수 있다.
의존성 설치(`pip install`)와 프로젝트별 README만으로
진입 장벽을 낮췄다.

### 패턴의 진화

저장소의 카테고리 구성은 LLM 앱 개발 패턴의 진화를
보여준다.

1. **단일 프롬프트 → RAG**: 외부 지식을 주입하여
   환각(hallucination)을 줄이는 단계
2. **RAG → 에이전트**: 도구 사용(tool use)과 자율적
   판단을 더해 복잡한 태스크를 수행하는 단계
3. **단일 에이전트 → 멀티 에이전트**: 역할 분담을 통해
   전문성과 신뢰성을 높이는 단계
4. **텍스트 → 멀티모달**: 음성, 이미지, 비디오 등
   다양한 입출력을 처리하는 단계
5. **클라우드 → 로컬**: 프라이버시와 비용을 고려한
   온디바이스 실행 단계

### 프레임워크 생태계

CrewAI, Phidata, OpenAI Agents SDK, Google ADK 등
다양한 프레임워크를 활용한 예제가 있다. 특정 프레임워크에
종속되지 않고 비교 학습이 가능하다는 점이 강점이다.

## 인사이트

### LLM 앱의 핵심은 오케스트레이션이다

모델 성능 자체보다 "모델을 어떻게 조합하고, 어떤 순서로
호출하며, 실패 시 어떻게 복구하는가"가 실전 앱의 품질을
결정한다. 이 저장소의 멀티 에이전트 팀 사례들(금융 팀,
법률 자문 팀 등)은 이를 잘 보여준다. 개별 에이전트의
능력보다 에이전트 간 역할 분담과 정보 흐름 설계가 결과를
좌우한다.

### RAG는 여전히 가장 실용적인 패턴이다

18개 이상의 RAG 변형이 존재한다는 것 자체가 이 패턴의
중요성을 증명한다. 특히 Corrective RAG(검색 결과를
검증·보정)와 Agentic RAG(에이전트가 검색 전략을 스스로
결정)는 단순 RAG의 한계를 넘어서는 실전적 발전이다.
기업 환경에서 LLM을 도입할 때 RAG는 거의 필수 패턴이다.

### MCP가 에이전트의 연결성을 표준화한다

MCP(Model Context Protocol) 카테고리가 별도로 존재하는
것은 이 프로토콜이 에이전트와 외부 시스템 간의 연결을
표준화하는 방향으로 빠르게 자리잡고 있음을 시사한다.
브라우저, GitHub, Notion 등 다양한 통합 사례가 이미
구현되어 있어, 커스텀 도구 통합(tool integration)의
보일러플레이트를 크게 줄일 수 있다.

### 로컬 실행이 새로운 차별화 요소다

클라우드 API에 의존하지 않는 로컬 RAG, 로컬 에이전트
구현이 포함되어 있다. 이는 비용 절감뿐 아니라 데이터
주권(data sovereignty), 오프라인 환경, 규제 대응 측면에서
점점 중요해지는 방향이다. Llama, Qwen 같은 오픈소스 모델의
성능 향상과 맞물려 로컬 실행의 실용성이 빠르게 높아지고
있다.

### 실전 도메인 적용이 핵심 가치다

단순 데모가 아니라 금융, 법률, 부동산, 의료, 채용 등
구체적 도메인에 적용한 사례가 많다. 이는 "LLM으로 무엇을
만들 수 있는가"라는 질문에 구체적 답을 제공한다.
특히 멀티 에이전트 팀 구성(예: AI VC 실사팀, 금융 분석팀)은
실제 업무 프로세스를 에이전트로 모델링하는 좋은 참고
사례다.
