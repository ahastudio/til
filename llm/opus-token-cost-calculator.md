# Opus 4.6과 4.7의 토큰 비용 비교

원문: <https://tokens.billchambers.me/leaderboard>

## 요약

Tokenomics는 Bill Chambers가 만든 커뮤니티 기반 토큰 비용 비교 도구로, Claude Opus
4.6과 4.7이 동일한 프롬프트를 어떻게 다르게 토큰화하는지 비교할 수 있다. 사용자들이
익명으로 프롬프트를 제출하면 두 모델의 토큰 소비량 차이를 보여주는 리더보드를
제공한다.

커뮤니티 데이터에 따르면 Opus 4.7의 새로운 토크나이저는 동일한 프롬프트에 대해 평균
37.4% 더 많은 입력 토큰을 소비한다. 제출된 데이터의 편차는 +19%에서 +86.2%까지
다양하다. 즉, 텍스트 내용을 전혀 바꾸지 않아도 모델 버전만 변경하면 요청당 비용이
증가한다.

이 도구는 오픈소스이며 저장되는 데이터는 익명 제출 ID만 포함한다.
Anthropic과는 공식적으로 무관하다.

GeekNews 댓글에서는 다양한 관점이 공유되었다. 한 사용자는 4.7이 출력 토큰과 추론
비용을 상당히 절감했기 때문에 전체 비용은 비교해야 한다고 지적했다. 다른 사용자들은
4.7의 적응형 사고(adaptive thinking)가 10분간 토큰을 소비하면서도 깊이 있는 분석을
하지 않는다고 비판했다. 긍정적 경험을 공유한 사용자는 4.7이 “시니어 팀원처럼”
더 자기비판적이고 품질이 높다고 평가했다.

## 분석

### 토크나이저 변경의 비용 영향

토크나이저는 텍스트를 모델이 처리할 수 있는 토큰 단위로 분할하는 전처리 단계다.
모델 버전 간 토크나이저가 변경되면 동일한 텍스트라도 토큰 수가 달라진다. Opus 4.7이
37.4% 더 많은 입력 토큰을 소비한다는 것은, 입력 가격이 동일하더라도 실질 비용이
37.4% 증가한다는 뜻이다.

이 변화가 의도적인 것인지 부수 효과인지에 따라 해석이 달라진다. 새 토크나이저가
더 세밀한 분해를 통해 모델 성능을 향상시키는 것이라면, 비용 증가는 품질 향상의
대가다. 단순히 비효율적인 토큰화라면 이용자 입장에서 순손실이다.

### 입력 vs 출력 비용의 트레이드오프

한 GeekNews 댓글이 지적한 대로, 입력 토큰 비용만으로 전체 비용을 판단할 수 없다.
4.7이 더 효율적인 출력을 생성하거나 추론 토큰을 줄인다면, 입력 비용 증가분이 상쇄될
수 있다. 그러나 이는 사용 패턴에 따라 다르다. 긴 컨텍스트를 입력하고 짧은 응답을
받는 사용 사례(코드 리뷰, 문서 분석 등)에서는 입력 비용 증가가 지배적이다.

### 커뮤니티 벤치마크의 가치와 한계

Anthropic이 공식적으로 제공하지 않는 비교를 커뮤니티가 자발적으로 수행하고 있다는
점이 주목할 만하다. 다만 커뮤니티 제출 데이터는 선택 편향(selection bias)이 있을
수 있다. 큰 차이를 발견한 사용자가 더 적극적으로 제출하기 때문에, 평균 37.4%라는
수치가 실제 평균보다 높게 왜곡되었을 수 있다.

## 비평

### 강점

단일 기능에 집중한 도구 설계가 효과적이다. “두 모델의 토큰 수 비교”라는 명확한
목적을 가지고 있어 사용이 직관적이다. 오픈소스이고 익명 데이터만 저장한다는
투명성도 신뢰를 높인다.

커뮤니티 기반으로 데이터를 수집하는 접근은 단일 벤치마크보다 다양한 실사용 패턴을
반영할 수 있다는 장점이 있다.

### 약점

이 도구는 토큰 수만 비교하고 비용은 직접 계산하지 않는다. “토큰 비용 계산기”라는
이름과 달리 실제 달러 비용 비교는 사용자가 별도로 해야 한다. 입력/출력 토큰
비율, 캐싱, 배치 할인 등을 고려한 실제 비용 비교가 있었다면 더 유용했을 것이다.

또한 토큰 수만으로는 모델의 가치를 판단할 수 없다. 같은 작업에 대해 4.7이 더 좋은
결과를 내면 토큰당 가치가 높으므로, 토큰 수 증가가 반드시 나쁜 것은 아니다.
이 맥락이 도구에서 충분히 전달되지 않는다.

## 인사이트

### 토크나이저는 숨겨진 가격 조정 메커니즘이다

AI 서비스의 가격은 토큰당 단가로 표시된다. 그런데 같은 텍스트가 더 많은 토큰으로
분해되면, 단가가 변하지 않아도 실질 비용은 올라간다. 이는 식품 업계의
“슈링크플레이션(shrinkflation)”과 구조적으로 동일하다. 가격을 올리는 대신
단위를 줄이는 것이다.

물론 토크나이저 변경이 의도적 가격 인상을 목적으로 한 것이라고 단정할 수는 없다.
모델 성능 향상을 위해 토크나이저를 바꾸는 것은 기술적으로 합리적인 결정이다.
그러나 결과적으로 사용자의 실질 비용이 증가하는 것은 사실이며, 이 변화가 가격
페이지에 명시적으로 공지되지 않는다면 투명성 문제가 된다.

AI API 비용을 관리하는 조직은 토큰 단가뿐 아니라 토크나이저 효율성까지 추적해야
한다. 모델을 업그레이드할 때 “토큰당 가격이 같다”는 것이 “비용이 같다”를 의미하지
않을 수 있다.

### 모델 업그레이드의 숨겨진 비용 구조

소프트웨어 업데이트는 일반적으로 같은 비용에 더 나은 성능을 기대하게 만든다.
그러나 AI 모델은 다르다. 새 버전이 더 많은 컴퓨트를 소비하면서 더 나은 결과를
내는 경우, “업그레이드 = 비용 증가”가 된다. GeekNews에서 5시간 Rate Limit을
2시간 만에 소진한다는 보고가 이를 잘 보여준다.

이는 사용자에게 새로운 종류의 의사결정을 요구한다. “최신 모델이 항상 최선”이라는
단순한 전략이 통하지 않고, 태스크별로 비용 대비 품질을 평가해야 한다. 간단한
태스크에 고사양 모델을 쓰는 것은 낭비이고, 복잡한 태스크에 저사양 모델을 쓰는
것은 품질 손실이다.

이 복잡성은 AI 모델 라우팅 — 태스크 복잡도에 따라 다른 모델로 분배하는 — 의
필요성을 더욱 부각시킨다. 단일 모델에 모든 것을 맡기는 시대는 지나가고 있다.

### 벤더 종속과 비용 투명성

AI API 사용자는 모델 내부 구현에 대한 통제권이 없다. 토크나이저가 변경되어도,
추론 과정에서 추가 토큰이 소비되어도, 사용자가 할 수 있는 것은 결과를 관찰하고
대응하는 것뿐이다. Tokenomics 같은 커뮤니티 도구가 등장하는 이유가 여기에 있다.
공급자가 제공하지 않는 투명성을 커뮤니티가 자체적으로 구축하는 것이다.

이는 클라우드 컴퓨팅 초기에 비용 모니터링 도구가 폭발적으로 성장한 패턴과 유사하다.
AWS의 복잡한 가격 구조를 이해하기 위해 CloudHealth, Spot.io 같은 서드파티 도구가
필요했듯, AI API의 실질 비용을 파악하기 위한 도구 생태계가 형성되고 있다.
장기적으로 이러한 커뮤니티 투명성 도구의 존재는 AI 공급자에게도 가격 정책의
명확한 커뮤니케이션을 압박하는 건강한 피드백 루프가 된다.
