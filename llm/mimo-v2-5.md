# MiMo-V2.5 — Xiaomi의 오픈소스 옴니모달 AI 모델

<https://huggingface.co/XiaomiMiMo/MiMo-V2.5>

## 소개

MiMo-V2.5는 Xiaomi가 공개한 오픈소스 옴니모달(Omnimodal) AI 모델로, 텍스트·이미지·영상·오디오를
단일 아키텍처에서 처리한다. MiMo-V2-Flash 백본에 전용 비전 인코더와 오디오 인코더를 결합한 구조로,
Sparse MoE(Mixture of Experts)를 채택해 표준 버전은 전체 310B 파라미터 중 15B만, Pro 버전은
전체 1.02T 파라미터 중 42B만 활성화된다. 최대 1M 토큰의 컨텍스트 윈도우를 지원하며 MIT 라이선스로
상업적 활용이 자유롭다.

## 아키텍처

| 항목              | MiMo-V2.5      | MiMo-V2.5-Pro    |
| ----------------- | -------------- | ---------------- |
| 전체 파라미터     | 310B           | 1.02T            |
| 활성화 파라미터   | 15B            | 42B              |
| 아키텍처          | Sparse MoE     | Sparse MoE       |
| Hidden Size       | 4096           | 6144             |
| 레이어 수         | 48 (1+47 MoE)  | 70 (1+69 MoE)    |
| Full Attention    | 9레이어        | 10레이어         |
| SWA 레이어        | 39레이어       | 60레이어         |
| 어텐션 헤드       | 64             | 128              |
| KV 헤드 (GA/SWA) | 8 / 4          | 8 (GQA)          |
| 컨텍스트 길이     | 1M             | 1M               |
| Routed Experts    | 256            | 384              |
| 토큰당 전문가     | 8              | 8                |
| SWA 윈도우        | 128 토큰       | 128 토큰         |

Hybrid Attention 구조(Sliding Window Attention + Global Attention, 5:1 비율)를 사용하며,
이를 통해 KV 캐시를 순수 Global Attention 대비 6배 줄인다. 비전 인코더는 729M 파라미터(28 레이어,
SWA 24개 + Full Attention 4개), 오디오 인코더는 261M 파라미터(24 레이어, SWA 12개 + Full
Attention 12개)로 구성된다.

## 학습 과정

1. 텍스트 사전 학습 (~48T 토큰, FP8 혼합 정밀도)
2. 멀티모달 프로젝터 워밍업 (오디오/비전 MLP 단기 적응)
3. 대규모 멀티모달 사전 학습 (텍스트·이미지·영상·오디오 통합)
4. SFT 및 에이전트 포스트 트레이닝 (컨텍스트 32K → 256K → 1M 단계적 확장)
5. RL 및 MOPD(Multi-teacher Online Policy Distillation)

## 성능 벤치마크

에이전트 작업 중심의 벤치마크에서 두각을 나타낸다.

- SWE-bench Pro: **56.1**
- TerminalBench 2.0: **65.8**
- ClawEval(Pro 모델): 63.8% 성공률

[GeekNews 댓글](https://news.hada.io/topic?id=29006)에 따르면 VentureBeat 테스트 결과
Pro 모델이 ClawEval 벤치마크에서 63.8% 성공률을 기록해 유사 규모의 오픈소스 대안들을
상회했으며, 동급 클로즈드 모델 대비 40~60% 적은 토큰으로 유사한 성능을 달성했다는 점이
주목받았다.

## 배포

SGLang(권장)으로 배포할 경우 예시:

```bash
python3 -m sglang.launch_server \
    --model-path XiaomiMiMo/MiMo-V2.5 \
    --served-model-name mimo-v2.5 \
    --log-level-http warning \
    --enable-cache-report \
    --pp-size 1 \
    --dp-size 2 \
    --tp-size 8 \
    --enable-dp-attention \
    --moe-a2a-backend deepep \
    --deepep-mode auto \
    --quantization fp8 \
    --context-length 262144 \
    --mm-enable-dp-encoder \
    --attention-backend fa3 \
    --mm-attention-backend fa3
```

vLLM에서는 `temperature=1.0`, `top_p=0.95` 샘플링 파라미터를 권장한다.

## 분석

### 단일 아키텍처로 네 가지 모달리티를 처리하는 설계

MiMo-V2.5의 핵심 설계 결정은 모달리티별 전용 모델을 조합하는 방식이 아니라 단일 LLM 백본에
전용 인코더를 결합하는 방식을 택했다는 것이다. 비전 인코더(ViT, 729M)와 오디오 인코더(261M)가
각자의 모달리티를 처리해 임베딩을 생성하고, 이것이 LLM 백본에 주입되는 구조다.

이 설계에서 주목할 점은 비전 인코더와 오디오 인코더 모두 LLM 백본과 동일한 Hybrid Attention
구조(SWA + Global Attention)를 채용했다는 것이다. 비전 인코더는 28레이어 중 24개가 SWA,
4개가 Full Attention이고, 오디오 인코더는 24레이어 중 12개가 SWA, 12개가 Full Attention이다.
SWA 윈도우 크기는 LLM 백본과 동일하게 128로 통일됐다. 이러한 아키텍처 통일성은 우연이
아니다. 모달리티별로 서로 다른 어텐션 메커니즘을 쓰면 백본 LLM과의 피처 정렬이 어려워지고
포스트 트레이닝에서 멀티모달 이해가 불안정해질 수 있다. 아키텍처를 일관되게 유지함으로써
인코더와 LLM이 비슷한 “언어”로 정보를 표현하게 된다.

비전 인코더의 윈도우 패턴은 `[-1] + [0,0,0,0,1,1,1,1,-1] × 3` 형태로, 전역 어텐션 레이어의
주기적 배치를 구조화했다. 이미지를 패치 시퀀스로 처리할 때 지역 특성(SWA)과 전역 관계(Global
Attention)를 균형 있게 포착하는 전략이다. 오디오 인코더에서 SWA와 Full Attention의 비율이
다른 것은 오디오 신호가 이미지보다 더 강한 전역 시간 의존성을 가지기 때문으로 해석할 수 있다.

### Hybrid Attention과 KV 캐시 6배 절감의 메커니즘

Hybrid Attention 구조의 핵심은 대부분의 레이어에서 슬라이딩 윈도우 어텐션(SWA)을 사용하고,
일부 레이어에서만 전체 시퀀스를 보는 Global Attention을 사용한다는 것이다. 표준 버전 기준으로
48레이어 중 39레이어가 SWA(128 토큰 윈도우), 9레이어만 Global Attention을 사용한다.

KV 캐시 절감의 원리는 다음과 같다. 표준 Self-Attention에서 각 레이어는 시퀀스 내 모든 이전
토큰의 Key와 Value를 캐시해야 한다. 1M 토큰 시퀀스라면 각 레이어마다 1M 개의 KV 쌍을
저장해야 한다. 반면 SWA를 사용하는 레이어에서는 윈도우 크기(128) 만큼의 KV만 활성 상태로
유지하면 된다. 슬라이딩 윈도우가 앞으로 이동하면서 이전 윈도우의 KV 슬롯은 해제된다.

48레이어 중 39개가 128 토큰 윈도우만 유지한다면, 이 레이어들의 KV 캐시는 1M이 아닌 128개다.
9개의 Global Attention 레이어만 1M 토큰의 KV를 유지한다. 총 KV 저장량을 단순 계산하면
`(39 × 128 + 9 × 1,000,000) / (48 × 1,000,000) ≈ 0.19`, 즉 순수 Global Attention 대비
약 19%다. “6배 감소”라는 수치는 이 원리에서 나온다.

이 설계의 트레이드오프는 SWA 레이어가 윈도우를 벗어난 장거리 의존성을 포착하지 못한다는 것이다.
MiMo-V2.5는 이를 주기적으로 배치된 Global Attention 레이어가 보완하는 방식으로 해결한다.
1M 토큰 처리를 실용적 수준으로 만드는 핵심 설계 결정이다.

### MOPD(Multi-teacher Online Policy Distillation)의 독창성

MiMo-V2.5의 포스트 트레이닝 파이프라인에서 가장 독창적인 요소는 MOPD다. 기존의 지식 증류
(Knowledge Distillation)는 대형 교사 모델(teacher)에서 소형 학생 모델(student)로 지식을
전달하는 오프라인 방식이다. MOPD는 여기서 두 가지를 확장한다.

첫째, “Multi-teacher”다. 단일 교사 모델이 아니라 여러 도메인에 특화된 교사 모델들을 동시에
활용한다. 코딩 전문 교사, 수학 추론 전문 교사, 에이전트 작업 전문 교사 등이 각 도메인에서의
올바른 행동 분포를 학생 모델에 전달한다. 이는 “박식한 단일 스승에게 배우는” 것이 아니라
“각 분야의 전문가에게 해당 영역을 배우는” 방식이다.

둘째, “Online Policy”다. 교사의 지식을 사전에 고정된 데이터셋으로만 전달하지 않고, 훈련 중
실시간으로 교사 모델이 생성한 롤아웃(rollout)을 학생 모델이 학습한다. 이를 통해 학생 모델의
현재 능력 수준에 맞춰 교사가 적절한 난이도의 예시를 제공하는 적응적 학습이 가능해진다.

이 접근은 RL(Reinforcement Learning)과 결합될 때 시너지가 생긴다. RL이 보상 신호를 통해
원하는 행동을 강화하는 동안, MOPD는 이미 검증된 고품질 행동의 분포를 직접 모방하게 해서
RL 탐색 공간을 효율적으로 좁힌다. 순수 RL에서 발생하는 탐색 비효율성을 MOPD가 보완하는
구조다.

### 컨텍스트 윈도우 단계적 확장 전략

SFT 및 에이전트 포스트 트레이닝 단계에서 컨텍스트를 32K → 256K → 1M으로 단계적으로 확장하는
전략은 표면상 단순해 보이지만 기술적으로 중요한 의미가 있다. 처음부터 1M 컨텍스트로 훈련하면
각 훈련 단계의 연산량이 극도로 증가해 배치 크기가 작아지고 학습 신호의 다양성이 줄어든다.
단계적 확장은 초기에는 다양한 짧은 예시들로 기본 패턴을 학습하고, 이후 더 긴 컨텍스트로 확장해
장거리 의존성을 점진적으로 학습하게 한다.

특히 256K에서 1M으로의 마지막 확장은 질적 도약이다. 256K 토큰은 일반적인 소설 한 권 분량에
해당하고, 1M 토큰은 여러 권의 책 혹은 중대형 코드베이스 전체에 해당한다. 이 규모에서는
단순한 패턴 매칭이 아니라 진정한 장기 기억(long-term memory)과 추론이 필요하며, 이를 위한
별도의 학습 데이터와 전략이 필요하다.

### 에이전트 특화 설계가 벤치마크에 미치는 영향

SWE-bench Pro 56.1과 TerminalBench 2.0의 65.8은 에이전트 작업에서 MiMo-V2.5의 강점을
보여주는 수치다. 이 벤치마크들은 단순한 질문-답변이 아니라 실제 소프트웨어 엔지니어링 작업
(버그 수정, 코드 수정)과 터미널 환경에서의 다단계 작업 수행 능력을 측정한다.

이 성능의 배경에는 에이전트 포스트 트레이닝이 있다. 에이전트 워크플로우에서 모델은 단일 응답을
생성하는 것이 아니라 환경과 반복적으로 상호작용하며(실행 → 결과 관찰 → 다음 행동 결정) 목표를
달성해야 한다. 이를 위해서는 긴 상호작용 히스토리를 정확하게 추적하는 능력, 오류 발생 시
방향 전환하는 적응성, 도구 사용과 코드 실행을 정확하게 계획하는 능력이 필요하다.

VentureBeat의 테스트에서 Pro 모델이 경쟁 모델 대비 40~60% 적은 토큰으로 유사한 성능을 달성했다는
점은 에이전트 작업에서 특히 중요하다. 에이전트 워크플로우는 많은 상호작용 턴을 포함하기 때문에
토큰 효율이 실제 운영 비용에 직접 영향을 미친다.

## 비평

### 강점: MIT 라이선스와 진정한 상업적 자유

상업적 제한 없이 활용할 수 있는 MIT 라이선스는 기업 도입의 주요 장벽을 제거한다. 많은 오픈소스
LLM이 “연구용” 또는 “비상업적 용도” 제한을 두거나, 일정 규모 이상의 기업에는 상업적 라이선스를
요구하는 것과 대비된다. MIT 라이선스에서는 모델 가중치를 수정하고, 파인튜닝하고, 프로덕션 서비스에
통합하는 모든 행위가 자유롭다. 이는 Llama 3가 커스텀 상업적 라이선스를 요구하거나 Mistral이
일부 버전에 제한을 두는 것과 구별되는 명확한 차별점이다.

에이전트 작업 벤치마크에서의 강점은 AI 에이전트 인프라가 빠르게 성장하는 현시점에 시의적절하다.
SWE-bench Pro 56.1은 실제 소프트웨어 엔지니어링 작업에서 충분히 유용한 수준이며, 이를 MIT
라이선스로 프로덕션 시스템에 통합할 수 있다는 조합은 엔터프라이즈 AI 에이전트 시장에서 중요한
가치 제안이다.

### 강점: 오디오 모달리티의 기본 통합

대부분의 멀티모달 LLM이 텍스트와 이미지에 집중하는 반면, MiMo-V2.5는 오디오 인코더(261M,
24레이어)를 기본 아키텍처에 포함했다. 이는 선택적 플러그인이 아니라 사전 학습 단계부터 통합된
네이티브 지원이라는 점에서 의미가 크다. 음성 인터페이스, 영상 콘텐츠 분석, 멀티미디어 에이전트
등 오디오 이해가 필수인 응용 분야에서 별도의 ASR(자동 음성 인식) 모델 없이 단일 모델로 처리할
수 있다.

### 약점: 중국 빅테크의 지속 가능성 우려

Xiaomi는 스마트폰·IoT 하드웨어 회사로, AI 연구가 핵심 사업 역량이 아니다. 삼성이 Exynos
관련 AI 연구를 축소하거나, 인텔이 여러 AI 연구 팀을 해체했던 사례처럼, 비AI 중심 기업의
AI 연구는 사업 전략 변화에 따라 축소될 리스크가 있다. 모델 업데이트, 버그 수정, 커뮤니티
지원이 장기적으로 유지될지는 검증이 필요하다. HN 토론(ID: 47930683)에서 커뮤니티 반응이
저조했다는 점(포인트 5, 댓글 0)은 오픈소스 AI 커뮤니티에서 아직 입지가 약하다는 신호일 수
있다.

### 약점: 재현 및 독립 검증 부족

벤치마크 수치가 공식적으로 독립 검증된 사례가 충분하지 않다. 특히 에이전트 벤치마크(SWE-bench,
TerminalBench)는 테스트 환경 구성에 따라 결과가 크게 달라질 수 있다. 어떤 버전의 Python
환경을, 어떤 시간 제한과 반복 횟수로 테스트했는지가 결과에 영향을 미친다. 제3자 재현 데이터와
다양한 실전 환경에서의 검증이 쌓여야 수치의 신뢰성을 판단할 수 있다.

### 약점: 훈련 데이터와 과정의 불투명성

DeepSeek가 훈련 불안정성까지 공개하는 것과 달리, MiMo-V2.5는 학습 과정에 대한 세부 정보가
제한적이다. 약 48T 토큰이라는 규모는 밝혔지만 데이터 구성, 품질 필터링 기준, 합성 데이터 비율,
멀티모달 데이터의 도메인 분포 등이 공개되지 않았다. 오픈소스로 모델 가중치를 공개하는 것과
훈련 파이프라인을 공개하는 것은 다른 차원의 투명성이다. 현재 MiMo-V2.5는 전자만 해당한다.

## 인사이트

### 옴니모달은 차세대 AI 인프라의 기본 단위가 될 것이다

텍스트, 이미지, 영상, 오디오를 별도 모델로 처리하던 시대에서 단일 모델이 모든 모달리티를 다루는
시대로 전환이 가속화되고 있다. GPT-4o, Gemini Ultra에 이어 MiMo-V2.5가 오픈소스 진영에서 이
방향을 따른다. 이 전환의 이면에는 기술적 이유와 실용적 이유가 모두 있다.

기술적으로, 단일 모델에서 여러 모달리티를 함께 훈련하면 교차 모달(cross-modal) 표현을 학습할
수 있다. 예를 들어 “강아지가 짖는 소리”를 텍스트로 묘사한 데이터와 실제 오디오 데이터를 함께
학습하면, 모델이 오디오-텍스트 간의 의미론적 연결을 내재화한다. 별도 모델에서 각 모달리티를
학습하고 이후 연결하려 할 때보다 더 깊은 이해가 가능하다.

실용적으로, 에이전트 워크플로우가 실시간 환경에서 화면 분석, 음성 명령, 문서 이해를 동시에
수행해야 하는 시나리오가 늘어나면서 단일 모델 처리는 지연 시간(latency)과 비용 면에서 구조적
이점을 가진다. 여러 모델을 조합하면 각 모델 간 컨텍스트를 직렬화해 전달해야 하는 오버헤드가
생기지만, 단일 모델은 모든 정보를 같은 표현 공간에서 처리한다.

역사적 유사 사례는 스마트폰의 등장이다. 휴대폰, MP3 플레이어, 디지털 카메라, GPS 네비게이터가
별도로 존재하다가 스마트폰이라는 단일 기기로 통합됐다. 초기에는 각 전용 기기가 품질 면에서
앞섰지만 통합의 편의성과 생태계 효과가 전용 기기를 압도했다. AI 모달리티 통합도 유사한 경로를
밟을 가능성이 높다. 옴니모달 능력은 선택이 아닌 필수 스펙이 될 것이다.

### Xiaomi의 AI 전략: 하드웨어-소프트웨어 수직 통합

Xiaomi가 스마트폰과 IoT 기기를 수억 대 이상 운용하는 기업임을 고려하면, MiMo-V2.5는 단순한
AI 연구 결과물이 아니다. 자사 기기에 탑재할 온디바이스 AI 또는 엣지 추론 파이프라인의 핵심
부품일 가능성이 높다.

Sparse MoE 구조에서 15B만 활성화되는 표준 버전은 이 전략적 맥락에서 이해된다. 310B 전체 파라미터
는 클라우드 서버에 적합하지만, 15B 활성화 파라미터는 고성능 엣지 추론에서 충분히 현실적인 수치다.
Xiaomi의 제조 규모를 고려하면 클라우드에서 엣지로의 점진적 배포 경로를 상상할 수 있다. 처음에는
클라우드 서비스로 시작해 데이터를 수집하고, 점진적으로 온디바이스 최적화 버전을 스마트폰, 스마트
스피커, 로봇 청소기 등 자사 기기에 탑재하는 수직 통합 전략이다.

MIT 라이선스로 외부 생태계를 키우면서 동시에 자사 하드웨어에 최적화된 AI 스택을 구축하는 이 전략은
Apple의 Neural Engine, Google의 TPU 전략과 유사하다. Apple이 A 시리즈 칩의 Neural Engine을
자사 ML 프레임워크(Core ML)에 최적화하고, 이를 통해 경쟁자가 따라오기 어려운 온디바이스 AI
성능을 달성한 것처럼, Xiaomi도 MiMo 시리즈를 자사 하드웨어와 공진화(co-evolution)시키는 방향을
추구할 수 있다.

단, Xiaomi는 Apple과 달리 하드웨어 설계에서 자체 AI 가속 칩이 없다는 근본적 차이가 있다. Qualcomm
Snapdragon에 의존하는 한 하드웨어 수직 통합의 완성도에는 한계가 있다. 이 한계를 극복하기 위해
Xiaomi가 자체 SoC 개발에 투자할지, 아니면 Qualcomm과의 긴밀한 협력으로 최적화를 달성할지는
중요한 변수다.

### 토큰 효율성이 오픈소스 AI의 새로운 경쟁 축이 된다

VentureBeat 테스트에서 MiMo-V2.5 Pro가 클로즈드 모델 대비 40~60% 적은 토큰으로 유사한 성능을
달성했다는 결과는 표면적 성능 수치보다 더 중요한 의미를 가진다. AI 서비스에서 운영 비용의 상당
부분이 토큰 처리 비용이기 때문에, 같은 결과를 내는 데 필요한 토큰 수가 절반이라면 운영 비용도
절반에 가까워진다.

이 효율성의 배경은 MOPD를 포함한 포스트 트레이닝 전략이다. 단순히 많은 토큰을 생성하는 것이
아니라 목표 달성에 필요한 최소한의 단계로 작업을 완료하는 능력이 강화됐다. 에이전트 작업에서
“최소 토큰으로 최대 효과”는 단순한 효율 지표가 아니라 모델의 계획 능력과 실행 정밀도를 반영하는
질적 지표다.

이 트렌드는 오픈소스 LLM의 경쟁 프레임을 바꾼다. 기존에는 “벤치마크 점수가 높은 모델”이
경쟁 우위를 가졌다면, 앞으로는 “같은 작업을 더 적은 비용으로 완료하는 모델”이 실제 채택
결정을 좌우할 것이다. 특히 에이전트 워크플로우처럼 많은 상호작용이 필요한 응용에서 토큰
효율은 클라우드 서비스 비용에 직접 영향을 미치고, 이는 고객 유치의 핵심 요인이 된다.

### 1M 컨텍스트와 진정한 장기 기억의 문제

MiMo-V2.5의 1M 토큰 컨텍스트 지원은 기술적으로 인상적이지만, 동시에 LLM의 근본적 한계를
드러낸다. 1M 토큰을 “기억”하는 것과 1M 토큰에서 올바른 정보를 “검색하고 추론”하는 것은
다른 문제다.

“잃어버린 중간(Lost in the Middle)” 현상이 대표적 문제다. LLM은 컨텍스트의 시작과 끝에
위치한 정보를 중간에 위치한 정보보다 훨씬 잘 활용하는 경향이 있다. 1M 토큰 컨텍스트에서
중간 50만 토큰 위치에 있는 중요한 정보가 올바르게 활용되는지는 별도로 검증해야 한다.
GraphWalks 평가를 1M 컨텍스트 벤치마크로 사용했다고 하나, 이것이 실제 다양한 응용에서의
장기 기억 신뢰성을 보장하지는 않는다.

더 근본적으로, 현재의 어텐션 메커니즘 기반 LLM은 컨텍스트를 처리할 때마다 전체를 다시 보는
구조다. 이는 인간의 장기 기억(뇌에 압축 저장, 필요시 검색)과 본질적으로 다른 방식이다.
1M 토큰을 진짜 기억하는 것이 아니라 매번 재처리하는 것이다. MiMo-V2.5가 1M 컨텍스트에서
KV 캐시를 6배 줄이는 데 성공했다 해도, 1M 토큰에 대한 어텐션 계산 자체는 여전히 O(n²)의
복잡도를 갖는 레이어들을 포함한다. 진정한 장기 기억 문제는 컨텍스트 길이 확장이 아니라
다른 아키텍처적 접근을 요구할 수 있다.
