# Gemma 4: 파라미터 대비 최고 성능의 오픈 모델

원문: <https://blog.google/innovation-and-ai/technology/developers-tools/gemma-4/>

## 요약

Google DeepMind가 2026년 4월 2일 Gemma 4를 발표했다.
"Byte for byte, the most capable open models"이라는 부제처럼
파라미터 대비 최고 성능을 핵심 가치로 내세운다.
Apache 2.0 라이선스로 공개되어 상업적 사용에 제한이 없다.

Gemini 3와 동일한 연구·기술 기반 위에 구축되었으며,
Gemini(프로프라이어터리)와 Gemma(오픈)를 조합하면
업계 최강의 도구 세트를 갖추게 된다는 것이 Google의 포지셔닝이다.

4개 크기로 제공된다.
E2B(유효 파라미터 2B)와 E4B(유효 파라미터 4B)는
엣지 디바이스용이고,
26B MoE(추론 시 3.8B만 활성화하여 빠른 토큰/초 제공)와
31B Dense(최대 품질, 파인튜닝 기반으로 적합)는
서버·PC급 워크로드를 담당한다.
31B는 비양자화 bfloat16 가중치가 80GB NVIDIA H100 한 장에 올라가고,
양자화 버전은 소비자 GPU에서도 실행 가능하다.
31B 모델은 Arena AI 텍스트 리더보드에서 오픈 모델 3위,
26B 모델은 6위를 기록했으며,
자기보다 20배 큰 모델을 능가한다고 주장한다.

전 모델이 네이티브 멀티모달을 지원한다.
가변 해상도 이미지와 비디오를 처리하며
OCR, 차트 분석에 강점이 있다.
E2B와 E4B는 음성 입력도 네이티브로 지원한다.
컨텍스트 윈도우는 엣지 모델 128K, 대형 모델 256K까지 지원한다.
학습 데이터에 140개 이상의 언어가 포함되었다.

에이전트 워크플로를 위한 기능도 내장했다.
함수 호출(function calling), 구조화된 JSON 출력,
네이티브 시스템 인스트럭션을 지원하며,
코드 생성 품질도 높아 로컬 AI 코드 어시스턴트로 활용 가능하다.

생태계 지원도 폭넓다.
Hugging Face, vLLM, llama.cpp, MLX, Ollama, NVIDIA NIM/NeMo,
LM Studio, Unsloth, SGLang, Keras 등 주요 프레임워크를
첫날부터 지원한다.
Google AI Studio(31B, 26B), AI Edge Gallery(E4B, E2B)에서
바로 실험할 수 있고,
Android Studio Agent Mode와 ML Kit GenAI Prompt API로
모바일 앱 개발에도 투입할 수 있다.
Gemma 시리즈는 첫 세대 출시 이후 4억 회 이상 다운로드되었고,
10만 개 이상의 커뮤니티 파인튜닝 변형이 존재한다.

파인튜닝 성공 사례로 INSAIT의 불가리아어 언어 모델(BgGPT)과
Yale 대학의 암 치료 경로 발견 연구(Cell2Sentence-Scale)가
언급되었다.

## 분석

### Per-Layer Embeddings(PLE)라는 아키텍처 혁신

소형 모델(E2B, E4B)에 적용된 PLE는
레이어나 파라미터를 늘리는 대신
각 디코더 레이어에 토큰별 소형 임베딩을 부여하는 기법이다.
모델 크기를 키우지 않고 성능을 끌어올리는 접근으로,
온디바이스 배포 효율을 극대화하려는 설계 의도가 명확하다.

### "유효 파라미터(Effective Parameters)"라는 명명 전략

E2B, E4B의 "E"는 Effective를 뜻한다.
실제 총 파라미터 수가 아니라 유효 파라미터 수를 모델명에 쓴 것은
"이 크기로 이 성능"이라는 효율성 서사를 강화하려는 브랜딩이다.
MoE 모델(26B-A4B)도 전체 26B 중 활성화되는 4B만 사용하므로
같은 맥락에서 이해할 수 있다.

### 엣지 디바이스를 향한 전략적 확장

E2B와 E4B는 스마트폰, Raspberry Pi, Jetson Orin Nano에서
오프라인으로 동작한다.
Google Pixel 팀과의 협업, Qualcomm·MediaTek과의 호환성 파트너십은
Google이 클라우드 API뿐 아니라 온디바이스 AI를
전략적 배포 채널로 본다는 증거다.
Android AICore Developer Preview에서 프로토타이핑하면
향후 Gemini Nano 4와의 전방 호환성도 확보된다.
하드웨어 쪽에서도 NVIDIA Jetson부터 Blackwell GPU,
AMD ROCm, Google Trillium/Ironwood TPU까지 폭넓게 지원한다.

### Apache 2.0으로의 라이선스 전환

이전 Gemma 모델은 Google의 자체 라이선스를 사용했지만
Gemma 4는 Apache 2.0으로 전환했다.
Hugging Face CEO Clément Delangue가
"거대한 마일스톤"이라고 평가한 것처럼,
커뮤니티가 명시적으로 요구한 변화이기도 하다.
Llama의 커스텀 라이선스, Mistral의 Apache 2.0 전략과 비교하면,
Google이 오픈소스 생태계에서의 채택률을
라이선스 완화로 가속하려는 판단을 한 것이다.
데이터·인프라·모델에 대한 완전한 통제권(디지털 주권)을
사용자에게 부여한다는 점도 엔터프라이즈 도입에 유리하다.

## 비평

### 벤치마크 선택의 편향 가능성

"Arena AI 텍스트 리더보드 오픈 모델 3위"라는 표현은
특정 벤치마크에서의 순위를 강조하면서
다른 벤치마크(예: 코딩, 에이전트 태스크, 장문 이해)에서의
성능은 명시하지 않았다.
"20배 큰 모델을 능가한다"는 주장도
어떤 태스크에서 어떤 모델과 비교한 것인지 구체적이지 않다.

### 엣지 모델의 실질적 성능 한계

E2B와 E4B가 스마트폰에서 돌아간다는 것은 인상적이지만,
실제 사용 시나리오에서의 응답 속도, 품질, 배터리 소모량 등
실용적 지표가 빠져 있다.
"near-zero latency"라는 표현은
추론 속도가 아니라 네트워크 지연이 없다는 뜻에 가깝고,
온디바이스 추론 자체의 지연은 다른 문제다.

### 멀티모달 성능의 검증 부족

전 모델이 이미지, 비디오, OCR을 지원한다고 하지만
각 모달리티에서의 성능을 독립적으로 검증한 벤치마크가
발표문에 충분히 포함되지 않았다.
특히 비디오 이해와 음성 입력은
텍스트에 비해 평가 기준이 덜 확립된 영역이어서
"네이티브 지원"이 실무에서 어떤 수준인지 판단하기 어렵다.

### 140개 언어 지원의 실질적 의미

학습 데이터에 140개 이상의 언어가 포함되었다고 하지만,
언어별 학습 데이터 비율과 성능 차이에 대한 정보가 없다.
영어 중심 벤치마크에서의 3위가
한국어, 아랍어, 스와힐리어 등에서도
비례적 성능을 보장하는 것은 아니다.

## 인사이트

### "파라미터당 지능"이 모델 경쟁의 새로운 축이 되고 있다

GPT-4 시대까지 모델 경쟁은 "더 크면 더 좋다"였다.
Gemma 4는 이 프레이밍을 정면으로 뒤집어
"같은 크기에서 더 똑똑하다"를 핵심 가치로 내세운다.
이는 단순한 마케팅 변화가 아니라
AI 산업의 물리적 제약을 반영한 구조적 전환이다.

GPU 비용, 전력 소비, 추론 지연은 모두 모델 크기에 비례한다.
클라우드에서 대형 모델을 호출하는 비용이
결국 토큰 단가로 사용자에게 전가되므로,
같은 품질을 더 작은 모델로 달성하는 것은
추론 비용을 직접적으로 줄이는 것과 같다.
"파라미터당 지능"의 경쟁은 곧 "토큰당 비용"의 경쟁이다.

이 흐름이 계속되면 모델 선택의 기준이 바뀐다.
"가장 똑똑한 모델"이 아니라
"이 작업에 필요한 최소 크기의 충분히 똑똑한 모델"을 고르는 것이
엔지니어링 판단의 핵심이 된다.
Gemma 4의 4단계 크기 라인업은 이 판단을 구조적으로 지원한다.

### 오픈 모델의 경쟁 무대가 클라우드에서 엣지로 이동하고 있다

GPT-5, Claude 같은 프론티어 모델은 클라우드 API로만 접근 가능하다.
오픈 모델이 클라우드에서 이 모델들과 정면 경쟁하는 것은
성능과 생태계 모두에서 불리하다.
Gemma 4가 엣지 디바이스에 집중하는 것은
"클라우드 모델이 갈 수 없는 곳"에서 승부하겠다는 전략이다.

스마트폰, IoT 디바이스, 자동차, 로봇 등
네트워크 접속이 불안정하거나 지연이 치명적인 환경에서는
온디바이스 추론이 유일한 선택지다.
Qualcomm, MediaTek과의 파트너십은
이 시장을 하드웨어 수준에서 선점하려는 움직임이다.

이 전략이 성공하면 AI 생태계에 흥미로운 분업이 생긴다.
복잡한 추론은 클라우드의 대형 모델이,
실시간 반응은 엣지의 소형 모델이 담당하는 계층 구조다.
Gemma 4의 E2B/E4B가 이 계층의 하단을 차지하려는 것이고,
Google은 상단(Gemini)과 하단(Gemma) 모두를 장악하여
전체 스택을 소유하려 한다.

### Apache 2.0은 "오픈 AI" 정의 전쟁에서의 결정적 한 수다

Meta의 Llama는 "오픈"을 표방하면서도
커스텀 라이선스로 상업적 사용에 조건을 달았다.
이 모호함은 "Llama가 진정한 오픈소스인가"라는
커뮤니티 논쟁을 지속시켰다.
Gemma 4가 Apache 2.0을 선택한 것은
이 논쟁에서 확실한 진영을 선언한 것이다.

Apache 2.0은 소프트웨어 업계에서 가장 관대한 라이선스 중 하나로,
수정, 재배포, 상업적 사용에 제한이 없다.
스타트업이 Gemma 4 위에 상업 제품을 구축할 때
법적 리스크가 사실상 제로라는 의미이므로,
생태계 확장에는 가장 유리한 선택이다.

이 결정의 2차 효과는 경쟁사에 대한 압력이다.
Gemma가 Apache 2.0으로 높은 성능을 제공하면,
제한적 라이선스를 가진 다른 오픈 모델은
"왜 더 제한적인 라이선스를 써야 하는가"라는
정당화 부담을 안게 된다.
4억 회 다운로드와 10만 개 커뮤니티 변형이라는 수치는
이 전략이 이미 효과를 내고 있음을 보여준다.
