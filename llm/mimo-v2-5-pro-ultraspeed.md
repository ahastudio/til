# MiMo-V2.5-Pro-UltraSpeed: 초당 1000토큰을 생성하는 1조 파라미터 모델

원문: <https://mimo.xiaomi.com/blog/mimo-tilert-1000tps>

HN 토론: <https://news.ycombinator.com/item?id=48446639> (605점, 454개 댓글)

GeekNews: <https://news.hada.io/topic?id=30307>

## 요약

샤오미(Xiaomi)와 TileRT가 공동 발표한 MiMo-V2.5-Pro-UltraSpeed는 1조(1T) 파라미터 모델에서 디코딩 속도 초당 1000토큰을 최초로 달성했다.
표준 8-GPU 서버 클러스터에서 실제 최대 1200 tokens/s를 기록했다.

기술 혁신은 세 요소의 조합이다.

첫째, **FP4 양자화**다.
Mixture-of-Experts(MoE) 파라미터를 선택적으로 4비트 정밀도로 줄이면서 다른 부분은 전체 정밀도를 유지한다.
메모리 풋프린트와 대역폭 요구사항을 극적으로 낮췄다.

둘째, **DFlash 추측적 디코딩**이다.
블록 수준 마스크 예측 방식으로 드래프트 모델이 단일 포워드 패스에서 마스크된 위치 블록 전체를 채울 수 있다.
순차적 토큰 생성을 병렬 처리로 전환하는 것이다.

셋째, **TileRT 시스템 아키텍처**다.
퍼시스턴트 커널 실행과 워프 특수화로 전통적 오퍼레이터 오버헤드와 마이크로초 단위 병목을 제거한다.

가격은 표준 MiMo-V2.5-Pro의 3배이지만 출력 속도는 약 10배 빠르다.
2026년 6월 9~23일(UTC+8) 시간 제한 API 액세스와 하루 10건 대기열, 30분 세션으로 제한된 무료 채팅을 제공했다.

## 분석

### 속도가 새로운 애플리케이션 패러다임을 여는 임계값이 있다

인퍼런스 속도의 의미는 선형적이지 않다.
초당 100토큰과 300토큰의 차이는 체감 속도의 차이지만, 초당 1000토큰은 새로운 사용 패턴을 가능하게 한다.

실시간 코딩 에이전트에서 여러 접근법을 병렬로 생성하고 가장 좋은 것을 선택하는 것이 가능해진다.
인간이 읽는 속도보다 빠르게 텍스트가 생성되면, 사용자 인터페이스 설계 자체가 달라진다.
스트리밍 표시가 아니라 순간적 완성이 UX의 기본이 될 수 있다.
의료 이미지 분석, 고빈도 거래 신호 같은 밀리초 단위 결정이 필요한 도메인이 LLM 활용 범위에 들어온다.

이것은 단순히 “더 빠른 같은 것”이 아니라 “가능해지는 것의 범위가 달라지는” 변화다.

### 1조 파라미터에서의 속도 달성이 의미하는 경쟁 구도 변화

Groq, Cerebras 같은 특수 하드웨어 기업들은 소형 모델에서의 고속 인퍼런스로 포지셔닝했다.
1조 파라미터 모델을 표준 GPU에서 초당 1000토큰으로 실행하는 것이 가능해지면, 특수 하드웨어의 경쟁 우위가 줄어든다.

[GeekNews 댓글](https://news.hada.io/topic?id=30307)에서 Groq, Cerebras와의 속도 비교가 논의됐다.
특수 하드웨어 없이 표준 GPU 클러스터에서 이 속도를 달성한다는 것은, 속도를 위해 비표준 인프라에 투자하지 않아도 되는 상황이 된다는 것을 의미한다.
인퍼런스 속도의 민주화가 특수 하드웨어 시장의 구조를 바꿀 가능성이 있다.

### 가격-속도 트레이드오프의 경제학

3배 가격에 10배 속도라는 조합의 경제적 의미는 사용 패턴에 따라 다르다.
속도가 중요하지 않은 배치 처리 작업에서는 비용 효율성이 낮다.
그러나 속도가 사용자 경험에 직접 영향을 미치는 실시간 애플리케이션에서는, 10배 빠른 응답이 3배 비용을 정당화할 수 있다.

더 중요한 것은 속도가 기능의 전제 조건이 되는 경우다.
초당 100토큰에서는 불가능한 애플리케이션이 초당 1000토큰에서는 가능하다면, 이 속도가 아닌 다른 속도는 비교 대상이 아니다.
가격은 기능 자체가 가능한지를 묻는 단계가 지난 후의 질문이 된다.

## 비평

### FP4 양자화의 품질 영향이 충분히 공개되지 않는다

4비트 양자화는 일반적으로 모델 품질에 영향을 미친다.
어떤 태스크에서, 어느 정도의 품질 저하가 발생하는지가 이 모델의 실용성을 판단하는 핵심 정보다.

발표에서 속도 달성에 초점을 맞추면서 품질 비교가 충분히 제시되지 않는다.
코딩, 추론, 지식 기반 태스크에서의 표준 벤치마크 비교가 없으면, “초당 1000토큰이지만 품질이 절반”과 “초당 1000토큰이고 품질은 거의 유지”를 구분할 수 없다.
속도 돌파를 선언하는 발표에서 품질 트레이드오프를 명시하지 않는 것은 선택적 공개다.

### 시간 제한 API의 마케팅 구조

6월 9~23일의 시간 제한, 하루 10건 대기열, 30분 세션 제한은 기술적 필요보다 마케팅 설계에 가까워 보인다.
희소성을 인위적으로 만들어 관심을 집중시키고, 제한된 기간 내에 미디어 커버리지를 최대화하는 방식이다.

이것이 나쁜 것은 아니지만, 독립적인 품질 검증이 이 기간에 이뤄지기 어렵게 만든다.
제한된 액세스 내에서 다양한 벤치마크를 수행하고 표준 MiMo-V2.5-Pro와 체계적으로 비교하기 어렵기 때문이다.
“초당 1000토큰”의 주장은 검증 없이 언론에 확산되고, 제한 기간이 끝난 후에야 더 체계적인 평가가 가능해진다.

## 인사이트

### 추측적 디코딩이 표준화되는 흐름에서의 의미

DFlash 추측적 디코딩은 Perry(더 작은 드래프트 모델이 토큰을 예측하고 큰 모델이 검증)의 원리를 블록 수준으로 확장한다.
이 기법 자체는 새로운 것이 아니지만, 1조 파라미터 모델에 적용하면서 유의미한 속도 향상을 달성한 것이 이번 발표의 핵심이다.

추측적 디코딩이 대형 모델의 표준 추론 기법으로 자리 잡는다면, 드래프트 모델의 품질이 전체 파이프라인 속도를 결정하는 병목이 된다.
소형 고품질 모델을 만드는 것이 대형 모델 속도를 높이는 간접적 방법이 된다.
이것은 소형 모델 연구와 대형 모델 연구가 분리된 경쟁이 아니라 상호 의존적 생태계를 구성한다는 것을 의미한다.

### 중국 AI 기업의 인프라 혁신 패턴

샤오미의 이번 발표는 중국 AI 기업들이 미국의 최첨단 GPU 접근 제한 속에서 소프트웨어 수준의 효율화에 집중하는 패턴을 보여준다.
하드웨어 제약이 혁신의 방향을 결정하는 방식이다.

최신 NVIDIA 칩에 대한 접근이 제한된 환경에서, 동일하거나 구형 하드웨어에서 더 효율적으로 실행하는 방법을 찾는 것이 경쟁 전략이 된다.
FP4 양자화, DFlash 디코딩, TileRT 커널 최적화는 모두 하드웨어 제약 내에서의 소프트웨어 혁신이다.
이것이 장기적으로 인퍼런스 효율화 기술의 수준을 높이는 방향으로 작용할 수 있다.
