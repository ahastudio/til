# Qwen3.6-35B-A3B -- 3B 활성 파라미터로 Dense 30B급 성능을 달성한 MoE 모델

원문: <https://qwen.ai/blog?id=qwen3.6-35b-a3b>

## 요약

Alibaba의 Qwen 팀이 Qwen3.6-35B-A3B를 발표했다.
총 35B 파라미터 중 추론 시 3B만 활성화되는
MoE(Mixture of Experts) 아키텍처를 사용하며,
전체 파라미터의 약 8.6%만으로 27~31B Dense 모델급 성능을 달성한다.

텍스트, 이미지, 비디오를 처리하는 네이티브 멀티모달 모델이다.
Thinking 모드(추론 과정을 보여주는 모드)와
Non-thinking 모드(직접 응답하는 모드)를 모두 지원한다.

벤치마크 성능이 인상적이다.
SWE-bench Verified에서 73.4, Terminal-Bench 2.0에서 51.5를 기록하여
훨씬 큰 Dense 모델들을 능가했다.
MMMU 81.7, RealWorldQA 85.3으로
유료 서비스인 Claude Sonnet 4.5를 넘어서는 수치를 보였다.

Claude Code, OpenClaw 등 Anthropic API 프로토콜과 호환되며,
API는 “coming soon” 상태로 아직 공개되지 않았다.
다만 범용 에이전트 태스크에서는 35.6점으로
이전 모델(41.8)보다 하락한 부분도 있다.

## 분석

### 8.6% 활성화율의 의미

전체 35B 파라미터 중 3B만 활성화한다는 것은
추론 시 계산량이 3B 모델 수준이라는 뜻이다.
메모리에는 35B 전체를 올려야 하지만
실제 연산 비용은 3B에 비례하므로
추론 속도와 처리량에서 큰 이점을 갖는다.
이는 MoE 아키텍처의 핵심 설계 원리인
“저장은 크게, 계산은 작게”를 극단까지 밀어붙인 사례다.

### 코드 및 터미널 태스크에서의 강점

SWE-bench Verified 73.4와 Terminal-Bench 2.0 51.5는
소프트웨어 엔지니어링과 터미널 환경 태스크에서
이 모델이 특히 강하다는 것을 보여준다.
3B 활성 파라미터로 이 수준의 코딩 성능을 내는 것은
MoE의 전문가 라우팅이 코드 관련 전문가를
효과적으로 선택하고 있음을 시사한다.

### 멀티모달 벤치마크의 성과와 맥락

MMMU 81.7과 RealWorldQA 85.3은
Claude Sonnet 4.5를 넘어서는 수치로 발표되었다.
다만 이 비교는 특정 벤치마크에 한정된 것이며,
실제 사용 환경에서의 멀티모달 품질은
벤치마크 점수와 괴리가 있을 수 있다.
특히 비디오 이해 능력에 대한 벤치마크는
별도로 제시되지 않았다.

### Thinking/Non-thinking 이중 모드

추론 과정을 명시적으로 보여주는 Thinking 모드와
직접 답변하는 Non-thinking 모드의 이중 구조는
사용 사례에 따른 유연한 배포를 가능하게 한다.
코딩이나 수학 같은 복잡한 추론이 필요한 태스크에서는
Thinking 모드가 정확도를 높이고,
단순 질의응답에서는 Non-thinking 모드로 지연을 줄일 수 있다.

### 범용 에이전트 성능의 하락

범용 에이전트 태스크에서 35.6점으로
이전 모델의 41.8점보다 하락한 것은 주목해야 할 부분이다.
코딩과 멀티모달에서의 강세와 대조적으로
이 수치는 특정 영역에 최적화하면서
범용성이 희생되었을 가능성을 시사한다.

## 비평

### 벤치마크 선별의 편향

발표에서 강조된 벤치마크는 모델이 강점을 보이는 영역에
집중되어 있다.
SWE-bench, Terminal-Bench, MMMU, RealWorldQA는
모두 이 모델이 우수한 점수를 기록한 항목이다.
반면 범용 에이전트 태스크의 하락(41.8 → 35.6)은
상대적으로 덜 강조되었다.
벤치마크를 선별적으로 보여주는 것은
모든 모델 발표에서 흔한 관행이지만,
전체 성능 프로파일을 판단하려면
약점 영역의 수치도 동등하게 검토해야 한다.

### “Claude Sonnet 4.5를 넘어선다”는 주장의 한계

MMMU와 RealWorldQA에서 Claude Sonnet 4.5를 넘었다는 비교는
두 개의 특정 벤치마크에 한정된 것이다.
Claude Sonnet 4.5는 범용 추론, 장문 이해, 에이전트 태스크 등
훨씬 넓은 영역에서 평가받는 모델이다.
벤치마크 두 개의 점수로 전체 모델 우위를 함축하는 것은
마케팅으로서는 효과적이지만 기술적으로는 오해를 유발할 수 있다.

### API 미공개 상태의 불확실성

“coming soon”이라는 API 상태는
외부 개발자가 모델을 직접 검증할 수 없다는 뜻이다.
발표된 벤치마크 수치는 자체 평가 결과이며,
독립적인 제3자 검증은 API 또는 가중치 공개 이후에야 가능하다.
Claude Code나 OpenClaw와의 호환성도
실제 사용 경험이 축적되기 전까지는 주장에 불과하다.

## 인사이트

### MoE의 극단적 희소성은 AI 추론 비용의 구조적 전환을 예고한다

전체 파라미터의 8.6%만 활성화한다는 설계는
MoE 아키텍처의 희소성을 새로운 수준으로 끌어올린 것이다.
Gemma 4의 26B-A4B(15.4% 활성화)와 비교해도
활성화 비율이 절반에 가깝다.
이 추세는 “모델의 지식 용량과 추론 비용을 분리한다”는
MoE의 근본 약속이 점점 현실화되고 있음을 보여준다.

이 방향의 2차 효과는 추론 비용 구조의 변화다.
Dense 모델에서는 성능을 두 배로 올리면 비용도 대략 두 배가 된다.
MoE에서는 전문가를 추가하여 지식 용량을 키우면서도
활성 파라미터 수를 유지할 수 있으므로
성능-비용 곡선이 근본적으로 달라진다.
3B 수준의 연산 비용으로 30B급 성능을 제공한다면,
토큰 단가가 10분의 1 수준으로 떨어질 수 있다.

역사적으로 컴퓨팅에서 비용이 10배 이상 떨어지면
새로운 사용 패턴이 창발한다.
클라우드 컴퓨팅이 서버 비용을 10배 줄였을 때
스타트업 생태계가 폭발했고,
스마트폰이 컴퓨팅 비용을 10배 줄였을 때
모바일 앱 경제가 탄생했다.
MoE의 극단적 희소화가 추론 비용을 구조적으로 낮추면,
현재는 비용 때문에 불가능한 AI 사용 패턴이 가능해진다.
상시 가동 AI 에이전트, 실시간 대화 기반 인터페이스,
대량의 문서를 동시에 처리하는 워크플로 등이
경제적으로 성립하는 시점이 앞당겨진다.

### 코딩 특화 성능은 AI 에이전트 시대의 핵심 경쟁 지표가 되고 있다

Qwen3.6이 SWE-bench Verified와 Terminal-Bench에서
대형 Dense 모델을 능가한다는 사실은
코딩 벤치마크가 모델 경쟁의 최전선이 되었음을 보여준다.
2024년까지 모델 경쟁은 MMLU, HellaSwag 같은
범용 지식·추론 벤치마크가 중심이었다.
2025년 이후 경쟁 축이 SWE-bench, Terminal-Bench 같은
실무 코딩 태스크로 이동한 것은 우연이 아니다.

이 전환의 배경에는 AI 에이전트의 부상이 있다.
Claude Code, Cursor, Windsurf 같은 코딩 에이전트가
개발자 워크플로의 핵심 도구가 되면서,
모델의 코딩 능력은 직접적인 제품 경쟁력으로 환산된다.
범용 벤치마크에서 1점 올리는 것보다
SWE-bench에서 1점 올리는 것이
실제 사용자 경험에 미치는 영향이 크다.

이 추세의 구조적 함의는 모델 평가 기준의 파편화다.
“최고의 모델”이라는 단일 순위가 의미를 잃고,
“코딩에 최고” “멀티모달에 최고” “장문 이해에 최고”처럼
영역별 챔피언이 공존하는 구도가 된다.
Qwen3.6은 “3B 활성 파라미터로 코딩에 최고”라는
매우 구체적인 포지션을 점유하려 하고 있다.
이는 범용 최강을 다투는 GPT-5, Claude 5와는
다른 차원의 경쟁이다.

### Anthropic API 호환성은 중국 AI의 글로벌 생태계 전략을 보여준다

Qwen3.6이 Claude Code, OpenClaw 등
Anthropic API 프로토콜과 호환된다는 사실은
기술적 세부사항 이상의 전략적 의미를 갖는다.
중국에서 개발된 모델이 미국 기업의 API 규격을
적극적으로 지원하는 것은
글로벌 AI 생태계에서의 채택률을 극대화하려는 판단이다.

이 전략은 안드로이드 생태계의 초기 전략과 유사하다.
안드로이드는 자체 생태계를 처음부터 구축하는 대신
기존 Java 개발자 생태계와 호환되는 길을 택했다.
Qwen이 Anthropic API 호환성을 내세우는 것은
Claude Code 사용자가 모델만 교체하면
기존 워크플로를 그대로 유지할 수 있게 하려는 것이다.
전환 비용을 제로에 가깝게 만들어
생태계 진입 장벽을 낮추는 클래식한 플랫폼 전략이다.

2차 효과로는 AI 모델 시장의 상품화(commoditization)가 가속된다.
API 규격이 사실상 표준화되면
모델은 교체 가능한 부품이 되고,
경쟁 요소는 성능과 가격으로 수렴한다.
Qwen3.6이 3B 활성 파라미터로 30B급 성능을 제공하면서
Anthropic API와 호환된다면,
“더 싸고 더 빠른 Claude 대체재”로 포지셔닝할 수 있다.
이는 Anthropic에게는 가격 하방 압력이 되고,
사용자에게는 선택지 확대라는 이점이 된다.

장기적으로 이 호환성 전략이 확산되면
AI 모델 시장은 클라우드 컴퓨팅 시장과 비슷한 구조로 수렴할 수 있다.
AWS가 만든 API 규격을 Azure, GCP가 호환하면서
멀티클라우드가 가능해진 것처럼,
OpenAI나 Anthropic이 만든 API 규격을
Qwen, Llama, Mistral이 호환하면서
멀티모델 전략이 표준이 되는 미래다.
