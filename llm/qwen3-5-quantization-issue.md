# Qwen3.5 모델 양자화, 커뮤니티 버전 성능 문제의 원인

원문: <https://twitter.com/Brooooook_lyn>

## 요약

Unsloth가 150개 이상의 벤치마크 실험을 통해 Qwen3.5 모델의 커뮤니티 양자화 버전에서
도구 호출 오류, 무의미한 출력, 환각 현상이 발생하는 근본 원인을 규명했다.

문제의 핵심은 균일 양자화(uniform quantization)의 구조적 한계다. 모든 레이어에 동일한
비트 수를 적용하면 중요하지 않은 부분에 정밀도를 낭비하고 정작 민감한 레이어를 파괴한다.
Qwen3.5의 `linear_attn.out_proj` 레이어는 4비트 압축 시 정보 손실에 극도로 민감하다.
이 레이어의 민감도는 출력 레이어 대비 약 120배에 달한다.

Unsloth가 제시한 해결책은 혼합 비트 양자화(mixed-bit quantization)다. MLP 레이어는 3비트,
어텐션 레이어는 5비트, 출력 레이어는 bf16 전체 정밀도를 유지한다. 캘리브레이션 데이터도
기존에 일반적으로 사용하던 위키피디아 텍스트 대신 실제 사용 환경(대화, 코딩, 도구 호출)을
반영한 데이터를 사용한다.

결론적으로 AI 모델 압축은 단순한 크기 축소가 아니라 내부 구조에 대한 깊은 이해를 요구한다.
커뮤니티에서 만들어지는 양자화 모델들이 흔히 쓰는 균일 양자화 방식은, 모델 구조에 따라
성능을 크게 훼손할 수 있다.

## 분석

### 균일 양자화의 구조적 한계

양자화는 부동소수점 가중치를 낮은 비트 정수로 변환해 모델 크기와 추론 속도를 개선하는
기법이다. 균일 양자화는 모든 레이어에 동일한 비트 수를 적용하는 가장 단순한 방식이다.
이 방식이 Transformer 기반 모델에서 문제를 일으키는 이유는, 레이어마다 가중치 분포와
정보 중요도가 크게 다르기 때문이다.

Qwen3.5의 경우 `linear_attn.out_proj` 레이어가 특히 취약하다. 이 레이어는 멀티헤드
어텐션의 출력을 통합하는 역할을 하는데, 4비트로 압축하면 세밀한 어텐션 패턴 정보가
손실된다. 민감도가 출력 레이어 대비 120배라는 수치는, 이 레이어의 가중치 변화가
모델 출력에 미치는 영향이 그만큼 크다는 의미다.

### 캘리브레이션 데이터의 중요성

양자화 과정에서 캘리브레이션 데이터는 어떤 정보를 보존할지 결정하는 기준이 된다.
위키피디아 텍스트로 보정하면 일반 지식 응답에는 적합하지만, 도구 호출이나 코딩처럼
특수한 출력 형식을 요구하는 작업에서는 최적화가 되지 않는다. 실제 사용 환경을 반영한
캘리브레이션 데이터가 필수인 이유다.

이것은 양자화가 단순한 압축 알고리즘이 아니라 모델의 사용 목적에 맞게 조정되어야 하는
과정임을 보여준다. "범용" 양자화 모델이라는 개념 자체가 제한적일 수 있다.

## 비평

### 강점

150개 이상의 벤치마크라는 실험 규모가 신뢰성을 높인다. 단순한 관찰이 아니라 체계적
실험을 통해 원인을 규명했다는 점에서, Unsloth의 분석은 커뮤니티에 실질적 기여를 한다.

혼합 비트 양자화라는 구체적 해결책을 제시하고 레이어별 비트 수까지 공개했다. 이를 통해
커뮤니티가 검증하고 적용할 수 있는 실용적 정보가 된다.

### 약점

트위터 포스트 기반의 발표이므로 전체 실험 방법론과 데이터가 공개되지 않았을 가능성이 높다.
150개 벤치마크의 구체적 내용, 비교 방법, 통계적 유의성 등을 독립적으로 검증하기 어렵다.

Qwen3.5에 특화된 발견이 다른 모델에 일반화될 수 있는지 불분명하다. 각 모델 아키텍처마다
민감한 레이어가 다를 수 있으므로, 이 분석이 Qwen3.5 외 모델에도 적용되는지 추가 검증이 필요하다.

## 인사이트

### 오픈소스 모델 생태계의 품질 문제

허깅페이스(HuggingFace) 등 모델 허브에는 커뮤니티 양자화 모델이 수없이 올라온다.
이 모델들의 품질을 검증하는 표준화된 프로세스가 없다. 일반 사용자는 모델 설명에 적힌
"quantized by X" 정보만 보고 사용 여부를 결정한다.

Unsloth의 분석이 보여주듯, 같은 모델의 양자화 버전 간 품질 차이가 도구 호출 오류나
환각 현상으로 나타날 수 있다. 이것은 단순한 성능 저하가 아니라 기능 자체가 작동하지
않는 수준의 차이다. 사용자가 이를 인식하지 못하면 "LLM은 도구 호출에 불안정하다"는
잘못된 결론에 도달한다. 실제 문제는 LLM이 아니라 양자화 품질이었는데도.

이 문제를 해결하려면 양자화 모델의 표준화된 품질 검증 체계가 필요하다. 특히 도구 호출,
코딩, 지시 따르기 같은 특수 기능에 대한 별도 벤치마크가 필요하다.

### 모델 압축 전문성이 희소 자원이 된다

대형 모델을 실용적으로 실행하려면 양자화가 필수적이다. 그러나 이 분석이 보여주듯 양자화는
단순 작업이 아니다. 레이어별 민감도 분석, 혼합 비트 전략 설계, 사용 목적에 맞는 캘리브레이션
데이터 구성이 모두 전문 지식을 요구한다.

이 전문성은 희소하다. 현재 Unsloth, llama.cpp 팀, 일부 연구 그룹만이 이 수준의 최적화를
수행할 수 있다. 모델은 공개되어 있지만 최적화 기술은 소수에게 집중된다. 이는 오픈소스
모델 민주화의 한계다. 모델 접근성은 개선되었지만, 모델을 실제로 잘 사용하기 위한 기술은
여전히 집중되어 있다.

장기적으로 이 전문성이 자동화되거나 더 광범위하게 보급되어야 오픈소스 AI의 실질적 민주화가
이루어진다. Unsloth 같은 팀의 기여가 단기적으로 이 격차를 메우고 있다.

### 균일 양자화의 종말

이 분석은 더 넓은 추세의 일부다. GPTQ, AWQ, GGUF 등 다양한 양자화 형식이 등장한 이유
중 하나가 균일 양자화의 한계를 극복하기 위해서였다. 혼합 비트 양자화는 그 다음 단계다.

흥미로운 점은 이 최적화가 모델 구조에 대한 깊은 이해를 요구한다는 것이다. 어떤 레이어가
중요한지는 수학적으로 분석할 수 있지만, 실제 작업 성능에 미치는 영향은 실험을 통해서만
알 수 있다. Unsloth가 150개 벤치마크를 실행한 이유가 여기에 있다. 이것은 자동화되기
어려운 영역이며, 당분간 전문가 집단의 경험 지식이 중요한 역할을 한다.
