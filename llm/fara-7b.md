# Fara-7B

<https://github.com/microsoft/fara>

<https://arxiv.org/abs/2511.19663>

<https://huggingface.co/microsoft/Fara-7b>

## 소개

Fara-7B는 Microsoft가 개발한 7B 파라미터 규모의 소형 언어 모델(SLM)로,
컴퓨터 사용(Computer Use) 작업을 위해 특별히 설계된 에이전트 모델이다.
텍스트 응답 대신 마우스 클릭, 키보드 입력, 스크롤 등 실제 컴퓨터 인터페이스
조작을 통해 사용자 대신 웹 기반 작업을 자동화한다.

핵심 특징은 다음과 같다.

- **시각 기반 동작**: 접근성 트리(accessibility tree)나 별도 파싱 모델 없이
  웹페이지를 직접 인식해 좌표를 예측하고 클릭·타이핑·스크롤을 수행한다.
- **온디바이스 배포 가능**: 7B 파라미터의 소형 모델로 로컬 배포가 가능해
  레이턴시 감소와 프라이버시 보호가 가능하다.
- **효율적인 작업 완수**: 유사한 모델이 평균 약 41 스텝을 소비하는 데 비해
  Fara-7B는 태스크당 평균 약 16 스텝만 사용한다.

[Qwen2.5-VL-7B](https://arxiv.org/abs/2502.13923)를 베이스로
[Magentic-One](https://www.microsoft.com/en-us/research/articles/magentic-one-a-generalist-multi-agent-system-for-solving-complex-tasks/)
멀티 에이전트 프레임워크로 생성한 145K 트라젝토리로 지도 학습(SFT) 파인튜닝을
진행했다.

## 성능

WebVoyager, Online-Mind2Web, DeepShop, WebTailBench 4개 벤치마크에서
동급 사이즈 모델 중 최고 성능을 달성했다.

| GLM-4.1V-9B-Thinking        | 9B       | 66.8       | 33.9       | 32.0     | 22.4         |
| OpenAI computer-use-preview | -        | 70.9       | 42.9       | 24.7     | 25.7         |

## 벤치마크

### WebTailBench

기존 벤치마크에서 과소 대표되거나 누락된 11가지 실세계 태스크 유형에 집중한
609개 태스크 평가셋이다. 항공편, 호텔, 레스토랑 예약, 쇼핑, 구인구직, 부동산
등 단일 사이트 작업 8종과 멀티스텝 혹은 크로스 사이트 작업 3종으로 구성된다.

Fara-7B는 컴퓨터 사용 모델 중 모든 카테고리에서 최고 성능을 기록했다 (매크로
평균 38.4%).

### CUAVerifierBench

에이전트 트라젝토리를 평가하는 “판사(verifier)” 모델 자체를 평가하는 벤치마크다.
Fara-7B 트라젝토리에 대한 사람 주석을 포함하며, Universal Verifier(`MMRubricAgent`)와
레거시 검증기 출력을 함께 수록해 검증기-사람 일치도를 측정할 수 있다.

## 사용법

```bash
# 저장소 클론 및 환경 설정
git clone https://github.com/microsoft/fara.git
cd fara
python3 -m venv .venv
source .venv/bin/activate
pip install -e .[vllm]
playwright install
```

vLLM으로 모델을 서빙한 뒤 CLI로 실행할 수 있다.

```bash
# 모델 서빙
vllm serve "microsoft/Fara-7B" --port 5000 --dtype auto

# 태스크 실행
fara-cli --task "whats the weather in new york now"
```

GPU가 없는 경우 Azure Foundry에서 관리형으로 배포하거나, LM Studio / Ollama를
통해 GGUF 버전을 로컬에서 실행할 수 있다.

## 분석

Fara-7B의 핵심 기술적 선택은 접근성 트리 의존을 제거한 것이다. 대부분의 웹
에이전트는 DOM 구조를 파싱해 레이블을 붙이는 별도 단계(Set of Marks 등)를
거치는데, Fara-7B는 스크린샷을 직접 인식하고 픽셀 좌표를 예측해 클릭한다.
이 방식은 모델이 단일 추론 패스로 “보고 → 행동”을 수행할 수 있게 해서 전체
스텝 수를 절반 이하로 줄이는 효과를 가져왔다.

학습 데이터 생성에는 Magentic-One 멀티 에이전트 프레임워크가 사용됐다. 대형
모델이 웹 브라우저를 조작하는 과정을 대량으로 기록해 145K 트라젝토리를 구성한
것으로, 다양한 웹사이트, 태스크 유형, 난이도를 포괄하도록 설계됐다. 이 합성
데이터 기반 접근법은 소형 모델이 특정 도메인에서 대형 모델에 근접하거나 능가할
수 있음을 보여준다.

평가 인프라 측면에서 BrowserBase를 활용한 실제 브라우저 세션 관리, 5회까지의
자동 재시도, 100 스텝 예산 상한 등 재현 가능한 온라인 평가 파이프라인을
구축한 점이 주목할 만하다. 특히 WebTailBench V2는 2025년 11월에 만료된
날짜 기반 태스크를 갱신해 벤치마크 스테일니스 문제를 적극적으로 해결했다.

## 비평

성능 수치는 인상적이지만 한계를 직시해야 한다. WebTailBench에서 Fara-7B의
38.4% 성공률은 여전히 GPT-4o-0513 기반 SoM 에이전트의 60.4%와 큰 격차가
있다. “컴퓨터 사용 모델 중 최고”라는 표현이 실제 실용성을 과장할 수 있다.

항공편, 부동산, 구인구직 등 복잡한 도메인에서 30% 미만의 성공률은 상용 배포에는
여전히 부족한 수준이다. 특히 BrowserBase 없이는 봇 트래픽 차단으로 인해 상당
수의 트라젝토리가 오류로 중단된다는 점에서, 실세계 환경에서의 신뢰성 문제가
남아 있다.

보안 측면의 경고도 유의할 필요가 있다. Microsoft 스스로 “실험적 릴리스”이며
샌드박스 환경에서 실행하고, 민감한 데이터나 고위험 도메인은 피하라고 권고한다.
자율적으로 브라우저를 조작하는 에이전트는 프롬프트 인젝션 공격에 취약하며,
악의적 웹 콘텐츠가 에이전트를 오용할 수 있는 위험이 항상 존재한다.

## 인사이트

### 소형 언어 모델의 도메인 특화 전략

Fara-7B는 범용 능력을 포기하고 하나의 도메인(컴퓨터 사용)에 집중함으로써
훨씬 큰 모델과 경쟁하는 전략을 택했다. 이는 소형 모델의 실용화를 위한 중요한
방향성을 제시한다. 7B 파라미터는 GPT-4 계열의 수백 분의 일 수준이지만, 특정
작업에 한해서는 충분히 유용한 성능에 도달할 수 있다는 것이 실증됐다.

이 전략은 엣지 컴퓨팅과 온디바이스 AI의 확산과 맞닿아 있다. 범용 LLM을 클라우드
API로 호출하는 방식은 지연 시간, 비용, 프라이버시 세 가지 문제를 동시에 갖는다.
도메인 특화 소형 모델은 이 세 문제를 한 번에 해결할 수 있는 잠재력을 갖는다.
앞으로 더 많은 도메인(코드 실행, 데이터 분석, 이미지 편집 등)에서 유사한
특화 소형 모델이 등장할 가능성이 높다.

다만 도메인 특화는 범용성 희생을 전제로 한다. Fara-7B는 웹 태스크 자동화 외의
대화, 추론, 창작 등에서 얼마나 잘 동작하는지 알려져 있지 않다. 특화 모델을
도입할 때는 사용 사례가 명확히 한정된 경우에만 적합하다는 점을 고려해야 한다.

### 합성 데이터로 에이전트 학습하기의 가능성과 한계

Magentic-One으로 생성한 145K 합성 트라젝토리로 경쟁력 있는 모델을 만들어냈다는
점은 에이전트 학습 데이터 생성 방법론에 중요한 시사점을 준다. 사람이 직접 웹을
탐색하며 수집하기 어려운 대규모 다양성을 멀티 에이전트 프레임워크가 자동으로
채워줄 수 있음을 보여준 것이다.

그러나 합성 데이터의 품질은 결국 생성 에이전트(Magentic-One)의 능력에 상한이
묶인다. 생성 에이전트가 잘 처리하지 못하는 태스크 유형은 학습 데이터에서도
과소 대표되거나 잘못된 트라젝토리가 포함될 가능성이 있다. Fara-7B가 복잡한
멀티스텝 태스크(compositional tasks)에서 상대적으로 낮은 성공률(23%)을 보이는
것은 이 한계를 반영할 수 있다.

더 나아가 합성 데이터 기반 학습은 “분포 밖(out-of-distribution)” 태스크에
대한 일반화 능력을 검증하기 어렵다는 근본적 문제를 안고 있다. 학습 분포를
벗어난 새로운 종류의 웹 인터페이스나 태스크 유형이 나타났을 때 모델이 얼마나
강건한지는 벤치마크 성능만으로는 알 수 없다.

### 에이전트 평가 인프라의 중요성

Fara-7B 프로젝트가 모델 자체만큼이나 중요하게 기여하는 것은 평가 인프라다.
WebTailBench, CUAVerifierBench, Universal Verifier 등 평가 도구 생태계를
함께 공개했다. 에이전트 평가는 일반 LLM 평가보다 훨씬 복잡한데, 실제 웹사이트가
매일 변하고 태스크가 스테일해지며 브라우저 오류가 발생하는 환경에서 재현 가능한
결과를 얻어야 하기 때문이다.

특히 “verifier를 평가하는 벤치마크(CUAVerifierBench)”를 별도로 만든 것은
메타 평가 관점에서 눈길을 끈다. LLM-as-a-judge 방식이 에이전트 평가의 주류로
자리 잡으면서, “판사가 얼마나 신뢰할 수 있는가”라는 질문도 중요해졌다. 판사
모델의 인간 일치도를 측정하는 별도 데이터셋을 구축한 것은 평가 파이프라인 전체의
신뢰성을 높이는 체계적 접근이다.

WebTailBench V2에서 만료된 날짜 기반 태스크를 갱신한 것도 주목할 만하다. 온라인
에이전트 벤치마크는 정적이 아니라 살아있는 데이터셋이어야 한다는 인식이 담겨
있다. 앞으로 에이전트 벤치마크 유지보수는 별도의 엔지니어링 노력이 필요한
지속적 작업으로 자리 잡을 것이다.

### 로컬 배포와 프라이버시 보호의 새로운 가능성

Fara-7B가 7B 파라미터로 온디바이스 배포를 가능하게 한다는 점은 프라이버시
민감한 자동화 시나리오에서 중요한 의미를 갖는다. 기존 클라우드 기반 컴퓨터 사용
모델은 사용자 화면 스크린샷이 외부 서버로 전송되는 구조다. 개인 금융 정보,
의료 기록, 기업 내부 시스템을 다루는 자동화 태스크에서는 이것이 심각한
프라이버시 우려를 낳는다.

GGUF 양자화 버전을 제공해 24GB 미만 VRAM 환경에서도 실행 가능하도록 한 것은
실용적 배포 장벽을 낮추는 선택이다. LM Studio나 Ollama 같은 친숙한 도구와의
호환성도 개발자 접근성을 높인다. 이를 통해 기업 내부망이나 에어갭 환경에서도
에이전트 자동화를 구축하는 시나리오가 현실화될 수 있다.

다만 로컬 배포가 곧 완전한 프라이버시 보호를 의미하지는 않는다. 에이전트가
BrowserBase 같은 외부 브라우저 세션 관리 서비스를 사용할 경우 트라픽은 여전히
제3자를 거친다. 진정한 온프레미스 배포를 원한다면 브라우저 레이어부터 검증
레이어까지 전체 스택을 자체 운용하는 추가 작업이 필요하다.
