# `*.가격.kr` - 한국어 키워드 기반 가격 비교 단축 URL

원문: <https://xn--o39aom.kr>

## 요약

가격.kr은 한국어 키워드를 서브도메인으로 사용하여
가격 비교 페이지로 리다이렉트하는 커뮤니티 기반 단축 URL 서비스다.
예를 들어 `만두.가격.kr`이나 `주식.가격.kr`을 입력하면
커뮤니티가 투표로 선정한 최적의 가격 비교 링크로 연결된다.

운영 방식:

- 사용자가 새로운 키워드를 제안하거나 목적지 변경을 요청
- 커뮤니티 투표로 각 키워드의 최적 링크를 결정
- 위키 스타일의 크라우드소싱 플랫폼

비활성화된 PriceWagon이라는 전신 서비스를 대체하며,
GitHub에 오픈소스로 공개되어 있다(github.com/price-kr/price.kr).
제작자는 이를 "소셜 실험"으로 설명한다.

## 분석

### 한글 도메인(IDN)의 창의적 활용

`가격.kr`은 퓨니코드로 `xn--o39aom.kr`이며,
서브도메인 와일드카드 DNS와 결합하여
임의의 한글 키워드를 URL로 변환하는 구조다.

### 검색 엔진과 다른 큐레이션 접근

커뮤니티 투표 기반 큐레이션은 검색 엔진과 다른 접근이다.
알고리즘이 아닌 사람의 판단으로 "최적의 가격 비교 링크"를 선정하므로
SEO 조작이나 광고 편향에 덜 취약하다.

### 개념 증명 단계의 프로젝트

"소셜 실험"이라는 표현에서 알 수 있듯이
지속 가능한 사업 모델보다는 개념 증명에 가까운 프로젝트다.

## 비평

### 콜드 스타트 문제

커뮤니티 기반 서비스의 고질적 문제인 콜드 스타트가 가장 큰 도전이다.
키워드별 투표가 활성화되려면 충분한 사용자 기반이 필요하지만,
사용자를 끌어오려면 이미 유용한 키워드 매핑이 존재해야 한다.

### 링크 부패(link rot) 관리

가격 비교 사이트의 URL 구조는 빈번히 변경되므로,
매핑된 목적지 링크의 유효성을 지속적으로 검증해야 한다.
커뮤니티 자율에 맡기면 오래된 매핑이 깨진 채 방치될 수 있다.

### 한글 도메인의 퓨니코드 변환 문제

한글 도메인은 공유 시 퓨니코드로 변환되는 경우가 많아
`xn--o39aom.kr`처럼 읽기 어려운 URL이 될 수 있다.
모바일 브라우저의 한글 도메인 지원도 일관적이지 않다.

## 인사이트

### 한글 도메인의 "프로그래머블 네임스페이스" 활용

대부분의 한글 도메인은 단순 브랜딩 목적이지만,
가격.kr은 서브도메인을 프로그래머블한 API 엔드포인트처럼 활용한다.
`{키워드}.가격.kr` 구조는 사실상 REST API의 경로 파라미터와 동일한 역할을 하며,
DNS를 라우팅 계층으로 사용하는 서버리스 아키텍처의 한 변형이다.
이 패턴은 `{도시}.날씨.kr`, `{약품}.효능.kr` 등으로 확장 가능하다.

### "검색 대안"으로서의 큐레이티드 리다이렉트

구글 검색 결과가 SEO 스팸과 광고로 오염되면서
특정 도메인의 "최선의 링크"를 커뮤니티가 큐레이션하는 수요가 존재한다.
가격.kr은 가격 비교라는 특정 의도(intent)에 대해
검색 엔진을 우회하여 직접 최적 결과로 연결하는 모델이다.
Reddit의 검색 결과 의존도가 높아지는 현상과 같은 맥락에서,
"의도 특화 큐레이션"이 검색의 대안으로 실험되고 있는 사례다.

### 커뮤니티 기반 서비스의 "최소 존속 가능 거버넌스" 문제

위키피디아에서 소규모 서브레딧까지, 커뮤니티 큐레이션의 성패는
기여자 수가 아니라 거버넌스 구조에 달려 있다.
가격.kr이 "소셜 실험"에서 지속 가능한 서비스로 전환되려면
악의적 매핑(피싱 사이트로의 리다이렉트), 상업적 이해충돌(특정 쇼핑몰 밀어주기),
비활성 키워드 관리 등의 거버넌스 문제를 해결해야 한다.
