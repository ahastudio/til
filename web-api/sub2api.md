# Sub2API: 구독 할당량을 API로 되파는 게이트웨이

<https://github.com/Wei-Shaw/sub2api>

## 소개

Sub2API는 구독 기반 AI 서비스의 할당량(quota)을 API 형태로 재분배하는
게이트웨이 플랫폼이다. 핵심 아이디어는 이름 그대로다. Claude Pro,
ChatGPT Plus, Gemini 같은 정액제 구독 계정을 여러 개 풀(pool)로 묶어두고,
그 계정들의 사용 권한을 플랫폼이 발급한 API 키로 잘게 쪼개어 나눠 준다.
최종 사용자는 각 제공사에 직접 가입하는 대신 Sub2API가 발급한 키 하나로
업스트림(upstream) 모델에 접근하고, 플랫폼은 인증, 과금, 로드 밸런싱,
요청 포워딩을 대신 처리한다.

기술 스택은 전형적인 현대 백엔드 구성이다. 백엔드는 Go 1.25.7에 Gin 웹
프레임워크와 Ent ORM을 쓰고, 프론트엔드 관리 대시보드는 Vue 3.4+,
Vite 5+, TailwindCSS로 만들었다. 데이터는 PostgreSQL 15+에 저장하고
Redis 7+를 캐시와 작업 큐로 쓴다. 배포는 GitHub Releases의 사전 빌드
바이너리를 받아오는 셸 스크립트 원클릭 설치, PostgreSQL과 Redis를 묶은
Docker Compose, 그리고 Go 소스 직접 빌드의 세 가지를 제공한다. 설치 후에는
브라우저에서 셋업 마법사(Setup Wizard)로 DB, Redis, 관리자 계정을 설정한다.

기능 목록을 보면 이 프로젝트가 단순한 프록시가 아니라 “재판매 플랫폼”을
지향한다는 점이 분명해진다. 멀티 계정 관리(OAuth와 API Key 두 방식 지원),
사용자별 API 키 발급, 토큰 단위 정밀 과금, 끈끈한 세션(sticky session)을
유지하는 지능형 계정 스케줄링, 사용자별/계정별 동시성 제한, 요청 및 토큰
레이트 리밋이 들어 있다. 결제는 EasyPay, Alipay, WeChat Pay, Stripe를
내장해 사용자가 직접 충전(self-service top-up)할 수 있다. 외부 시스템을
iframe으로 끼워 넣어 관리 대시보드를 확장하는 기능까지 있다.

지원 업스트림은 Anthropic Claude, Google Gemini, OpenAI Codex이며
Antigravity 계정도 언급된다. Nginx를 리버스 프록시로 쓸 때
`underscores_in_headers on;`을 켜야 한다는 주의사항이 README에 따로
적혀 있는데, 이는 `session_id` 같은 언더스코어 포함 헤더가 끊겨
멀티 계정의 sticky session 라우팅이 깨지기 때문이다. 즉 “어느 요청을
어느 구독 계정으로 흘려보낼지”를 헤더 기반으로 고정하는 것이 이 시스템의
핵심 동작이라는 뜻이다.

특기할 점은 README 맨 위에 굵은 경고문이 붙어 있다는 것이다. 이 프로젝트가
Anthropic을 비롯한 업스트림 제공사의 약관(ToS)을 위반할 수 있고, 그로 인한
계정 정지, 서비스 중단, 데이터 손실의 모든 책임은 사용자에게 있으며, 본
프로젝트는 “기술 학습과 연구 목적”으로만 제공된다는 면책 조항이다. 그리고
README의 절반 가까이가 후원사(sponsor) 표로 채워져 있는데, 대부분이
Claude Code / Codex / Gemini를 “공식가의 2~38%”에 파는 릴레이 서비스,
계정 판매상, 주거용 프록시 IP 공급사, 멀티 계정용 안티 디텍션 브라우저다.
이 후원사 명단 자체가 프로젝트의 실제 용도를 가장 정직하게 설명한다.

## 분석

### 이 프로젝트는 구독제와 종량제 사이의 차익을 노린다

Sub2API의 존재 이유는 기술적 우아함이 아니라 가격 구조의 틈이다. AI 제공사는
개인 구독에 정액 요금을 매기면서 대체로 “한 사람의 일상적 사용”을 가정해
넉넉한 한도를 준다. 반면 API는 토큰 단위 종량제로, 정액 구독보다 단가가
훨씬 비싸다. Sub2API는 이 두 과금 모델의 차이를 차익 거래(arbitrage)한다.
정액 구독 계정 한 개의 한도를 여러 사용자가 토큰 단위로 나눠 쓰게 하면,
구독료를 토큰당 단가로 환산했을 때 공식 API보다 압도적으로 싸진다.
후원사들이 “공식가의 10%, 2.2%”를 외치는 근거가 바로 여기에 있다.

따라서 이 플랫폼이 공들여 구현한 기능들(정밀 과금, 동시성 제한, sticky
session 스케줄링)은 모두 하나의 목표를 향한다. 정액 구독 계정의 한도를
최대한 빨아내되, 제공사의 탐지(detection)에 걸려 계정이 정지되지 않도록
사용 패턴을 분산하고 위장하는 것이다. sticky session은 효율이 아니라
위장을 위한 장치에 가깝다. 한 대화가 여러 계정을 오가면 비정상 패턴으로
보이므로, 같은 세션은 같은 계정에 고정해 “한 사람이 자연스럽게 쓰는 것처럼”
보이게 한다.

### 핵심 가정은 “구독 약관이 종량제 재판매를 막지 못한다”는 베팅이다

이 시스템이 성립하려면 두 가지가 참이어야 한다. 첫째, 구독 계정을 OAuth나
API Key 형태로 추출해 외부에서 호출할 수 있어야 한다. 둘째, 제공사가 그
사용을 약관 위반으로 보더라도 즉각 차단하지 못하거나, 차단하더라도 계정을
계속 갈아끼우는 비용이 차익보다 작아야 한다. Sub2API는 이 두 가정에 베팅한
구조물이다.

멀티 계정 관리와 계정 스케줄링이 1급 기능으로 올라온 것이 그 증거다.
정상적인 API 게이트웨이라면 단일 크레덴셜로 안정적으로 호출하면 그만이다.
계정을 “관리”하고 “스케줄링”한다는 발상 자체가, 개별 계정이 언제든 죽을 수
있는 소모품이라는 전제를 깔고 있다. README가 직접 인정하듯 이는 약관 위반
가능성이 높은 회색지대 행위이며, 이 플랫폼은 그 위반을 효율적으로 운영하기
위한 인프라다.

### 회색시장 API 생태계의 “운영 계층”에 위치한다

리버스 엔지니어링된 비공식 API 생태계는 보통 세 층으로 나뉜다. 가장 아래는
계정 자체를 조달하는 층(계정 판매상, 주거용 프록시, 안티 디텍션 브라우저),
중간은 그 계정을 호출 가능한 형태로 묶는 운영 층, 위는 최종 사용자에게
파는 판매 층이다. Sub2API는 정확히 가운데 운영 층을 차지하는 오픈소스
도구다. 후원사 표가 아래층(BmoPlus의 계정 판매, Bestproxy의 주거용 IP,
RoxyBrowser의 멀티 계정 브라우저)과 위층(각종 릴레이 서비스)을 모두
포함하는 것은 우연이 아니다. Sub2API는 이 분업 구조의 접착제다.

흥미로운 점은 Sub2ApiPay가 본체에 흡수되어 결제가 내장됐고, 모바일 관리
콘솔(sub2api-mobile)까지 생태계로 등장했다는 것이다. 이는 단순한 해킹
도구를 넘어 “구독 재판매 사업을 차리는 SaaS 키트”로 제품화되고 있음을
보여준다. Stripe와 Alipay 결제를 내장한 게이트웨이는 취미 프로젝트가
아니라 상업 운영을 전제한 물건이다.

### “기술 학습 목적” 면책은 책임을 사용자에게 떠넘기는 장치다

README의 경고문과 면책 조항은 양면적이다. 한편으로는 정직하다. 다른 회색
도구들이 위험을 숨기는 것과 달리, 약관 위반 가능성과 계정 정지 위험을 맨
앞에 명시한다. 다른 한편으로 이 문구는 법적 방패다. “연구 목적”, “모든
책임은 사용자에게”라는 표현은 토렌트 클라이언트나 에뮬레이터가 오래
써온 면책 공식과 같다. 도구를 만든 쪽은 중립적 기술 제공자로 남고, 책임은
운영자에게 전가된다. 결제 시스템까지 내장한 플랫폼이 “학습 목적”이라는
주장은, 그 진정성을 곧이곧대로 받아들이기 어렵게 만든다.

## 비평

### 약관 위반과 정지 위험은 우회 대상이지 해결된 문제가 아니다

이 플랫폼의 가장 근본적인 취약점은 기술이 아니라 자신이 의존하는 자원의
적법성이다. 정액 구독은 개인 사용을 전제로 발급되며, 여러 사용자에게
재판매하거나 종량제로 쪼개 파는 것은 거의 모든 주요 제공사의 약관이 명시적
으로 금지한다. Sub2API의 모든 정교함(스케줄링, sticky session, 동시성
제한)은 이 금지를 우회하기 위한 것이지 해소하기 위한 것이 아니다.

제공사 입장에서 보면 이는 명백한 남용 신호의 집합이다. 단일 구독 계정에서
나오는 비정상적으로 높은 토큰 소비, 다양한 지역의 IP, 동시 다발 요청은
모두 탐지 모델의 입력이 된다. 후원사 명단에 주거용 프록시와 안티 디텍션
브라우저가 들어 있다는 사실 자체가, 단순 게이트웨이만으로는 탐지를 피할 수
없어 외부 위장 수단에 의존해야 함을 자백한다. 즉 Sub2API 단독으로는
지속 가능하지 않으며, 회색 생태계 전체를 동원해야 겨우 버틴다.

더 큰 문제는 위험의 비대칭이다. 도구 제작자는 면책 조항 뒤에 숨고, 계정을
사들여 운영하는 사람이 정지·환불 분쟁·법적 책임을 모두 떠안는다. 종량제로
선결제를 받아둔 운영자가 계정 무더기가 정지되는 순간, 그가 판 사용자들에
대한 채무는 그대로 남는다. 결제 내장은 이 채무 위험을 운영자에게 구조적으로
집중시킨다.

### 구독 계정을 API처럼 쓰는 것은 본질적으로 불안정한 추상화다

Sub2API는 구독 서비스를 마치 안정적인 API 엔드포인트처럼 다루지만, 그 둘은
신뢰성 보장 수준이 전혀 다르다. 공식 API에는 SLA, 버전 정책, 하위 호환성
약속이 있다. 반면 구독 계정의 백엔드 인증 흐름, OAuth 토큰 형식, 세션 헤더
규약은 제공사가 사용자 통보 없이 언제든 바꿀 수 있는 내부 구현이다.
Sub2API는 공식 계약이 아니라 관찰된 동작에 기대어 만든 추상화다.

이 불안정성은 README가 스스로 드러낸다. Nginx가 언더스코어 헤더를 떨군다는
이유로 `session_id` 라우팅이 깨진다는 주의사항은, 이 시스템이 얼마나 깨지기
쉬운 세부에 의존하는지를 보여준다. 헤더 이름 하나, 인증 플로우 한 단계만
바뀌어도 전체 라우팅이 무너질 수 있다. 정상 API라면 이런 것이 계약으로
고정되어 있어 클라이언트가 신경 쓸 필요가 없다.

따라서 운영자가 약속하는 “안정성”은 본질적으로 허구다. 그것은 제공사가
아직 그 우회 경로를 막지 않았다는, 시간 한정 보장일 뿐이다. 제공사가 인증을
강화하거나 비정상 사용을 차단하는 순간, 그 위에 쌓인 모든 SLA 약속은
하룻밤 사이에 증발한다. 종량제 선결제 모델은 이 허구의 안정성 위에 실제
화폐를 얹는 것이어서 위험이 더 크다.

### 지속 가능성은 제공사의 무관심이라는 외생 변수에 달려 있다

이 사업 모델의 수명은 운영자의 역량이 아니라 제공사의 대응 의지라는 외부
변수가 결정한다. 제공사가 단속을 본격화하기로 마음먹는 순간, 차익은
사라진다. 제공사에게는 충분한 동기가 있다. 재판매는 정액 구독의 단가
가정을 무너뜨려 직접적인 매출 잠식을 일으키고, API라는 고마진 채널을
잠식하기 때문이다.

역사적으로 이런 차익 거래는 제공사가 “허용 비용 < 단속 비용”으로 판단하는
동안만 존재했다. 무료 체험 남용, 학생 할인 재판매, 게임 계정 공유가 모두
같은 패턴을 거쳐 결국 강한 디바이스 바인딩, 행동 분석, 법적 조치로 막혔다.
Sub2API가 의존하는 틈도 영구적이지 않다. 제공사가 구독에 디바이스 핑거프린트,
사용 패턴 분석, 더 강한 인증을 도입하면 우회 비용이 차익을 넘어선다.

비판의 핵심은 이것이다. Sub2API는 제공사가 “아직 진지하게 막지 않은” 창문
안에서만 작동하는 도구이며, 그 창문이 닫히는 시점을 통제할 수단이 전혀 없다.
정교한 스케줄링과 결제 시스템은 이 근본적 외생성을 가릴 뿐 없애지 못한다.

### 후원 구조가 프로젝트의 인센티브를 회색시장에 묶어 둔다

README의 절반이 회색시장 행위자들의 후원 광고라는 사실은 단순한 자금 조달을
넘어 인센티브 구조의 문제를 드러낸다. 계정 판매상, 프록시 공급사, 안티
디텍션 브라우저, 릴레이 서비스가 후원하는 오픈소스 도구는, 그 후원자들의
사업이 번성할 때만 자신도 번성한다. 즉 이 프로젝트는 약관 위반 생태계의
성장에 경제적으로 정렬되어 있다.

이는 “기술 학습과 연구 목적”이라는 자기 규정과 정면으로 충돌한다. 연구
도구라면 후원자가 계정 판매상일 이유가 없다. 후원 표는 이 프로젝트의 실제
사용자가 누구이고 어떤 가치 사슬에 봉사하는지를 면책 조항보다 훨씬 정직하게
말해 준다. 코드의 중립성을 주장하면서 수익 모델은 특정 남용 생태계에
묶여 있는 구조다.

결과적으로 프로젝트의 진화 방향도 이 인센티브를 따른다. 결제 내장, 모바일
콘솔, 외부 시스템 iframe 통합 같은 기능 추가는 모두 “재판매 운영을 더 쉽게”
만드는 쪽으로 흘러간다. 탐지 회피와 운영 편의가 로드맵을 지배하는 한,
이 도구가 합법적 용도로 선회할 유인은 구조적으로 존재하지 않는다.

## 인사이트

### 구독제의 “넉넉한 한도”는 이런 도구를 만나면 사라진다

Sub2API가 성공적으로 확산되면 그 결과로 자신의 토대를 스스로 파괴한다.
이는 글이 예상하지 않은 2차 효과다. 정액 구독의 넉넉한 한도는 평균 사용자가
한도를 거의 다 쓰지 않는다는 통계적 가정 위에 성립한다. 한 명이 한도의
20%만 쓴다면 제공사는 나머지 80%를 여유로 두고 가격을 책정할 수 있다.

재판매 도구는 바로 이 여유분을 정확히 겨냥해 토큰 한 방울까지 빨아낸다.
재판매가 흔해지면 구독 계정의 평균 사용률이 100%에 수렴하고, 제공사의 단가
가정이 무너진다. 제공사의 대응은 정해져 있다. 한도를 줄이거나, 가격을
올리거나, 디바이스·행동 바인딩을 강화하는 것이다. 어느 쪽이든 정직한 개인
사용자가 피해를 본다.

따라서 이 도구의 진짜 비용은 운영자의 계정 정지 위험이 아니라, 생태계
전체에 가해지는 외부효과다. 무임승차가 임계점을 넘으면 공유지의 비극이
발동해 모두에게 더 비싸고 더 제약된 구독이 돌아온다. Sub2API는 그 임계점을
앞당기는 가속기다.

### 이것은 API 스크래핑의 고양이-쥐 게임의 최신 라운드일 뿐이다

비공식 API를 둘러싼 우회와 차단의 반복은 인터넷 역사에서 오래된 패턴이다.
RSS 시절의 비공식 트위터 API, libgen과 sci-hub, 무료 SMS 게이트웨이,
넷플릭스 계정 공유, 게임 사설 서버가 모두 같은 줄거리를 따랐다. 처음에는
제공사가 묵인하고, 도구가 번성하고, 차익이 커지고, 마침내 제공사가 인증을
강화하며 막는다. Sub2API는 이 서사의 LLM 버전이다.

이 구조적 유추가 주는 통찰은 결말이 거의 정해져 있다는 것이다. 고양이-쥐
게임에서 장기적으로 이기는 쪽은 늘 자원과 통제력을 쥔 플랫폼이었다. 제공사는
인증 스택 전체를 소유하므로 언제든 규칙을 바꿀 수 있고, 우회자는 매번
반응적으로 따라갈 수밖에 없다. 트위터가 API를 봉쇄하자 수백 개의 클라이언트가
하룻밤에 죽었던 일이 그대로 반복될 수 있다.

다만 한 가지 새로운 변수가 있다. LLM 추론은 제공사에게도 막대한 변동 비용을
발생시킨다는 점이다. 과거의 스크래핑은 제공사에게 한계 비용이 거의 0이었지만,
재판매된 토큰은 실제 GPU 시간을 태운다. 그래서 제공사의 단속 동기가 과거보다
훨씬 강하고 즉각적이다. 이 라운드의 고양이-쥐 게임은 이전보다 짧고 격렬하게
끝날 가능성이 높다.

### 오픈소스 “중립 도구”라는 방패는 결제 내장 앞에서 무너진다

Sub2API는 “코드는 중립적이며 용도는 사용자의 책임”이라는 오래된 방어
논리에 기댄다. 이 논리는 범용 도구에서는 어느 정도 통한다. 그러나 글이
얼버무리는 트레이드오프가 있다. 기능을 특정 남용 시나리오에 맞춰 특화할수록
중립성 주장은 약해진다는 것이다. 토큰 단위 과금, 셀프 충전 결제, sticky
session 위장은 “AI 게이트웨이 일반”에 필요한 기능이 아니라 “구독 재판매
운영”에 정확히 맞춰진 기능이다.

이는 역사적으로 검증된 법적 경계다. 그록스터(Grokster) 판결과 여러
저작권 소송에서 법원은 도구가 단지 침해에 쓰일 수 있다는 사실보다, 침해를
유도·조장하도록 설계·홍보되었는지를 따졌다. 결제 시스템을 내장하고
계정 판매상의 후원을 받는 게이트웨이는 “유도(inducement)”의 정황을 스스로
쌓고 있다. 면책 조항은 이 정황을 지우지 못한다.

장기적으로 이 트레이드오프는 프로젝트의 생존을 좌우한다. 진짜 중립을
지키려면 재판매 특화 기능을 덜어내야 하는데, 그러면 후원 생태계와 사용자가
떠난다. 반대로 운영 편의를 계속 높이면 법적·플랫폼적 표적이 커진다.
Sub2API는 이 딜레마의 어느 쪽도 풀 수 없으며, “중립 도구”라는 자기 규정은
제품이 성숙할수록 점점 더 유지하기 어려워진다.

### 비공식 재판매의 번성은 공식 종량제 가격이 비합리적이라는 신호다

마지막으로, 이 현상을 도덕적 일탈로만 보면 더 중요한 시장 신호를 놓친다.
Sub2API 같은 도구와 그 후원 생태계가 이만큼 번성한다는 것은, 공식 구독제와
공식 API 사이의 가격 격차가 시장이 받아들이기 어려울 만큼 크다는 방증이다.
구독을 종량제로 환산했을 때 API보다 수십 배 싸다면, 그 격차 자체가 차익
거래를 빨아들이는 진공이다.

이는 제공사에게 두 갈래의 선택지를 제시한다. 단속을 강화해 격차를 강제로
유지하거나, 가격 구조를 손봐 격차 자체를 줄이는 것이다. 헤비 유저용 종량제
요금, 구독과 API의 중간 티어, 합법적 할당량 공유 같은 정식 상품이 나오면
회색시장의 존재 이유가 상당 부분 사라진다. 무료 음악 공유가 결국 스트리밍
정액제를 끌어낸 것과 같은 동학이다.

따라서 Sub2API의 가장 큰 의의는 그 도구 자체가 아니라, 그것이 노출하는
가격 비효율이다. 회색시장은 늘 공식 시장이 비워둔 자리를 채우며, 그 위치가
어디인지를 가장 정확히 가리키는 지표다. 제공사가 이 신호를 단속으로만
읽으면 같은 도구가 계속 형태를 바꿔 재등장할 것이고, 가격 신호로 읽으면
시장을 합법 영역으로 흡수할 기회를 얻는다. 어느 쪽을 택하느냐가 다음
라운드의 양상을 결정한다.
