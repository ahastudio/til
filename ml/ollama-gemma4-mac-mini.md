# Mac mini에서 Ollama와 Gemma 4 설정

> 원문: <https://gist.github.com/greenstevester/fc49b4e60a4fef9effc79066c1033ae5>

## 요약

Apple Silicon Mac mini에서 Ollama를 이용해 Gemma 4 모델을 로컬로 실행하고,
부팅 시 자동 시작하도록 구성하는 방법을 다룬 가이드다. Homebrew로 Ollama를 설치하고
LaunchAgent를 통해 모델을 메모리에 미리 올려두는 방식을 사용한다.

핵심 교훈은 26B 모델이 24GB Mac mini에서 시스템을 비반응 상태로 만들었기 때문에
**8B 모델(약 9.6GB)을 권장**한다는 것이다. Ollama v0.19+의 MLX 백엔드가 Apple Silicon
환경에서 추론 성능을 향상시킨다.

## 분석

설정의 핵심 구성 요소:

- **설치**: `brew install ollama` → `ollama pull gemma4`
- **자동 시작**: 시스템 설정의 "Launch at Login" 활성화
- **모델 상주**: `OLLAMA_KEEP_ALIVE="-1"` 환경 변수로 모델을 메모리에 무한 유지
- **API 엔드포인트**: `localhost:11434`
- **권장 메모리**: 16GB 통합 메모리 이상

GeekNews 댓글에서는 실무 활용의 한계가 주로 논의되었다. 최신 모델을 사용하려면
추론 엔진을 자주 업데이트해야 하고, 양자화 버전이 바뀔 때마다 재다운로드가 필요하다.
출시 초기에는 tokenizer 버그와 도구 호출 실패가 빈번하다는 점도 지적되었다.

Claude Sonnet 대체재로서의 한계도 분명하다. 복잡한 작업에서는 한계가 뚜렷하며,
프라이버시가 중요한 경우를 제외하면 API 서비스가 여전히 유리하다는 평가가 많다.

## 비평

"로컬 LLM"의 가장 큰 매력은 프라이버시와 오프라인 운용이지만, 실질적인 성능 격차가
여전히 크다. 26B 파라미터 모델조차 24GB Mac mini에서 시스템을 멈추게 한다는 사실은,
클라우드 API와 경쟁 가능한 수준의 로컬 추론이 소비자 하드웨어에서 가능해지려면
아직 하드웨어 세대가 더 필요하다는 것을 보여준다.

가이드 자체는 실용적이지만, "Gemma 4 26B"라는 제목과 달리 실제 권장 사항은 8B로
끝나는 점이 아이러니하다. 제목에서 기대하는 것과 실제 결론이 다르다.

## 인사이트

**로컬 LLM의 진짜 적은 모델 크기가 아니라 메모리 대역폭이다.** Apple Silicon의
통합 메모리 아키텍처가 로컬 LLM에 유리하다고 알려져 있지만, 26B 모델이 24GB
시스템을 마비시킨다는 현실은 메모리 대역폭의 한계를 드러낸다. 향후 소비자 로컬 AI의
핵심 경쟁 요소는 VRAM이 아닌 메모리 대역폭이 될 것이다.

**"ollama pull 한 줄"이 만들어낸 민주화.** 복잡한 CUDA 드라이버 설정 없이 모델을
한 줄로 내려받아 실행하는 경험은 로컬 AI의 진입 장벽을 획기적으로 낮췄다. 이 접근성이
LM Studio 등 경쟁 도구를 자극하고 있으며, 로컬 AI 런타임 시장의 경쟁이 심화되고 있다.

**프라이버시 우선 AI 워크플로우의 등장.** 기업 보안 정책이나 민감 데이터 처리 요건
때문에 클라우드 API를 쓸 수 없는 상황에서 로컬 LLM 수요가 있다. 성능이 아닌
컴플라이언스가 로컬 LLM 채택의 주 동인이 되는 영역이 분명히 존재한다.
