# OCTOMO: 역방향 SMS 인증 API

<https://octomo.octoverse.kr/>

Show GN: [OCTOMO — 고객이 직접 문자를 보내는(MO) 방식의 휴대폰 본인인증 API](https://news.hada.io/topic?id=30281)

## 소개

OCTOMO는 SMS 인증의 비용 구조를 뒤집은 MO(Mobile Originated) 방식의 휴대폰
본인인증 API 서비스다. 기존의 MT(Mobile Terminated) 방식에서는 서비스가
사용자에게 인증 문자를 발송하지만, OCTOMO에서는 사용자가 직접 지정된 번호로
인증 코드를 담은 문자를 보낸다. 서비스는 `POST /octomo/v1/public/message/exists`
API를 호출해 5분 이내에 해당 문자가 수신되었는지 확인한다. 구조가 역전되면서
SMS 발송 비용이 서비스 운영자가 아닌 사용자의 요금제에서 차감된다.

비용 구조는 완전 무료다. 기본료, 건당 비용, API 사용료 모두 0원이며 무제한
트래픽을 지원한다고 명시되어 있다. 기존 SMS 인증 서비스가 건당 9~50원, 월
기본료 5~20만 원을 부과하는 것과 비교하면, 월 1만 건 기준으로 최대 70만 원의
비용을 절감할 수 있다는 것이 서비스의 핵심 주장이다. 계약이나 심사 없이
회원가입 후 즉시 API 키를 발급받아 사용할 수 있어 MVP 개발 단계에서 진입
장벽이 낮다.

보안 측면에서는 “점유 인증” 강화를 장점으로 내세운다. 사용자가 실제로 해당
기기를 물리적으로 소지하고 있어야 문자를 발송할 수 있으므로, 전화번호 도용
시 기기 없이는 인증을 완료할 수 없다는 논리다. 국내 주요 은행, 카드사, 홈택스
등 금융권과 공공기관에서 이미 MO 인증을 사용하고 있다는 점도 기술 신뢰성의
근거로 제시한다. 단, 현재 국내 휴대폰 번호만 지원한다.

GeekNews 댓글에서는 점유인증과 본인인증이 충족하는 법적 요건이 다르다는 점이
지적되었다. 전자서명법, 정보통신망법, 전자금융거래법 등 각 법령이 요구하는
인증 수준이 다르기 때문에, 어떤 서비스에서 점유인증만으로 충분한지 명확히
해야 한다는 것이다. 서비스 운영자는 이를 문서에 추가하겠다고 응답했다.

## 분석

### 비용 구조의 역전이 가진 의미

OCTOMO의 핵심은 기술 혁신이 아니라 비용 귀속의 재설계다. MO 인증 기술 자체는
이미 금융권과 공공기관에서 오래전부터 사용해온 방식이다. 새로운 것은 이 기술을
API로 포장해 스타트업이 쉽게 접근할 수 있게 만들었다는 점, 그리고 “발송 비용이
0원”이라는 프레이밍이다.

그러나 “비용이 사라진 것”이 아니라 “비용을 지불하는 주체가 바뀐 것”이다. 기존
MT 방식에서는 서비스 운영자가 SMS 발송 비용을 부담했다. OCTOMO 방식에서는
같은 비용을 사용자가 부담한다. 무제한 요금제가 보편화된 환경에서는 실질적인
부담이 크지 않을 수 있지만, 구조적으로는 비용이 외부화된 것이다. 이 전환이
사용자에게 명시적으로 고지되어야 하는지, 그리고 이것이 서비스 UX에 미치는
영향은 별개로 다루어야 할 문제다.

### “무료”라는 비즈니스 모델의 지속 가능성

완전 무료 API를 표방하면서 어떻게 수익을 창출하는지는 이 서비스의 핵심 물음이다.
현재 공개된 구조를 보면, Free 플랜(월 1만 건)은 영구 무료이고, Pro 플랜(월 10만
건)은 9,900원/월이다. 수신 문자를 처리하는 인프라 비용(수신 전용 번호 운영,
API 서버, 데이터 저장)은 사용량에 비례하는데, 무료 사용자 다수를 지원하면서
소수의 Pro 사용자 수익으로 이를 충당하는 구조다.

이 구조는 개발자 도구 시장에서 흔한 PLG(Product-Led Growth) 전략이다. 소규모
서비스가 무료로 익숙해지다가 트래픽이 증가하면 Pro 또는 Enterprise로 전환하는
경로를 의도한다. 그러나 SMS 인증은 스케일업할수록 수신 처리 인프라 비용이
선형 이상으로 증가할 수 있다. 특히 악성 트래픽(스팸 문자, 무작위 인증 시도)이
발생하면 수신 인프라가 그 비용을 모두 흡수해야 한다. 이 비용이 현재 요금제에
충분히 반영되어 있는지 불투명하다.

### MT에서 MO로: UX 마찰의 방향이 바뀐다

기존 SMS 인증에서 사용자는 문자를 받고 코드를 입력한다. 총 동작은 앱 전환
없이 코드 확인 후 붙여넣기 정도다. OCTOMO 방식에서는 사용자가 별도 앱(기본
문자앱)을 열고, 지정된 번호를 수신자로 입력하고, 제공된 코드를 문자 내용으로
입력하고, 전송 버튼을 누른다. 이것은 최소 4~5단계의 추가 조작이다.

서비스 측에서는 이를 “오입력 감소”로 긍정적으로 포장하지만, UX 마찰과 오입력
감소는 동전의 양면이다. 오입력이 줄어드는 이유는 사용자가 직접 내용을 쓰는
것이 아니라 화면에 표시된 코드를 그대로 복사하기 때문이다. 즉, 마찰이 없는
UX가 오입력도 없는 것이지, OCTOMO 방식이 마찰과 오입력을 모두 줄이는 것은
아니다. 전환율에 민감한 서비스라면 이 추가 마찰이 이탈률에 어떤 영향을 미치는지
실측 데이터 없이는 판단하기 어렵다.

### 점유인증과 본인인증의 법적 지형

GeekNews 댓글에서 제기된 법적 요건 문제는 이 서비스의 실제 적용 범위를 결정하는
핵심이다. 한국의 인터넷 서비스는 법령에 따라 인증 수준 요건이 다르다.
정보통신망법 하의 일반 회원가입에서는 점유인증만으로 충분한 경우가 있지만,
전자금융거래법이 적용되는 금융 서비스나 의료법이 적용되는 의료 서비스에서는
더 강한 본인확인 수단이 요구된다. 특히 주민등록번호 수집 금지 이후 강화된
본인확인기관 연동이 필요한 서비스들은 점유인증 단독으로 법적 요건을 충족하지
못한다.

서비스 측이 “본인인증”이라는 용어를 쓰면서 실제로는 점유인증을 제공한다는 점은
사용자가 적용 가능 범위를 오해할 위험을 만든다. 마케팅 언어와 법적 의미 사이의
간극이 명확히 정리되지 않으면, 이를 도입한 서비스가 법적 리스크에 노출될 수 있다.

## 비평

### “발송 비용 0원”은 비용 소멸이 아니라 비용 전가다

서비스의 핵심 가치 제안—“SMS 발송 비용 0원”—은 정확하지만 불완전한 진술이다.
서비스 운영자의 발송 비용이 0원인 것은 맞다. 그러나 문자 발송 비용이 사용자의
요금제에서 차감된다는 사실은 서비스 소개 페이지에서 작은 글씨로만 언급된다.
프리미엄 요금제 사용자에게는 체감 비용이 없을 수 있지만, 종량제 요금제 사용자나
해외 로밍 중인 사용자에게는 유의미한 비용이다.

이 구조가 문제인 이유는 비용 부담이 서비스 운영자에서 사용자로 이전될 때
사용자는 그 사실을 알고 동의하는 선택을 할 기회가 없다는 점이다. 전통적인
MT 방식에서 인증 문자를 받는 데 사용자에게 비용이 발생하지 않는다는 것은
암묵적 기대다. OCTOMO 방식이 이 기대를 깨면서도 명시적 고지를 강제하지 않는
구조는 서비스 운영자가 이 사실을 사용자에게 고지하지 않을 경우 소비자 불만의
씨앗이 된다.

더 깊이 보면 이 비즈니스 모델은 서비스 운영자의 비용 절감을 사용자 비용 부담으로
정당화한다. 그러나 그 비용은 서비스 운영자가 인증 서비스를 무료로 사용하는 대가로
궁극적으로 사용자가 지불하는 것이다. 이것이 악의적인 구조는 아니지만, 마케팅
언어가 이 구조를 투명하게 드러내지 않는다는 점은 비판적으로 읽힐 수 있다.

### 보안 강점 주장이 간과하는 MO 방식의 고유 취약점

서비스는 MO 방식이 “고객이 직접 기기에서 문자를 발송하므로 실제 기기 소지를
확인할 수 있다”고 주장한다. 이것은 부분적으로 옳다. 그러나 MT 방식과 비교한
보안 우위를 주장하기 위해서는 MO 방식 고유의 위협 모델도 함께 다루어야 한다.

MT 방식의 주요 위협은 SIM 스와핑(SIM swapping)과 SS7 취약점 공격이다. 공격자가
피해자의 전화번호를 자신의 SIM에 이전하면 인증 문자를 가로챌 수 있다. MO 방식은
이 공격을 방어하는 것이 아니라 다른 종류의 공격에 노출된다. SIM 스와핑에 성공한
공격자는 MO 방식에서도 피해자의 번호로 문자를 발송할 수 있어 인증을 통과한다.
즉, SIM 레벨 공격에 대한 MO 방식의 보안 우위는 없다.

새로운 위협으로는 자동화된 문자 발송 봇이 있다. MT 방식에서 서비스가 발송하는
문자는 서비스 측에서 속도 제한을 걸 수 있다. MO 방식에서 문자를 발송하는 것은
사용자(또는 봇)이므로, 대규모 자동화 테스트나 무작위 인증 시도를 막기 위해
수신 측 필터링이 중요해진다. 이에 대한 보호 방법이 서비스 문서에서 명확하지 않다.

### 기술 적용 범위가 실제 수요와 일치하지 않을 수 있다

서비스가 홍보하는 7가지 활용 사례(회원가입, 2차 인증, 정보 변경, 고객 문의, 이벤트,
투표, 출석 인증)는 인증 강도 요구사항이 크게 다른 작업들을 동일선에 두고 있다.

회원가입 본인인증이나 로그인 2차 인증은 보안 요건이 높고 법적 규제를 받을 수 있는
영역이다. 반면 이벤트 응모번호 접수나 스터디 출석 인증은 보안 요건이 거의 없는
영역이다. 이 두 가지를 같은 서비스로 묶어 마케팅하면 보안 요건이 높은 사용 사례에
대한 적합성이 실제보다 과장될 위험이 있다.

MVP 단계에서 비용 없이 “일단 붙여보자”는 수요를 잡는 데는 이 포지셔닝이 효과적이다.
그러나 이후 서비스가 성장하면서 법적 요건을 검토할 때, 점유인증만으로는 부족한
상황에서 인증 체계를 전면 교체해야 하는 기술 부채를 떠안을 수 있다. 초기에 잘못
선택한 인증 방식을 교체하는 것은 단순한 API 교체가 아니라 사용자 재인증, 데이터
마이그레이션, 법적 검토를 수반하는 큰 작업이다.

## 인사이트

### 비용 구조의 역전은 플랫폼 생태계 전반에서 반복되는 패턴이다

OCTOMO의 비용 역전 구조—서비스가 부담하던 비용을 사용자에게 이전하면서 전체
비용이 줄었다고 프레이밍하는 방식—는 디지털 서비스 생태계에서 반복되는 패턴이다.
과거 통화 서비스에서 VoIP가 등장했을 때, 통신사가 부담하던 인프라 비용이 사용자의
인터넷 데이터로 전환되었다. 이를 두고 “통화가 무료가 됐다”고 말했지만 실제로는
비용의 형태가 바뀐 것이다.

OCTOMO는 이 패턴을 인증 영역에 적용했다. 흥미로운 점은 이 전환이 가능한 이유가
국내 무제한 요금제의 보편화 때문이라는 것이다. 문자 발송 비용이 체감되지 않는 환경이
이 비즈니스 모델의 전제 조건이다. 즉, 이 서비스는 특정 통신 시장 구조에 기생하는
모델이다. 종량제 문자 요금이 보편적인 시장—예를 들어 미국의 구형 요금제 사용자나
개발도상국 시장—에서는 같은 모델이 작동하지 않는다.

이 패턴의 두 번째 함의는 “무료”가 된 서비스의 외부성이 결국 어딘가에서 흡수된다는 것이다.
MT SMS 인증의 비용은 서비스 운영자가 흡수했다. MO 방식에서는 그 비용이 사용자의
요금제(개인 비용)와 통신사의 망 처리 용량(집합적 비용)으로 분산된다. 무료처럼
보이는 혁신은 종종 비용의 사회화를 통해 달성된다.

### 인증 방식의 선택이 만드는 조기 기술 부채는 과소평가된다

스타트업이 MVP 단계에서 인증 방식을 결정할 때, 보안 요건보다 연동 속도와 비용을
우선시하는 경향이 있다. OCTOMO의 포지셔닝—“계약 없이 5분 연동”—은 정확히 이
경향을 공략한다. 그러나 인증 방식은 서비스 성장 이후에 교체하기 가장 어려운
컴포넌트 중 하나다.

사용자 수가 많아질수록 인증 방식 교체에는 기존 사용자 재인증, 법적 동의 재수집,
신원 확인 데이터 마이그레이션이 필요하다. 특히 한국의 경우 서비스가 특정 단계에
도달하면 전자서명법, 개인정보보호법, 전자금융거래법 등의 적용을 받게 되고,
그 시점에 점유인증이 법적으로 불충분하다는 것을 발견하게 될 수 있다. 이때의 전환
비용은 절감한 인증 비용의 수십 배가 될 수 있다.

이 패턴은 데이터베이스 선택, 메시징 큐 선택 등 인프라 결정에서도 반복된다. “일단
빠르게”가 나쁜 결정은 아니지만, 선택의 조기 기술 부채를 의식하고 전환 시점을
계획하는 팀과 그렇지 않은 팀의 결과는 장기적으로 크게 갈린다. OCTOMO 같은
서비스가 이 결정을 쉽게 만들어주는 동시에 그 결과에 대한 책임은 도입 서비스에
있다는 점을 명확히 해야 한다.

### MO 인증 API의 상품화는 통신사 역할의 잠식이다

OCTOMO가 제공하는 서비스의 실체는 수신 전용 번호를 임차하고, 해당 번호로 들어오는
문자를 API로 노출하는 것이다. 이것은 통신사가 법인 고객에게 오래전부터 제공해온
기업용 MO 서비스를 재포장한 것이다. 차이는 접근성이다. 기존 통신사 직접 계약은
법인 심사, 계약, 정산 설정에 1~2주가 소요됐다면, OCTOMO는 회원가입 후 즉시 사용
가능하다.

이 상품화가 의미하는 것은 통신사 B2B 서비스의 마진이 압박받기 시작했다는 신호다.
통신사가 제공하던 부가가치 서비스—번호 임차, 문자 수신 처리, API 제공—를 중개
레이어가 더 저렴하고 빠르게 제공하기 시작하면, 통신사는 도매 인프라 공급자로
하향 이동하는 압력을 받는다. 이는 클라우드가 IDC를 대체하고, OTT가 방송사를
대체한 것과 같은 산업 레이어 재편의 작은 선례다.

장기적으로 이 트렌드는 통신사가 네트워크 레이어를 유지하면서 서비스 레이어를
제3자에게 개방하는 방향으로 이어질 수 있다. OCTOMO 같은 서비스가 성장하면
통신사는 직접 인증 서비스를 제공하는 경쟁자가 되거나, 또는 OCTOMO 같은 서비스의
인프라 공급자로 포지셔닝을 재설정해야 한다. 이 구조 재편이 어떤 속도로 진행되는지는
통신사의 규제 지위와 기업 민첩성에 달려 있다.

### 인증 UI 패턴의 변화가 사용자 기대 모델을 바꾼다

“SMS를 받는 것”에서 “SMS를 보내는 것”으로의 인증 패턴 전환은 기술적 변화이지만,
그 2차 효과는 사용자의 인증 심리 모델 변화다. 현재 대부분의 사용자는 “인증 = 코드를
받아 입력하는 것”으로 학습되어 있다. 이 학습된 패턴이 강력한 이유는 수십 년간의
반복 경험 때문이다.

MO 방식이 보편화된다면, 이 학습된 패턴이 깨지고 “인증 = 문자를 보내는 것”이라는
새로운 패턴이 형성된다. 그 과정에서 발생하는 사용자 혼란과 지원 비용은 선도 서비스가
흡수해야 한다. 초기 도입 서비스일수록 이 혼란 비용을 더 많이 부담한다는 뜻이다.

더 위험한 것은 이 새로운 패턴을 피싱 공격자가 활용할 수 있다는 점이다. “지정된
번호로 코드를 보내달라”는 요청이 정상 인증 절차처럼 느껴지게 된다면, 공격자가
이 패턴을 모방해 사용자로 하여금 특정 번호에 문자를 보내도록 유도하는 사회공학
공격이 쉬워진다. MT 방식에서 “코드를 입력하라”는 피싱은 인식이 높아졌지만,
“문자를 보내라”는 피싱은 아직 낯선 패턴이라 방어가 취약하다. 새로운 UX 패턴은
항상 새로운 피싱 벡터를 열어준다.
