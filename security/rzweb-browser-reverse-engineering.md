# rzweb - 브라우저 기반 리버스 엔지니어링 플랫폼

원문: <https://github.com/indalok/rzweb>

## 요약

rzweb는 오픈소스 리버스 엔지니어링 프레임워크인 Rizin을 WebAssembly로 컴파일하여
브라우저에서 직접 바이너리 분석을 수행할 수 있게 한 플랫폼이다.
파일 업로드 없이 모든 처리가 클라이언트 측에서 이루어진다.

핵심 기능:

- **영속적 Rizin 세션**: 분석 상태가 인터랙션 간에 유지됨
- **전체 터미널 접근**: 명령 자동완성, Tab 완성, 화살표 키 네비게이션
- **전용 시각화 패널**: 디스어셈블리, CFG(제어 흐름 그래프), 헥스 덤프,
  문자열, 임포트/익스포트, 섹션, 바이너리 메타데이터
- **분석 캐싱**: 바이너리 해시 기반 IndexedDB 캐싱으로 빠른 재분석
- **지원 포맷**: ELF, PE/PE+, Mach-O, Raw 바이너리

기술 스택은 React + TypeScript + Tailwind CSS(프론트엔드),
rzwasi(Rizin의 WebAssembly 빌드, 별도 저장소)로 구성되어 있다.

## 분석

### 구조적 프라이버시 모델

rzweb의 아키텍처는 명확한 프라이버시 모델을 제공한다.
바이너리가 서버로 전송되지 않으므로 기밀 바이너리 분석에도 사용할 수 있다.
이는 기업 환경에서 특히 중요한데,
리버스 엔지니어링 대상 바이너리는 종종 민감한 지적 재산이기 때문이다.

### rzwasi를 통한 영속적 분석 세션

Rizin을 WebAssembly로 컴파일한 rzwasi 프로젝트가 핵심 의존성이다.
Rizin의 CLI 인터페이스와 영속적 세션 API를 모두 노출하여
단순한 웹 래퍼가 아닌 상태를 유지하는 분석 워크플로우를 가능하게 한다.

### IndexedDB 기반 캐싱의 실용성

바이너리 해시를 키로 사용하므로 동일 바이너리 재분석 시
전체 분석 과정을 반복하지 않아도 된다.

## 비평

### WebAssembly 단일 스레드의 성능 한계

WebAssembly의 단일 스레드 실행은 대형 바이너리 분석에서 심각한 병목이 된다.
1MB 이상 파일에 대한 자동 분석 제한이 있다는 것 자체가
실무에서의 활용 범위를 제한한다.
실제 리버스 엔지니어링 대상은 수십~수백 MB인 경우가 흔하다.

### 정적 분석만 가능하다는 한계

`ptrace` 시스콜을 사용하는 디버깅 기능이 브라우저 샌드박스에서 불가능하므로,
정적 분석만 가능하다.
동적 분석이 필요한 경우 결국 네이티브 도구로 돌아가야 한다.

### Rizin 생태계의 상대적 협소함

Rizin 자체가 radare2 포크로 생태계가 Ghidra나 IDA Pro에 비해 작고,
플러그인 호환성이나 시그니처 데이터베이스 규모에서도 차이가 크다.

## 인사이트

### "설치 없는 전문 도구"라는 WebAssembly의 킬러 유스케이스

리버스 엔지니어링 도구의 설치와 환경 설정은 악명 높게 복잡하다.
IDA Pro는 라이센스 관리, Ghidra는 JDK 의존성, radare2는 빌드 설정이 진입 장벽이다.
rzweb는 URL 하나로 이 모든 장벽을 제거한다.
이는 Figma가 데스크톱 디자인 도구를 브라우저로 옮긴 것과 같은 패턴이며,
교육 현장이나 CTF 대회에서 특히 파괴적인 영향력을 가진다.

### 프라이버시를 "약속"에서 "구조"로 바꾸는 클라이언트 사이드 처리

클라우드 기반 분석 도구는 "데이터를 보지 않겠다"고 약속하지만, 기술적으로는 볼 수 있다.
rzweb는 데이터가 아예 서버에 도달하지 않으므로 프라이버시가 구조적으로 보장된다.
이 패턴은 보안 도구에만 국한되지 않는다.
의료 영상 분석, 법률 문서 검토, 금융 데이터 처리 등
데이터 주권이 중요한 모든 분야에서 동일한 아키텍처 수요가 존재한다.

### 전문 도구의 "80/20 브라우저 버전"이라는 새로운 시장 카테고리

rzweb는 IDA Pro를 대체하지 못한다.
그러나 빠른 트리아지, 교육, 초기 분석에는 충분하다.
Ghidra가 IDA Pro의 무료 대안으로 시장을 만든 것처럼,
rzweb는 "설치조차 필요 없는 더 낮은 계층"을 개척한다.
전문 도구 시장은 점점 "전문가용 고기능 네이티브"와 "범용 접근성 웹"으로
양극화되는 추세다.
