# 대한민국 정부의 GPKI 인증서, Firefox 루트 저장소 재도전

원문: <https://bugzilla.mozilla.org/show_bug.cgi?id=1845081>

## 배경

행정안전부(MOIS)와 한국지역정보개발원(KLID)이 Mozilla Root Store에 “MOIS SSL Root CA” 인증서를 등록해 달라는 요청을 제출했다.
2023년 2월에 발급된 이 루트 인증서는 정부 및 공공기관 웹사이트에 사용할 목적으로 만들어졌다.

이 요청이 처음이 아니다.
Bug 1226100과 Bug 1377389에서 두 차례 거절당한 전례가 있다.
거절 이유는 한국 정부 PKI(GPKI)의 인증서 오발급(misissuance) 사고였다.
더불어 Bug 1451235에서는 Mozilla가 감사 회사인 딜로이트 안진의 WebTrust 감사 결과를 신뢰하지 못하겠다고 공식적으로 선언하기까지 했다.

이 역사를 알면, 이번 요청이 왜 단순한 행정 절차가 아닌지 분명해진다.

## 현황

버그 리포트는 현재 ASSIGNED 상태(우선순위 P2)로, 추가 정보 제출 대기 중이다.
Mozilla의 루트 저장소 포함 심사는 까다롭고 길다.
단순히 인증서를 제출한다고 끝나는 것이 아니라,
CA/Browser Forum 요구사항 준수, 독립적인 WebTrust 감사 통과, 커뮤니티 검토를 모두 거쳐야 한다.

현재 제기된 주요 우려 사항은 네 가지다.

첫째, 감사 기관 신뢰성 문제다.
이번 신규 계층(hierarchy) 감사를 맡은 회사가 딜로이트 코리아/안진이다.
Mozilla가 과거에 불신(distrust)을 선언했던 바로 그 회사다.

둘째, 인프라 접근성 문제다.
CRL(인증서 폐기 목록)과 OCSP(온라인 인증서 상태 프로토콜) 엔드포인트가
한국 외부의 네트워크에서는 접속되지 않는다.
루트 CA 인증서는 전 세계 브라우저에서 검증 가능해야 하므로, 이는 치명적인 결함이다.

셋째, 문서화 문제다.
CPS(인증서 실무준칙)에 내부 모순이 있고, 인증서 프로파일이 구식이며,
버전 관리가 투명하지 않다.

넷째, 거버넌스 문제다.
과거에 문제를 일으킨 GPKI 시스템으로부터 새 계층이 실질적으로 독립되어 있는지 불분명하다.
커뮤니티의 다수 기술 전문가들이 명시적으로 반대 의견을 제출한 상태다.

## 분석

[GeekNews 댓글](https://news.hada.io/topic?id=30329)에서 stypr은 구체적인 기술 증거를 제시했다.
CRL/OCSP 해외 IP 접속 버그, 2016-2017년과 동일한 감사 회사 사용,
WebTrust 인증이 2023년에서 갱신되지 않아 2024-2025년 감사 여부를 알 수 없다는 점,
CPS의 모호한 책임 면제 조항을 지적했다.
이 중 어느 하나만으로도 Mozilla가 재검토를 요구할 충분한 이유가 된다.

CRL/OCSP 접속 차단 문제는 단순한 설정 실수로 보기 어렵다.
공공기관 인프라가 기본적으로 외부 접속을 차단하는 방향으로 설계되어 있다면,
이는 루트 CA 운영 철학 자체의 문제다.
루트 CA는 전 세계 어디서든 인증서 상태를 확인할 수 있어야 하며,
이것은 선택적 요구사항이 아니라 기본 조건이다.

감사 기관 문제는 더 복잡하다.
딜로이트 안진이 이전 GPKI 감사에서 실패를 보이자 Mozilla가 불신을 선언했는데,
같은 감사 기관이 새 계층을 다시 감사했다.
감사 기관 교체 없이 같은 체계를 유지하면서 “새로운 시작”을 주장하는 셈이다.
이는 Mozilla 커뮤니티가 납득하기 어려운 접근이다.

[GeekNews 댓글](https://news.hada.io/topic?id=30329)에서 yeobi222는 흥미로운 현실적 맥락을 짚었다.
어차피 외부 대민 사이트에 자체 인증서를 탑재할 계획이 없기 때문에,
이 시도가 성공하더라도 실질적 변화는 크지 않을 수 있다는 것이다.
결과적으로 Chrome 의존도만 높아지고 있다는 지적도 덧붙였다.

## 비평

chickendreamtree는 [GeekNews 댓글](https://news.hada.io/topic?id=30329)에서
제한된 정보로 무능함을 판단하지 말고 시스템적 제약과 조직적 압박을 이해해야 한다고 했다.
이 시각은 공감할 수 있다.
공공기관에서 일하는 담당자들이 기술적 이상을 실현하기 어려운 구조적 현실이 있다.

그러나 Mozilla 루트 저장소 포함 심사는 선의나 노력을 보상하는 자리가 아니다.
글로벌 브라우저가 신뢰하는 루트 CA는 기술적 기준을 충족해야 하며,
조직 내 어려움은 외부 보안 기준의 완화 근거가 될 수 없다.

chcv0313, regentag, qwkjsan99가 [GeekNews 댓글](https://news.hada.io/topic?id=30329)에서
시스템적·조직적 문제가 반복 실패의 원인이라고 지적한 것은 핵심을 찌른다.
기술 문제가 아니라 거버넌스 문제라는 진단이다.
CRL 엔드포인트 하나를 외부에서 접근 가능하게 만드는 것은 기술적으로 어렵지 않다.
이것이 수년간 고쳐지지 않는다면, 기술 역량이 아니라 의사결정 구조의 문제로 봐야 한다.

WebTrust 감사가 2023년에 멈춰 있다는 점도 심각하다.
루트 CA는 매년 감사를 받아야 하며, 이것은 전 세계 CA 공동체가 합의한 기본 요건이다.
갱신되지 않은 감사 인증은 인증서 자체가 아무리 기술적으로 완벽해도
Mozilla Root Program에서 수락될 수 없는 결격 사유다.

## 인사이트

이 버그 리포트는 단순히 인증서 하나의 수락 여부 문제가 아니다.
한국 인터넷 인프라의 글로벌 신뢰성 구축 실패가 누적된 결과다.

가장 우려되는 2차 효과는 Chrome 중심성의 강화다.
Firefox가 거절하면 공공기관 웹사이트는 결국 Chrome에만 의존하게 된다.
이는 브라우저 다양성을 줄이고, Google의 웹 표준 결정권에 대한 국가 인프라의 종속성을 높인다.
정부가 브라우저 독립성을 원한다면 Mozilla가 요구하는 기준을 충족하는 것이 유일한 경로다.

또 하나의 문제는 이 상황이 반복된다는 점이다.
2015년(Bug 1226100), 2017년(Bug 1377389), 그리고 2023년.
매번 비슷한 지적이 반복되고, 근본 원인은 해결되지 않은 채 새 요청이 제출된다.
감사 기관을 바꾸지 않고, CRL 인프라를 고치지 않고, CPS의 모순을 제거하지 않은 상태에서
새 계층을 만들어 다시 도전하는 방식은 Mozilla 커뮤니티의 신뢰를 더욱 소진시킨다.

공공 PKI를 진지하게 운영하려면 국제 기준에 맞는 감사 체계 구축이 먼저다.
딜로이트 안진이 이 역할을 할 수 있는지 여부는 Mozilla가 이미 한 번 판단했다.
같은 기관과 같은 관계를 유지하면서 다른 결과를 기대하는 것은 무리다.
