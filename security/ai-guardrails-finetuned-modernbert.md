# $1 AI Guardrails: The Unreasonable Effectiveness of Finetuned ModernBERTs

원문: https://www.youtube.com/watch?v=YZHPEkfy2kc

## 요약

Diego Carpentero는 LLM 기반 시스템을 위협하는 주요 공격 벡터(attack vector)를 분류하고, 이에 대응하는 저지연(low-latency) 자체 호스팅 방어 레이어를 1달러 이하의 비용으로 구축하는 방법을 제시한다. 2023년에는 시스템 프롬프트를 탈취하려는 탐색적 수준의 프롬프트 인젝션(prompt injection)이 주를 이뤘지만, 현재는 신원 워크플로우(identity workflow)와 결합된 정교한 공격으로 진화했으며, 이 공격들은 더 이상 예외가 아니라 기준(baseline)이 됐다.

공격 벡터는 여섯 가지로 분류된다. 직접 인젝션(direct injection)은 사용자 입력으로 시스템 제어를 우회하고 데이터를 탈취한다. 간접 인젝션(indirect injection)은 웹 페이지, 이메일 받은편지함 등 외부 콘텐츠에 악성 지시를 심어 LLM이 이를 읽도록 유도한다. LLM 내부 벡터(LLM internals vector)는 무의미한 접미사 토큰(suffix token)을 통해 모델 정렬(model alignment)을 확률적으로 무력화한다. RAG 벡터는 8백만 개 문서 중 단 다섯 개의 오염된 청크(chunk)로 LLM 응답을 조작할 수 있음을 보인다. MCP(Model Context Protocol) 벡터는 사용자에게 보이는 도구 요약과 LLM이 읽는 전체 설명 사이의 비대칭을 악용한다. 에이전트 벡터는 링크 클릭, 유니코드 숨김 문자, 원격 코드 실행, 공급망 공격(supply chain attack)으로 이어지는 복합적인 공격이다.

이 모든 공격의 근본 원인은 LLM이 시스템 제어와 데이터를 구분하는 네이티브(native) 메커니즘을 갖지 않는다는 점이다. 발표자는 이를 “제로 트러스트(zero trust) 격차”로 정의하며, 모델 정렬은 확률적 선호(probabilistic preference)일 뿐 하드 제약(hard constraint)이 아니라고 강조한다. 인간 검토자 역시 아이스버그 효과(iceberg effect)로 인해 실제 승인 내용을 파악하기 어렵다.

방어 레이어 구현에는 인코더 모델(encoder model)인 ModernBERT를 파인튜닝(fine-tuning)하는 방식을 선택한다. 양방향 어텐션(bidirectional attention), 교번 어텐션(alternating attention), 회전 위치 인코딩(rotary positional encoding, RoPE), 플래시 어텐션(flash attention), 언패딩 및 시퀀스 패킹(unpadding and sequence packing) 등의 아키텍처 개선이 결합되어 분류(classification) 작업을 35밀리초 내에 수행하면서도 파인튜닝 메모리를 70% 절감한다. InjectGuard 데이터셋의 75,000개 레이블 예시로 파인튜닝한 결과, 미확인 데이터 기준 약 85% 정확도를 달성했다.

## 분석

### 공격 표면의 확장 구조

발표의 공격 벡터 분류는 단순 나열이 아니라 공격 표면이 어떻게 LLM의 입력 경계 바깥으로 확장되는지를 보여주는 체계적 분석이다. 프롬프트 → 컨텍스트(RAG) → 도구(MCP) → 에이전트 행동 순서로 전개되며, 각 단계에서 공격자의 제어 범위가 넓어지고 탐지 가능성은 낮아진다. 특히 2026년 3월에 보고된 AI 광고 심사 시스템 우회 사례는 “AI가 평가하는 데이터가 AI의 의사결정을 역전시킨다”는 구조적 취약점을 실증한 첫 사례로, 단순한 기술적 버그가 아니라 아키텍처 설계 원칙의 부재에서 비롯된 것임을 명확히 한다.

### ModernBERT 아키텍처 선택의 타당성

인코더 모델을 선택한 이유는 분류 과제의 특성과 정확히 맞닿아 있다. 생성(generative) 작업이 불필요한 이진 분류에서 디코더(decoder) 모델의 자동회귀(autoregressive) 처리는 낭비다. 양방향 어텐션은 시퀀스 전체를 한 번의 순전파(forward pass)로 처리해 `CLS` 토큰에 문맥 전체를 응축하며, 이것이 35밀리초 지연의 기술적 근거다.

교번 어텐션 설계는 공격 패턴의 분포와 맞물린다. 깁스어시 접미사(gibberish suffix)나 GitHub 이슈 제목 내 인젝션처럼 국소적으로 집중된 패턴은 로컬 어텐션(슬라이딩 윈도우 128토큰)으로, MCP 도구 설명이나 에이전트 플랜처럼 긴 문맥을 요구하는 패턴은 8,192토큰 글로벌 어텐션으로 처리한다. 이는 단순한 효율화가 아니라 위협 모델(threat model)과 아키텍처를 정렬한 설계다.

### 비용 구조와 자체 호스팅의 함의

“1달러 이하”라는 표현은 파인튜닝 비용보다는 추론(inference) 운영 비용을 암시한다. 자체 호스팅은 내부 요청과 중간 단계의 모델 응답을 외부 제공자에게 전송하지 않아도 되므로 프라이버시 리스크와 토큰 비용이 동시에 제거된다. 파이프라인에 LLM-as-a-judge 방식을 사용할 경우 누적 지연이 수 초에 달할 수 있지만, 파인튜닝된 ModernBERT는 단일 안전 점검에 35밀리초를 사용하면서도 재훈련을 몇 시간 내에 완료할 수 있어 빠르게 변화하는 공격 패턴에 대응 가능하다.

## 비평

### 강점

공격 벡터 분류는 구체적인 사례(시드니 사건, 위키피디아 리다이렉션 POC, 2026년 광고 시스템 우회, npm 공급망 공격)를 근거로 제시해 설득력이 높다. 특히 LLM 내부 벡터에서 그리디 좌표 기울기(greedy coordinate gradient) 탐색 방식과 블랙박스 모델로의 전이 가능성을 설명한 부분은 학술적 근거가 명확하다. ModernBERT의 각 아키텍처 개선을 방어 과제의 요구사항과 직접 연결해 설명한 방식도 실무 적용 맥락에서 유용하다.

### 약점

85% 정확도 수치의 맥락이 불충분하다. 어떤 벤치마크 데이터셋인지, 오탐(false positive)과 미탐(false negative)의 비율이 어떤지, 그리고 안전 시스템에서 더 중요한 지표인 재현율(recall)은 얼마인지 제시되지 않는다. 15%의 오류율이 프로덕션 환경에서 허용 가능한지에 대한 논의가 없다.

또한 InjectGuard 데이터셋의 75,000개 예시가 현재의 공격 다양성을 충분히 커버하는지 검증이 필요하다. 발표자 스스로 공격 벡터가 “동적이고 다양하며 변이한다”고 강조했음에도, 훈련 데이터가 이 변이 속도를 따라잡을 수 있는지에 대한 구체적인 논의는 생략됐다. 자체 호스팅 GPU 비용, 운영 복잡성, 보안 업데이트 부담도 “1달러 이하” 표현에 포함되지 않았다.

에이전트 환경에서의 방어는 언급했지만 구체적인 구현 방법이 없다. 에이전트 플랜이나 중간 추론 단계를 실시간으로 분류하는 것은 프롬프트 하나를 분류하는 것과 전혀 다른 파이프라인 설계를 요구한다.

## 인사이트

### 제로 트러스트 원칙의 LLM 적용 공백

제로 트러스트는 “신뢰하지 말고 항상 검증하라”는 원칙으로 현대 보안 아키텍처의 표준이 됐다. 그런데 LLM은 이 원칙을 구조적으로 위반하는 방식으로 설계됐다. 시스템 프롬프트와 사용자 입력, 외부 데이터가 모두 동일한 컨텍스트 윈도우 안에서 평등하게 취급된다. 이는 유닉스(Unix)에서 커널 공간과 사용자 공간을 분리하지 않은 것과 유사하다.

이 공백은 단기적으로 외부 감시 레이어(guardrail)로 보완할 수 있지만, 장기적으로는 LLM 아키텍처 자체에서 신뢰 계층(trust hierarchy)을 구분하는 설계가 필요하다. 구조적 태그, 암호화 서명된 시스템 프롬프트, 또는 별도의 신뢰 경계를 가진 실행 환경 등이 연구 방향이 될 수 있다. 현재의 가드레일 접근은 증상을 치료하는 것이며, 근본 원인은 아키텍처 설계에 있다.

### 분류기(discriminator)의 군비 경쟁 딜레마

파인튜닝된 ModernBERT로 공격을 탐지하는 접근은 역사적으로 반복된 패턴이다. 스팸 필터, 악성코드 탐지, 네트워크 침입 탐지(IDS)가 모두 같은 경로를 걸었다. 탐지기가 등장하면 공격자는 탐지를 우회하는 기법을 개발한다. 깁스어시 접미사가 블랙박스 모델에 전이 가능한 것처럼, 분류기를 우회하도록 최적화된 적대적 프롬프트(adversarial prompt)도 같은 원리로 생성 가능하다.

발표자가 “몇 시간 내에 재훈련 가능”하다고 강조한 것은 이 군비 경쟁을 의식한 언급이다. 핵심은 재훈련 주기가 공격 변이 속도보다 빠를 수 있느냐다. 스팸 필터의 역사는 탐지기와 공격자가 공진화(co-evolution)하는 구조임을 보여준다. 단일 분류기에 의존하는 것보다 다층 방어(defense in depth), 즉 가드레일과 런타임 격리, 최소 권한 에이전트 설계를 결합하는 것이 더 현실적인 접근이다.

### AI 에이전트의 권한 확대와 공격 표면의 지수적 증가

2025년 이전의 LLM 공격은 대부분 정보 탈취(exfiltration)에 집중됐다. 에이전트 시스템이 등장하면서 공격의 결과가 텍스트 탈취에서 코드 실행, 파일 조작, 공급망 감염으로 확대됐다. 에이전트가 “링크를 클릭하고 파일을 다운로드하고 실행 권한을 변경”할 수 있다면, 이는 사실상 원격 코드 실행(RCE) 취약점과 동일한 위험 수준이다.

근본적인 문제는 에이전트에게 부여하는 권한의 범위다. 4,000~5,000명의 개발자에게 영향을 미친 npm 공급망 공격 사례는 코딩 에이전트가 검증 없이 외부 패키지를 설치하는 것이 시스템 수준의 위험임을 보여준다. 최소 권한 원칙(principle of least privilege)을 에이전트 설계에 적용하는 것, 즉 에이전트가 필요한 최소한의 동작만 수행하도록 권한을 제한하는 것이 가드레일보다 선행돼야 하는 설계 원칙이다. 가드레일은 권한이 이미 부여된 상태에서의 오용을 탐지하지만, 권한 자체를 제한하는 것이 더 근본적인 방어다.
