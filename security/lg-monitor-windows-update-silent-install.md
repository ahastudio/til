# LG 모니터가 Windows Update로 소프트웨어를 몰래 설치한다

원문: <https://videocardz.com/newz/lg-monitors-silently-install-software-through-windows-update-without-user-consent>

HN 토론: <https://news.ycombinator.com/item?id=48956688> (1196점, 603개 댓글)

## 요약

LG 모니터를 Windows PC에 연결하면 사용자 동의 없이 LG 제조사 소프트웨어가
자동으로 설치되는 현상이 확인됐다.
설치 경로는 Windows Update의 드라이버 메타데이터 배포 메커니즘이다.
사용자가 아무 설치 버튼도 누르지 않아도 모니터를 HDMI로 연결하는 것만으로
백그라운드에서 소프트웨어가 설치된다.

이 현상은 여러 LG 모니터 모델에서 발생하며 전문가용 제품군도 예외가 아니다.
설치되는 소프트웨어는 LG 모니터 제어 앱이며 샌드박스 없이 시스템에 전체 접근 권한을
갖는 일반 앱으로 동작한다.
GamersNexus가 이 문제를 영상으로 다루면서 더 넓은 주목을 받았다.

Windows는 하드웨어 제조사가 장치 메타데이터와 함께 연관 앱을 패키지로 제출할 수 있도록
허용하고 있다.
LG는 이 채널을 사용자 동의 없는 소프트웨어 배포에 활용한 것이다.
이를 막으려면 그룹 정책 편집기에서 “장치 메타데이터 연관 앱 자동 다운로드 방지”를 활성화하거나,
홈 에디션에서는 `sysdm.cpl` → 하드웨어 탭 → 장치 설치 설정에서 제조사 앱 자동 설치를
비활성화해야 한다.

## 분석

### Windows Update가 소프트웨어 배포 채널로 사용되는 구조

이 사건의 핵심은 LG만의 문제가 아니라 Windows의 장치 메타데이터 배포 구조에 있다.
Microsoft는 하드웨어 제조사가 장치 드라이버와 함께 연관 앱을 제출하도록 허용하고,
Windows가 장치 감지 시 이 앱을 자동으로 설치하는 기능을 제공한다.
원래 의도는 새 장치를 연결할 때 필요한 드라이버나 보조 앱을 편리하게 설치하는 것이었다.

문제는 이 채널이 “편의”를 넘어 “무동의 배포”로 쓰일 때 발생한다.
드라이버는 하드웨어 동작에 필요한 최소한의 코드지만,
앱은 사용자 인터페이스와 네트워크 통신을 포함하는 별도 소프트웨어다.
이 둘을 같은 신뢰 수준으로 자동 설치하도록 허용한 것이 이 구조의 근본적인 문제다.

### 책임 소재를 둘러싼 논쟁

HN 토론에서 책임 소재를 두고 두 입장이 갈렸다.
한쪽은 LG가 사용자 동의 없이 소프트웨어를 배포하기로 결정했으므로 LG의 책임이라는 입장이다.
다른 쪽은 이 경로 자체를 만들고 허용한 Microsoft의 책임이 더 크다는 입장이다.[^gkbrk]
tialaramex는 Microsoft에 직접 압력을 넣는 것이 더 효과적이라고 지적했다.
기업 고객이 Microsoft와의 계약 협상에서 이 문제를 제기하면
제조사 앱 자동 설치 정책을 강화하도록 압박할 수 있다는 논리다.[^tialaramex]

두 입장 모두 맞다.
LG는 허용된 채널을 사용자 친화적이지 않은 방식으로 활용했고,
Microsoft는 그 채널을 충분한 제한 없이 개방해 두었다.
그러나 최종 사용자 입장에서 실질적인 해결책을 찾으려면
Microsoft 정책 변화가 더 근본적인 경로다.

## 비평

### “드라이버”와 “앱”의 경계를 허문 것이 핵심 문제다

이 사건을 단순히 “나쁜 제조사”의 문제로 보면 본질을 놓친다.
USB 자동 실행 취약점과 유사하게,[^gkbrk] 이 문제는 OS가 하드웨어 신호를
소프트웨어 설치 트리거로 사용하도록 설계된 데서 온다.
드라이버 설치와 앱 설치를 같은 신뢰 경로로 묶은 결정이 이 취약점의 원천이다.

보안 관점에서 하드웨어 연결은 코드 실행의 트리거가 되어서는 안 된다.
이 원칙은 USB 자동 실행 사태 이후 공론화됐지만,
장치 메타데이터를 통한 앱 배포에는 같은 기준이 적용되지 않았다.
같은 실수가 다른 인터페이스에서 반복되고 있는 셈이다.

### 기업 환경에서 실제로 더 심각하다

개인 사용자에게 이 사건은 불편함이지만, 기업 엔드포인트 관리 측면에서는 통제 실패다.
MDM 정책으로 관리되는 장치에서 Windows Update 경로를 통해 승인되지 않은 소프트웨어가
설치된다면 자산 관리, 보안 감사, 컴플라이언스 체계 전체에 구멍이 생긴다.
모니터를 교체했을 뿐인데 엔드포인트 소프트웨어 목록이 바뀌는 상황은
기업 보안팀에게 설명하기 어려운 사고다.

## 인사이트

### 드라이버 공급망이 앱 배포 채널이 됐다

이 사건은 드라이버 공급망이 사실상 소프트웨어 배포 채널로 확장된 현실을 보여준다.
하드웨어 제조사가 OS 업데이트 경로를 통해 앱을 배포할 수 있다면,
엔드포인트 보안은 앱 설치 목록 감시만으로 충분하지 않다.
드라이버, 펌웨어, 장치 메타데이터 연관 앱까지 감사 범위에 들어와야 한다.

이 확장은 앞으로도 계속될 가능성이 높다.
IoT 장치, 스마트 주변기기, 네트워크 장비도 유사한 자동 설치 경로를 요구하거나
이미 사용하고 있을 수 있다.
연결만으로 코드가 실행되는 표면적이 넓어지고 있다.

### 사용자 동의의 경계가 다시 협상되고 있다

LG가 선택한 배포 방식은 법적으로는 허용된 것일 수 있다.
Windows 생태계 내에서 사용 가능한 경로를 이용했기 때문이다.
그러나 사용자 관점에서 “동의한 적 없는 소프트웨어 설치”는 악성 소프트웨어의 정의와
구분하기 어렵다.[^kingleopold]

이 사건은 사용자 동의의 최소 기준이 기술적 가능성이 아니라
사용자의 인식과 통제 가능성으로 정의돼야 한다는 논쟁을 다시 불러온다.
하드웨어를 연결한다는 행위가 소프트웨어 설치에 대한 묵시적 동의가 아님은
대부분의 사용자가 동의하는 원칙이지만,
그 원칙을 기술 표준과 규제로 강제하는 일은 아직 불완전하다.
