# GLM 5.2가 보안 벤치마크에서 Claude를 앞서다

원문: <https://semgrep.dev/blog/2026/we-have-mythos-at-home-glm-52-beats-claude-in-our-cyber-benchmarks/>

HN 토론: <https://news.ycombinator.com/item?id=48709670> (428점, 208개 댓글)

## 요약

Semgrep이 IDOR(Insecure Direct Object Reference) 취약점 탐지를 주제로
다양한 AI 모델의 성능을 비교한 벤치마크 결과를 발표했다.
데이터셋, 평가 지표(F1 스코어), 시스템 프롬프트를 동일하게 고정한 채
모델과 처리 환경만 바꿔가며 실험했다.

F1 스코어 기준 순위는 다음과 같다.
Semgrep 멀티모달 + GPT 5.5가 61%로 1위였고,
Semgrep 멀티모달 + Opus 4.8이 53%로 그 뒤를 이었다.
오픈 웨이트 모델인 GLM 5.2는 39%를 기록하며
Claude Code(Opus 4.6)의 37%와 Claude Code(Opus 4.8/4.7)의 28%를 모두 앞질렀다.

GLM 5.2는 취약점 하나를 탐지하는 데 약 $0.17이 들었다.
연구팀은 이를 Claude Code 대비 “1/6 비용”이라고 표현했다.
GLM 5.2는 MoE(Mixture-of-Experts) 아키텍처로 총 파라미터 약 750B,
활성 파라미터는 토큰당 약 40B이며 100만 토큰 컨텍스트 윈도우를 제공한다.

연구팀은 결과의 한계를 명확히 밝혔다.
“하나의 태스크, 하나의 데이터셋, 한 번의 실행”이라는 전제 하에
결과의 일반화 가능성이 제한적임을 인정했다.
Semgrep 멀티모달 파이프라인은 엔드포인트 열거와 가이드 내비게이션 같은
추가 스캐폴딩을 제공했지만, 오픈 웨이트 모델은 최소한의 지원만 받았다.

## 분석

### 벤치마크가 드러낸 진짜 비교 대상

이 실험에서 가장 주목할 비교는 GLM 5.2 대 Claude가 아니다.
진정한 대조는 Semgrep의 독점 멀티모달 파이프라인과
단순 프롬프트 기반 오픈 웨이트 모델 사이의 성능 격차다.
전자는 엔드포인트 열거와 가이드 내비게이션이라는 도메인 특화 지원을 받았고,
후자는 시스템 프롬프트만 받았다.
이 격차가 22%포인트(61% vs 39%)인데, 이것이 Semgrep 자체 기술의 가치를 보여주는 수치다.

반면 GLM 5.2가 Claude Code를 넘어선 것은 다른 층위의 발견이다.
모두 프롬프트만 받은 조건에서 오픈 웨이트 모델이 상업 에이전트 하네스를 앞선 셈이다.
여기서 상기해야 할 점은 WithinReason이 댓글에서 지적한 것처럼[^WithinReason]
Claude Code는 LLM이 아닌 에이전트 하네스라는 것이다.
즉 에이전트 오케스트레이션 없이 GLM 5.2 단독이 에이전트화된 Claude를 이겼다는 해석도 가능하지만,
이는 에이전트 하네스가 이 특정 태스크에서 오히려 방해가 됐을 수도 있음을 시사한다.

### 비용 효율성의 구조적 의미

$0.17/취약점이라는 수치는 단순한 가격 비교가 아니다.
이것이 의미하는 바는 보안 스캐닝의 경제적 모델이 바뀌고 있다는 점이다.
기존에는 비용이 높아서 지속적 스캐닝보다 주기적 감사를 택했다면,
이 비용 수준에서는 PR마다, 커밋마다 IDOR 스캔을 돌리는 것이 경제적으로 타당해진다.
보안 스캐닝이 “비싼 일회성 이벤트”에서 “저렴한 지속 프로세스”로 이동하는 변곡점이다.

SwellJoe는 자체 보안 버그 헌팅 벤치마크에서 DeepSeek V4 Pro가
캐싱 효율로 비용 면에서 가장 경쟁력 있다고 보고했다.[^SwellJoe]
또한 semgrep 도구 통합이 성능을 개선하지 못했다고 언급하며,
모델이 낯선 도구 마스터링과 보안 문제 해결을 동시에 요구받을 때 어려움을 겪는다고 추측했다.
도구 통합이 오히려 성능을 저하시킬 수 있다는 이 관찰은
에이전트 설계에서 도구 복잡도 관리의 중요성을 부각한다.

### 오픈 웨이트 모델의 보안 도메인 진입

GLM 5.2는 중국 연구소 ZhipuAI에서 개발한 모델이다.
solenoid0937가 지적했듯[^solenoid0937] 미국 상무부가 OpenRouter나 Hugging Face를 통한
특정 오픈 모델 배포에 제재를 가할 가능성이 있다.
GLM 5.2가 보안 취약점 탐지에서 경쟁력을 보인 것은
이 문제에 지정학적 차원을 더한다.
미국 사이버보안 인프라가 중국 오픈 웨이트 모델에 의존하는 구도는
정책 입안자들이 수출 통제를 논의할 때 또 다른 변수가 된다.

## 비평

### 단일 실행이 숨기는 것

연구팀 스스로 “하나의 태스크, 하나의 데이터셋, 한 번의 실행”이라는 한계를 명시했지만,
이 자기 고백이 결과 해석에 충분히 반영되지 않는 경향이 있다.
F1 스코어의 통계적 안정성을 확보하려면 동일 조건에서 최소 수십 회 실행의 평균이 필요하다.
LLM의 출력은 확률적이므로 단일 실행 결과는 높은 분산을 가진다.
39%와 37%의 차이는 실행 간 자연스러운 편차 범위 안에 있을 수 있다.

bArray가 댓글에서 GLM 5.2의 750B 파라미터 로컬 실행에 필요한 하드웨어를 물었다.[^bArray]
이 질문 자체가 실용적 맹점을 드러낸다.
로컬 실행이 불가능하다면 API를 통해야 하고,
그 경우 비용 구조는 논문에서 제시한 수치와 달라질 수 있다.
또한 API 가용성, 레이턴시, 처리량 제한도 실제 운영 환경에서의 적용 가능성에 영향을 준다.

### 파이프라인 불균등이 만드는 착시

Semgrep 멀티모달 파이프라인은 엔드포인트 열거와 가이드 내비게이션을 제공했지만
단순 프롬프트 기반 모델들은 그런 지원 없이 경쟁했다.
이는 “모델 vs 모델”이 아니라 “파이프라인 vs 모델”의 비교다.
GLM 5.2도 동일한 멀티모달 파이프라인에서 실행하면 어떤 성능을 낼지 알 수 없다.
반대로 Claude Code도 동일한 단순 프롬프트 조건이라면 결과가 달라질 수 있다.

이 비교 구조는 Semgrep 자체 파이프라인의 가치를 증명하는 데는 유효하지만,
모델 간 순수 성능 비교로 인용할 때는 오해를 낳는다.
“GLM 5.2가 Claude를 이겼다”는 제목은 정확하지 않으며,
더 정확하게는 “멀티모달 파이프라인 없이 GLM 5.2가 에이전트 하네스 Claude Code를 앞섰다”여야 한다.

### IDOR에 한정된 결과의 확장 불가능성

IDOR는 사이버보안의 극히 일부다.
이 취약점 클래스는 접근 제어 로직이 비교적 명확하고,
패턴을 학습하기 용이한 편이다.
SQL 인젝션, 비즈니스 로직 취약점, 제로데이 등 다른 취약점 유형에서
동일한 순위가 유지될지는 전혀 불명확하다.
himata4113은 Windows 커널 관련 취약점에서 자신이 달성한 성능이
여기 제시된 수치보다 훨씬 높다고 언급하며[^himata4113]
도메인 특화 지식이 범용 벤치마크 성능보다 실제 보안 작업에서 더 중요할 수 있음을 시사했다.

## 인사이트

### 벤치마크 경쟁이 재구성하는 AI 시장 구조

“GLM 5.2가 Claude를 이겼다”는 뉴스가 HN에서 428점을 받았다는 사실 자체가
오픈 웨이트 모델이 상업 모델을 앞서는 사례가 이제 놀라운 일이 아닌
“기대되는 일”이 됐음을 보여준다.
1년 전이라면 동일한 뉴스가 더 큰 충격을 줬을 것이다.
이 정상화는 상업 AI 회사들이 가격을 경쟁 요소로 점점 더 강하게 의식해야 한다는 압박을 의미한다.

XCSme가 보고한 것처럼[^XCSme] GLM 5.2가 Opus 4.8보다 약간 낮은 성능이지만
5배 저렴하고 3배 느린 특성은 흥미로운 트레이드오프를 제시한다.
배치 처리나 비실시간 보안 스캐닝 워크플로에서는
이 트레이드오프가 오히려 유리할 수 있다.
실시간 응답이 필요 없는 CI/CD 파이프라인 통합이나 야간 배치 스캔에서는
비용 효율이 레이턴시보다 훨씬 중요하다.

### 보안 AI의 결정론적 신뢰 문제

F1 스코어 39%는 무엇을 의미하는가.
취약점 10개 중 약 4개를 찾고, 찾아낸 것 중 일부는 거짓 양성이라는 뜻이다.
이 수치가 보안 컨텍스트에서 실용적인가 하는 질문은
벤치마크 성능 자체와 다른 층위에 있다.
보안 엔지니어가 AI 스캐너를 신뢰하기 위해서는
거짓 음성과 거짓 양성의 패턴이 예측 가능해야 한다.
LLM 기반 스캐너는 확률적이기 때문에 동일 코드에 대해
다음 실행에서 다른 결과를 낼 수 있다.
이 비결정론성이 보안 감사의 필수 조건인 “재현 가능한 결과”와 어떻게 공존할 수 있는지는
아직 답이 없는 질문이다.

### 오픈 웨이트 모델의 지정학적 가속

GLM 5.2 같은 중국 오픈 웨이트 모델이 보안 도메인에서 경쟁력을 가진다는 발견은
AI 수출 통제 논의에 새로운 복잡성을 더한다.
모델 가중치가 이미 공개됐다면 수출 통제의 실효성이 제한되고,
오히려 미국 사용자들이 경쟁력 있는 오픈 모델에 접근하지 못하는 결과를 낳을 수 있다.
이는 규제가 혁신 속도를 따라잡지 못할 때 발생하는 정책 역설이다.
보안 AI 경쟁에서 국가별 우위가 모델 성능이 아닌 접근성 제한으로 결정될 수 있다는 시나리오는
오픈 소스 AI 생태계의 장기 지속 가능성에 대한 더 큰 질문을 제기한다.
