# Agent Safehouse - macOS에서 LLM 에이전트를 커널 수준으로 격리하기

> 원문: <https://www.agent-safehouse.dev/>

## 요약

Agent Safehouse는 macOS의 `sandbox-exec`를 이용해 LLM 코딩 에이전트를 커널 수준에서
격리하는 오픈소스 도구다. "Go full `--yolo`. We've got you"라는 슬로건처럼, 에이전트에게
광범위한 권한을 주면서도 프로젝트 디렉토리 밖으로의 쓰기 접근을 OS 커널이 물리적으로
차단한다.

Bash와 macOS 외에 별도 의존성이 없으며, Homebrew로 설치하거나 단일 셸 스크립트로
바로 사용할 수 있다. Claude Code, OpenAI Codex, GitHub Copilot CLI, Gemini CLI,
Aider, Cursor Agent, Goose 등 13개 이상의 에이전트를 지원한다.

핵심 설계 원칙은 거부 우선(deny-by-default)이다. SSH 키, AWS 자격증명, 다른 저장소는
기본적으로 차단되며, 명시적으로 허용된 경로만 접근 가능하다. LLM 프롬프트를 활용해
개인 환경에 맞는 최소 권한 프로필을 자동 생성하는 기능도 제공한다.

## 분석

Agent Safehouse의 접근법은 프롬프트 수준이 아닌 운영체제 경계를 보안 레이어로
사용한다는 점에서 근본적으로 다르다. 에이전트에게 "실수하지 마"라고 지시해도
LLM은 확률적으로 동작하기 때문에 결국 사고가 발생할 수 있다. macOS `sandbox-exec`는
커널이 직접 접근을 거부하므로 모델 품질과 무관하게 외부 시스템 손상이 불가능해진다.

```bash
cd ~/projects/my-app
safehouse claude --dangerously-skip-permissions
```

셸 래퍼 함수를 추가하면 에이전트 실행이 자동으로 샌드박싱된다. 개발자 입장에서는
기존 워크플로 변경 없이 보안 레이어가 투명하게 삽입된다.

HN 댓글에서 789점·175댓글이라는 높은 반응은 에이전트 보안을 프롬프트 엔지니어링
문제가 아닌 시스템 설계 문제로 봐야 한다는 공감대가 이미 형성됐음을 보여준다.
"토큰 저장 위치, IPC 경로, 임시 파일 처리까지 같이 잠그지 않으면 우회면이 그대로
남는다"는 지적도 나왔다.

## 비평

`sandbox-exec`는 macOS 전용이고 Apple이 공식적으로 deprecated로 분류한 API다.
즉, 지금은 동작하지만 미래의 macOS 버전에서 제거되거나 제한될 수 있다는 위험이
있다. 장기적으로 신뢰할 수 있는 기반인지는 불확실하다.

또한 네트워크 접근 제어는 포함되지 않는다. 파일 시스템 쓰기를 막더라도 에이전트가
자격증명을 읽어 외부로 전송하는 경로는 여전히 열려 있다. "읽기 차단 없이 쓰기만
차단"하는 모델은 데이터 유출 시나리오를 완전히 다루지 못한다.

그럼에도 현재 시점에서 실질적인 보호를 즉시 제공한다는 점은 분명한 가치다. 완벽한
샌드박스가 아니더라도 "실수로 `/` 하위를 지우는" 가장 흔한 사고는 방지한다.

## 인사이트

### LLM 에이전트 보안의 계층 모델이 정립되고 있다

프롬프트 가이드라인(1계층),
애플리케이션 레벨 권한 요청(2계층), OS 커널 격리(3계층)라는 계층이 형성되고 있다.
Agent Safehouse는 3계층에 해당한다. 브라우저 보안이 JavaScript 실행 제한에서
OS 프로세스 격리로 발전했듯, 에이전트 보안도 같은 경로를 따를 것이다.

### "--yolo 모드를 안전하게"라는 역설이 핵심 가치 제안이다

에이전트를 제한적으로
쓰면 생산성 이점이 줄고, 자유롭게 쓰면 위험이 커진다는 딜레마를 OS 격리로
해소한다. 이 접근법은 에이전트 활용의 심리적 마찰을 줄여준다. 개발자가 에이전트를
더 적극적으로 쓸 수 있게 되는 2차 효과가 직접적인 보안 효과만큼 중요하다.

### 자격증명 보호가 파일 쓰기 보호보다 더 중요할 수 있다

`~/.ssh`, `~/.aws`를
읽기조차 차단하지 않으면 에이전트가 자격증명을 읽어 외부 API를 호출하거나 탈취할
수 있다. 진정한 에이전트 샌드박스는 파일 쓰기 차단이 아니라 비밀 관리 시스템과의
통합이 핵심이 될 것이다. 1Password나 Vault 같은 시크릿 매니저와 에이전트 런타임을
연결하는 표준이 등장할 가능성이 높다.
