### 2. Using go fix to modernize Go code

- **출처**: Go Blog — <https://go.dev/blog/gofix>
- **한 줄 요약**: Go 1.26에 맞춰 `go fix`가 전면 개편되어 최신 문법과
  라이브러리로 자동 현대화한다.
- **왜 주목받나**: HN에서 많은 추천과 댓글을 받았고, 레거시 Go 코드의 대량
  개선을 자동화할 수 있어서.
- **개발자 관점 인사이트**:
  - 실무 영향: 대규모 리팩터링의 인력 부담과 리뷰 비용이 감소.
  - 즉시 활용: `go fix -diff` 로 변경분을 검토한 뒤 릴리스마다 적용.
  - 방향: 조직별 규칙을 분석기로 구현하는 "셀프 서비스" 도구 확산.
  - 주의: 빌드 태그별로 여러 번 실행하고, 깨끗한 Git 상태에서 적용.

### 5. pg_background: Make Postgres do the long work

- **출처**: vibhorkumar.wordpress.com —
  <https://vibhorkumar.wordpress.com/2026/02/16/pg_background-make-postgres-do-the-long-work-while-your-session-stays-light/>
- **한 줄 요약**: `pg_background`로 SQL을 백그라운드 워커에서 실행해 세션을
  가볍게 유지하는 방법을 설명.
- **왜 주목받나**: HN에서 실무형 Postgres 팁으로 반응이 높았고, 별도 큐 없이 긴

- **개발자 관점 인사이트**:
  - 실무 영향: DB 내부에서 장시간 작업을 분리해 응답성을 확보.
  - 즉시 활용: VACUUM, 대규모 리포팅, 인덱스 작업을 백그라운드로 실행.
  - 방향: Postgres 내부 백그라운드 작업 패턴 확산.
  - 주의: `max_worker_processes` 한도와 권한 설정을 사전에 점검.
