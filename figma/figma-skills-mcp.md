# Figma Skills for MCP — MCP 서버와 연동하는 Figma 스킬 가이드

<https://help.figma.com/hc/en-us/articles/39166810751895-Figma-skills-for-MCP>

## 소개

Figma Skills는 AI 어시스턴트에게 일반적인 Figma 작업을 안정적으로 처리하는 방법을
가르치는 사전 구축된 지시 파일이다.
Figma MCP(Model Context Protocol) 서버와 함께 사용하며, 스킬을 설치하면 매번 작업을
설명하지 않고도 명령만으로 반복 작업을 수행할 수 있다.

## 기본 스킬

Figma가 공식 제공하는 6개의 기본(foundation) 스킬이다.

| 스킬                              | 역할                                             |
| --------------------------------- | ------------------------------------------------ |
| `figma-use`                       | 캔버스에 프레임, 컴포넌트, 변수 생성 및 편집    |
| `figma-use-figjam`                | FigJam 보드에 스티키, 섹션, 커넥터 추가·수정   |
| `figma-code-connect-components`   | 디자인과 코드 구현을 연결                        |
| `figma-create-design-system-rules`| 팀의 디자인 규칙 저장 및 관리                   |
| `figma-create-new-file`           | 새 Figma 파일 또는 FigJam 생성                  |
| `figma-implement-design`          | 디자인을 실제 코드로 변환                        |

## 예시 스킬

더 복잡한 워크플로를 위한 예시 스킬이다.

| 스킬                     | 역할                                              |
| ------------------------ | ------------------------------------------------- |
| `figma-generate-library` | 코드베이스에서 디자인 시스템 라이브러리 생성      |
| `figma-generate-design`  | 실제 컴포넌트를 사용한 전체 페이지 화면 작성      |

## 설치 방법

### 방법 1: 플러그인 (권장)

Claude Code나 Cursor 같은 AI 도구에서 Figma 플러그인을 설치하면 스킬이 자동으로
포함된다.

### 방법 2: 수동 다운로드

GitHub 저장소에서 스킬 마크다운 파일을 직접 내려받아 개발 환경에 설치할 수 있다.

## 권한 및 플랜 요건

- `figma-use`, `figma-use-figjam`: 해당 파일에 대한 편집 권한 필요
- `figma-code-connect-components`: 팀 라이브러리 게시 필수, Organization 이상 플랜 필요
- Dev 시트: 드래프트 외부 파일에 읽기 전용 접근만 허용
- 일부 기능은 베타 기간 중 무료 제공

커뮤니티에서 추가 스킬을 찾거나, 팀 특화 워크플로를 위한 커스텀 스킬을 직접 개발할
수도 있다.

## 분석

### 스킬과 MCP의 역할 분리

Figma MCP 서버는 Figma API에 대한 접근 계층(access layer)을 제공하고, 스킬은
그 위에서 작동하는 행동 명세(behavioral specification) 계층이다.
MCP 없이는 스킬이 Figma 캔버스를 조작할 수 없고, 스킬 없이는 MCP가 어떤 작업을
어떤 순서로 수행해야 하는지 AI가 판단해야 한다.
두 계층의 분리는 “연결”(MCP)과 “행동”(스킬)을 독립적으로 업데이트할 수 있게 한다.

### 기본 스킬의 구성 논리

6개 기본 스킬은 Figma 워크플로의 주요 진입점을 망라한다.
캔버스 조작(`figma-use`), FigJam 조작(`figma-use-figjam`), 코드 연결
(`figma-code-connect-components`), 디자인 시스템 규칙, 파일 생성, 코드 변환이다.
이 분류는 “무엇을 만드는가”(캔버스/FigJam)와 “무엇과 연결하는가”(코드/디자인 시스템)를
축으로 설계되어 있다.

### 예시 스킬의 위치

`figma-generate-library`와 `figma-generate-design`은 기본 스킬을 조합해 더 복잡한
워크플로를 수행하는 상위 레벨 스킬이다.
예시(example)로 분류된 것은 이 스킬들이 팀이나 프로젝트마다 커스터마이징이 필요할
가능성이 크기 때문으로 보인다.
기본 스킬은 범용적이고, 예시 스킬은 특정 워크플로 패턴의 출발점으로 제공된다.

## 비평

### 강점

스킬을 플러그인 설치 시 자동 포함하는 방식은 진입 장벽을 낮추는 실용적 선택이다.
기본 6개 스킬이 Figma 핵심 워크플로를 포괄하면서도 각 스킬의 범위가 명확하게
분리되어 있다.
커스텀 스킬 개발 가능성을 열어두어, 표준화와 유연성을 동시에 확보했다.

### 약점 및 한계

`figma-code-connect-components`가 Organization 이상 플랜을 요구하는 점은 소규모 팀이나
개인 개발자에게 실질적 제약이다.
“예시 스킬”과 “기본 스킬”의 경계가 문서에서 충분히 설명되지 않아, 언제 어느 스킬을
써야 하는지 판단이 어려울 수 있다.
스킬 파일의 실제 내용(마크다운 문법, 구조, 사용 가능한 지시 형식)에 대한 상세
명세가 이 문서에는 없다.

### 관점의 공백

여러 AI 도구(Claude Code, Cursor, Copilot 등) 간의 스킬 호환성 차이가 언급되지
않는다.
스킬이 의도대로 동작하지 않을 때의 디버깅 방법이나 오류 시나리오가 빠져 있다.
Dev 시트의 읽기 전용 제한이 `figma-implement-design` 스킬의 실제 동작에 어떤 영향을
주는지 명확하지 않다.

## 인사이트

### MCP + 스킬 조합이 만드는 “도구 사용 지식의 표준화”

Figma가 MCP 서버와 스킬을 분리한 설계는 소프트웨어 아키텍처의 관심사 분리
(separation of concerns) 원칙을 AI 에이전트 통합에 적용한 사례다.
MCP는 “Figma와 어떻게 통신하는가”를 담당하고, 스킬은 “Figma에서 무엇을 어떻게
해야 하는가”를 담당한다.
이 분리는 AI 모델이 교체되거나 Figma API가 변경되어도 나머지 계층에 영향을 최소화한다.

이 패턴은 소프트웨어 도구 통합의 새로운 표준이 될 수 있다.
기존에는 도구 통합을 위해 SDK 또는 API 클라이언트 라이브러리를 작성했고, 이를
유지 관리하는 비용이 있었다.
MCP + 스킬 모델은 통신 계층(MCP)과 행동 지식(스킬)을 독립적으로 관리할 수 있게
하여 유지 보수 비용을 분산시킨다.
어떤 AI 에이전트든 MCP 프로토콜을 지원하면 스킬을 소비할 수 있다는 점에서
공급업체 독립성(vendor neutrality)도 확보된다.

### 플랜 제한이 드러내는 AI 워크플로 접근성의 균열

`figma-code-connect-components`가 Organization 플랜을 요구하는 구조는 AI 에이전트
워크플로의 접근성이 기존 SaaS 가격 체계를 그대로 상속한다는 점을 드러낸다.
AI가 강력한 기능을 자동화할수록, 그 기능이 프리미엄 플랜에 묶여 있으면 AI 워크플로의
혜택도 고가 플랜 사용자에게 집중된다.
이는 “AI 도구가 민주화를 가져온다”는 통념과 실제 제품 설계 사이의 긴장이다.

역사적으로 강력한 개발 도구는 먼저 엔터프라이즈 고객에게 제공된 후, 시간이 지나면서
접근성이 확대되는 패턴을 보였다.
Git이 처음에는 Linux 커널 팀의 도구였다가 GitHub를 통해 누구나 사용하게 된 것처럼,
AI 에이전트 기능도 엔터프라이즈에서 개인 개발자로 확산되는 경로를 따를 가능성이
높다.
단, 그 확산 속도가 기술 발전 속도를 따라가지 못하면, AI 생산성 격차는 조직 규모에
따라 벌어질 수 있다.

### 커스텀 스킬 생태계가 만들어낼 새로운 전문 직군

Figma가 커스텀 스킬 개발 가능성을 열어둔 것은 새로운 전문 직군의 출현을 예고한다.
팀 특화 워크플로를 마크다운 스킬로 인코딩하는 작업은 디자인 지식, 개발 워크플로,
AI 프롬프트 엔지니어링을 모두 이해하는 사람이 담당하게 된다.
이 역할은 현재 존재하지 않는 포지션이지만, Figma Skills 생태계가 성숙하면 자연스럽게
수요가 생길 것이다.

이것은 새로운 소프트웨어 계층이 등장할 때마다 반복되는 패턴이다.
웹이 등장하자 웹 개발자가 생겼고, 스마트폰 앱이 등장하자 모바일 개발자가 생겼다.
AI 에이전트 워크플로가 조직 내 표준 도구가 되면, “AI 워크플로 엔지니어” 혹은
“스킬 개발자”라는 직군이 기업 내 역할로 자리잡을 것이다.
Figma Skills는 그 역할의 초기 형태를 Figma 도메인에서 구체화한 사례다.
