# Penpot — 오픈소스 디자인·코드 협업 플랫폼

<https://github.com/penpot/penpot>

HN 토론: <https://news.ycombinator.com/item?id=46064757> (775점, 207개 댓글)

## 소개

Penpot은 스페인 Kaleidos가 개발한 오픈소스 디자인 플랫폼으로, 디자인과 코드 협업을
하나의 도구에서 실현하는 것을 목표로 한다. 2015년 12월 공개 이래 47,000개에 가까운
GitHub 스타를 받았으며(2026년 4월 기준), MPL 2.0 라이선스로 배포된다. Verified
Digital Public Good(DPG)으로 인증받은 공공재 소프트웨어다.

Figma의 오픈소스 대안으로 주목받는다. 그러나 “Figma 대체제”라는 포지셔닝을 넘어,
디자인 산출물을 SVG·CSS·HTML로 직접 표현함으로써 디자이너와 개발자 간의 번역
비용을 없애는 것이 핵심 철학이다. 디자인이 곧 코드인 구조(design expressed as
code)를 지향한다.

## 주요 기능

### 오픈 표준 기반 디자인

모든 디자인 요소가 SVG, CSS, HTML, JSON으로 표현된다. Inspect 탭에서 선택한 요소의
SVG/CSS/HTML 코드를 즉시 확인할 수 있어 개발자가 별도의 해석 없이 코드를 가져갈
수 있다. 파일 형식이 오픈 표준이므로 벤더 종속 없이 데이터를 이관하거나 외부 도구와
연동하기 쉽다.

### CSS Grid · Flex Layout

반응형 인터페이스를 실제 코드처럼 설계할 수 있는 레이아웃 시스템이다. CSS Grid와
Flexbox를 디자인 툴에서 직접 다룸으로써, 디자인 시안과 실제 구현 사이의 간극을
줄인다.

### 네이티브 Design Tokens

W3C 표준을 따르는 네이티브 Design Tokens 기능을 제공한다. 디자인과 개발 사이의
단일 진실 공급원(single source of truth)으로 작동해 디자인 시스템 일관성을 유지하고
복잡한 시스템의 관리 비용을 낮춘다.

### MCP 서버와 AI 워크플로우

Penpot MCP 서버는 디자인과 코드 사이의 다방향 워크플로우를 가능하게 한다. AI
에이전트가 Penpot 파일을 읽고 디자인 데이터를 코드 생성에 활용할 수 있으며, 오픈
API와 플러그인 시스템이 자동화와 AI 통합을 지원한다.

### 셀프 호스팅

Docker, Kubernetes, Elestio 등 다양한 방식으로 자체 서버에 배포할 수 있다. SaaS
버전(`design.penpot.app`)도 운영하지만, 완전한 데이터 소유권을 원하는 기업이나
컴플라이언스 요구사항이 있는 조직을 위해 셀프 호스팅이 설계의 핵심 축이다.

### 실시간 협업과 플러그인

Figma처럼 실시간 멀티유저 협업을 지원한다. 플러그인 시스템으로 외부 서비스와
통합하거나 커스텀 워크플로우를 구성할 수 있으며, 웹훅과 API를 통해 개발 도구 체인에
편입된다.

## 기술 스택

백엔드는 Clojure, 프론트엔드는 ClojureScript로 작성됐다. 단일 코드베이스에서 JVM
과 브라우저 모두를 대상으로 컴파일하는 구조다. SVG를 렌더링 기반으로 사용하므로
디자인 요소가 DOM에 직접 매핑된다.

## 분석

### “디자인 = 코드”라는 철학의 실용적 의미

Penpot의 핵심 차별점은 도구 철학에 있다. Figma는 디자인을 독자적인 표현 방식으로
저장하고 개발자가 이를 코드로 “번역”한다. Penpot은 디자인 자체를 SVG와 CSS로
저장하므로 번역 단계가 사라진다. Inspect 탭의 코드가 단순한 참고 자료가 아니라
실제 산출물이 된다.

이 철학은 CSS Grid와 Flex Layout 지원에서 더 명확해진다. 디자이너가 Flex 컨테이너를
설정하면 그것이 곧 CSS `display: flex`가 된다. 반응형 동작을 디자인 단계에서 명세할
수 있다는 의미다. 기존 디자인 도구가 “어떻게 보이는가”를 표현했다면, Penpot은
“어떻게 동작하는가”까지 표현할 수 있다.

### MCP 시대의 포지셔닝

Penpot MCP 서버 출시는 AI 에이전트가 디자인 파일을 직접 읽고 쓸 수 있는 인터페이스를
제공한다. 이는 “AI 코딩 에이전트가 디자인 시안을 이해하고 코드를 생성”하는 워크플로우를
가능하게 한다. Figma도 유사한 통합을 제공하지만, Penpot은 파일 형식 자체가 오픈
표준이므로 AI 파싱 정확도가 높다. SVG/CSS로 표현된 디자인은 언어 모델이 이미 학습
데이터로 대량 접한 형식이기 때문이다.

### 오픈소스 디자인 도구 생태계의 공백 채우기

Figma가 2022년 Adobe에 인수 시도(최종 무산) 이후 대안 도구에 대한 수요가 폭발했다.
Penpot은 그 수혜를 받은 대표적인 프로젝트로, 해당 시기에 GitHub 스타가 급증했다.
디지털 공공재(DPG) 인증은 정부 기관, 교육 기관, NGO 등 오픈소스 요건이 있는 조직의
도입 근거가 된다. Kaleidos가 스페인 회사라는 점에서 유럽의 데이터 주권(data
sovereignty) 요구와도 잘 맞는다.

### 회사·비즈니스 모델 맥락

Penpot 공동창립자 Pablo가 HN 토론에 직접 답변하며 회사 배경과 로드맵을 정리했다.
2011년 설립된 유럽 스타트업으로 2021년에 제품 중심으로 피벗했고, 현재 약 45명
규모다. 비즈니스 모델은 “Open Nitrate(역(逆)-오픈 코어)” — 일반적인 오픈 코어가
핵심을 닫고 주변을 여는 것과 반대로, 핵심을 열고 주변 SaaS 운영 편의 기능에 가격을
매기는 형태로 자기 정의한다. 더 중요한 발표는 새 렌더링 엔진의 오픈 베타가 곧
공개될 예정이라는 점으로, 아래 비평 섹션에서 다루는 성능 이슈에 대한 직접적인 응답
신호다.[^diacritica]

### 단순한 벡터 편집기로의 활용 결도 있다

“Figma 대체”라는 큰 서사 외에, UI 요소 디자인이나 커스텀 아이콘·그래픽 요소 제작
같은 단순한 벡터 편집기로도 잘 쓰인다는 사용자 보고가 있다. 단위·레이아웃 처리가
깔끔하고, 익스포트 시스템이 정리에 유용하며, 초기 Sketch가 주던 “생산성이 빠르게
나오는” 느낌을 다시 준다는 평가다.[^boriskourt] 이는 본문이 강조한 “디자인=코드”
워크플로우 외에, “Sketch 후속 자리”라는 또 다른 채택 경로가 있음을 보여준다.

## 비평

### 강점

셀프 호스팅을 1급 시민으로 취급한다는 점이 엔터프라이즈 채택의 핵심 유인이다. SaaS
도구에 민감한 금융·의료·공공 부문이 데이터 소유권을 유지하면서 디자인 도구를 도입할
수 있다. MPL 2.0 라이선스는 내부 사용에 제한이 없고 상업적 서비스에도 적용 가능해
채택 장벽이 낮다.

Clojure/ClojureScript 스택은 함수형 프로그래밍 모델로 UI 상태 관리와 SVG 렌더링을
다룬다는 점에서 기술적으로 일관성이 높다. 오픈 표준 파일 포맷 채택은 장기적으로
벤더 종속 리스크를 제거한다.

### 약점 및 한계

Figma의 완성도와 생태계에는 아직 격차가 있다. 플러그인 수, 커뮤니티 템플릿, 서드파티
통합 면에서 Figma 생태계를 따라잡으려면 시간이 필요하다. 특히 대형 디자인 팀이
이미 Figma 기반 워크플로우를 구축했다면 마이그레이션 비용이 상당하다.

Clojure/ClojureScript 스택은 기여자 풀을 좁힌다. JavaScript/TypeScript 중심의 오픈소스
생태계에서 Clojure 개발자 수는 상대적으로 적다. 이것이 기능 개발 속도와 외부 기여의
다양성에 구조적 제약이 될 수 있다.

셀프 호스팅의 운영 부담도 고려해야 한다. Docker/Kubernetes 구성과 업그레이드 관리,
데이터 백업 등 인프라 운영 역량이 없는 소규모 팀에게는 SaaS보다 총소유비용이 높을
수 있다. 평범한 디자이너에게 “디자인을 쓰려면 서버 관리부터 배워야 한다”는 진입
장벽이 비합리적이라는 비판도 강하게 제기된다 — Blender처럼 크라우드펀딩으로 데스크톱
앱을 정공법으로 운영하는 길이 더 적합하지 않냐는 문제 제기다.[^comezkandirali]

성능과 안정성 면에서 현장 보고가 갈린다. 5–6 페이지 규모의 작업과 적은 동시 사용자
환경에서도 64GB RAM 전용 서버가 빠르게 점유되며 결국 팀 전체에 lag가 퍼지고
크래시한다는 강한 부정 보고가 있다.[^v3ss0n] 페이지 간 이동만으로 예제 문서조차
일부가 이상하게 변하는 현상을 8개월 뒤에도 재현한 사용자도 있다.[^supermatt] 다른
대안 사이의 비교에서 Penpot보다 훨씬 매끄러운 렌더링과 큰 파일 처리 성능을 보였던
Motiff가 법적 분쟁으로 종료된 후 결국 Figma로 회귀했다는 사례도 보고된다.[^sreekanth850]
즉 “Figma의 오픈소스 대안”이라는 포지션은 이론적으로는 강하지만, 실제 일상 사용성에서
Figma 수준의 부드러움에 도달했는가는 아직 사용자 경험에 큰 편차가 있다.

리소스 측면의 약점도 있다. 디자인 툴 선택의 “결정적 요인”으로 종종 거론되는 무료
컴포넌트·템플릿 패키지 수에서 Figma 생태계와 격차가 크다는 점이다.[^insane_dreamer]
이는 단순히 기능 비교가 아니라 채택 결정의 실제 변수에 해당한다.

## 인사이트

### 오픈소스가 SaaS 락인에 대한 헤지(hedge)가 되는 시대

Figma의 Adobe 인수 시도는 디자인 업계에 충격을 줬다. 단일 벤더의 SaaS 도구에 핵심
워크플로우를 의존할 때의 리스크가 실제로 현실화된 순간이었다. 인수가 무산됐지만,
그 사건은 “만약 Figma가 가격을 올리거나 기능을 제한하거나 폐쇄된다면?”이라는 질문을
모든 팀이 갖게 만들었다. Penpot은 그 질문에 대한 구조적 답변이다.

이 패턴은 디자인 도구에만 국한되지 않는다. 코드 편집기(VS Code의 오픈소스 포크들),
CI/CD(Jenkins → GitHub Actions → self-hosted runners), 프로젝트 관리(Jira 대안으로서의
Linear와 Plane) 등 소프트웨어 개발 도구 전반에서 “오픈소스 자기 호스팅 대안”의 수요가
높아지고 있다. 개발 도구의 오픈소스화는 조직이 워크플로우 주권을 확보하는 핵심 전략이
됐다.

장기적으로 이 트렌드는 SaaS 도구 벤더에게 가격 결정력 제한으로 작용한다. 설득력 있는
오픈소스 대안이 존재하는 영역에서는 SaaS 가격이 오픈소스 셀프 호스팅 비용을 크게
초과하기 어렵다. Penpot의 존재 자체가 Figma의 가격 정책에 간접적 영향을 미친다.

이 헤지의 가치를 명확하게 정리한 코멘트가 있다. 웹 스택과 셀프 호스팅의 “성능세”를
감수하더라도 디자인 파일이 폐쇄형 클라우드 사일로에 인질로 잡히지 않는 쪽이 낫다는
것이다. 성능은 코드라 시간이 지나면 최적화될 수 있지만, 폐쇄형 라이선스 조건은 사용자
입장에서 최적화할 수 없다 — 법무 영역이지 코드 영역이 아니기 때문이다.[^leo_e] 같은
시기에 “FOSS 도구의 황금기에 와 있다”는 인식, 그리고 지역 크리에이티브 커뮤니티가
서버 비용을 분담해 자체 인스턴스를 운영하면 모두에게 큰 비용 절감이 된다는 시도
역시 같은 흐름의 표출이다.[^closingreunion] 즉 Penpot이 호명되는 맥락은 단순한
“Figma 대체”가 아니라, 워크플로우 주권을 직접 운영해 보려는 의지의 응집 지점에 가깝다.

특히 이 흐름은 디자인 업계와 개발자 업계의 비대칭을 드러낸다. 개발자 세계는 오픈
소스를 디폴트 가까이 사용하는 반면, 디자인 세계는 폐쇄형 도구를 압도적으로 선호하는
경향이 강하다. Penpot의 존재는 그 비대칭에 균열을 내는 흥미로운 신호다.[^maelito]

### MCP가 디자인-개발 워크플로우에 가져올 구조적 변화

Penpot MCP 서버는 디자인 파일을 AI 에이전트가 읽고 쓸 수 있는 인터페이스로 만든다.
표면적으로는 “AI가 디자인을 보고 코드를 생성한다”는 이야기처럼 들리지만, 더 깊은
함의는 디자인이 기계 판독 가능한(machine-readable) 명세가 된다는 것이다.

Penpot의 SVG/CSS 기반 파일 포맷은 이 전환에 특히 유리하다. 언어 모델은 이미 방대한
SVG와 CSS 데이터로 학습되어 있어, Penpot 디자인 파일을 분석하는 데 별도의 특수
지식이 필요하지 않다. Figma의 독점 포맷과 달리, Penpot 파일은 LLM이 “이해”하는
표준 언어로 쓰여 있다. 이것이 AI-first 디자인 워크플로우에서 오픈 표준의 전략적
우위를 만든다.

2~3년 내에 주류가 될 워크플로우는 이런 형태일 것이다. 디자이너가 Penpot에서 컴포넌트를
수정하면, MCP를 통해 AI 코딩 에이전트가 변경사항을 감지하고 해당 컴포넌트의 React/
Vue/Swift 구현을 자동으로 업데이트한다. 디자인과 코드가 서로를 동기화하는 양방향
파이프라인이 된다. Penpot의 오픈 표준 철학은 이 미래를 향한 가장 직접적인 경로다.

### 공공 디지털 인프라로서의 디자인 도구

Penpot이 Verified Digital Public Good(DPG)으로 인증받은 사실은 단순한 마케팅 이상의
의미가 있다. DPG 인증은 UN의 지속가능발전목표(SDG) 달성에 기여하는 오픈소스 프로젝트를
식별하는 프레임워크다. 이 인증을 받은 소프트웨어는 정부 조달, 국제 개발 원조, 교육
기관 도입에서 우선 고려 대상이 된다.

디자인 도구가 공공재로 분류된다는 것은 디지털 인프라의 정의가 확장되고 있음을 보여준다.
전통적으로 공공 디지털 인프라는 인터넷 프로토콜, 운영체제, 데이터베이스 같은 저수준
기술로 한정됐다. 그런데 이제 디자인 도구, 협업 플랫폼, 콘텐츠 관리 시스템도 공공재
논의의 범주에 들어오고 있다. 이는 소프트웨어 도구의 접근성과 주권이 사회 인프라의
일부로 인식되기 시작했다는 신호다.

실무적으로 이것은 공공 기관과 교육 기관의 Penpot 도입에 정당성을 부여한다. 유럽 각국
정부가 디지털 주권 정책을 강화하는 흐름 속에서, DPG 인증을 받은 셀프 호스팅 가능한
디자인 도구는 조달 요건을 충족하는 실질적 선택지가 된다. Kaleidos가 스페인 회사라는
점은 EU의 데이터 지역화(data localization) 트렌드와 맞물려 유럽 공공 부문 채택을
가속할 것이다.
