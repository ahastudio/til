# Claude Code to Figma Design

<https://twitter.com/zoink/status/2023760388530307167>

Figma CEO Dylan Field가 Claude Code에서 Figma로 작업물을 가져올 수 있는 기능을
발표했다.

## 사용 방법

Figma MCP를 설치한 후, Claude Code에서 "Send this to Figma"라고 입력하면
브라우저의 렌더링 상태가 자동으로 완전히 편집 가능한 Figma 레이어로 변환된다.

반대로 Figma MCP를 통해 디자인 변경 사항을 코드베이스로 다시 가져올 수도 있다.

<https://figma.bot/ClaudeCodetoFigma>

## 핵심 메시지

디자인 캔버스는 IDE/ADE에서 프롬프트를 입력하는 것보다 다양한 가능성을 탐색하는
데 더 적합하다. 캔버스에서는 발산적으로 사고하고, 접근 방식을 나란히 비교하며 큰
그림을 볼 수 있다. 직접 조작(Direct Manipulation)으로 세부 사항을 편집할 수도
있다.

## 워크플로의 변화

이전에는 제품 디자인과 개발 워크플로가 선형적이었다. 아이디어 브레인스토밍 →
디자인 → 코드 순서로 진행됐다. 이제는 어디서든 시작해서 어디로든 갈 수 있다.
터미널, 프롬프트 박스, 비주얼 UI, 손으로 그린 스케치 등 어디서 시작하든 Figma가
모든 것이 모이는 장소가 되길 원한다고 밝혔다.

## 인사이트

**양방향 워크플로의 실현.** 코드 → 디자인, 디자인 → 코드 양방향 전환이
가능해지면서 "디자인 먼저" vs "코드 먼저"라는 이분법이 무의미해진다.
프로토타이핑을 코드로 빠르게 만든 뒤 Figma에서 다듬고, 다시 코드로 반영하는
반복(Iteration) 사이클이 훨씬 빨라진다.

**터널 비전 탈출.** AI로 코드를 쉽게 만들 수 있는 시대에 "첫 번째 버전이 곧 최종
버전"이 되는 함정을 경고한다. Figma 캔버스에서 여러 대안을 시각적으로 비교하며
줌 아웃할 수 있다는 점이 이 문제의 해결책으로 제시된다.

**디자인의 재정의.** AI가 어떤 가능성이든 구현할 수 있는 세상에서 진짜 차별화
요소는 디자인, 크래프트, 관점이다. 핵심 작업은 거의 무한한 가능성 공간에서
최적의 솔루션을 찾는 것으로 변한다. 이는 곧 "디자인을 배워야 한다"는 메시지다.

**Figma의 전략적 포지셔닝.** Cursor, Windsurf, Claude Code 등 ADE가 부상하면서
디자인 도구의 입지가 좁아질 수 있는 상황에서, Figma는 오히려 이들과 통합하여
"가능성 탐색의 허브"로 자리매김하려 한다. 코딩 도구의 경쟁자가 아닌 보완재로서의
포지셔닝이 인상적이다.
