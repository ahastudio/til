# DigitalPlat FreeDomain: 모두를 위한 무료 도메인

<https://github.com/DigitalPlatDev/FreeDomain>

## 소개

DigitalPlat FreeDomain은 개인과 단체에게 무료 도메인을 제공하는 서비스다.
“디지털 정체성은 특권이 아니라 기본권”이라는 철학 아래 Edward Hsing이
15세에 개인 DNS 실험으로 시작해 현재 500,000개 이상의 도메인을 보유한
대규모 서비스로 성장했다. GitHub 레포지토리 스타 167,000개를 기록했다.

제공되는 도메인 확장자는 다음과 같다.

| 확장자        | 비고                  |
| ------------- | --------------------- |
| `.dpdns.org`  | DigitalPlat 자체 도메인 |
| `.us.kg`      | 키르기스스탄 국가 도메인 |
| `.qzz.io`     |                       |
| `.xx.kg`      | 키르기스스탄 국가 도메인 |
| `.qd.je`      | 저지 섬 국가 도메인    |

사용자는 대시보드(dash.domain.digitalplat.org)에서 도메인을 신청하고,
Cloudflare, FreeDNS by Afraid.org, Hostry 등 원하는 DNS 제공자와 연동한다.
자체 DNS 인프라 운영도 지원한다. 계정당 기본 1개 도메인이 허용되며,
등록된 도메인 아래에 `example.foo.us.kg` 형식으로 서브도메인을 무제한 생성할 수 있다.

서비스 운영 역사는 간단하다. 15세의 Hsing이 친구에게 서브도메인을 빌려주면서
시작됐고, 입소문으로 수백 명에서 수만 명으로 성장했다.
초기 기술 스택은 VPS 위의 BIND9, Python과 Flask 백엔드로 구성됐다.
이후 남용 방지를 위해 자동화된 검토 파이프라인, 행동 패턴 점수 시스템,
GitHub 기반 신원 확인을 도입했다.

## 분석

### 무료 도메인의 기술적 구조와 DNS 위임

FreeDomain의 기술 구조는 DNS 위임(delegation)을 핵심으로 한다.
서비스가 `.us.kg` 같은 최상위 도메인 또는 2단계 도메인의 네임서버를 통제하고,
사용자에게 그 아래 3단계 도메인(`username.us.kg`)을 할당한다.
사용자는 자신의 DNS 제공자(Cloudflare 등)에서 이 도메인의 레코드를 관리한다.

이 위임 구조는 서비스가 도메인 공간의 “중간 계층”을 소유한다는 의미다.
Cloudflare Pages의 `*.pages.dev`, GitHub Pages의 `*.github.io`와 구조적으로 같다.
차이는 FreeDomain이 특정 호스팅 서비스에 종속되지 않고
어떤 DNS 제공자, 어떤 호스팅 서비스와도 연동 가능하다는 점이다.

### GitHub 기반 신원 확인의 설계 선택

남용 방지를 위해 GitHub 계정을 신원 확인 수단으로 사용하는 것은
흥미로운 설계 선택이다. KYC(Know Your Customer)처럼 실명이나 신분증을
요구하지 않으면서도 계정 생성에 마찰을 추가한다.
GitHub 계정은 만들기 어렵지 않지만, 스팸봇이 대규모로 생성하는 것은
이메일 계정보다 훨씬 어렵다.

이 접근법은 “익명성과 책임 사이의 균형”이라는 오래된 인터넷 문제에 대한
실용적 해답이다. GitHub 계정은 개발자 커뮤니티에서 신뢰의 신호이기도 하다.
스타, 기여 이력, 팔로워가 있는 계정과 방금 만든 빈 계정은 다르게 취급될 수 있다.
이 맥락을 활용하면 추가적인 신원 확인 비용 없이 남용 위험을 낮출 수 있다.

### 계정당 1도메인 제한과 서브도메인 모델

초기에는 더 많은 도메인을 허용했다가 남용 문제로 계정당 1개로 제한했다는 것이
FAQ에 명시돼 있다. 그러나 서브도메인은 무제한이다.
이 정책이 만드는 실제 사용 패턴은 흥미롭다.
`myproject.us.kg` 하나를 등록하고 Cloudflare에서 와일드카드 레코드를 설정하면
`api.myproject.us.kg`, `docs.myproject.us.kg`, `staging.myproject.us.kg`를
추가 비용 없이 운영할 수 있다.

단일 도메인 제한이 서브도메인 모델의 채택을 자연스럽게 촉진한다.
이것은 기술적 제약이 아키텍처 패턴에 영향을 미치는 사례다.
도메인을 여러 개 쓸 수 없다면, 서브도메인 기반 멀티서비스 아키텍처가
더 합리적인 선택이 된다.

## 비평

### 강점: 진입 장벽 없는 DNS 학습 환경

FreeDomain의 가장 큰 가치는 학습 맥락에서 드러난다.
도메인이 없어 자신의 서비스를 외부에 공개하지 못했던 개발자,
Cloudflare나 Nginx 설정을 연습하고 싶었지만 도메인 비용이 부담됐던 학생,
사이드 프로젝트에 도메인을 쓰기 위해 연간 요금을 지출하기 꺼렸던 개인에게
실질적인 장벽을 제거한다. `.us.kg` 도메인이 `.com`만큼 기억하기 좋지는 않지만,
학습과 실험 목적에서는 완전히 기능적이다.

### 한계: 서비스 지속성과 도메인 의존성 위험

500,000개 도메인이 이 서비스에 의존하고 있다. 서비스가 종료되면
모든 도메인이 동시에 작동을 멈춘다. 비슷한 서비스들의 역사를 보면
이 위험은 현실적이다. 개인이 운영하는 무료 서비스는 자금 조달, 운영자의 생활 변화,
법적 문제 등으로 예고 없이 종료될 수 있다. Telegram 보안 사고가 이미
서비스 운영에서 하나의 위기를 경험했음을 시사한다.

생산 서비스에 FreeDomain 도메인을 사용하는 것은 이 위험을 인지하고
감수하는 결정이어야 한다. 학습과 실험, 개인 프로젝트에는 적합하지만
중요한 서비스의 주 도메인으로는 신중해야 한다.

### 보완이 필요한 관점: 남용 대응의 구체성 부재

“자동화된 검토 파이프라인, 행동 패턴 점수 시스템”을 사용한다고 하지만
이것이 실제로 무엇을 의미하는지 공개 문서에 없다.
피싱, 스팸, 악성코드 배포에 FreeDomain 도메인이 사용되는 빈도,
보고된 남용에 대한 평균 응답 시간, 도메인 취소 기준이 공개되지 않는다.
남용 신고 이메일(abusereport@digitalplat.org)이 있지만
처리 투명성이 낮다는 점은 신뢰 구축의 약점이다.

## 인사이트

### 도메인 시장의 구조적 문제를 무료 서비스가 드러낸다

도메인 가격은 기술적 원가와 무관하게 결정된다. `.com` 도메인의 유지 비용은
연간 수 달러 수준이지만 소매가는 10~20달러다. 이 마진은 ICANN, 레지스트리,
레지스트라 세 계층의 수익 배분 구조에서 나온다. 더 중요한 사실은
이 가격이 개발도상국 사용자, 학생, 초기 단계 프로젝트에게는 실질적 장벽이 된다는 것이다.

FreeDomain이 500,000개 도메인을 무료로 운영할 수 있다는 사실은
도메인의 기술적 운영 원가가 소매가보다 훨씬 낮다는 증거다.
이 서비스가 존재하고 성장한다는 것은 수요가 실재하며, 기존 시장이
그 수요의 일부를 가격으로 차단하고 있었음을 보여준다.
GitHub의 Stars가 167,000개에 달하는 것은 이 수요의 크기를 시사한다.

유사한 패턴이 반복된다. Let's Encrypt가 SSL 인증서를 무료로 만들기 전까지
HTTPS 배포율은 낮았다. 무료화 후 웹의 HTTPS 비율은 급격히 상승했다.
도메인도 비슷한 전환점을 맞을 수 있다. 무료 도메인이 충분히 신뢰받는다면,
“내 서비스는 도메인을 살 여유가 없어서 IP로 운영한다”는 상황이 줄어든다.

### 개인 프로젝트에서 공공 인프라로의 전환이 만드는 책임

15세의 개인 실험이 400,000명이 의존하는 서비스가 됐을 때 발생하는 것은
기술적 문제가 아니라 책임의 문제다. 친구 몇 명이 쓰는 서비스와
50만 도메인을 운영하는 서비스는 근본적으로 다른 종류의 약속이다.

이 전환이 가장 어려운 이유는 그것이 의도적이지 않다는 점이다.
Edward Hsing은 공공 DNS 인프라를 운영하기로 결정한 것이 아니라,
사람들이 서비스를 쓰기 시작했고 멈출 수 없게 됐다.
오픈소스 프로젝트 메인테이너들이 겪는 번아웃의 구조와 같다.
시작은 자발적이지만, 사용자가 의존하기 시작하면 중단이 다른 사람들을 해친다.

이 구조적 문제의 해결책은 명확하지 않다. FreeDomain이 더 지속 가능하려면
자금 조달(스폰서, Pro 티어), 조직화(재단 설립), 또는 다른 조직으로의 이전이 필요하다.
README에 “DigitalPlat Foundation”이 언급되는 것은 이 방향을 의식한 움직임일 수 있다.
오픈소스 지속 가능성의 문제는 소프트웨어에만 국한되지 않는다.
DNS 인프라도 동일한 도전을 직면한다.

### 국가 코드 도메인(.kg, .je)의 전략적 활용과 위험

FreeDomain이 제공하는 도메인 중 `.us.kg`, `.xx.kg`는 키르기스스탄,
`.qd.je`는 저지 섬의 국가 코드 최상위 도메인(ccTLD)이다.
이 도메인들을 사용할 수 있는 이유는 해당 국가의 도메인 레지스트리가
이 도메인들의 2단계 혹은 3단계 영역을 제3자에게 위임하거나 판매했기 때문이다.

이 의존성은 외부 위험을 만든다. `.us.kg`를 제공하는 능력은
키르기스스탄 레지스트리의 정책에 달려 있다. 정책 변경, 계약 만료,
지정학적 이유로 인한 도메인 압수는 FreeDomain의 통제 밖에 있다.
.io 도메인이 영국령 인도양 지역(BIOT) 반환 협상으로 불확실성에 처한 사례처럼,
ccTLD는 본질적으로 해당 국가의 정치적 상황에 노출된다.

역설적으로 이 위험성이 FreeDomain의 무료 모델을 가능하게 한다.
`.com`처럼 안정적이고 신뢰받는 도메인은 수요가 높고 가격이 비싸다.
리스크가 있는 ccTLD는 상업적 수요가 낮아 더 저렴하게 확보할 수 있다.
무료 서비스는 이 위험을 사용자와 공유한다. 사용자는 안정성 대신 비용 제로를 선택한다.
이것이 무료 서비스가 항상 “공짜가 아닌” 이유다. 위험을 가격 대신 지불한다.
