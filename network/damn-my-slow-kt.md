# damn-my-slow-kt - KT 인터넷 SLA 보상 자동화 도구

원문: <https://github.com/kargnas/damn-my-slow-kt>

## 요약

damn-my-slow-kt는 KT 인터넷의 SLA(Service Level Agreement) 보상을
자동으로 청구해주는 도구다.
KT의 SLA 정책에 따르면 측정 속도가 계약 속도의 50% 미만일 때
일별 환불을 받을 수 있지만, 매일 직접 측정하고 수동으로 청구해야 한다.
이 도구는 하루 최대 10회(2시간 간격) 속도 측정과 보상 청구를 자동화한다.

기술 스택은 TypeScript, Playwright(헤드리스 Chromium으로 KT 웹사이트
자동화), Commander/Inquirer CLI, YAML 설정, SQLite(또는 JSON 폴백),
Discord/Telegram 웹훅 알림으로 구성된다.

## 분석

보상 기준은 30분 이내 최소 5회 속도 측정, 60% 이상의 측정이 최저 속도
미달, KT 공식 측정 소프트웨어 사용이다.
30일 연속 미달 시 월 이용료 전액 면제, 월 5일 이상 미달 시 위약금 없이
계약 해지가 가능하다.

macOS 또는 Windows 환경이 필요하며(KT 측정 소프트웨어가 Linux 미지원),
Node.js 20 이상과 유선 LAN 연결이 필수다.
Wi-Fi 환경에서의 측정은 공식 보상 대상에서 제외된다.

현재 Windows는 미검증 상태이며, Linux/Docker/NAS는 지원하지 않는다.

## 비평

소비자 권리 행사를 자동화한다는 아이디어는 훌륭하지만, 몇 가지 구조적
한계가 존재한다.
KT 계정 비밀번호를 로컬에 저장해야 하는 보안 문제, KT가 SLA 정책 자체를
변경할 가능성, 그리고 같은 건물 내 다수 사용자가 동시 측정 시 네트워크
품질이 오히려 저하될 수 있다는 점이 커뮤니티에서도 지적됐다.

SK브로드밴드나 LG유플러스 버전에 대한 요청도 있었으나, 각 ISP마다
측정 방식과 보상 정책이 다르므로 단순 포팅은 어렵다.

## 인사이트

### 소비자 권리의 자동화가 만드는 비대칭 해소

통신사의 SLA 정책은 형식적으로는 소비자 보호를 위해 존재하지만,
실질적으로는 청구 과정의 번거로움이 진입 장벽으로 작용한다.
매일 속도를 측정하고 수동으로 보상을 신청하는 행위는 대부분의 소비자에게
비현실적이며, 통신사는 이 마찰 비용을 사실상 방어 기제로 활용해왔다.

이런 구조는 항공사의 지연 보상, 보험 청구, 세금 환급 등 다양한 영역에서
반복되는 패턴이다.
권리는 존재하지만 행사 비용이 높아 실질적 보호가 이루어지지 않는 상황은
제도적 허점이라 할 수 있다.

자동화 도구가 이 마찰 비용을 제거하면, 기업과 소비자 사이의 정보·행동
비대칭이 해소되기 시작한다.
이는 단순한 기술적 편의를 넘어 시장의 인센티브 구조 자체를 변화시키는
힘을 갖는다.

### 규제 차익거래와 기업의 적응적 대응

이 도구가 널리 퍼지면 KT는 두 가지 선택지에 직면한다.
네트워크 품질을 실제로 개선하거나, SLA 정책의 조건을 까다롭게 변경하는
것이다.
역사적으로 기업은 후자를 선택하는 경향이 강하다.

미국의 넷 뉴트럴리티(Net Neutrality) 논쟁에서도 유사한 패턴이 나타났다.
규제가 강화되면 ISP는 기술적 우회로를 찾거나 규정의 해석을 좁히는
방식으로 대응했다.
한국 통신사도 측정 소프트웨어의 사양 변경, 측정 시간대 제한, 약관 개정
등으로 자동화 도구를 무력화할 수 있다.

그러나 이런 대응은 규제 당국의 관심을 끌 수 있으며, 소비자 권리를
축소하는 방향의 약관 변경은 공정거래위원회나 방송통신위원회의 개입을
유발할 수 있다.
자동화 도구의 존재 자체가 정책 논의를 촉발하는 촉매 역할을 하게 된다.

### 브라우저 자동화의 시민 기술(Civic Tech)로서의 가능성

Playwright 같은 브라우저 자동화 기술은 본래 테스트 자동화를 위해
만들어졌지만, 이 프로젝트는 그것을 시민의 권리 행사 도구로 전용했다.
이런 접근은 시빅 해킹(Civic Hacking)의 전통과 맞닿아 있다.

Code for America 같은 시빅 테크 운동이 정부 서비스의 접근성을 개선하는
데 집중했다면, damn-my-slow-kt는 기업 서비스에 대한 소비자 권리 행사의
접근성을 개선한다.
웹 인터페이스 뒤에 숨겨진 복잡한 절차를 자동화함으로써, 기술적 역량이
있는 소수만 행사할 수 있던 권리를 보다 넓은 범위로 확장한다.

다만, 이 모델이 확장되려면 비밀번호 관리 같은 보안 문제를 해결해야 한다.
OAuth 기반 인증이나 공식 API 제공이 이상적이지만, 통신사가 자발적으로
이를 제공할 인센티브가 없다는 점이 근본적 딜레마다.
결국 시민 기술의 발전은 제도적 변화 없이는 항상 기업의 기술적 장벽과
충돌하게 된다.
