# We're Making Bunny DNS Free

원문: <https://bunny.net/blog/were-making-bunny-dns-free/>

HN 토론: <https://news.ycombinator.com/item?id=48657030> (905점, 267개 댓글)

## 요약

Bunny.net은 2026년 6월 자사 DNS 서비스의 쿼리 비용을 전면 폐지하고
계정당 최대 500개 도메인을 무료로 호스팅할 수 있다고 발표했다.
현재 Bunny DNS는 월 2,000억 건의 쿼리를 처리하며 30만 개 이상의 도메인을 관리한다.
저자들은 “사용자와 서비스 사이를 잇는 근본적인 시스템이 프리미엄 부가기능이어서는 안 된다”고 주장하며
DNS를 인프라의 공공재로 재정의한다.

이번 발표와 함께 여러 기능이 추가되었다.
자동 존(zone) 스캔으로 도메인 마이그레이션이 간소화되고,
DNS 레코드에서 직접 Bunny CDN을 활성화하는 1-Click Acceleration과
Bunny Shield 보안을 즉시 적용하는 1-Click Security가 도입된다.
IPv6가 네임서버 전체에 지원되며,
DNSSEC는 NSEC Black Lies 기법으로 존 열거(zone enumeration) 공격을 방어한다.

신규 지원 레코드 타입으로 HTTPS, SVCB, TLSA, CDS, CDNSKEY가 추가되었다.
HTTPS와 SVCB는 브라우저가 HTTP/3나 ECH(Encrypted Client Hello)를 협상할 수 있게 하는 최신 표준이고,
TLSA는 DANE(DNS-based Authentication of Named Entities)으로 TLS 인증서를 DNS에서 검증하는 메커니즘이며,
CDS와 CDNSKEY는 상위 레지스트리와 DNSSEC 키 교환을 자동화하는 레코드다.

가격 정책의 구체적 조건은 이렇다.
DNS 쿼리 비용은 없으나 계정에는 월 $1의 최소 지출이 유지된다.
즉, DNS만 단독으로 사용하는 경우에도 월 최소 $1이 필요하다.
Bunny는 이를 DNS 사업의 지속 가능성을 위한 최소 장벽으로 설명한다.

## 분석

### DNS 무료화는 상위 서비스 유입을 위한 깔때기 전략이다

DNS를 무료로 제공하는 결정의 직접적 수혜자는 Bunny의 CDN과 Shield 사업이다.
DNS는 사용자의 트래픽이 어디로 향할지를 결정하는 계층이다.
이 계층을 장악하면 트래픽의 관문을 장악한다.
Cloudflare가 2018년 1.1.1.1 리졸버를 무료로 공개하고
그 이후 Zero Trust, Workers, R2를 판매하는 구조와 동일한 패턴이다.

Bunny의 1-Click Acceleration과 1-Click Security는 이 전략을 명시적으로 실현한다.
DNS 레코드 관리 화면에서 버튼 하나로 CDN과 Shield가 활성화된다.
무료 DNS를 쓰는 사용자가 Bunny CDN을 채택하는 마찰이 사실상 제거된다.
이 설계는 DNS를 독립 서비스가 아니라 플랫폼 진입점으로 위치시킨다.

HN에서 Bender는 이 전략을 직접 명명했다[^Bender].
“Cloudflare가 사용한 것과 동일한 판매 기법이다.
한 번 DNS가 거기서 관리되면 CDN 기능 활성화는 버튼 클릭만으로 충분하다.”
전략을 인식한 사용자도 편의성 앞에서 같은 선택을 한다는 것이
이 기법의 실질적 효력을 보여준다.

### NSEC Black Lies는 DNSSEC의 프라이버시 딜레마를 해결하는 기법이다

DNSSEC는 DNS 응답의 무결성을 보장하지만 구조적 프라이버시 취약점을 갖는다.
NSEC(Next Secure) 레코드는 존재하지 않는 이름에 대한 부정 응답을 증명하는 방식인데,
표준 NSEC는 “이 이름부터 다음 이름까지 사이에는 레코드가 없다”는 범위를 공개한다.
공격자는 이 정보를 이용해 존 전체를 열거(zone walk)할 수 있다.

NSEC Black Lies는 RFC 4470에 기반한 최적화다.
존재하지 않는 이름에 대해 “A부터 B까지 없다”는 실제 범위 대신
“요청한 이름 직전부터 직후까지만 없다”는 합성(synthesized) NSEC를 반환한다.
쿼리마다 범위를 최소화해 존 목록을 노출하지 않으면서도 DNSSEC 검증을 통과시킨다.
이 기법은 Cloudflare도 채택하고 있으며, 대형 DNS 사업자의 사실상 표준이 되어가고 있다.

### 최신 레코드 타입 지원은 웹 프로토콜 현대화의 인프라 선결 조건이다

HTTPS와 SVCB 레코드는 최신 웹 성능과 보안의 핵심이다.
브라우저가 HTTP/3를 협상하려면 DNS 레이어에서 HTTPS 레코드를 통해
서버의 ALPN 프로토콜과 포트를 미리 알려야 한다.
ECH(Encrypted Client Hello)도 DNS로 공개키를 배포한다.
이 레코드들을 지원하지 않는 DNS 제공자는 이 기능들의 활성화를 막는 병목이 된다.

DANE과 DNSSEC를 결합하면 CA(인증기관) 없이 TLS 인증서를 검증하는 경로가 열린다.
TLSA 레코드는 특정 서버가 제시해야 할 인증서의 지문을 DNS에 게시하는 방식이다.
이메일 보안(SMTP over DANE)에서 이미 실전 사용되고 있으며,
Bunny가 이를 지원함으로써 메일 인프라 관리자에게도 유효한 선택지가 된다.

## 비평

### “DNS는 무료여야 한다”는 철학적 주장이 비용 구조를 은폐한다

이 글의 핵심 논거는 “DNS는 인터넷의 근본 인프라이므로 무료여야 한다”다.
그러나 2,000억 건의 월간 쿼리를 처리하는 글로벌 인프라에는 실제 비용이 발생한다.
서버, 대역폭, 운영, 엔지니어링 — 이 비용은 사라지지 않는다.
“무료”는 비용이 제거된 것이 아니라 다른 수익원으로 전가된 것이다.

이 글은 그 전가 구조를 명시하지 않는다.
Bunny CDN과 Shield에서 발생하는 마진이 DNS 비용을 보전한다는 것은
비즈니스 모델을 이해하면 추론할 수 있지만, 글에서 명확히 서술되지 않는다.
“인터넷을 더 빠르게 만들겠다”는 수사는 사업 전략을 공공재 담론으로 포장한다.
독자는 이 프레이밍을 이타주의가 아니라 경쟁 전략으로 독해해야 한다.

### $1/월 최소 지출 조건은 “완전 무료”의 주장을 약화시킨다

발표의 핵심 메시지는 DNS가 무료라는 것이지만,
계정에는 월 $1의 최소 지출이 유지된다.
순수하게 DNS만 사용하는 경우에도 이 최소 지출은 적용된다.
즉, 실질적으로 DNS는 월 $1에 제공되는 서비스이며 “무료”라는 표현은 정확하지 않다.

이 조건이 실제 사용에서 큰 장벽이 아닐 수 있다.
CDN이나 다른 Bunny 서비스를 함께 쓰는 사용자에게는 $1이 어차피 초과된다.
그러나 독립형 DNS 제공자로 Bunny를 평가하는 사용자에게는
Route 53, Cloudflare, Hurricane Electric의 진정한 무료 DNS와
비교할 때 조건의 차이가 드러난다.
마케팅 언어와 실제 조건 사이의 이 간극은 독자가 주의해야 할 지점이다.

1dom은 이 불일치를 더 날카롭게 비판했다[^1dom].
“DNS가 무료라는 거창한 메시지와 실제로 계정을 만들어도 무료 DNS를 쓸 수 없다는 현실 사이의
불일치는 솔직하지 못한 것에 가깝다.”
EU의 Cloudflare 대안을 표방하면서 미국 빅테크식 소인수 조건 마케팅을 쓰는 역설도 지적했다.

### 예측 불가능한 트래픽 과금 위험이 제품 간 일관되게 처리되지 않는다

Diti가 지적한 것은 DNS 무료화보다 더 근본적인 Bunny의 청구 구조 문제다[^Diti].
LLM 크롤러나 봇으로 인한 예상치 못한 트래픽 폭증 시 수천 유로가 청구될 수 있다는 우려다.
Bunny CDN에는 일정 금액 초과 시 요청을 차단하는 청구 상한선 기능이 있지만,
이 기능이 Bunny DNS나 Bunny Stream 등 다른 제품에는 존재하지 않는다.

기능 추가의 우선순위가 문제다.
“DNS를 무료로 만든다”는 결정이
“모든 제품에서 청구 상한선을 설정할 수 있다”는 기본 안전망 정비보다 앞섰다.
비용 통제 수단이 없는 상태에서 무료 진입을 허용하는 것은
예상치 못한 트래픽에 노출된 사용자에게 비대칭적 위험을 부과한다.

### 1-Click 통합의 편의성은 벤더 고착(vendor lock-in)의 마찰을 의도적으로 높인다

1-Click Acceleration이 DNS 레코드에서 Bunny CDN을 즉시 활성화한다는 것은
강력한 UX 개선이지만 동시에 플랫폼 의존성을 심화시킨다.
DNS 레코드와 CDN 설정이 단일 인터페이스에서 결합되면,
나중에 CDN을 교체하려 할 때 DNS 설정 재검토가 함께 필요해진다.
Cloudflare의 “오렌지 구름(orange cloud)” 개념이 CDN 이탈의 마찰로 작동하는 것과 같은 메커니즘이다.

이것이 나쁜 설계라는 의미가 아니다.
편의성과 이동성의 트레이드오프는 모든 플랫폼이 다루는 문제이고,
Bunny가 이 트레이드오프에서 편의성을 선택한 것은 사업 논리상 타당하다.
그러나 이 글은 그 트레이드오프를 언급하지 않는다.
독자가 스스로 인식해야 할 이동성 비용이 투명하게 제시되지 않는다.

## 인사이트

### DNS 계층 장악이 클라우드 인프라 경쟁의 새로운 전선이다

Cloudflare는 DNS 리졸버(1.1.1.1)와 권한 DNS 모두를 무료로 제공하면서
수억 개의 도메인과 수십억 명의 사용자를 자사 관측 지점으로 끌어들였다.
이를 기반으로 Zero Trust, Workers, Pages, R2가 판매된다.
Bunny의 DNS 무료화는 같은 경쟁 논리를 소규모 CDN 시장에 적용하는 시도다.

DNS 계층을 장악하면 경쟁자 CDN으로의 전환 마찰이 자동으로 높아진다.
도메인의 A 레코드, CNAME, HTTPS 레코드가 모두 Bunny 콘솔에서 관리되면
Fastly나 Akamai로 이동하는 결정이 DNS 설정 전체의 재검토를 의미하게 된다.
인프라 시장에서 전환 비용은 경쟁 우위다.
무료 DNS는 이 전환 비용을 선행 투자로 낮추는 전략이다.

teekert는 DNS 전용 API 키가 제공하는 권한 최소화(least privilege) 효과를 실제 사례로 보여줬다[^teekert].
Hetzner DNS를 사용할 때 API 키가 유출되면 공격자가 VM을 임대하거나 신용카드를 남용할 수 있다.
Bunny DNS로 이전하면 유출된 키로 할 수 있는 것이 DNS 레코드 변경으로 제한된다.
DNS 관리를 클라우드 인프라 공급자에서 분리하는 것이
계정 권한 침해 시의 피해 반경을 구조적으로 줄인다.

### 최신 DNS 레코드 지원이 HTTP/3와 ECH 보급률을 결정한다

HTTPS, SVCB, ECH는 DNS 없이는 작동하지 않는다.
브라우저가 HTTP/3를 우선 시도하려면 HTTPS 레코드에서 `h3` ALPN을 읽어야 한다.
ECH가 SNI를 암호화하려면 DNS로 서버의 공개키를 배포해야 한다.
DNS 제공자가 이 레코드를 지원하지 않으면 이 기능들은 그 도메인에서 비활성화된다.

지금까지 이 기능들의 보급을 막는 주요 병목 중 하나가 DNS 지원 부재였다.
대형 레지스트리 관리 DNS(예: GoDaddy, Namecheap)는 HTTPS 레코드 지원에 느렸다.
Bunny가 무료 DNS를 제공하면서 이 현대 레코드를 포함했다는 것은
마이그레이션 인센티브와 프로토콜 현대화가 결합된 구조다.
개발자가 무료로 이동하면서 자동으로 더 나은 웹 프로토콜 지원을 얻게 된다.

### “손실 리더로서의 DNS”는 장기적 지속 가능성 질문을 남긴다

AWS가 S3를 원가 수준으로 제공해 상위 서비스의 데이터 중력을 만들었듯,
DNS 무료화는 손실 리더(loss leader) 전략의 인프라 버전이다.
이 전략은 주력 수익원이 건재한 동안에는 효과적이다.
Bunny CDN의 마진이 DNS 운영 비용을 충분히 보전하는 한 이 구조는 유지된다.

그러나 CDN 시장의 가격 경쟁은 계속 심화되고 있다.
Cloudflare, AWS CloudFront, Fastly, 그리고 수십 개의 신규 진입자가 마진을 압박한다.
CDN 마진이 충분히 줄어들면 무료 DNS의 지속 가능성이 흔들린다.
사용자는 이 비즈니스 모델의 의존 관계를 인식하고 도메인 이동 계획을 항상 유지해야 한다.

khurs는 Bunny의 자금 조달 구조가 이 지속 가능성 논의에 중요한 맥락을 제공한다고 지적했다[^khurs].
Bunny는 2022년 단 한 차례의 소규모 $600만 투자 유치 이후 유기적 성장을 추구하는 사기업이다.
Cloudflare가 상장 전 수억 달러의 VC 자금으로 손실 리더 전략을 펼친 것과 달리,
Bunny는 CDN 수익만으로 DNS 무료 서비스를 감당해야 한다.
역설적으로 이것이 더 지속 가능한 구조일 수 있다 — 투자자 회수 압박 없이 자체 수익성으로 운영되므로.

역설적으로, 무료 서비스에 더 많이 의존할수록 이동 비용이 커진다.
DNSSEC, HTTPS 레코드, CDN 통합이 모두 Bunny 콘솔에 묶인 도메인은
서비스 중단이나 정책 변경 시 긴급 마이그레이션의 복잡도가 높다.
인프라의 공공재화를 표방하는 서비스에도 대안 경로를 유지하는 것이 운영 위생이다.
