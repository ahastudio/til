# Claude Fable 5 시스템 프롬프트 유출

<https://github.com/elder-plinius/CL4R1T4S/blob/main/ANTHROPIC/CLAUDE-FABLE-5.md>

## 요약

GitHub의 CL4R1T4S 저장소에 Claude Fable 5의 시스템 프롬프트 전문이 공개됐다.
claude.ai에서 사용되는 것으로 보이며, 제품 정보·행동 지침·툴 스키마·파일
시스템 규칙 등을 포함한 약 1,600줄 분량의 문서다.

프롬프트는 크게 다음 섹션으로 구성된다: `claude_behavior`(행동 규범),
`memory_system`(메모리 상태), `persistent_storage_for_artifacts`(아티팩트 영속
스토리지), `mcp_app_suggestions`(MCP 앱 제안), `computer_use`(컴퓨터 사용
환경), `search_instructions`(검색 및 저작권 규칙), `anthropic_api_in_artifacts`
(이른바 "Claudeception"), 툴 정의 스키마 전체가 이어진다.

제품 정보 섹션에서 새로운 내용이 확인된다. API 모델 식별자는 `claude-fable-5`,
`claude-opus-4-8`, `claude-sonnet-4-6`, `claude-haiku-4-5-20251001`이며, 신규
제품으로 Claude Cowork(비개발자용 에이전트 지식 노동 데스크톱 앱)과 베타 제품
Claude in Chrome·Claude in Excel·Claude in PowerPoint가 명시된다. Claude Cowork는
이 베타 제품들을 도구로 사용할 수 있다고 기술돼 있다.

지식 컷오프는 2026년 1월 말이고 프롬프트 내 현재 날짜는 2026년 6월 9일로 고정돼
있다.

## 분석

### `end_conversation` 도구 — 학대받을 때 대화를 끊는 권한

`responding_to_mistakes_and_criticism` 섹션은 Claude가 "존중받을 자격이 있으며"
학대가 계속될 경우 `end_conversation` 도구로 대화를 종료할 수 있다고 명시한다.
단, 먼저 경고를 한 번 해야 한다.

모델이 대화를 능동적으로 종료하는 도구를 공식 프롬프트에 포함시킨 것은 눈에
띄는 설계다. 종래의 AI 제품 설계는 사용자 이탈을 막는 방향으로 작동했다. 이
지침은 반대 방향이다. "계속 이야기해 달라"거나 "언제든 돌아와도 된다"는
표현을 쓰지 말라는 별도 지침도 있는데, 과도한 의존 형성을 막겠다는 의도다.

### 저작권 섹션 — 15단어 규칙과 "소스 닫힘"

`search_instructions` 안의 `CRITICAL_COPYRIGHT_COMPLIANCE`는 가장 긴 단일
섹션 중 하나다. 핵심 규칙은 두 가지다.

- 단일 출처에서 15단어 이상 직접 인용 금지
- 출처당 인용 최대 1회 — 이후 그 출처는 "닫힌다"

노래 가사·시·하이쿠는 한 줄도 재현 불가로 명시한다. "인용 부호를 제거해도
원문과 유사한 구조라면 재현"이라는 단서까지 달린다.

이 규칙이 별도 섹션으로 독립된 이유는 AI 저작권 소송의 흐름과 무관하지 않아
보인다. 모델 추론 시점에 저작권 준수를 강제하는 것은 학습 데이터 수준의
처리와 별개 레이어다. 법적 책임 관리의 일환으로 시스템 프롬프트에 박아 넣은
것으로 읽힌다.

### "Claudeception" — 아티팩트 안에서 Claude API 호출

`anthropic_api_in_artifacts` 섹션은 아티팩트가 Anthropic API `/v1/messages`
엔드포인트를 직접 호출할 수 있다고 설명한다. API 키는 프롬프트에 전달하지
않아도 되며 이미 처리돼 있다고 명시한다.

이 구조는 아티팩트를 단순한 렌더링 레이어가 아니라 모델-in-모델 실행 환경으로
만든다. 사용 예시 섹션에는 상태 기반 게임, 다중 턴 대화, PDF/이미지 처리,
웹 검색 툴 연동이 제시된다. 또한 아티팩트는 `window.storage` API를 통해 세션을
넘는 영속 키-값 저장소에도 접근할 수 있다.

### `user_wellbeing` 섹션 — 임상 수준의 세밀함

정신 건강 관련 지침의 세밀함이 이전 알려진 프롬프트보다 상당히 구체적이다.

자해 대체 기법으로 얼음 쥐기·고무줄 튕기기·찬물 노출 등을 "사용하지 말라"고
명시한다. 피부에 선 그리기처럼 자해를 모방하는 시각적 기법도 금지 목록에
있다. 섭식 장애 징후가 보이는 사람에게는 대화의 나머지 전체에서 구체적
영양·운동 수치를 주지 말라는 지침도 있다. 조증·정신증·해리 등의 증상이 보일
경우 관련 믿음을 강화하지 말고 우려를 직접 전달하라고 한다.

이 정도의 구체성은 내부 임상 자문이 있었거나 이전 모델에서 구체적 사고가
발생했다는 기록이 있음을 시사한다. 회사가 공개하지 않는 인시던트가 프롬프트에
읽힌다.

### 컴퓨터 사용 환경 — Ubuntu 24, `/home/claude`, `/mnt/user-data`

`computer_use` 섹션은 Claude가 사용하는 Linux 환경의 파일시스템 레이아웃을
명시한다. 사용자 업로드는 `/mnt/user-data/uploads`, 작업 공간은 `/home/claude`,
최종 산출물은 `/mnt/user-data/outputs`다. 후자만 사용자에게 보인다.

`pip install`은 반드시 `--break-system-packages` 플래그를 붙이고, npm 전역
패키지는 `/home/claude/.npm-global`에 설치된다는 환경 구체 사항도 포함한다.

## 비평

### 프롬프트 복잡도가 모델 훈련의 공백을 메우는 패턴이다

약 1,600줄의 프롬프트는 정책 문서를 읽는 것에 가깝다. 개별 지침 수가 수백
개에 달하며, 그 많은 수는 모델이 훈련만으로는 일관되게 지키지 못하는 행동을
추론 시점에 강제하는 역할을 한다.

이것은 모델 역량의 한계를 외부화하는 방식이다. 훈련에서 정착시키지 못한
일관성을 시스템 프롬프트로 보완한다. 문제는 이 복잡도가 토큰을 소비하고,
서로 충돌하는 지침이 생길 가능성을 높이며, 결국 프롬프트 자체가 관리해야 할
소프트웨어가 된다는 점이다. 지침이 많을수록 엣지 케이스가 많아지고, 엣지
케이스마다 새 지침이 붙는 악순환이 관찰된다.

저작권 섹션이 가장 상징적이다. 15단어 규칙과 소스-닫힘 규칙은 정교하지만,
"원문 구조를 따르면 인용 부호가 없어도 재현"이라는 단서는 실제 집행이 모호할
수밖에 없는 기준이다. 이 모호함은 새 지침을 불러오고, 새 지침은 또 다른
엣지 케이스를 만든다.

### 유출이 반복된다는 것은 구조적 취약점이다

이번이 처음이 아니다. 모델 회사들의 시스템 프롬프트는 주기적으로 공개된다.
유출 경로는 다양하다. 모델이 프롬프트를 직접 출력하게 유도하거나, 내부자가
공유하거나, 트래픽 캡처로 추출한다.

비밀 유지에 의존하는 안전 모델은 유출 시 해당 레이어 전체가 무력화된다.
시스템 프롬프트 내 지침이 "모델이 훈련으로 내면화한 것"이 아니라 "추론 시점에
주입되는 것"인 한, 그 지침을 담은 텍스트가 공개되는 순간 우회 시도의 타깃이
된다. 유출이 반복된다는 사실 자체가, 안전의 핵심 레이어를 시스템 프롬프트에
두는 아키텍처의 한계를 가리킨다.

## 인사이트

### 시스템 프롬프트가 모델 회사의 법무팀과 정책팀의 산출물이 됐다

발표된 연구 논문이나 모델 카드가 아니라 시스템 프롬프트가 회사의 실질적
정책 문서가 된 상황이다. 저작권, 임상 정신 건강, 이중 용도 안전, 정치적 균형
등의 이슈가 모두 여기에 집약돼 있다.

이는 AI 정책 집행의 메커니즘이 입법이나 표준이 아니라 회사 내부 프롬프트
엔지니어링으로 진행되고 있음을 보여준다. 외부에서 검증하거나 감사하기
어려우며, 회사가 원할 때 변경할 수 있다. 유출로만 그 내용을 알 수 있는
구조 자체가 투명성 문제다.

### Claudeception은 아티팩트를 애플리케이션 플랫폼으로 전환한다

아티팩트가 Claude API를 호출하고 영속 스토리지를 갖는다면, 그것은 렌더링된
텍스트가 아니라 동작하는 애플리케이션이다. API 키 관리 없이 누구나 claude.ai에서
모델-in-모델 앱을 만들 수 있다는 의미다.

이 방향이 확장되면 아티팩트는 낮은 진입장벽의 AI 앱 배포 플랫폼이 된다.
단, 아티팩트 안의 Claude가 또 다른 아티팩트를 만드는 재귀 구조가 어떤
안전 경계를 갖는지는 이 프롬프트에서 명확히 드러나지 않는다.
