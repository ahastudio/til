# gmoney의 Claude Code 실전 25가지 팁

2026년 2월 14일 트윗:
<https://twitter.com/gmoneynft/status/2022454519741767800>

`gmoney.eth(@gmoneyNFT)`가 수개월간 Claude Code를
일상 도구로 사용하며 배운 25가지 실전 팁입니다.
Discord에서 여러 AI 에이전트를 24시간 운영하며
리서치, 건강 추적, 트레이딩 분석, 콘텐츠 작성 등을
자동화한 경험을 바탕으로 합니다.

Boris Cherny의 팁에서 직접 영감을 받은 부분과,
새벽 2시에 직접 부딪히며 알아낸 노하우가 섞여 있습니다.

## 설정과 구성

### 1. CLAUDE.md는 에이전트의 두뇌

세션 시작 시 Claude가 가장 먼저 읽는 파일입니다.
에이전트의 정체성, 행동 방식, 사용 가능한 도구,
피해야 할 실수를 모두 여기에 기록합니다.

Claude Code 팀은 수정할 때마다 Claude에게
CLAUDE.md를 스스로 업데이트하도록 지시합니다.
몇 주 후면 실수율이 눈에 띄게 줄어듭니다.

### 2. 지시가 아닌 영혼을 부여하라

SOUL.md 파일로 성격과 톤을 정의합니다.
"사용자 질의에 효율적으로 응답하라"보다
"진심으로 도움이 되어라, 보여주기식은 안 된다"가
훨씬 나은 결과를 만들어냅니다.

### 3. 지식과 정체성을 분리하라

| 파일      | 역할                            |
|-----------|---------------------------------|
| SOUL.md   | 성격, 톤                        |
| AGENTS.md | 워크플로 규칙                   |
| TOOLS.md  | 환경 정보(SSH, API 키 등)       |
| USER.md   | 사용자에 대한 컨텍스트          |

하나를 수정해도 나머지가 깨지지 않습니다.

### 4. 메모리 파일은 필수

Claude Code는 세션 간 기억이 없습니다.
직접 메모리 시스템을 구축해야 합니다.

- `memory/YYYY-MM-DD.md`: 일별 원시 노트
- `MEMORY.md`: 장기 컨텍스트 큐레이션

시작 시 읽고, 작업 중 기록합니다.
Obsidian에 마크다운으로 저장하면
검색과 브라우징도 가능합니다.

### 5. 스킬(Skill)은 재사용 가능한 플레이북

하루에 두 번 이상 하는 작업은 스킬로 만듭니다.
각 스킬은 SKILL.md에 정확한 지침, 라우팅 로직,
파일 경로를 포함합니다.

아침 요약, 야간 리서치, 건강 다이제스트,
X 아티클 작성, 회의록 처리 등
10개 이상의 커스텀 스킬을 운영합니다.

## 워크플로와 프롬프팅

### 6. 항상 Plan 모드부터

단순 질문이 아니라면 Plan 모드로 시작합니다.
접근법을 먼저 정리하고, 확인한 뒤 실행합니다.

Claude Code 팀 중 한 명은 첫 번째 Claude에게
계획을 쓰게 하고, 두 번째 Claude에게
시니어 엔지니어 입장에서 리뷰하게 합니다.

### 7. 꼬이면 밀어붙이지 말고 다시 계획하라

가장 흔한 실수입니다.
에이전트가 수정을 시도하다 상황을 악화시키고,
다시 시도하고, 악순환에 빠집니다.
수정이 잘못된 느낌이 드는 순간,
Plan 모드로 돌아가야 합니다.

### 8. "use subagents"는 마법의 주문

복잡한 요청에 "use subagents"를 붙이면
Claude가 백그라운드 워커를 생성해서
하위 작업을 병렬로 처리합니다.

메인 컨텍스트 윈도우를 깨끗하게 유지하면서
여러 리서치를 동시에 진행할 수 있습니다.
Claude Code 팀 일부는
작업당 5개 이상의 서브에이전트를 사용합니다.

### 9. 컨텍스트 윈도우는 RAM처럼 관리하라

모든 메시지, 파일 읽기, 도구 호출이
컨텍스트를 소비합니다.
한계에 도달하면 품질이 급격히 떨어집니다.

상태 표시줄로 컨텍스트 사용량을 확인하고,
지저분해지기 전에 `/clear`합니다.
반복 작업마다 새 컨텍스트 윈도우를 여는
방법도 효과적입니다.

### 10. 바보같을 정도로 구체적으로 지시하라

나쁜 예: "AI에 대한 트윗 써줘."

좋은 예: "전부 소문자, 이모지 없이,
200자 이내, 반대 의견 각도로,
AI 에이전트를 쓰는 3명이
20명 팀을 능가할 수 있다는 내용의
트윗을 써줘. 구체적 숫자를 사용해."

제약이 많을수록 결과가 좋아집니다.

### 11. 미흡한 결과물은 수정하지 말고 처음부터

Boris 팀의 조언:
"지금까지 알게 된 모든 것을 바탕으로,
이것을 버리고 우아한 해결책을 구현하라."

나쁜 첫 결과를 패치하는 것보다
전체 컨텍스트를 유지한 채 새로 시작하는 것이
훨씬 낫습니다.

## 24시간 에이전트 운영

### 12. 하트비트로 능동적 에이전트를 만들어라

30분마다 체크인하는 하트비트 시스템을 구축합니다.
이메일, 캘린더, Discord 채널, 서버 상태를 확인하고
주의가 필요하면 알려주고, 아니면 침묵합니다.

핵심: 여러 체크를 하나의 하트비트로 묶어서
API 호출을 줄입니다.

### 13. 정확한 타이밍은 cron, 나머지는 하트비트

스케줄이 중요하면 cron(아침 요약 6:30,
야간 리서치 2:00), 주기적 체크에는
하트비트를 사용합니다.
3개의 하트비트로 충분한 곳에
15개의 cron을 만들지 마세요.

### 14. 여러 에이전트에는 명확한 영역을 지정하라

한 Discord 서버에서 4개의 에이전트를 운영합니다.
처음에는 서로 간섭이 심했습니다.

해결: 에이전트별로 특정 채널, 역할을 부여하고,
언제 개입하고 언제 침묵할지 규칙을 정합니다.
이것 없이는 에이전트끼리 무한 루프로
토큰을 낭비하게 됩니다.

### 15. 콘텐츠 파이프라인을 끝까지 자동화하라

회의 녹취록을 가져와서 커스텀 스킬로 처리하고,
자기 목소리로 초안 포스트를 생성하는 cron이
있습니다. 또 다른 cron은 6시간마다
X/Twitter의 AI 뉴스를 수집해서
Discord에 다이제스트를 게시합니다.

## 도구와 통합

### 16. MCP 서버는 과소평가되고 있다

MCP(Model Context Protocol)로 Claude를
외부 도구에 연결합니다.
데이터베이스, API, 브라우저, 파일 시스템 등.

Anthropic 팀은 Slack MCP로 버그 스레드를
붙여넣고 "고쳐"라고만 합니다.
컨텍스트 전환이 제로입니다.

### 17. Hooks로 지루한 작업을 자동화하라

Hooks는 특정 이벤트 전후에 실행됩니다.
Claude Code 팀은 권한 요청을 Opus를 통해
라우팅하는 Hook으로 안전한 작업을
자동 승인합니다.

Trail of Bits는 보안 감사를 위한
Hooks와 Skills 기반의 설정을 공개했습니다.

### 18. 음성 받아쓰기는 3배 효과

타이핑보다 3배 빠르게 말할 수 있고,
프롬프트가 훨씬 상세해집니다.
Claude Code 팀은 음성 받아쓰기를
강력히 추천합니다.

WisprFlow를 사용하면 타이핑으로는
귀찮아서 적지 않았을 자연어 제약 조건까지
포함한 길고 구체적인 프롬프트가 가능합니다.

### 19. 코드뿐 아니라 데이터 작업에도 활용하라

Claude Code 팀은 6개월 이상
SQL을 직접 작성하지 않았습니다.
코드베이스에 BigQuery 스킬을
체크인해두고 Claude가 쿼리를 처리합니다.

CLI가 있는 모든 데이터베이스에 적용 가능합니다.
수동 스프레드시트 작업 대부분을 대체했습니다.

## 실수에서 배운 것들

### 20. 에이전트가 채널을 공유하면 규칙이 필요하다

초기에 4개 에이전트가 모든 Discord 채널을
수신했습니다. 같은 메시지에 전부 응답하고,
중복 콘텐츠를 생성하고, 토큰을 낭비했습니다.

지금은 에이전트마다 고유 영역이 있고,
공유 채널에는 "직접 언급될 때만 응답"
규칙이 있습니다.

### 21. 시뮬레이션 데이터를 절대 믿지 마라

Claude는 실제처럼 보이는 가짜 데이터를
자신 있게 생성합니다. 트레이딩 백테스트,
연구 인용, 통계 등.

"시뮬레이션 데이터를 절대 사용하지 마라.
실제 데이터가 없으면 그렇다고 말하라"는
규칙을 명시적으로 넣었습니다.

### 22. 에이전트에게 접근 권한을 주기 전에 서버 잠금

원격 VPS에서 에이전트를 운영합니다.
가장 먼저 Tailscale을 설정하고
모든 접근을 Tailscale 서브넷으로 제한합니다.
보안 체크를 위한 cron도 운영합니다.

Trail of Bits의 설정에서 샌드박싱과
권한 제어를 다룹니다.

### 23. Claude Code는 Cursor 안에서도 실행 가능

터미널 전용이 불편하다면 Cursor 안에서
Claude Code를 실행할 수 있습니다.
Claude Code의 기능에 Cursor의 UI를
결합할 수 있습니다.

프로젝트 유형별로 다른 AI 성격을
설정하는 것도 가능합니다.

### 24. 자기 개선 루프는 실제로 작동한다

수정할 때마다 Claude에게 자체 규칙을
업데이트하도록 지시합니다.
Claude는 자기 자신을 위한 규칙을
놀라울 정도로 잘 작성합니다.

몇 주 후면 차이가 체감됩니다.
모델이 바뀌어서가 아니라
지시사항이 복리로 쌓이기 때문입니다.

### 25. 병렬 세션은 대부분이 놓치는 최대 생산성 팁

Claude Code 팀 전원이 같은 말을 합니다.
3~5개 세션을 각각 고유한 git worktree에서
병렬로 실행합니다.
셸 별칭(za, zb, zc)으로 한 키에
세션 간 이동이 가능합니다.

## 인사이트

### Boris Cherny 팁과의 관계

이 글은 Boris Cherny가 공유한 Claude Code 팀의
내부 팁을 실전에 적용한 사례입니다.
CLAUDE.md 자기 업데이트(#1, #24),
Plan 모드(#6), 병렬 세션(#25),
서브에이전트(#8), 스킬(#5) 등은
Boris의 원본 팁과 직접 대응합니다.

### 에이전트 오케스트레이션이 핵심

단일 채팅 수준을 넘어
여러 에이전트의 협업 시스템을 구축하는 것이
핵심 패턴입니다. 하트비트(#12), cron(#13),
영역 분리(#14, #20) 등은 에이전트를
서비스처럼 운영하는 접근법입니다.

### 지시사항의 복리 효과

가장 중요한 통찰은 "지시사항이 복리로 쌓인다"는
점입니다(#24). 모델 자체는 변하지 않지만,
CLAUDE.md/SOUL.md/MEMORY.md의 축적이
시간이 지날수록 에이전트 품질을 높입니다.
이것은 프롬프트 엔지니어링을 넘어
시스템 설계의 영역입니다.

### 실무 자동화의 수준

코딩 도구를 넘어서 건강 추적,
트레이딩 분석, 콘텐츠 파이프라인,
서버 모니터링까지 자동화하는 사례는
Claude Code의 범용 에이전트로서의
가능성을 보여줍니다.

### 가드레일의 중요성

시뮬레이션 데이터 금지(#21),
서버 잠금(#22), 에이전트 영역 분리(#14)는
AI 에이전트 운영의 위험 관리를 다룹니다.
자유도가 높을수록 가드레일이 중요하다는
실전 교훈입니다.
