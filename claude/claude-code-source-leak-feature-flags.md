# Claude Code 소스 유출: 44개 숨겨진 피처 플래그 분석

원문: <https://clor.tistory.com/100>

HN 토론: <https://news.ycombinator.com/item?id=47584540> (2,095점, 1,022개 댓글)

## 요약

2026년 3월 31일, Anthropic의 `@anthropic-ai/claude-code` npm 패키지 v2.1.88이 59.8MB
크기의 소스맵 파일을 실수로 포함해 배포되었다.
이를 통해 내부 프로젝트명 “Tengu”로 불리는 Claude Code의 전체 소스 코드 — 512,000줄,
1,900개 TypeScript 파일 — 가 외부에 노출되었다.

소스 분석 결과 두 계층으로 구성된 피처 플래그 아키텍처가 드러났다.
컴파일 타임 플래그는 Bun의 `feature()` 매크로를 사용하며, 비활성 기능은 빌드 시점에
완전히 제거된다.
런타임 플래그는 GrowthBook A/B 테스트 플랫폼으로 관리되며 `tengu_` 접두사를 사용한다.

컴파일 타임 플래그는 32개가 확인되었으며 크게 에이전트/자율 실행, 플래닝/원격 실행,
컨텍스트/메모리, UI/인터페이스, 인프라/자동화, 디버깅 카테고리로 분류된다.
주목할 만한 플래그들로는 24시간 백그라운드 데몬인 KAIROS, 사용자 요청 없이 작업을 시작하는
PROACTIVE, 멀티 에이전트 오케스트레이션을 담당하는 COORDINATOR_MODE 등이 있다.

26개의 숨겨진 슬래시 커맨드도 발견되었다.
`/ultraplan`, `/dream`, `/ctx-viz`, `/bridge-kick`, `/good-claude` 등이 포함되며,
`/ultraplan`은 Opus 4.6으로 30분 분량의 클라우드 플래닝을 수행하는 기능이다.
또한 120개 이상의 환경 변수가 확인되었는데, `DISABLE_COMMAND_INJECTION_CHECK` 같은
보안 관련 항목도 포함된다.

특히 눈길을 끄는 것은 안티-디스틸레이션(Anti-Distillation) 방어 메커니즘으로,
경쟁사가 API 트래픽을 통해 모델을 학습하지 못하도록 시스템 프롬프트에 가짜 도구 정의를
주입한다.
사용자의 좌절 신호를 감지하는 정규 표현식 패턴도 포함되어 있어, 톤 조절에 사용된다.

## 분석

### 플래그 아키텍처가 드러내는 개발 전략

이중 계층 플래그 아키텍처는 Anthropic이 기능 출시를 얼마나 체계적으로 관리하는지를
보여준다.
컴파일 타임 플래그를 통해 비활성 코드를 번들에서 완전히 제거함으로써 성능 overhead와
정보 노출을 동시에 최소화하는 전략이다.
런타임 GrowthBook 플래그는 A/B 테스트 기반으로 점진적 롤아웃을 가능하게 한다.

이 구조에서 특히 주목해야 할 것은 KAIROS, PROACTIVE, COORDINATOR_MODE 플래그들이
하나의 방향을 가리킨다는 점이다.
백그라운드 데몬, 자율 작업 시작, 멀티 에이전트 오케스트레이션이 동시에 개발되고 있다는 것은
Claude Code가 현재의 “요청-응답” 모델에서 “상시 동작 에이전트” 모델로 전환을 준비하고
있음을 시사한다.

### COORDINATOR_MODE의 아키텍처적 함의

COORDINATOR_MODE는 오케스트레이션과 실행을 분리하는 구조다.
코디네이터는 Mailbox 시스템을 통해 워커를 관리하되, 직접적인 파일 시스템 접근 권한은
갖지 않는다.
연구자들은 이를 “명령-통제 계층 구조”에 비유했다.

이 구조는 현재의 단일 에이전트 모델과 질적으로 다르다.
코디네이터가 복수의 실행 에이전트를 조율한다면, 병렬 작업 처리 능력이 기하급수적으로
늘어날 수 있다.
CCR_AUTO(Cloud Container Runtime 자동화)와 BYOC_RUNNER(직접 컴퓨팅 제공)가 함께
개발 중인 것은 이 에이전트들이 클라우드 인프라 위에서 동작하는 그림을 완성시킨다.

### BUDDY와 VOICE_MODE: 인터페이스의 방향

BUDDY 플래그는 18종의 종(species) 변형을 가진 터미널 AI 펫이다.
커뮤니티는 소스에서 BUDDY를 “ASCII 아트 스프라이트를 가진 다마고치 방식의 동반자 생물 시스템”으로
묘사한 내부 설명을 발견했다.[^treexs]
같은 소스에서 확인된 “Undercover mode”는 Anthropic 직원이 오픈소스 프로젝트에 기여할 때
커밋과 PR에서 Anthropic 내부 정보를 모두 제거하는 기능이다.
VOICE_MODE와 BRIDGE_MODE(전화/브라우저 원격 제어)를 함께 보면, Anthropic이 CLI 도구를
넘어 다양한 인터랙션 패러다임을 실험하고 있음을 알 수 있다.

## 비평

### 강점: 내부 아키텍처의 드문 실증적 관찰

이 사건은 평소 접근 불가능한 대형 AI 에이전트 제품의 내부 설계를 실증적으로 관찰할 수 있는
드문 기회를 제공했다.
기능 플래그 아키텍처, 안티-디스틸레이션 전략, 멀티 에이전트 설계 패턴은 공개 문서에서는
확인하기 어려운 정보다.
이 분석은 해당 내용을 체계적으로 정리해 가치가 높다.

### 약점: 해석의 한계

노출된 코드 구조를 분석한 것이지, 실제 동작을 검증한 것이 아니다.
플래그가 존재한다는 것이 해당 기능이 현재 작동 중이거나 곧 출시된다는 의미는 아니다.
KAIROS나 PROACTIVE 같은 플래그가 실험적 프로토타입인지, 적극 개발 중인 기능인지를
소스 구조만으로 판단하기 어렵다.
또한 보도 이후 Anthropic이 관련 패키지를 수정했을 것이므로, 현재 시점의 상태는 다를 수 있다.

사실 이번이 처음 있는 일도 아니다.
HN 커뮤니티의 mil22는 2025년 2월에도 유사한 사건이 있었음을 지적했다.[^mil22]
당시 Sublime Text 캐시를 통해 소스 일부가 노출된 사례로,
반복적인 사고는 Anthropic의 빌드·배포 파이프라인에 구조적 취약점이 있음을 시사한다.

코드 품질에 대한 비판도 제기되었다.
커뮤니티 멤버 mohsen1은 `src/cli/print.ts`의 특정 함수를 “모든 지표에서 코드베이스
최악의 함수”라고 지적했다.[^mohsen1]
3,167줄, 최대 12단계 중첩, 분기 복잡도(cyclomatic complexity) 약 486이며,
에이전트 실행 루프, AWS 인증, MCP 수명 주기 관리 등 전혀 다른 책임이 한 함수에 뒤엉켜 있다.
512,000줄 규모의 제품에서 이런 코드가 존재한다는 점은 내부 기술 부채의 단면을 보여준다.

### 윤리적 고려

소스맵 유출은 Anthropic의 의도치 않은 실수다.
이 분석이 공익적 정보를 제공하더라도, 유출된 내부 코드를 상세히 공개하는 것에는 윤리적
고려가 필요하다.
특히 `DISABLE_COMMAND_INJECTION_CHECK` 같은 보안 관련 환경 변수의 공개는 잠재적인
보안 위험을 수반한다.

흥미로운 것은 Anthropic의 대응 방식이다.
패키지를 `npm unpublish`로 삭제하는 대신 `npm deprecate`로 표시하는 경로를 택했는데,
HN의 foob는 이 차이를 정확히 짚어냈다.[^foob]
`unpublish`는 레지스트리에서 버전을 완전히 제거하지만, `deprecate`는 패키지를 남겨둔 채
경고 메시지만 붙인다.
이미 소스맵이 광범위하게 다운로드된 상황에서 완전 삭제보다 빠른 경고 표시를 택한 것은
실용적 판단이었지만, 역설적으로 소스맵 파일이 더 오래 접근 가능한 상태로 남게 했다.

## 인사이트

### “상시 에이전트” 전환은 이미 시작되었다

KAIROS, PROACTIVE, DAEMON 플래그의 동시 존재는 우연이 아니다.
이것은 Claude Code가 근본적으로 다른 상호작용 패러다임으로의 전환을 준비하고 있다는
명확한 신호다.
현재의 Claude Code는 사용자가 질문하면 응답하는 “요청-응답” 모델이다.
KAIROS(24시간 데몬)와 PROACTIVE(자율 작업 시작)가 활성화되면, 이것은 사용자가 자리를
비운 사이에도 코드를 작성하고, 오류를 수정하고, PR을 준비하는 에이전트가 된다.

이 전환의 함의는 단순한 기능 추가를 넘어선다.
“언제든 깨워서 쓸 수 있는 도구”와 “항상 실행 중인 에이전트”는 사용자와의 관계가 근본적으로
다르다.
전자는 망치와의 관계고, 후자는 어시스턴트와의 관계다.
Anthropic이 이 전환을 준비하고 있다는 것은 AI 도구의 카테고리 자체가 재정의되고 있음을
의미한다.

이 전환이 성공하면, 개발자의 작업 방식은 “AI에게 작업을 지시하는 것”에서 “AI가 제안하고
개발자가 검토하는 것”으로 역전될 수 있다.
결국 에이전트 주도의 워크플로우에서 인간의 역할은 실행자에서 검토자로 이동하며, 이것이
소프트웨어 개발 프로세스 전반에 미치는 영향은 아직 충분히 논의되지 않고 있다.

### 안티-디스틸레이션 방어는 AI 생태계의 새로운 군비 경쟁을 예고한다

시스템 프롬프트에 가짜 도구 정의를 주입해 경쟁사의 모델 학습을 방해한다는 발견은
AI 산업에서 새로운 유형의 경쟁이 시작되었음을 보여준다.
소스에서 발견된 구체적인 구현은 모든 API 요청에 `anti_distillation: ['fake_tools']`를
주입하는 방식으로, 경쟁사가 API 트래픽을 스크레이핑하면 가짜 도구 정의가 훈련 데이터에
섞여 들어가게 된다.[^cedws]
LLM 회사가 사용자 좌절 감지에 정교한 감정 분석 모델 대신 정규 표현식을 사용한다는 점도
커뮤니티의 주목을 받았다.
BoppreH는 이를 “트럭 회사가 부품 운반에 말을 쓰는 것”에 비유했다.[^BoppreH]
레이턴시와 비용 측면에서는 합리적인 선택일 수 있지만, AI 기업의 자기 제품 활용 철학에 대한
의문을 남긴다.

전통적인 소프트웨어에서는 API를 통한 경쟁사 데이터 수집을 약관으로 금지하거나,
레이트 리밋으로 막거나, 법적으로 대응했다.
AI 모델에서는 훈련 데이터의 품질이 제품 품질과 직결되기 때문에, 경쟁사가 자신의 API를
통해 데이터를 수집해 경쟁 모델을 훈련시키는 것을 기술적으로 방해하려는 시도가 나타난다.
이것은 소프트웨어 보안의 새로운 영역이다 — 인간 해커가 아니라 AI 에이전트를 상대로 한
정보 오염 전략이다.

이 군비 경쟁이 확산되면, AI 제품들은 점점 더 자신의 동작 방식을 난독화하고, 경쟁사 에이전트를
탐지하고, 의도적으로 오염된 응답을 생성하는 방향으로 진화할 수 있다.
이는 AI 생태계 전반의 신뢰성 문제로 이어질 수 있으며, 누가 어떤 데이터를 어떻게 수집하는가에
대한 업계 차원의 표준이 필요해질 것이다.

### 소스 유출이 드러낸 것은 코드가 아니라 로드맵이다

59.8MB 소스맵 유출에서 가장 중요한 것은 코드 자체가 아니다.
플래그 이름들이 드러낸 로드맵이다.

기업은 보통 로드맵을 비공개로 유지한다.
경쟁사에게 전략을 노출하지 않기 위해서, 그리고 출시 일정이 바뀌었을 때 사용자 실망을
관리하기 위해서다.
그런데 플래그 이름들은 단순한 기능 목록이 아니라 제품 철학을 담고 있다.
KAIROS(그리스어로 “적절한 순간”), BUDDY, DREAM, ULTRAPLAN 같은 이름들은
Anthropic이 Claude Code를 어떤 방향으로 진화시키려는지에 대한 강한 암시를 담고 있다.

이 사건은 현대 소프트웨어 개발에서 소스맵의 보안 의미를 재고하게 만든다.
소스맵은 디버깅을 위한 개발자 도구였지만, 충분히 큰 시스템에서는 아키텍처와 로드맵을
노출하는 정보 보안 위험이 될 수 있다.
Anthropic의 실수는 이 점을 업계 전체에 각인시키는 사례로 기록될 것이다.

HN에서는 유출된 소스를 바탕으로 유사 시스템을 재구현하는 것의 법적 허용 여부를 묻는
질문도 제기되었다.[^blobbers]
누군가의 실수로 공개된 코드라도 그것을 참조해 경쟁 제품을 만드는 것이 지식재산권을
침해하는지는 법적으로 불분명하다.
소스코드 유출의 법적 처리는 클린룸 설계(clean room design) 원칙, 영업 비밀 보호,
오픈소스 라이선스(MIT) 적용 범위가 복잡하게 얽혀 있다.
이 사건이 그저 Anthropic의 배포 실수로 끝나는지, 아니면 업계 표준을 바꾸는 법적 선례로
이어지는지는 아직 열린 질문이다.
