# .claude/ 폴더 구조 분석

- 원문: <https://blog.dailydoseofds.com/p/anatomy-of-the-claude-folder>

## 요약

Claude Code가 생성하고 사용하는 `.claude/` 디렉터리의 구조와 각 구성
요소의 역할을 해부한 글이다. CLAUDE.md, 설정 파일, 메모리, 스킬, 훅의
동작 방식을 설명한다.

GeekNews에서 50포인트를 기록했다.

## 주요 구성 요소

| 경로                   | 역할                                  |
| ---------------------- | ------------------------------------- |
| `CLAUDE.md`            | 에이전트 지침·컨텍스트 주입           |
| `.claude/settings.json`| Claude Code 동작 설정                 |
| `.claude/memory/`      | 세션 간 지속 메모리                   |
| `.claude/skills/`      | 재사용 스킬 정의                      |
| `.claude/hooks/`       | 이벤트 기반 자동 실행 훅              |

## 인사이트

### 1. .claude/는 에이전트를 위한 설정 레이어다

`.env`가 앱 실행 환경을 제어하듯, `.claude/`는 에이전트 실행 환경을
제어한다. CLAUDE.md는 "이 프로젝트에서 이렇게 행동하라"는 지침서이고,
skills는 재사용 가능한 도구 모음이며, hooks는 이벤트 트리거다.

이 구조를 이해하면 Claude Code를 단순 채팅이 아니라 프로그래밍 가능한
에이전트로 운용할 수 있다.

### 2. 에이전트 설정도 코드와 함께 버전 관리해야 한다

`.claude/`를 git에 커밋하면 팀 전체가 동일한 에이전트 동작을 공유한다.
"왜 내 Claude는 이렇게 동작하는데 너는 달라?"라는 혼선이 사라진다.

에이전트 지침도 코드 리뷰 대상이 된다. 프롬프트 엔지니어링이 팀 작업이
되는 것이다.

### 3. 계층적 컨텍스트 주입이 핵심 설계다

전역(`~/.claude/`) → 프로젝트(`.claude/`) → 하위 디렉터리(`CLAUDE.md`)
순서로 지침이 합쳐진다.

공통 규칙은 전역에, 프로젝트별 규칙은 로컬에 두는 관심사 분리가 가능하다.
모노레포에서 패키지별로 다른 지침을 줄 수도 있다.

### 4. 메모리와 훅이 에이전트를 "학습 가능한" 시스템으로 만든다

단발성 도구가 아니라 사용할수록 맥락이 쌓이고 자동화가 늘어나는 시스템이다.
메모리는 대화 간 연속성을, 훅은 반복 작업의 자동화를 담당한다.

이 설계는 에이전트가 단순 보조에서 팀의 일원으로 진화하는 방향을 보여준다.
