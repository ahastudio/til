# Claude-Ads — Claude Code로 광고 감사를 자동화하는 스킬

<https://github.com/AgriciDaniel/claude-ads>

## 소개

Claude-Ads는 Claude Code 스킬(Skill)로 구현된 광고 감사(Audit) 자동화 도구다. 7개 광고 플랫폼에 걸쳐 250개 이상의 감사 항목을 실행하며, 전통적인 대행사가 2-4주 걸리는 감사를 3-5분에 완료한다고 주장한다. 6개의 병렬 서브에이전트가 플랫폼별로 동시 분석을 수행하며, 모든 처리는 로컬에서 이뤄진다. MIT 라이선스로 공개된 오픈 소스 프로젝트다.

## 지원 플랫폼과 감사 항목 수

| 플랫폼               | 감사 항목 수 | 주요 커버리지                           |
| -------------------- | ------------ | --------------------------------------- |
| Google Ads           | 80개         | Search, PMax, AI Max, Demand Gen, CTV   |
| Meta Ads             | 50개         | Pixel/CAPI, 크리에이티브, 타기팅        |
| Apple Ads            | 35+개        | ASA, 앱 스토어 최적화                   |
| TikTok Ads           | 28개         | 크리에이티브, 픽셀, 입찰 전략           |
| LinkedIn Ads         | 27개         | B2B 타기팅, Matched Audiences           |
| Microsoft/Bing Ads   | 24개         | 검색 파트너, 쇼핑 캠페인                |
| YouTube/크로스플랫폼 | 별도         | 영상 광고, 통합 어트리뷰션              |

## 핵심 분석 기능

**Ads Health Score:**
계정 전체에 0-100점을 부여하는 채점 시스템으로, 심각도 가중치(Severity-Weighted) 알고리즘을 적용한다. 점수 구간은 A(90-100, “사소한 최적화만 필요”)부터 F(40점 미만, “즉각 개입 필요”)까지 6단계로 나뉜다.

**품질 게이트(Quality Gates):**
규칙 기반 가드레일을 통해 위험한 설정 조합을 자동으로 플래깅(Flagging)한다. Smart Bidding 없이 Broad Match를 권장하지 않도록 방지하고, CPA가 목표의 3배를 초과하면 경고하며, 최소 예산 충족 여부를 강제 검사하고, 학습 단계(Learning Phase) 캠페인의 조기 편집을 차단한다.

**추가 분석 기능:**

- 산업별 벤치마크 비교 및 캠페인 템플릿 (SaaS, 이커머스, 지역 서비스, B2B)
- PPC 재무 모델링 및 계산기 (CPA, ROAS, LTV:CAC)
- A/B 테스트 설계 프레임워크
- 경쟁사 인텔리전스 분석
- PDF 감사 보고서 생성 (`reportlab` 사용)
- 크로스플랫폼 크리에이티브 품질 평가

## 기술 아키텍처

### 3-레이어 오케스트레이션 구조

Claude-Ads는 오케스트레이터(Orchestrator) → 서브 스킬(Sub-Skill) → 에이전트(Agent)의 3계층 구조로 동작한다. 각 레이어는 명확히 분리된 책임을 가진다.

**오케스트레이터 레이어**는 `/ads audit` 같은 최상위 명령을 수신하고, 어떤 플랫폼 에이전트를 활성화할지, 어떤 참조 데이터를 로드할지, 어떤 순서로 결과를 집계할지를 결정한다. Claude Code의 에이전트 위임(Agent Delegation) 기능을 통해 6개 플랫폼 에이전트를 동시에 실행시킨다.

**서브 스킬 레이어**는 각 플랫폼별 도메인 전문성을 캡슐화한다. Google Ads 서브 스킬은 PMax 캠페인의 자산 그룹 구성, AI Max의 검색어 매칭 설정, Demand Gen의 크리에이티브 규격 등 Google 특유의 80개 감사 항목 로직을 내장한다. 이 레이어는 25개의 참조 파일에서 현재 벤치마크와 플랫폼별 규격 데이터를 선택적으로 로드한다.

**에이전트 레이어**는 실제 분석 작업을 수행한다. 내보낸 데이터, 스크린샷, 또는 붙여 넣은 지표를 입력으로 받아 각 감사 항목을 검사하고 Health Score에 기여하는 점수를 산출한다. MCP(Model Context Protocol) 서버를 통해 Google Ads API, Meta Marketing API, LinkedIn API와 직접 연동도 가능해, 수동 데이터 내보내기 없이 자동화할 수 있다.

### 병렬 에이전트 오케스트레이션 패턴

6개 에이전트의 병렬 실행은 Claude Code의 서브에이전트(Subagent) 기능에 기반한다. 이 패턴의 핵심은 각 에이전트가 독립적인 컨텍스트 창(Context Window)을 갖는다는 점이다. 단일 에이전트가 7개 플랫폼을 순차적으로 처리하면, 앞서 분석한 플랫폼의 데이터가 컨텍스트를 점유해 후속 분석의 품질이 저하될 수 있다. 반면 플랫폼별 독립 에이전트는 해당 플랫폼에 특화된 데이터와 규칙만을 컨텍스트에 유지하므로, 분석 깊이와 속도 모두에서 이점이 있다.

오케스트레이터는 각 에이전트의 완료를 기다리며 결과를 집계한다. 단순 병합이 아니라 플랫폼 간 교차 분석도 수행한다. 예를 들어 Google Ads와 Meta 양쪽에서 유사한 타기팅을 사용하고 있다면, 어트리뷰션 중복 가능성을 크로스플랫폼 인사이트로 제시한다.

### 로컬 실행과 프라이버시 모델

Claude-Ads의 설계 원칙 중 하나는 광고 계정 데이터가 외부 서버로 전송되지 않는다는 점이다. 모든 분석은 로컬 머신에서 Claude Code CLI를 통해 실행된다. MCP 서버 연동도 로컬 MCP 서버를 통해 이뤄지므로, 플랫폼 API에 대한 인증 토큰은 로컬에만 존재한다.

광고 성과 데이터, 예산 정보, 타기팅 설정, 전환율 등은 경쟁사 유출 시 직접적 비즈니스 피해로 이어지는 민감 정보다. SaaS 형태로 제공되는 광고 분석 도구들이 클라우드에 데이터를 업로드해야 하는 것과 달리, 로컬 실행 방식은 기업 보안 정책과의 충돌 없이 채택할 수 있다는 점에서 엔터프라이즈 도입 장벽을 낮춘다.

## 분석

### 광고 감사의 규칙 기반 vs. 창의적 판단 경계

광고 감사 업무를 크게 두 범주로 나누면, Claude-Ads가 자동화할 수 있는 영역과 여전히 인간이 필요한 영역이 명확해진다.

규칙 기반 감사는 자동화에 적합하다. “Smart Bidding 없이 Broad Match를 사용하고 있는가”, “CPA가 목표의 3배를 초과했는가”, “캠페인이 광고 예산을 충분히 소진하고 있는가”, “광고 품질 점수(Quality Score)가 임계값 미만인가” 같은 항목들은 명확한 규칙으로 표현되며, 데이터가 주어지면 기계적으로 검사할 수 있다. 이것이 Claude-Ads가 250개 이상의 감사 항목을 다루는 방식이다.

반면 창의적 판단이 필요한 영역은 다르다. “이 광고 크리에이티브가 브랜드 정체성에 부합하는가”, “이 캠페인 메시지가 경쟁 환경에서 차별화되는가”, “이 타기팅 전략이 6개월 후 시장 변화에 적응할 수 있는가” 같은 판단은 단순한 규칙 체크가 아니라 시장, 경쟁사, 브랜드에 대한 깊은 이해를 요구한다. Claude-Ads는 이 영역에 대해서도 분석을 제공하지만, 그 품질은 제공된 컨텍스트의 풍부함에 의존한다.

현재 구현에서 가장 큰 제약은 데이터 입력 방식이다. MCP 서버 직접 연동이 가능하지만, 설정이 없는 기본 모드에서는 사용자가 내보낸 데이터, 스크린샷, 또는 지표를 붙여 넣어야 한다. 이 수동 단계가 남아있는 한, “3-5분 완성”의 전제 조건으로 사용자의 준비 시간이 포함되지 않는다는 점을 고려해야 한다.

### AI가 전문 서비스를 자동화하는 역사적 패턴

Claude-Ads가 “광고 대행사를 대체”한다고 주장하는 것은 AI의 역사에서 반복되는 패턴의 최신 사례다. 전문가 서비스의 자동화는 항상 규칙화 가능한 절차적 작업부터 시작됐다.

1980년대와 1990년대, 회계 소프트웨어(Lotus 1-2-3, QuickBooks)는 부기사(Bookkeeper)의 일상 업무를 대체했다. 고객 원장 정리, 대차대조표 계산, 세금 양식 작성 같은 규칙 기반 작업이 소프트웨어로 이관됐다. 회계사의 역할은 소프트웨어가 생성한 데이터를 해석하고 전략적 조언을 제공하는 방향으로 이동했다.

2000년대와 2010년대, LegalZoom 같은 서비스는 표준화된 법률 문서(설립 계약, 유언장, 상표 출원)를 자동 생성해 법률 서비스의 진입 장벽을 낮췄다. 그러나 복잡한 소송, 협상, 창의적 법률 전략 수립은 여전히 변호사의 영역으로 남았다.

광고 감사는 이 흐름의 연장선에 있다. 광고 대행사가 수행하는 감사 업무의 상당 부분이 체크리스트 기반 점검이라는 점에서, Claude-Ads가 주장하는 자동화는 기술적으로 타당하다. 그러나 진짜 대행사 가치의 핵심, 즉 시장 이해, 창의적 아이디에이션, 클라이언트 관계 관리, 위기 대응은 여전히 자동화의 경계 밖에 있다. Claude-Ads는 대행사를 대체하는 것이 아니라, 대행사가 했어야 할 기계적 작업을 대체함으로써 대행사(또는 클라이언트)가 더 높은 가치 활동에 집중할 수 있게 하는 도구로 보는 것이 더 정확하다.

## 비평

### 강점: 플랫폼 특화 깊이와 품질 게이트

단순히 “광고를 분석한다”가 아니라 Google Ads에서는 PMax 자산 그룹, AI Max 검색어 매칭, Demand Gen 크리에이티브, CTV까지, Meta에서는 Pixel/CAPI 설정, Advantage+ 캠페인 구성, 크리에이티브 피로도(Creative Fatigue)까지 세분화된 항목을 다룬다. 이 깊이는 범용 AI 도구와 차별화되는 도메인 전문성을 보여준다.

품질 게이트 개념도 주목할 만하다. 단순히 “이렇게 하면 좋다”는 권고에 그치지 않고, 특정 조건 조합(Smart Bidding 없는 Broad Match)을 명시적으로 금지하는 가드레일은 초보자가 실수로 비효율적인 설정을 따르는 것을 방지한다. 이는 규칙의 단순 목록이 아니라 인과 관계를 이해한 시스템 설계를 보여준다.

### 약점: 데이터 입력의 마찰과 검증 부재

MCP 서버 설정이 없는 기본 사용에서는 사용자가 직접 데이터를 준비하고 붙여 넣어야 한다. 광고 데이터를 플랫폼별로 내보내고, 형식을 맞추고, Claude Code에 입력하는 과정이 여전히 수동이다. “3-5분”이라는 주장이 이 준비 시간을 포함하지 않는다면, 전체 워크플로의 소요 시간은 더 길다.

“광고비의 15-30% 낭비 발견”, “월 $7,500-$15,000 절감” 같은 수치는 구체적이지만 검증이 필요하다. 이 수치들이 실제 사용자 사례에서 측정된 것인지, 또는 광고 업계 일반 통계를 바탕으로 한 이론적 추정인지 문서에서 명확하지 않다. 오픈 소스 프로젝트로서 실제 사용 사례와 검증된 결과를 커뮤니티에서 공유하는 것이 신뢰성 제고에 도움이 될 것이다.

### 약점: AI 생성 분석의 환각(Hallucination) 위험

광고 캠페인 분석에서 잘못된 권고는 실질적인 비용 손실로 이어진다. AI 모델이 플랫폼 정책이나 벤치마크 데이터를 잘못 기억하거나, 데이터 패턴을 잘못 해석하는 환각 리스크가 있다. Claude-Ads가 25개의 참조 파일을 사용해 최신 벤치마크를 주입한다는 것은 이 위험을 줄이기 위한 설계지만, 참조 파일이 얼마나 자주 업데이트되는지, 플랫폼 정책 변경을 어떻게 추적하는지에 대한 명확한 설명이 없다.

## 인사이트

### Claude Code 스킬이 수직 시장 자동화의 새로운 배포 단위가 되고 있다

Claude-Ads는 Claude Code의 스킬 시스템이 단순한 개발 보조 도구를 넘어 도메인 특화 자동화 플랫폼(Domain-Specific Automation Platform)으로 진화할 수 있음을 보여주는 구체적 사례다. 스킬 시스템의 핵심은 도메인 지식을 코드와 함께 패키징해 배포할 수 있다는 점이다.

전통적인 소프트웨어 개발에서 도메인 전문성은 개발자가 별도로 학습하거나, 도메인 전문가와 협업해야 얻을 수 있었다. Claude Code 스킬은 이 전문성을 재사용 가능한 컴포넌트로 캡슐화한다. Claude-Ads의 경우, 광고 업계 10년 경험이 250개 감사 항목, 25개 참조 파일, 품질 게이트 규칙 집합으로 표현됐다. 이 패키지를 `/plugin install` 한 줄로 설치하면, 광고를 전혀 모르는 개발자도 전문적인 광고 감사 도구를 갖게 된다.

이 패턴은 광고를 넘어 다양한 수직 시장으로 확장될 것이다. 법률 계약서 검토 스킬, ISO 27001 보안 감사 스킬, IFRS 재무 보고서 분석 스킬, 의료 기기 규정 준수 검사 스킬 등이 동일한 방식으로 구현될 수 있다. Claude Code 플러그인 마켓플레이스는 이러한 도메인 지식 패키지의 거래 장소가 될 것이며, 이는 소프트웨어 배포의 새로운 계층을 만들어낸다.

이 새로운 배포 단위의 함의는 크다. 지금까지 도메인 전문 소프트웨어는 개발팀, 제품팀, 세일즈팀을 갖춘 회사들이 만들었다. Claude Code 스킬은 개인 도메인 전문가가 자신의 지식을 소프트웨어로 패키징해 배포할 수 있게 한다. 광고 전문가가 자신의 방법론을 스킬로 만들고 마켓플레이스에서 판매하는 모델이 가능해진다. 이것은 지식 노동의 상품화 방식을 근본적으로 바꾸는 변화다.

### 에이전시 서비스의 “커모디티화(Commoditization)” 가속과 새로운 분업

Claude-Ads가 명시적으로 “광고 대행사 대체”를 목표로 한다는 점은 광고 산업의 구조 변화를 예고한다. 그러나 이 변화를 “대행사 소멸”로 이해하는 것은 역사적 패턴에 맞지 않는다.

회계 소프트웨어가 등장했을 때 부기사가 사라지고 회계사는 오히려 성장했다. 기계적 계산 업무가 소프트웨어로 이관되면서, 회계사는 더 높은 가치를 창출하는 분석과 조언 업무에 집중할 수 있게 됐다. LegalZoom이 등장했을 때 표준화된 계약서 작성 업무는 커모디티화됐지만, 복잡한 M&A 법률 자문이나 소송 전략은 오히려 프리미엄이 높아졌다.

광고 감사의 커모디티화는 대행사가 감사 업무에 투입하던 초급 인력의 역할을 축소시킬 것이다. 그러나 그 자리에는 Claude-Ads 같은 도구를 운영하고, 출력 결과를 해석하며, 클라이언트에게 전략적 맥락을 제공하는 새로운 역할이 생긴다. AI 도구를 활용해 생산성을 10배 높인 소규모 광고 전문가 집단이 기존 대형 대행사와 경쟁할 수 있게 되는 구조 변화가 더 가능성 높은 결과다.

이 분업의 새로운 경계선은 “규칙으로 표현 가능한 전문 지식” vs. “맥락과 관계에 의존하는 판단”이다. 전자는 AI 스킬로 커모디티화되고, 후자는 더욱 가치 있어진다. 광고 대행사가 생존하려면 자신의 가치가 어느 쪽에 있는지를 명확히 파악하고, AI 도구와 협력하는 방식을 빠르게 정립해야 한다.

### 멀티 에이전트 오케스트레이션이 AI 소프트웨어의 기본 설계 패턴이 된다

Claude-Ads의 6개 병렬 에이전트 구조는 현재 AI 소프트웨어 엔지니어링의 선도적인 패턴을 반영한다. 단일 대형 언어 모델이 복잡한 작업을 처리하는 모놀리식(Monolithic) 접근에서, 특화된 에이전트들이 협업하는 분산 에이전트(Distributed Agent) 아키텍처로 패러다임이 이동하고 있다.

이 이동의 이유는 근본적이다. 언어 모델의 컨텍스트 창은 유한하고, 여러 도메인의 복잡한 정보를 동시에 고품질로 처리하는 것은 단일 모델에게 과도한 부담이다. 플랫폼별 특화 에이전트는 해당 도메인의 데이터와 규칙만을 컨텍스트에 유지함으로써 분석 품질을 높이고, 병렬 실행으로 전체 처리 시간을 단축한다.

더 중요한 것은 이 패턴이 소프트웨어 설계 원칙과 정렬된다는 점이다. 단일 책임 원칙(Single Responsibility Principle), 관심사 분리(Separation of Concerns), 수평 확장(Horizontal Scaling)이 AI 에이전트 설계에도 동일하게 적용된다. Claude-Ads의 오케스트레이터-서브스킬-에이전트 3계층 구조는 마이크로서비스(Microservice) 아키텍처와 개념적으로 동형(Isomorphic)이다.

이 패턴이 성숙하면서, 멀티 에이전트 오케스트레이션을 위한 도구와 프레임워크가 표준화될 것이다. 에이전트 간 통신 프로토콜, 상태 관리, 오류 처리, 결과 집계 방식이 컨벤션으로 확립되고, 이 위에서 도메인 특화 스킬이 더 쉽게 구현될 수 있게 될 것이다. Claude-Ads는 이 방향으로 나아가는 초기 사례 중 하나로, 향후 AI 소프트웨어 아키텍처 진화의 방향을 미리 보여준다.
