# Claude Code의 숨겨진 강력한 기능 15가지

> 원문:
> <https://twitter.com/bcherny/status/2038454336355999749>

## 요약

Claude Code 창시자 Boris Cherny가 공유한 15가지 숨겨진 기능 목록이다. 모바일
코딩, 세션 이동, 자동 스케줄링, 병렬 팬아웃 등 일반 사용자가 쉽게 발견하기
어려운 고급 기능을 다룬다.

## 기능 목록

### 1. 모바일 앱

iOS/Android의 Code 탭으로 노트북 없이 코딩이 가능하다. 이동 중에 세션을
시작하고 나중에 데스크탑에서 이어받을 수 있다.

### 2. 세션 텔레포트

`--teleport` 명령으로 모바일, 웹, 데스크탑, 터미널 간 클라우드 세션을 이전한다.
`/remote-control`로 원격 제어도 가능하다.

### 3. 자동 스케줄링

`/loop`와 `/schedule`로 최대 1주일 단위 자동 실행을 설정한다. 코드 리뷰, Slack
피드백 수집 등 반복 작업을 자동화할 수 있다.

### 4. Hooks

에이전트 라이프사이클에 결정론적 로직을 삽입한다. SessionStart, PreToolUse,
PermissionRequest 등 이벤트에 셸 스크립트를 연결할 수 있다.

### 5. Dispatch

Claude Desktop의 보안 원격 제어 기능으로, 원격 작업을 수행한다.

### 6-7. 브라우저 확장

Chrome 확장과 Desktop 앱으로 웹 개발 결과를 직접 검증한다.

### 8. 세션 포크

`/branch` 또는 CLI로 기존 세션을 분기한다. 하나의 세션에서 여러 방향을 동시에
탐색할 수 있다.

### 9. /btw

작업 중 빠른 사이드 쿼리를 처리한다. 메인 작업 흐름을 끊지 않고 부수적인 질문을
던질 수 있다.

### 10. Git Worktrees

동일 저장소에서 다수의 병렬 작업을 지원한다. Boris Cherny가 이전 팁에서도
강조한 핵심 생산성 기법이다.

### 11. /batch

수십에서 수천 개의 워크트리 에이전트를 병렬로 팬아웃한다. 대규모 마이그레이션이나
일괄 변경 작업에 적합하다.

### 12. --bare 플래그

SDK 시작 속도를 최대 10배 향상시킨다. 자동화 스크립트나 CI 환경에서 유용하다.

### 13. --add-dir

추가 폴더에 대한 접근 권한을 부여한다. 모노레포나 멀티 프로젝트 환경에서
여러 디렉토리를 동시에 참조할 수 있다.

### 14. --agent

커스텀 에이전트를 정의하고 실행한다.

### 15. /voice

음성으로 코딩한다. 키보드 없이 자연어 음성 입력으로 작업을 지시할 수 있다.

## 분석

이 목록은 Claude Code가 단순한 CLI 코딩 도구에서 **멀티디바이스, 멀티세션,
멀티에이전트 플랫폼**으로 진화하고 있음을 보여준다. 15개 기능의 스펙트럼이
모바일(1번)에서 대규모 병렬 처리(11번)까지 걸쳐 있다는 것 자체가 제품의
야심을 드러낸다.

특히 주목할 점은 기능의 절반 이상이 "단일 세션, 단일 작업"이라는 기존
CLI 패러다임을 벗어나는 것들이라는 점이다. 텔레포트, 포크, 배치, 스케줄링은
모두 **세션의 시공간적 제약을 해체**하는 방향으로 수렴한다.

## 비평

### Boris Cherny 효과

제품 창시자가 직접 "숨겨진 기능"을 공개하는 행위는 마케팅과 교육의 경계에 있다.
"숨겨진"이라는 프레이밍이 효과적인 이유는, 사용자에게 "나는 이 도구의 10%만
쓰고 있었구나"라는 자각을 유도하기 때문이다. 이는 기능 자체의 가치와 별개로,
사용자의 탐색 의지를 자극하는 전략이다.

### 발견 가능성의 문제

15가지 기능이 "숨겨져" 있다는 것은 곧 발견 가능성(discoverability)의 실패를
의미하기도 한다. 강력한 기능이 있어도 사용자가 모르면 없는 것과 같다. 이 목록이
트윗으로 바이럴되어야 한다는 것 자체가, CLI 도구의 기능 노출 메커니즘이
불충분함을 역설적으로 증명한다.

### 실용성의 편차

텔레포트, /batch, Dispatch 같은 기능은 소수의 파워 유저에게만 유의미하다.
대부분의 사용자에게 일상적으로 가치 있는 기능은 /btw, Hooks, 세션 포크 정도다.
15라는 숫자가 주는 인상과 실제 일상적 활용도 사이에 간극이 있다.

## 인사이트

### 도구의 표면적과 깊이의 역설

CLI 도구는 GUI와 달리 모든 기능이 문자열 뒤에 숨어 있다. 이것은 단점이자
장점이다. 단점은 발견 가능성의 부재. 장점은 **숙련도에 따른 능력 곡선이
가파르다**는 것이다. 같은 도구를 쓰더라도 초보자와 전문가의 생산성 차이가
극단적으로 벌어진다. 이 15가지 목록은 그 곡선의 기울기를 가시화한 것이다.

vim이 30년간 살아남은 이유도 같은 구조다. 표면적으로는 단순하지만, 깊이
들어가면 거의 무한한 조합이 가능하다. Claude Code가 이 경로를 밟고 있다면,
장기적 사용자 충성도에는 유리하지만 온보딩 장벽은 높아진다.

### 세션이 아니라 워크플로가 단위다

텔레포트, 포크, 스케줄링, 배치를 관통하는 공통 주제는 **"세션"이라는 단위의
해체**다. 전통적 CLI 사용은 "터미널을 열고, 작업하고, 닫는다"는 단일 세션
모델이다. Claude Code는 세션을 디바이스 간 이동시키고, 분기시키고, 예약하고,
수천 개로 복제한다.

이것은 작업의 기본 단위가 "세션"에서 "워크플로"로 전환되고 있음을 의미한다.
워크플로는 여러 세션에 걸쳐 있고, 여러 디바이스를 오가며, 시간적으로도
비동기적이다. 개발자의 작업 모델 자체가 바뀌는 것이다.

### /batch의 전략적 함의

수천 개의 에이전트를 병렬 팬아웃하는 /batch는 단순한 편의 기능이 아니다.
이것은 **개인 개발자에게 조직 규모의 실행력을 부여하는** 기능이다. 대규모
마이그레이션, API 버전 업그레이드, 코드 스타일 일괄 변경 같은 작업은
전통적으로 팀 단위의 스프린트가 필요했다. /batch는 이것을 한 사람이 한 번의
명령으로 처리할 수 있게 만든다.

이것의 2차 효과는 흥미롭다. 대규모 변경을 "쉽게" 할 수 있게 되면,
이전에는 "비용 대비 가치가 낮아서" 방치되던 기술 부채 청산이 경제적으로
합리적인 선택이 된다. 도구의 능력이 의사결정의 경제학을 바꾸는 것이다.

### 음성 코딩이 열어젖히는 접근성의 문

/voice가 15번째, 마지막에 배치된 것은 우연이 아닐 수 있다. 가장 실험적이고
가장 미래지향적인 기능이기 때문이다. 음성 코딩은 "키보드를 안 쳐도 된다"는
편의의 문제가 아니다. 손을 쓸 수 없는 상황—이동 중, 요리 중, 산책 중—에서도
코딩 세션을 유지할 수 있다는 것은 **개발이라는 활동의 물리적 제약을 근본적으로
변경**한다.

더 중요한 것은 접근성(accessibility) 측면이다. RSI(반복성 긴장 손상)로
키보드를 장시간 사용하기 어려운 개발자, 시각 장애가 있는 개발자에게 음성
인터페이스는 생산성 도구가 아니라 **직업 유지의 수단**이 될 수 있다.

### 기능의 조합이 만드는 창발적 워크플로

15개 기능 각각도 강력하지만, 진짜 가치는 조합에서 나온다. 모바일에서 /voice로
아이디어를 시작하고(1+15), 텔레포트로 데스크탑에 이어받아(2) Plan 모드로
구체화한 뒤, /batch로 수천 개의 워크트리에 팬아웃하고(11), /schedule로 야간에
테스트를 돌리는(3) 워크플로를 상상해보라. 한 사람이 잠자는 동안에도 수천 개의
에이전트가 일하는 구조다.

이런 조합적 워크플로는 문서화되지 않는다. 사용자가 스스로 발견해야 한다.
Claude Code의 진짜 경쟁력은 개별 기능이 아니라, 기능 간 조합의 표면적이
얼마나 넓은가에 달려 있다.
