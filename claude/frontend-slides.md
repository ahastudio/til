# Frontend Slides: Claude Code로 HTML 프레젠테이션 생성하는 스킬

<https://github.com/zarazhangrui/frontend-slides>

## 소개

Frontend Slides는 Claude Code에서 `/frontend-slides` 명령으로 실행하는 스킬로,
HTML/CSS/JS 기반의 애니메이션이 풍부한 웹 프레젠테이션을 생성한다.
GitHub 별 19,000개 이상, 포크 1,500개 이상을 기록하고 있다.
비디자이너가 CSS나 JavaScript 지식 없이도 완성도 높은 웹 슬라이드를 만들 수 있도록
설계됐다.

핵심 특징은 다섯 가지다.
의존성 없음(Zero Dependencies)—npm, 빌드 도구, 프레임워크 없이 단일 HTML 파일에
CSS/JS를 인라인으로 포함한다.
시각적 스타일 발견(Visual Style Discovery)—취향을 언어로 설명하는 대신, 생성된
3가지 미리보기 중 하나를 선택하는 방식을 쓴다.
PPT 변환—기존 PowerPoint 파일을 이미지와 콘텐츠를 보존하며 웹 슬라이드로 변환한다.
Anti-AI-Slop—흔한 AI 결과물 미학(보라색 그라디언트 등)을 의도적으로 피하는
큐레이션된 스타일을 제공한다.
프로덕션 품질—접근성을 고려하고, 16:9 고정 비율, 주석이 달린 커스터마이즈 가능한
코드를 생성한다.

설치는 두 가지 방법을 지원한다.
Claude Code 플러그인 마켓플레이스에서 `/plugin marketplace add
zarazhangrui/frontend-slides` 명령으로 직접 설치하거나, GitHub 저장소를
`~/.claude/skills/frontend-slides`에 클론해 수동 설치할 수 있다.

스타일은 Dark Themes(Bold Signal, Electric Studio, Creative Voltage, Dark Botanical),
Light Themes(Notebook Tabs, Pastel Geometry, Split Pastel, Vintage Editorial),
Specialty(Neon Cyber, Terminal Green, Swiss Modern, Paper & Ink) 12가지 기본
프리셋을 포함한다.
추가로 `beautiful-html-templates`에서 가져온 34가지 Bold Template Pack도 선택적으로
사용할 수 있다.

## 분석

### “보여주기, 설명하지 않기” 인터페이스 원칙

스타일 발견 과정의 핵심은 “show, don't tell”이다.
사용자에게 “어떤 색상을 원하는가”, “어떤 폰트 스타일을 원하는가”를 묻는 대신,
AI가 브리프에서 분위기를 추론해 시각적 미리보기 세 가지를 생성하고 사용자가 고른다.
이것은 언어로 미적 취향을 표현하는 데 어려움을 느끼는 대부분의 사람들에게 효과적인
접근이다.
디자인 툴의 인터페이스 패러다임을 AI 스킬에 적용한 것이다.

### 점진적 템플릿 로딩 전략

Bold Template Pack의 로딩 전략이 설계상 주목할 만하다.
스킬은 먼저 compact index를 읽고, 후보 템플릿의 소형 `preview.md` 카드만 로드해
타이틀 슬라이드 미리보기를 만든다.
사용자가 선택한 하나의 템플릿에 대해서만 전체 `design.md`를 로드한다.
토큰 소비를 최소화하면서 풍부한 선택지를 제공하는 지연 로딩(lazy loading) 패턴이다.
컨텍스트 창 크기가 제한된 AI 에이전트 환경에서 리소스를 효율적으로 활용하는 방식이다.

### 단일 HTML 파일 아키텍처의 함의

의존성 없는 단일 HTML 파일 출력은 기술적 선택이자 배포 철학이다.
결과물을 이메일로 첨부하거나, USB에 담거나, 정적 호스팅에 올리는 데 어떤 추가
도구도 필요하지 않다.
반면, 파일이 커질수록 로딩 시간이 길어지고, 슬라이드 간 공유 리소스를 재사용하지
못하는 비효율이 있다.
이 트레이드오프는 “프레젠테이션 파일은 독립 실행 가능해야 한다”는 원칙을 기술로
구현한 것이다.
PowerPoint가 단일 `.pptx` 파일에 모든 것을 담는 것과 같은 철학이다.

### PPT 변환 기능의 실용적 위치

PPT 변환은 “기존 콘텐츠를 가진 사용자”라는 현실적 사용자 세그먼트를 위한 온램프다.
완전히 새로운 프레젠테이션 작업 방식을 요구하는 대신, 기존 파일을 그대로 가져올
수 있게 한다.
신규 도구가 기존 워크플로우에 통합되는 진입 장벽을 낮추는 전략적 선택이다.

## 비평

### 강점: 진입 장벽 최소화와 즉각적 결과물

비디자이너가 CSS/JS 없이 완성도 있는 슬라이드를 얻을 수 있다는 것이 가장 큰 강점이다.
단일 HTML 파일 출력은 결과물의 휴대성을 높이고, 별도 런타임 환경이 필요 없다.
Anti-AI-Slop 방향성은 AI 생성 콘텐츠가 범람하는 환경에서 차별화 포인트가 된다.
Claude Code 플러그인 마켓플레이스 지원으로 설치 마찰도 최소화했다.

### 한계: Claude Code 생태계 종속성

Claude Code를 사용하지 않는다면 이 스킬은 완전히 접근 불가다.
CLI 기반 인터페이스는 비기술 사용자에게 여전히 진입 장벽이 될 수 있다.
생성 결과의 품질이 Claude 모델의 버전과 프롬프트 해석에 따라 달라질 수 있어
재현성이 보장되지 않는다.
34가지 Bold Template Pack이 외부 저장소(`beautiful-html-templates`)에 의존하므로,
그 저장소의 변경이 스킬에 영향을 줄 수 있는 외부 의존성 위험이 있다.

### 보완이 필요한 관점: 슬라이드 수정 워크플로우

생성 이후 특정 슬라이드만 수정하거나 콘텐츠를 교체하는 워크플로우가 명확하지 않다.
HTML 파일을 직접 편집하거나 스킬을 재실행하는 것이 유일한 선택인지,
특정 슬라이드만 재생성하는 인크리멘털 편집이 가능한지가 문서에서 불분명하다.
또한 “프로덕션 품질”이라고 주장하지만, 생성된 코드의 접근성 검증이나 성능 기준이
구체적으로 명시되어 있지 않다.

## 인사이트

### 바이브 코딩의 방향 전환: 기능에서 심미성으로

Frontend Slides가 등장한 맥락은 바이브 코딩(vibe coding)의 확산이지만,
그 방향은 기존과 다르다.
기존 바이브 코딩 도구들은 주로 “동작하는 코드”를 빠르게 생성하는 데 초점을 맞췄다.
Frontend Slides는 “아름다운 결과물”을 생성하는 데 초점을 맞춘다.
이것은 AI 코딩 도구의 다음 전선이 기능 구현에서 심미적 품질로 이동하고 있음을
시사한다.

“Anti-AI-Slop”이라는 명시적 포지셔닝이 이를 뒷받침한다.
이미 AI 생성 콘텐츠의 미적 균질화가 시장에서 감지될 만큼 심각해졌다는 의미다.
보라색 그라디언트, 산세리프 제목, 진부한 아이콘—이것들이 AI 결과물의 시각적
클리셰가 됐다.
그 클리셰를 피하겠다는 기치를 내걸고 큐레이션된 스타일을 제공하는 것은,
AI 도구가 단순히 “생성”을 넘어 “큐레이션”으로 가치를 이동시키는 패턴의 선례다.
다음 세대의 AI 생성 도구 경쟁은 생성 속도나 다양성이 아니라, 큐레이션의 질로
결판날 것이다.

### 스킬 생태계로서의 Claude Code: 앱스토어 효과

Frontend Slides가 Claude Code 플러그인 마켓플레이스를 통해 배포된다는 사실은,
Claude Code가 단순한 코딩 도구를 넘어 스킬 생태계 플랫폼으로 진화하고 있음을 보여준다.
이것은 iOS 앱스토어나 VS Code 마켓플레이스와 동일한 플랫폼 전략이다.
핵심 플랫폼이 서드파티 개발자의 생태계를 품으면, 플랫폼의 가치는 플랫폼 자체가
아니라 그 위에 쌓인 스킬의 다양성과 질로 결정된다.

Frontend Slides처럼 Github 별 19,000개를 받은 스킬이 등장하면, 이것은 Claude Code
자체의 채택을 견인하는 킬러 스킬이 된다.
과거 Homebrew가 macOS를 개발자 친화적으로 만든 것처럼, 커뮤니티가 만드는 스킬이
AI 코딩 도구의 실용적 가치를 결정하게 된다.
이 생태계가 성장할수록 네트워크 효과가 발생하고, Claude Code를 떠나는 전환 비용이
높아진다.
스킬 생태계는 기능 경쟁이 아니라 락인(lock-in) 전략이기도 하다.

### “Show, Don't Tell” 인터페이스는 새로운 HCI 패러다임이다

미적 취향을 언어로 표현하는 것은 인간에게도 어렵다.
“현대적이고 세련된 느낌”이라는 설명은 사람마다 다르게 해석되고, AI도 마찬가지다.
Frontend Slides가 택한 “3가지 시각적 미리보기 중 선택” 방식은 이 언어적 모호성
문제를 근본적으로 우회한다.

이것은 AI 인터페이스 설계에서 중요한 선례다.
AI에게 “원하는 것을 언어로 설명하라”고 요구하는 대신, AI가 복수의 해석을 생성하고
인간이 그 중 하나를 선택하는 인터페이스 패턴이다.
이 패턴은 언어적 표현 능력과 기술 지식에 덜 의존하며, 더 많은 사람이 AI 도구를
효과적으로 사용할 수 있게 한다.
디자인 도구, 음악 생성, 건축 설계, 제품 기획 등 심미적 판단이 필요한 모든 도메인에
이 패턴이 적용될 수 있다.
인간이 언어로 명세(specification)를 작성하는 대신 AI가 생성한 결과물 중 하나를
선택하는 방식은, AI와 인간의 협력 방식 자체를 재정의한다.
제안하는 AI와 선택하는 인간—이 분업은 “AI가 인간의 지시를 따른다”는 기존 패러다임을
“AI가 옵션을 열고 인간이 방향을 결정한다”는 패러다임으로 전환한다.
