## 한 줄 요약

Anthropic의 그로스 마케팅 팀은 10개월간 **단 1명**이었고,
그 한 명이 Claude Code와 서브 에이전트, MCP 서버, 메모리 시스템을
조합해 풀 마케팅 팀의 업무를 수행했다.

## 요약

Anthropic(시가총액 3,800억 달러)의 그로스 마케팅은
10개월간 비기술직 1명이 운영했다.
유료 검색, 유료 소셜, 앱스토어, 이메일 마케팅, SEO를
한 사람이 전부 담당했다.

핵심은 Claude Code를 중심으로 구축한 **자동화 루프**다.

### 1단계: 데이터 분석과 카피 생성

기존 광고의 성과 데이터(CTR, 전환율, 지출 등)를
CSV로 추출해 Claude Code에 넣는다.
저성과 광고를 식별한 뒤, 두 개의 특화 서브 에이전트로 분리한다.

- **헤드라인 에이전트**: 30자 제한
- **설명문 에이전트**: 90자 제한

하나의 프롬프트에 다 넣는 대신
각 제약 조건에 맞춘 에이전트를 따로 두어 품질을 높인다.

### 2단계: 크리에이티브 자동화

Figma 플러그인을 만들어서
생성된 헤드라인과 설명문을 템플릿에 자동으로 삽입한다.
배치당 0.5초, 최대 100개의 광고 변형을 생성한다.
수작업으로 프레임 복제하고 텍스트 복사-붙여넣기 하던 시간이 사라졌다.

### 3단계: 성과 모니터링

Meta Ads API에 연결된 MCP 서버를 구축해
Claude가 실시간 캠페인 데이터에 직접 접근한다.
"이번 주 전환율이 가장 높은 광고는?"
"어디서 예산을 낭비하고 있지?" 같은 질문을
대시보드 열지 않고 바로 물어본다.

### 4단계: 학습 루프

모든 가설과 실험 결과를 기록하는 메모리 시스템을 구축했다.
다음 배치를 생성할 때 이전 라운드의 성공/실패를 자동으로 참조한다.
사이클마다 시스템이 학습한다.

### 수치

| 항목             | 이전    | 이후     |
| ---------------- | ------- | -------- |
| 광고 제작 시간   | 2시간   | 15분     |
| 크리에이티브산출 | 기준    | 10배     |
| 테스트 범위      | 제한적  | 다채널   |

## 분석

### 이것은 "AI가 일자리를 대체한다" 서사가 아니다

이 사례를 "AI가 마케터를 대체한다"로 읽으면 핵심을 놓친다.
Austin은 해고당한 팀원이 아니라
**처음부터 혼자였던** 사람이다.
AI가 사람을 대체한 것이 아니라,
한 사람이 AI로 **팀 규모의 업무량을 소화한** 것이다.

이 차이는 중요하다.
"1명이 10명 몫을 했다"와 "10명이 잘렸다"는
결과가 같아 보여도 조직적 의미가 완전히 다르다.

### 워크플로 설계가 핵심이다

기술적으로 가장 인상적인 부분은 개별 도구가 아니라
**전체 루프의 설계**다.

`1. 데이터 추출 → 2. 분석 → 3. 생성 → 4. 디자인 적용 → 5. 배포 → 6. 성과 측정 → 7. 학습 → 다시 1로.`

각 단계는 단순하지만, 이것을 하나의 순환 시스템으로 엮은 것이
진짜 레버리지다.
서브 에이전트 분리, MCP 서버, 메모리 시스템은
모두 이 루프를 **자동으로 돌리기 위한** 부품이다.

### 비기술직이라는 점

트윗에서 "non-technical person"이라고 강조한다.
이것은 Claude Code의 접근성을 보여주는 동시에,
**도메인 전문성이 기술력보다 중요하다**는 점을 시사한다.
어떤 광고가 저성과인지, 어떤 헤드라인이 전환을 이끄는지,
어떤 채널에 예산을 투입할지를 판단하는 것은
코딩 능력이 아니라 마케팅 감각이다.

## 비평

### 생존자 편향

이 트윗은 성공 사례만 보여준다.
Anthropic이라는 맥락이 있다.
자사 제품을 자사 업무에 쓴 것이고,
Claude Code 팀과의 거리가 가까우며,
문제가 생기면 즉시 피드백 루프가 작동한다.
**일반 기업에서 동일한 결과를 기대하기는 어렵다.**

### 품질 vs. 양

10배의 크리에이티브 산출이 10배의 성과를 의미하지는 않는다.
광고 변형이 많아도 본질적으로 다른 메시지가 아니라면
A/B 테스트의 탐색 공간만 넓어질 뿐이다.
"더 많은 변형"이 "더 나은 광고"인지는 별개의 문제다.

### 그로스 마케팅 ≠ GTM

Austin 본인이 강조했듯이,
그로스 마케팅은 전체 GTM(Go-To-Market)과 다르다.
Anthropic의 마케팅 조직 전체가 1명인 것은 아니다.
브랜드, PR, 콘텐츠, 이벤트, 파트너십 등은 별도 팀이 있을 것이다.
트윗의 바이럴 제목("$380B company, one person")은
의도적으로 과장된 프레이밍이다.

### MCP와 메모리 시스템의 한계

MCP 서버로 Meta Ads API에 접근하는 것은 인상적이지만,
이것이 **의사결정의 질**을 얼마나 높이는지는 불명확하다.
"전환율 높은 광고"를 찾는 것은 대시보드로도 가능하다.
진짜 가치는 "왜 이 광고가 전환율이 높은가"에 대한
인과적 분석인데, 현재 LLM이 이것을 잘 하는지는 의문이다.

메모리 시스템도 마찬가지다.
"이전에 뭐가 됐고 안 됐는지"를 기억하는 것과
"다음에 뭘 해야 하는지"를 아는 것은 다르다.
상관관계 기반 학습이 인과적 통찰로 이어지려면
사람의 해석이 여전히 필요하다.

## 인사이트

### 1. AI 레버리지의 단위는 "도구"가 아니라 "루프"다

개별 AI 도구(서브 에이전트, MCP, 메모리)는 부품이다.
진짜 레버리지는 이 부품들을 **피드백 루프**로 엮을 때 생긴다.
데이터 → 분석 → 생성 → 실행 → 측정 → 학습의 순환을
자동화하면, 사이클 속도가 곧 경쟁 우위가 된다.

### 2. "서브 에이전트 분리" 패턴은 범용적이다

헤드라인 30자, 설명문 90자로 에이전트를 분리한 것은
단순하지만 효과적인 패턴이다.
하나의 프롬프트에 여러 제약을 넣으면 품질이 떨어지고,
제약별로 에이전트를 분리하면 각각이 더 잘 작동한다.
이 패턴은 광고 카피뿐 아니라
**제약 조건이 서로 다른 모든 생성 작업**에 적용할 수 있다.

### 3. 도메인 전문가 + AI가 최강의 조합이다

이 사례의 주인공은 프로그래머가 아니라 마케터다.
Claude Code가 기술적 장벽을 낮추면서,
**도메인 지식을 가진 사람이 직접 시스템을 구축하는** 시대가 왔다.
중간에 개발자를 거칠 필요 없이
"이 데이터를 이렇게 분석하고, 이런 결과물을 만들어라"를
마케터 본인이 지시하고 실행한다.

### 4. Anthropic의 독파밍(Dogfooding)

자사 제품으로 자사 마케팅을 운영한 것은
가장 강력한 형태의 독파밍이다.
이 사례 자체가 Claude Code의 마케팅이 되는 구조다.
"우리 마케팅 팀이 1명인데 우리 도구로 잘 되고 있어요"는
어떤 고객 사례 연구보다 설득력 있다.

### 5. 조직 구조에 대한 질문

한 사람이 풀 팀의 업무를 수행할 수 있다면,
**팀의 정의 자체가 바뀌어야** 하는 것 아닌가?
전통적 마케팅 팀은 역할별로 사람을 배치한다.
AI 시대에는 **워크플로별로 시스템을 설계하고**,
각 시스템의 오케스트레이터가 1명이면 충분할 수 있다.
이것은 "인원 감축"이 아니라 "조직 설계의 패러다임 전환"이다.
