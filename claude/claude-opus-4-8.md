# Anthropic, Claude Opus 4.8 출시

원문: <https://www.anthropic.com/news/claude-opus-4-8>

## 요약

Anthropic이 Claude Opus 4.7의 점진적 업그레이드인 Claude Opus 4.8을
출시했다.
코딩, 에이전트 추론, 판단력 전반이 향상됐으며, 특히 정직성(honesty)이
강화돼 자기 코드의 결함을 놓치는 빈도가 Opus 4.7 대비 약 4배 줄었다.
다단계 작업의 신뢰성과 오류 교정 능력도 개선됐다.

벤치마크에서 에이전트 작업 완수와 도구 호출 효율을 선도하며, 브라우저
자동화 벤치마크인 Online-Mind2Web에서 84%를 기록하고 법률 벤치마크에서
처음으로 10%를 돌파했다.
가격은 Opus 4.7과 동일하게 입력 100만 토큰당 5달러, 출력 100만 토큰당
25달러로 유지된다.
Fast 모드는 입력 10달러·출력 50달러로, 이전 fast 모드보다 약 2.5배 빠르고
3배 저렴해졌다.

새 기능으로는 응답에 들이는 사고 깊이를 조절하는 **Effort control**,
대규모 코딩 작업을 위해 수백 개의 서브에이전트를 병렬 처리하는 **Dynamic
workflows**(리서치 프리뷰), 그리고 프롬프트 캐싱을 깨지 않고 작업 도중
지시를 갱신할 수 있는 Messages API의 system 항목 지원이 추가됐다.
정렬(alignment) 측면에서도 오정렬 행동 비율이 Opus 4.7보다 상당히 낮아졌고,
사용자 자율성·이익을 존중하는 친사회적 특성이 개선됐다.
Anthropic은 후속으로 Opus급 성능의 저비용 모델과, 일부 조직 대상 보안
테스트 중인 새 지능 등급 "Claude Mythos"를 예고했다.

## 분석

### 점진적 버전업의 무게중심이 "정직성"으로 이동했다

이번 릴리스에서 가장 눈에 띄는 수치는 벤치마크 점수가 아니라 "자기 코드
결함을 놓치는 빈도가 4배 감소"라는 정직성 지표다.
모델의 원초 능력(raw capability)이 이미 높은 수준에 도달한 단계에서,
경쟁 축이 "더 똑똑한가"에서 "자기 출력을 얼마나 믿을 수 있게 검증하는가"로
이동하고 있음을 보여준다.
이는 에이전트가 다단계로 자율 실행될 때 누적되는 오류를 줄이는 데 직결되는
특성으로, 단순 추론 점수보다 실전 신뢰성에 가깝다.

### Dynamic workflows — 하니스를 모델 제품에 내장하다

수백 개 서브에이전트를 병렬로 돌리는 Dynamic workflows는 그동안 외부
스캐폴딩이 담당하던 fan-out 오케스트레이션을 제품 기능으로 끌어들인 것이다.
이는 "Agent = Model + Harness"라는 등식에서, 모델 공급자가 하니스의 일부를
흡수하기 시작했다는 신호다.
[harness engineering 종합 보고서](../agentic-coding/harness-engineering-comprehensive-report.md)가
지적한 "모델이 똑똑해질수록 하니스를 흡수한다"는 긴장이 실제 제품에서
구체화된 사례로 읽을 수 있다.

## 비평

### 강점: 가격 동결과 효율 개선의 결합

성능을 올리면서 가격을 동결하고 fast 모드를 더 싸게 만든 것은, 토큰 비용이
에이전트 운용의 핵심 제약이라는 현실을 정확히 겨냥한다.
서브에이전트를 수백 개 띄우는 워크플로는 토큰을 폭발적으로 소비하므로,
fast 모드 단가 인하는 단순한 가격 정책이 아니라 dynamic workflows를
실용 가능하게 만드는 전제 조건이다.

### 약점: 벤치마크 선택과 실사용 괴리

[GeekNews 댓글](https://news.hada.io/topic?id=29960)에서는 4.7의 문제가
드디어 해결될지 기대하는 반응과 함께, Claude Code에서 여전히 문제를 겪는다는
보고, 점진적 릴리스가 실질 개선인지 마케팅인지에 대한 논쟁, 벤치마크 선택의
자의성에 대한 우려가 동시에 제기됐다.
공급자가 제시하는 벤치마크는 유리한 지표를 고른 것일 수 있으므로, 84%·10%
같은 단일 숫자보다 실사용 신뢰성 보고가 더 설득력 있다.
또한 Claude Mythos 같은 상위 등급의 기능 제약·라이선스 조건이 불투명하다는
점도 지적됐다.

## 인사이트

### 능력 곡선이 평탄해질수록 "정직성"이 차별화 축이 된다

모델 간 원초 능력 격차가 좁혀지는 국면에서, "자기 결함을 놓치지 않는다"는
정직성이 전면에 나선 것은 우연이 아니다.
생성 능력이 충분히 좋아지면 남는 병목은 "이 출력이 정말 맞는가"를 스스로
판정하는 능력이고, 이는 self-verification의 신뢰성 문제와 직결된다.
코드 결함 4배 감소라는 지표는, 모델이 생성자이자 검증자 역할을 겸할 때
검증자로서의 품질이 제품 가치를 좌우하기 시작했음을 시사한다.

이 흐름을 끝까지 밀면, 차세대 경쟁은 "얼마나 창의적으로 만드는가"가 아니라
"얼마나 자기 실수를 신뢰성 있게 잡아내는가"로 수렴한다.
behaviour harness — 의도한 기능을 올바르게 수행하는지 판정하는 계층 — 가
AI 코딩의 마지막 병목이라면, 모델 자체의 자기검증 능력 향상은 그 병목을
모델 내부로 끌어들이려는 시도다.
정직성 지표가 마케팅 카피의 맨 앞에 오는 날, 능력 경쟁은 검증 경쟁으로
전환된 것이다.

### 모델 공급자가 하니스를 흡수하기 시작했다

Dynamic workflows를 제품에 내장한 것은 추상화 계층의 가치 이동이라는
IT 역사의 반복되는 패턴이다.
초기에 외부 도구(LangGraph, 커스텀 오케스트레이터)가 담당하던 멀티에이전트
fan-out을 모델 공급자가 일급 기능으로 제공하면, 하니스 스타트업이 쥐고
있던 차별화의 일부가 commodity화된다.
컴파일러가 발전하며 손수 작성하던 어셈블리 최적화가 사라졌듯, 모델이
오케스트레이션을 내재화하면 그 계층의 외부 스캐폴딩은 가치를 잃는다.

그러나 역사가 주는 다른 교훈은, 표준화된 계층의 바로 위가 새 격전지가
된다는 것이다.
모델이 fan-out을 흡수하면 차별화는 그 위층 — 컨텍스트 큐레이션 전략,
평가 루프, 도메인 특화 워크플로 설계 — 으로 다시 밀려 올라간다.
하니스를 만드는 팀은 모델이 흡수할 만한 범용 기능에 해자를 걸지 말고,
모델이 따라잡기 어려운 도메인 지식과 검증 정책에 투자해야 한다는 실전
교훈이 여기서 나온다.

### "Mythos" 예고는 등급 분화 전략의 신호다

후속으로 예고된 Claude Mythos라는 새 지능 등급은, 단일 플래그십이 아니라
보안·접근 제어로 분화된 등급 체계로 가려는 전략을 드러낸다.
일부 조직 대상 보안 테스트라는 단서는, 최상위 능력이 누구에게나 열린
API가 아니라 심사를 거친 접근으로 게이팅될 가능성을 시사한다.
이는 능력이 곧 위험이 되는 프런티어 영역에서, 능력 향상과 오용 통제를
동시에 다루려는 시도다.

함의는 시장의 분화다.
범용 Opus/Sonnet/Haiku 라인이 commodity 경쟁을 벌이는 동안, 최상위 등급은
별도 트랙으로 분리되어 가격·접근·라이선스가 차등화된다.
사용자 입장에서는 "어떤 모델을 쓰는가"가 단순 성능 선택이 아니라, 접근
권한과 신뢰 등급의 문제로 바뀌는 미래를 예고하는 셈이다.
