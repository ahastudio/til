# Claude Code (유출본) 소스 코드 분석서

> 원문: <https://wikidocs.net/338204>

## 요약

약 1,884개의 TypeScript/React 소스 파일로 구성된 Claude Code 유출 소스를
분석한 문서다.
실제 코드를 인용하지 않고 아키텍처, 기능, 설계 패턴에 집중했다.

핵심 구조:

- **쿼리 엔진(query.ts, 68KB):** 사용자 요청을 반복적 턴으로 처리하는 중추 신경계
- **도구 시스템:** 45개 이상의 내장 도구, Zod 스키마 검증 + 10단계 실행 파이프라인
- **실행 모드:** REPL, 헤드리스, 코디네이터, 브릿지, 어시스턴트/카이로스
- **자동 압축:** 토큰 한계 접근 시 이미지 제거 → 메시지 그룹화 → 서브 에이전트 요약
- **TUI:** React 커스텀 리컨실러로 문자 그리드를 렌더링, 더블 버퍼링과 프레임 스로틀링

## 분석

이 분석서의 가장 큰 가치는 Claude Code를 하나의 소프트웨어 시스템으로서
구조적으로 이해할 수 있게 해준다는 점이다.

**10단계 퍼미션 파이프라인**은 에이전트 안전성의 실제 구현을 보여준다.
도구 조회 → 중단 신호 확인 → 입력 검증 → 프리훅 → 퍼미션 검증 →
실행 → 결과 매핑 → 대형 결과 디스크 저장 → 포스트훅 → 텔레메트리.
"안전한 에이전트"가 추상적 원칙이 아니라 구체적 게이트의 연쇄임을 확인할 수 있다.

**Bash 도구의 Tree-sitter 파싱**은 주목할 만하다.
명령어의 AST 구조를 분석하여 허용된 구문만 실행하는 "fail-closed" 보안 모델이다.
정규식 기반 필터링보다 훨씬 견고한 접근이다.

**도구 동시성 파티셔닝**도 실용적 설계다.
Read, Grep 같은 안전한 도구는 최대 10개 병렬 실행하고,
Edit, Bash 같은 위험 도구는 순차 실행한다.
이 단순한 분류가 성능과 안전성을 동시에 확보한다.

**8가지 반복 패턴**(제너레이터 스트리밍, 피처 게이트, 메모이즈드 컨텍스트,
보류 후 복구, 지연 임포트, 불변 상태, 인터럽션 복원력, 의존성 주입)은
대규모 에이전트 시스템의 설계 어휘를 제공한다.

작성자(jaehong747)는 "Claude로 Claude Code를 분석했다"고 밝혔는데,
이는 도구가 자기 자신을 설명할 수 있을 때
분석의 정확도가 달라진다는 흥미로운 사례다.

## 비평

유출 코드를 직접 인용하지 않은 것은 윤리적으로 적절한 판단이지만,
검증 가능성을 제한한다.
분석 내용이 실제 구현과 얼마나 정확히 대응하는지
제3자가 확인할 방법이 없다.

문서 작성에 "좋은 모델 + 많은 토큰 + 의지"가 필요했다는 작성자의 언급은
이런 종류의 분석이 아직 자동화되기 어려우며,
인간의 편집 판단이 품질을 좌우한다는 점을 역설적으로 보여준다.

유출본에서 발견된 `/buddy` 기능이 이후 공식 출시되었다는 점은
유출 분석이 단순한 호기심 충족을 넘어
제품 로드맵을 예측하는 데 활용될 수 있음을 시사한다.

## 인사이트

- **"방어적 프로그래밍 90%"라는 수치의 의미.**
  Claude Code의 코드 중 90%가 방어적 프로그래밍이라는 관찰은
  확률적 시스템 위에 결정론적 행동을 구축하는 비용을 정량화한다.
  이는 LLM 기반 제품의 "빙산 아래 복잡성"이며,
  바이브 코딩으로는 절대 도달할 수 없는 엔지니어링 깊이를 보여준다.
  에이전트를 만드는 것과 신뢰할 수 있는 에이전트를 만드는 것 사이의 격차가
  코드량 기준 10배라는 뜻이다.

- **에이전트의 자기 분석 가능성.**
  Claude로 Claude Code를 분석했다는 것은
  LLM이 자신의 런타임 인프라를 이해할 수 있다는 것을 의미한다.
  이는 단순한 편의를 넘어, 에이전트가 자신의 제약을 인식하고
  이를 우회하는 전략을 스스로 제안할 수 있는 가능성을 열어준다.
  자기 참조적 도구 개선의 출발점이다.

- **유출이 만든 역설적 투명성.**
  Anthropic은 Claude Code를 오픈소스로 공개하지 않았지만,
  소스맵 유출로 인해 커뮤니티가 내부 구조를 상세히 분석하게 되었다.
  이 과정에서 10단계 퍼미션 파이프라인, fail-closed Bash 보안 등
  안전성 설계가 오히려 긍정적으로 평가받았다.
  투명성이 신뢰를 구축한다는 오픈소스의 명제가
  의도치 않은 경로로 증명된 셈이다.
