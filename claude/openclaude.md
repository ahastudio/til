# OpenClaude: Claude Code 소스 유출로 탄생한 멀티 모델 포크

원문: <https://github.com/Gitlawb/openclaude>

## 요약

OpenClaude는 2026년 3월 31일 npm 소스맵을 통해 유출된
Claude Code 소스 코드를 기반으로,
Anthropic의 Claude 대신 임의의 LLM 프로바이더를 사용할 수 있게 만든 포크다.
GPT-4o, DeepSeek, Gemini, Llama, Mistral, Ollama 등
OpenAI Chat Completions API 호환 모델이면 무엇이든 연결할 수 있다.

핵심 구현은 `openaiShim.ts`(724줄)라는 번역 레이어다.
Anthropic 메시지 구조를 OpenAI 포맷으로,
tool_use/tool_result를 function call로 변환하며,
나머지 Claude Code 코드는 원본 그대로 동작한다.
총 786줄 추가, 새 의존성 0개로 구현되었다.

Bash 실행, 파일 읽기/쓰기/편집, grep, glob, 웹 검색,
에이전트/태스크 관리, MCP, LSP, 노트북, 메모리 등
Claude Code의 모든 도구가 그대로 작동한다.
슬래시 커맨드(/commit, /review, /compact, /diff, /doctor),
서브에이전트 생성, 비전 모델 지원도 포함된다.

GitHub 스타 7,600개 이상, 포크 2,900개 이상을 기록했다.
다만 라이선스는 "교육 및 연구 목적"으로만 제공되며,
원본 소스는 Anthropic의 이용약관에 종속된다고 명시하고 있다.

## 분석

### 786줄로 프로바이더 잠금을 해제한 구조

Claude Code의 아키텍처가 모델 호출 레이어를
얼마나 깔끔하게 분리해두었는지를 역설적으로 보여준다.
번역 레이어 하나만 끼워넣으면 전체 도구 체인이
다른 모델에서도 작동한다는 것은
Claude Code의 가치가 모델이 아니라
에이전트 프레임워크(도구, UX, 워크플로)에 있다는 증거다.

### GeekNews 댓글에서 드러난 법적·윤리적 논란

GeekNews 댓글에서는 부정적 반응이 지배적이었다.
"유출된 기업 소스 코드 기반이면 금지 아닌가"(kurthong),
"불법 복제 게임 유통과 다를 바 없다"(dbs0829),
"Claude는 등록 상표"(t7vonn) 등
법적·윤리적 우려가 다수를 차지했다.
"바이럴 가능성을 논하면서 윤리를 무시하는 풍조"(sea715)라는
메타 비판도 있었다.

### Extended Thinking 비활성화의 의미

Anthropic 고유 기능인 확장 사고(Extended Thinking),
프롬프트 캐싱, 베타 헤더가 모두 비활성화되어 있다.
이는 Claude Code의 성능 중 상당 부분이
모델 고유 기능에 의존한다는 뜻이기도 하다.
다른 모델로 대체했을 때 도구 호출 품질이
어느 정도 유지되는지는 모델별로 크게 다르다.

## 비평

### 기술적 실행력은 인상적이나 지속 가능성이 없다

786줄로 프로바이더 잠금을 해제한 것은
엔지니어링적으로 깔끔한 작업이다.
그러나 유출된 상업 소프트웨어의 소스 코드를 기반으로 하므로
Anthropic의 법적 대응 한 번이면 프로젝트가 종료된다.
"교육 및 연구 목적"이라는 면책 문구는
상업적 소프트웨어의 무단 복제에 대한 실질적 방어가 되지 못한다.

### "모든 도구가 작동한다"는 주장의 검증 부재

README는 모든 Claude Code 도구가 그대로 작동한다고 하지만,
모델별 도구 호출 성공률이나 품질에 대한 체계적 벤치마크가 없다.
GPT-4o와 DeepSeek-V3만 "Excellent" 등급이고
7B 이하 모델은 "Limited"라고 했지만,
"Excellent"의 기준이 무엇인지 정의되지 않았다.

### 상표권 침해 리스크

프로젝트명에 "Claude"를 포함하고 있어
저작권과 별개로 상표권 침해 이슈가 추가된다.
GeekNews 댓글에서도 지적되었듯이
유사 프로젝트들(Clawdbot → OpenClaw)이
이미 이름 변경을 강제당한 선례가 있다.

## 인사이트

### 에이전트 프레임워크의 가치는 모델이 아니라 도구 체인에 있다

OpenClaude가 증명한 가장 중요한 사실은
Claude Code의 핵심 가치가 Claude 모델 자체가 아니라
그 위에 구축된 에이전트 프레임워크라는 것이다.
파일 편집, Bash 실행, 검색, MCP 통합, 메모리 시스템 등
도구 체인과 오케스트레이션이 실제 생산성을 만든다.

이는 Anthropic의 비즈니스 모델에 대한 질문을 던진다.
모델 접근권을 통해 에이전트 프레임워크의 가치를 포획하는 전략이
번역 레이어 하나로 우회된다면,
장기적으로 프레임워크 자체를 오픈소스화하고
모델 API 소비로 수익화하는 방향이 더 합리적일 수 있다.

실제로 Claude Code가 "모델은 바꿀 수 없지만
프레임워크는 최고"라는 포지셔닝을 유지하려면,
프레임워크의 차별화(도구 품질, UX, 생태계)를
모델 잠금이 아닌 본질적 우위로 만들어야 한다.

### 소스 유출은 "오픈소스 압력"의 극단적 형태다

Llama의 가중치 유출이 Meta의 오픈소스 전략을 가속했듯,
Claude Code 소스 유출은 Anthropic에
에이전트 프레임워크의 오픈소스화 압력을 가한다.
이미 소스가 공개된 상태에서
폐쇄적 전략을 유지하는 것은 비용만 높이고 실익이 줄어든다.

OpenClaude 7,600스타, claw-code 등 파생 프로젝트의 등장은
"Claude Code의 프레임워크를 다른 모델에서 쓰고 싶다"는
시장 수요가 실재함을 보여준다.
Anthropic이 이 수요를 공식적으로 흡수할지,
법적 대응으로 억누를지가 향후 전략의 분기점이 된다.

역사적으로 유출 후 법적 대응만 한 기업보다
유출을 계기로 오픈 전략을 채택한 기업이
생태계 확장에서 더 나은 결과를 얻었다.

### "번역 레이어" 패턴이 AI 도구 시장의 탈중앙화를 가속한다

OpenClaude의 openaiShim.ts는
두 API 규격 사이의 번역기에 불과하지만,
이 패턴이 일반화되면 AI 도구 시장의 구조가 바뀐다.
에이전트 프레임워크가 특정 모델에 종속되지 않게 되면,
사용자는 작업별로 최적의 모델을 자유롭게 선택할 수 있다.

LiteLLM이 API 레벨에서 이미 이 역할을 하고 있고,
OpenClaude는 그것을 완성된 에이전트 제품 레벨로 올린 것이다.
이 흐름이 확산되면 에이전트 도구 회사는
"어떤 모델을 쓰느냐"가 아니라
"얼마나 좋은 도구 체인을 제공하느냐"로만 경쟁하게 된다.
모델 잠금 전략의 수명이 짧아지는 것이다.
