# Claude Code Unpacked — 비주얼 가이드

> 원문: <https://ccunpacked.dev>

## 요약

Claude Code의 유출 소스 코드를 시각적으로 분석한 비공식 프로젝트다.
804개 파일, 22만 줄 이상의 코드, 40개 이상의 명령어, 22개 이상의 도구로 구성된
Claude Code의 내부 구조를 다이어그램과 분류표로 정리했다.

주요 분석 내용:

- **에이전트 루프:** 입력에서 응답 렌더링까지 11단계 프로세스
- **도구 시스템:** 파일 작업, 실행, 검색, 에이전트/태스크, 계획, MCP, 유틸리티, 실험적 기능
- **명령어 카탈로그:** 설정, 워크플로우, 코드 리뷰, 디버깅, 고급 기능 등 40개 이상
- **숨겨진 기능:** Buddy(가상 펫), Kairos(영속 메모리), UltraPlan(30분 실행),
  Coordinator(병렬 워커), Bridge(원격 제어), Daemon(백그라운드),
  UDS Inbox(세션 간 통신), Auto-Dream(활동 회고)

## 분석

이 프로젝트의 가치는 코드 분석이 아니라 시각화에 있다.
22만 줄의 소스를 읽는 것과 구조화된 다이어그램으로 보는 것은
인지 부하에서 차원이 다르다.

GeekNews 댓글에서 제기된 "50만 줄의 90%가 방어적 프로그래밍"이라는 관찰은
LLM 기반 제품의 구조적 특성을 드러낸다.
정규식 패턴, 컨텍스트 정리, 재시도 루프가
확률적 시스템을 결정론적으로 제어하기 위한 비용이다.

다른 에이전트 도구와의 비교 데이터도 흥미롭다.
OpenCode 67만 줄, Codex 72만 줄, Gemini 57만 줄로,
이 규모가 비정상이 아니라 에이전트 시스템의 일반적 복잡도임을 보여준다.

숨겨진 기능 목록은 Claude Code의 로드맵을 간접적으로 드러낸다.
특히 Kairos(영속 메모리 모드), Auto-Dream(활동 회고),
Coordinator(병렬 워커) 등은 에이전트의 자율성을 높이는 방향이며,
향후 공식 출시될 가능성이 높다.

## 비평

유출 코드 기반이라는 태생적 한계가 있다.
작성자도 "유출 몇 시간 만에 사이트를 만들었다"고 밝혔는데,
속도가 정확성을 희생시켰을 가능성이 있다.
Anthropic 공식 문서와 교차 검증이 필요하다.

"클라이언트는 단순한 도구셋을 받고, 서버가 빠르게 혁신한다"는
아키텍처 옹호 의견은 타당하지만,
22만 줄의 클라이언트 코드가 "단순한 도구셋"인지는 의문이다.
서버-클라이언트 경계가 모호한 에이전트 시스템에서
전통적인 아키텍처 논리를 적용하는 것은 한계가 있다.

## 인사이트

- **코드 고고학이라는 새로운 분석 장르.**
  유출 또는 공개된 소스를 시각화하여 제품의 내부 논리를 해독하는 작업이
  하나의 콘텐츠 장르로 자리잡고 있다.
  이는 리버스 엔지니어링의 대중화이며,
  "코드를 읽을 수 있는 사람"과 "아키텍처를 볼 수 있는 사람"의
  차이를 시각화 도구가 메우고 있다.

- **방어적 프로그래밍 90%는 "신뢰 세금"이다.**
  확률적 시스템 위에 결정론적 행동을 보장하기 위해
  전체 코드의 90%를 방어에 쓴다는 것은
  LLM을 신뢰하지 못하는 비용이 코드에 반영된 것이다.
  모델이 더 신뢰할 수 있게 되면 이 90%는 점진적으로 줄어들 것이고,
  그때 에이전트 시스템의 진짜 혁신이 시작된다.

- **숨겨진 기능은 미래의 기본 기능이다.**
  Buddy, Kairos, Auto-Dream, Coordinator 같은 실험적 기능들은
  "에이전트가 감정을 가지고, 기억하고, 반성하고, 협력한다"는
  에이전트 진화의 다음 단계를 스케치하고 있다.
  현재는 숨겨진 기능이지만,
  1-2년 후에는 에이전트의 기본 사양이 될 가능성이 높다.
