# Free Claude Code

원문: <https://github.com/Alishahryar1/free-claude-code>

## 소개

Free Claude Code는 Claude Code CLI와 VSCode 확장이 Anthropic API로 보내는 요청을
중간에 가로채어 다른 LLM 제공업체로 라우팅하는 경량 프록시(proxy)다.
Anthropic API 키 없이도 Claude Code를 사용할 수 있게 해주며,
Python과 FastAPI로 작성되었다.

지원하는 제공업체는 NVIDIA NIM(분당 40회 무료), OpenRouter(수백 가지 모델),
DeepSeek(직접 API), LM Studio(완전 로컬), llama.cpp(로컬 경량 추론 엔진)다.
각 모델 등급(Opus / Sonnet / Haiku)마다 서로 다른 백엔드를 지정할 수 있고,
제공업체도 혼합해서 사용 가능하다.

핵심 동작 방식은 단순하다. Claude Code가 `ANTHROPIC_BASE_URL` 환경 변수를
프록시 서버(기본 포트 8082)로 가리키면, 프록시가 Anthropic 형식의 SSE(Server-Sent
Events) 요청을 받아 각 제공업체의 OpenAI 호환 형식으로 변환한 후 스트리밍으로
응답을 돌려준다. 설정은 환경 변수 두 개(`ANTHROPIC_BASE_URL`, `ANTHROPIC_AUTH_TOKEN`)만
변경하면 되며, Claude Code 자체 코드는 수정하지 않는다.

요청 최적화 기능도 포함한다. 할당량 탐침(quota probe), 제목 생성, 접두어 감지,
제안 모드, 파일 경로 추출 등 5가지 범주의 사소한 API 호출을 로컬에서 가로채어
처리함으로써 실제 API 할당량 소비를 줄인다. 스마트 속도 제한(rolling-window throttle,
429 지수 백오프, 동시 연결 수 제한)도 내장되어 있다.

Discord 및 Telegram 봇 통합도 지원한다. 봇을 통해 Claude Code를 원격으로 제어하고,
트리 구조 스레드 관리, 세션 지속성, 사고 토큰(thinking token) 및 도구 호출 결과의
실시간 스트리밍을 제공한다. 음성 메모 전사(Whisper 로컬 또는 NVIDIA NIM)도 지원한다.

## 분석

### 프록시 아키텍처의 단순함

이 프로젝트의 핵심 통찰은 Claude Code가 `ANTHROPIC_BASE_URL` 환경 변수를 존중한다는
사실에서 시작한다. Anthropic이 공식적으로 허용한 우회 지점을 활용한 것이다.
프록시는 Anthropic API 형식과 OpenAI 호환 형식 사이의 변환 계층 역할을 하며,
Claude Code 자체를 변경하지 않기 때문에 Claude Code 업데이트에 상당히 강건하다.

### 생각 토큰(Thinking Token) 처리

많은 오픈소스 추론 모델이 `<think>` 태그나 `reasoning_content` 필드로 사고 과정을
출력한다. 이 프록시는 이를 Claude의 네이티브 thinking 블록 형식으로 변환하여,
Claude Code가 추론 과정을 인식하고 활용할 수 있게 만든다.
이는 단순한 포맷 변환을 넘어, 외부 모델의 사고 흐름을 Claude Code의 UI에
자연스럽게 통합하는 기능이다.

### 모델별 라우팅과 제공업체 혼합

Opus/Sonnet/Haiku 각 등급에 서로 다른 모델과 제공업체를 매핑할 수 있다.
예를 들어 Opus 요청은 NVIDIA NIM의 고성능 모델로, Haiku 요청은 OpenRouter의
무료 경량 모델로 보내는 식이다. Claude Code가 내부적으로 작업 복잡도에 따라
다른 모델 등급을 선택한다는 점을 역으로 이용한 설계다.

### 하uristic 도구 파서

일부 오픈소스 모델은 도구 호출(tool call)을 구조화된 JSON 대신 평문 텍스트로
출력한다. 이 프록시는 휴리스틱 파서를 내장하여 텍스트로 출력된 도구 호출을
Claude Code가 기대하는 구조화된 형식으로 자동 변환한다. 이는 도구 사용 능력이
완전하지 않은 모델과의 호환성을 높이는 실용적인 접근법이다.

### Discord/Telegram 봇을 통한 원격 에이전트

로컬 프록시를 Discord/Telegram 봇과 연결하면, 메신저에서 Claude Code 세션을
원격으로 제어하는 자율 코딩 에이전트가 된다. 트리 기반 스레딩으로 대화 분기를
관리하고, 서버 재시작 후에도 세션을 복원한다. 이는 개인 개발 인프라를
챗봇 인터페이스 뒤에 숨기는 흥미로운 패턴이다.

## 비평

### 강점

이 프로젝트의 가장 큰 강점은 Claude Code의 공개된 확장 지점만 활용한다는 점이다.
`ANTHROPIC_BASE_URL` 재지정은 Anthropic이 공식적으로 문서화한 기능이며,
이를 활용하는 것은 기술적으로 정당하다. Claude Code 소스 코드를 수정하거나
패치하지 않으므로 업스트림 업데이트에도 상대적으로 안정적이다.

모듈화 설계도 눈에 띈다. `BaseProvider`와 `MessagingPlatform`을 추상 기반 클래스(ABC)로
정의하여 새 제공업체나 메시징 플랫폼을 추가하기 쉽다. 코드 품질 도구(Ruff,
타입 체커 Ty, Pytest)를 적극적으로 사용하는 점도 오픈소스 프로젝트로서 긍정적이다.

### 약점 및 한계

근본적인 약점은 모델 품질의 불일치다. NVIDIA NIM이나 OpenRouter의 무료 모델은
Claude Sonnet/Opus와 코딩 능력, 도구 사용 신뢰성, 컨텍스트 이해에서 상당한
격차가 있다. 휴리스틱 도구 파서는 이 격차를 부분적으로 메우지만, 복잡한
다단계 에이전트 작업에서는 실패 빈도가 높아질 수 있다.

또한 이 프로젝트는 “무료”라는 말에 함의된 것보다 더 많은 설정 과정을 요구한다.
NVIDIA NIM 키 발급, 프록시 서버 실행, 환경 변수 설정 등 기술적 역량이 필요하며,
일반 사용자에게는 진입 장벽이 존재한다. 서버가 항상 실행 중이어야 한다는 점도
간과하기 쉬운 운영 비용이다.

서비스 약관(ToS) 측면의 회색 지대도 있다. NVIDIA NIM과 OpenRouter 모두
프록시를 통한 API 재판매나 상업적 무제한 사용을 명시적으로 허용하지 않을 수 있다.
개인 사용에는 문제가 없겠지만, 팀 단위로 공유하거나 상업적으로 활용할 경우
제공업체의 이용 정책을 별도로 확인해야 한다.

## 인사이트

### AI 도구의 “API 호환성”이 새로운 상품화 축이 되다

이 프로젝트는 Claude Code라는 UI 레이어와 Anthropic 추론 백엔드 사이의 틈새를
정확히 공략한다. Claude Code가 OpenAI 호환 API를 직접 지원했다면 이 프로젝트는
필요 없었을 것이다. Anthropic이 `ANTHROPIC_BASE_URL` 재지정을 허용하면서도
Claude 모델만 동작을 보장한다고 암묵적으로 가정한 설계 결정이, 역설적으로
이 생태계를 가능하게 했다.

이는 더 넓은 패턴을 보여준다. AI 애플리케이션이 성숙해지면서 UI/UX 레이어와
추론 레이어 사이의 분리가 일어나고, 그 틈새에서 호환성 레이어, 라우터, 게이트웨이
프로젝트가 번성한다. LiteLLM, LM Studio, Ollama, 그리고 이 프로젝트 모두 같은
축을 따라 작동한다. 향후 기업용 AI 인프라에서 “어떤 모델이 실제로 실행되는가”는
“어떤 API 인터페이스를 제공하는가”만큼 중요하지 않을 수 있다.

NVIDIA NIM이 분당 40회 무료 요청을 제공하는 것도 이 구도 속에서 읽혀야 한다.
NVIDIA는 GPU 하드웨어 회사로서 추론 수요를 늘리는 것 자체가 이해관계에 부합한다.
무료 NIM 할당량은 개발자를 NVIDIA 생태계로 끌어들이는 획득 비용(customer acquisition
cost)이며, free-claude-code 같은 프로젝트는 이 전략의 의도치 않은 수혜자다.

### “무료”의 실제 비용은 신뢰성 불확실성이다

40 req/min 무료 할당량은 Claude Code의 전형적인 사용 패턴에서 빠르게 소진된다.
Claude Code는 파일 읽기, 검색, 도구 호출 등 수십 번의 API 요청을 단일 작업에서
발생시킨다. 이를 위해 프록시는 사소한 요청 5가지 범주를 로컬에서 가로채지만,
이 최적화 자체가 Claude Code의 내부 요청 패턴에 대한 역공학을 전제로 한다.
Claude Code 업데이트로 이 패턴이 변경되면 최적화가 무력화될 수 있다.

더 근본적으로, 무료 또는 저가 모델은 코딩 에이전트에게 요구되는 신뢰성
임계값을 충족하지 못하는 경우가 많다. 코드 편집, 터미널 명령 실행, 파일 시스템
조작을 수행하는 에이전트에서 도구 호출 실패나 잘못된 응답은 단순한 불편함이
아니라 데이터 손실이나 시스템 손상으로 이어질 수 있다. 따라서 진정한 비용은
API 요금이 아니라 불안정한 모델로 인한 작업 재시작, 오류 수정, 신뢰 저하다.
“무료”는 화폐 비용의 절감이지, 총 소유 비용(TCO)의 절감이 아니다.

### 오픈 인터페이스가 폐쇄형 생태계보다 더 큰 가치를 창출한다

이 프로젝트가 GitHub에서 6,000개 이상의 스타를 받은 사실은 단순한 비용 절감
욕구를 넘어서는 무언가를 시사한다. 개발자들은 자신이 사용하는 도구의 내부를
이해하고, 교체하고, 확장할 수 있는 능력을 원한다. Claude Code가 아무리 훌륭해도
불투명한 단일 백엔드에 묶여 있다면 일부 개발자는 본능적으로 탈출구를 찾는다.

이는 Anthropic에게 역설적인 신호다. Claude Code의 인기가 이 프로젝트 수요를
만들었고, 이 프로젝트는 다시 Claude Code의 보급 확대에 기여한다. Claude Code
인터페이스가 다양한 백엔드 위에서 작동할수록, Claude Code 자체가 업계 표준
코딩 에이전트 인터페이스로 자리잡을 가능성이 높아진다. Anthropic이 서드파티
모델 라우팅을 공식 지원한다면, 오히려 Claude Code를 플랫폼으로 확장하는
전략적 기회가 될 수 있다. VS Code가 특정 언어 서버에 종속되지 않고 LSP(Language
Server Protocol) 표준 위에 번성했듯이.
