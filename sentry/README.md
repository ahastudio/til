# Sentry

> Application Performance Monitoring & Error Tracking Software

<https://sentry.io/>

Sentry는 애플리케이션의 에러를 실시간으로 추적하고 모니터링하는
오픈소스 플랫폼입니다.
프로덕션 환경에서 발생하는 예외를 자동으로 수집하고,
에러 발생 맥락(스택 트레이스, 사용자 정보, 브라우저 정보 등)을
상세하게 기록해 줍니다.

## 주요 기능

### 에러 추적(Error Tracking)

- 실시간 에러 알림
- 스택 트레이스와 breadcrumbs로 에러 발생 경로 추적
- 에러 발생 빈도 및 영향받은 사용자 수 통계

### 성능 모니터링(Performance Monitoring)

- 트랜잭션별 응답 시간 측정
- 느린 API 호출 및 데이터베이스 쿼리 탐지
- 프론트엔드 렌더링 성능 분석

### 릴리스 추적(Release Tracking)

- 배포 버전별 에러 발생 추이 비교
- 새 배포로 인한 회귀 버그 조기 발견

## Self-Hosted vs SaaS

Sentry는 두 가지 방식으로 사용할 수 있습니다.

**Self-Hosted** - 오픈소스 버전을 직접 서버에 설치하여 운영.
데이터를 자체 서버에서 관리할 수 있습니다.

**SaaS** - Sentry에서 제공하는 클라우드 서비스 이용.
인프라 관리 부담 없이 바로 사용 가능하며, 무료 플랜도 제공됩니다.

## 장점

- 에러를 놓치지 않고 실시간으로 파악
- 사용자가 신고하기 전에 문제를 선제적으로 인지
- 에러 재현에 필요한 충분한 컨텍스트 제공
- 팀 협업을 위한 이슈 할당 및 코멘트 기능
- CI/CD 파이프라인과 통합 가능

Sentry를 사용하면 프로덕션 환경의 안정성을 크게 높일 수 있습니다.
