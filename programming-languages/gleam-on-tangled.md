# Gleam Is Now on Tangled

원문: <https://tangled.org/gleam.run/gleam>

HN 토론: <https://news.ycombinator.com/item?id=48959143> (244점, 153개 댓글)

## 요약

Gleam 언어 프로젝트가 GitHub에서 Tangled로 이동했다.
Tangled는 탈중앙화 식별자(DID)를 기반으로 한 분산형 Git 호스팅 플랫폼으로,
기존 중앙화된 Git 서비스에 대한 대안을 제시한다.

Gleam은 타입 안전한 확장 가능 시스템 구축을 위한 언어로,
기업 소유가 아니라 커뮤니티 후원으로 운영된다.
Tangled로의 이동은 이 커뮤니티 중심 운영 원칙과 일치하는 결정으로 보인다.
저장소는 현재 Rust 92.9% 비중으로 구성돼 있으며 11,000개 이상의 커밋이 쌓여 있다.

## 분석

### 탈중앙화 플랫폼의 실제 사용성 문제

HN 토론에서 Tangled 가입 경험에 관한 구체적인 불편함이 제기됐다.
pbjerkeseth는 패스워드 매니저 비호환성, 비직관적인 인증 흐름,
사용자 이름으로 직접 로그인이 안 되는 구조, 낯선 소셜 URL 프로토콜 요구 등을 꼽았다.[^pbjerkeseth]
프로젝트 자체에 관심이 있음에도 가입 과정에서 이탈할 뻔했다는 피드백이다.

탈중앙화 플랫폼은 기술적 참신함과 사용성 사이의 긴장을 항상 안고 있다.
DID 기반 인증이 기존 이메일+패스워드보다 장기적으로 더 강건한 구조일 수 있지만,
기존 개발자 도구에 익숙한 사용자에게 새 패러다임을 설득하는 과정이 필요하다.

### 오픈소스 커뮤니티가 플랫폼 다각화를 선택하는 이유

Gleam만이 아니다.
Codeberg, Sourcehut, Tangled 등으로 이동하는 프로젝트가 늘고 있다.
GitHub Copilot이 공개 저장소 코드를 학습 데이터로 사용하는 것에 대한 불편함,
Microsoft 소유권에 대한 거버넌스 우려, 단일 플랫폼 집중에 대한 위험 분산이
복합적으로 작용하는 것으로 보인다.

Gleam이 “기업이 소유하지 않은” 언어를 강조하는 만큼,
기업 소유의 플랫폼에 의존하는 것이 프로젝트 정체성과 일치하지 않는다는 판단도 있었을 것이다.

## 비평

### 이동의 실제 비용이 드러나지 않는다

저장소 이동은 코드만의 문제가 아니다.
이슈 추적, PR 워크플로, CI 통합, 의존성 패키지 등록, 기여자 온보딩 경로가 모두 영향받는다.
발표 페이지는 이 전환 비용을 거의 다루지 않는다.

외부 의존 프로젝트가 패키지를 Gleam에서 가져간다면
저장소 URL 변경이 빌드 파이프라인에 영향을 줄 수 있다.
생태계 크기가 작은 언어일수록 이런 이동이 기여자를 혼란스럽게 할 위험이 있다.

## 인사이트

### 언어 커뮤니티의 플랫폼 선택이 생태계 신호가 된다

언어가 어디서 호스팅되느냐는 그 언어가 어떤 가치를 지향하는지 보여준다.
Gleam이 탈중앙화 분산 플랫폼을 선택한 것은
Rust, Go, Elixir 같은 기성 언어가 GitHub를 당연히 여기는 것과 다른 길을 택했다는 의미다.

이 선택이 Gleam 사용자 커뮤니티에게 어떻게 받아들여지는지,
새 기여자 유입에 실제 장벽이 생기는지, 아니면 오히려 공유 가치를 가진 기여자를 모으는지가
몇 년 뒤에 평가될 흥미로운 실험이다.
