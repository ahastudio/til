# C++: 다큐멘터리 — 세계에서 가장 중대한 프로그래밍 언어

원문: <https://herbsutter.com/2026/06/04/c-the-documentary-released-today/>

HN 토론: <https://news.ycombinator.com/item?id=48408016> (392점, 286개 댓글)

## 요약

Herb Sutter가 *C++: The Documentary*의 공개를 발표했다.

[The Story of C++: The World's Most Consequential Programming Language | The Official Story](https://www.youtube.com/watch?v=lI7tMxzSJ7w)

68분 분량의 이 다큐멘터리는
AT&T 벨 연구소에서의 탄생부터 현재의 전 세계적 채택까지 40년에 걸친 C++의 진화를
연대기적으로 다룬다. Bjarne Stroustrup(C++ 창시자), Alexander Stepanov(STL 설계자),
John Romero(Doom/Quake 공동 창작자), Chris Lattner(LLVM/Swift 창시자), Brian
Kernighan(C 언어 공동 저자) 등 14명의 핵심 인물들이 출연한다.

다큐멘터리는 2025년 3분기 데이터를 인용하며 C++가 “상위 4개 언어 중 가장 빠르게
성장하는 언어”이고 지난 3.5년간 사용자가 +90% 증가했다고 밝힌다. 21개의 챕터가
발명, 표준화, “C++ 겨울”, 현대 C++, 미래까지의 여정을 구조적으로 커버한다.

GeekNews에 공유된 다큐멘터리에 대한 반응은 복합적이다. C++의 역사적 궤적을 잘
담아낸 점은 높이 평가받지만, Zortech C++의 DOS 채택 기여나 Andrei Alexandrescu의
현대 C++ 설계에 대한 더 깊은 탐구를 원하는 의견도 있었다. 또한 메모리 안전성 문제가
여전히 해결되지 않은 점을 비판하는 시각과, 다큐멘터리가 C++의 성취를 지나치게
찬양적으로 다루며 비판적 균형을 잃었다는 평가도 공존했다.

## 분석

### C++ 르네상스의 역설

다큐멘터리가 강조하는 +90% 성장이라는 수치는 흥미롭다. C++는 오랫동안 “사멸
예정 언어”로 분류되었다. Java와 C#의 등장, 2000년대 CPU 주파수 상승으로
성능 이점이 줄면서 채택이 감소했다. 그러나 멀티코어 시대와 AI 가속 컴퓨팅(CUDA)의
부상이 성능의 중요성을 다시 전면에 세웠다. 하드웨어를 직접 제어하는 언어의 가치는
추상화가 극한까지 올라간 환경에서 역설적으로 더 부각된다.

ElenaDaibunny[^ElenaDaibunny]의 반응 — “C++가 현재 상위 4개 언어 중 가장 빠르게
성장하고 있다는 게 믿기 어렵다” — 은 C++ 성장을 낯설게 바라보는 외부 시각을
대표한다. garyrob[^garyrob]은 대조적인 사례를 공유했다. 25년간 C++를 쓰지 않은
상태에서 AI를 사용해 하루 만에 9,000줄짜리 커스텀 리마인더 앱을 완성했다. AI
도구가 “쓰기 어렵지만 사용하기 좋은” C++의 접근성 문제를 다른 방향에서 해결하고
있다는 역설이다.

### 표준화의 무게

527명으로 구성된 ISO C++ 표준 위원회라는 수치는 언어의 규모와 거버넌스 복잡성을
동시에 보여준다. C++26의 경계 검사와 리플렉션 기능은 메모리 안전성 우려에 대한
응답이지만, 위원회 규모가 의사결정의 속도와 일관성에 미치는 영향은 지속적인
과제다. Rust의 등장은 C++가 수십 년간 해결하지 못한 메모리 안전성 문제를 언어
설계 수준에서 접근한 직접적인 도전이다.

## 비평

### 다큐멘터리의 성취와 한계

40년의 기술 역사를 68분에 담는 것은 압축을 요구한다. Ken Thompson이 “C++는 일관성
없고 복잡하다”고 비판한 것처럼, 언어의 그림자 — 복잡성 증가, 컴파일 시간, 학습
곡선 — 에 대한 균형 잡힌 시각이 충분하지 않을 수 있다. 찬양적 다큐멘터리는
C++ 커뮤니티 내부의 자기 이해를 강화하지만, 외부에서 바라보는 비판적 시각을
포용하지 못한다. mirmor23[^mirmor23]은 Thompson의 비판이 여전히 공명한다고 지적하며,
현대 C++ 숙달에 여러 권의 전문 서적이 필요한 언어는 그 자체로 문제가 있다고 주장한다.
orsenthil[^orsenthil]은 다른 종류의 빈틈을 지적한다. 게임 엔진의 C++ 역사를 다루면서
John Romero 대신 John Carmack을 포함시키지 않은 이유에 의문을 제기한다. 동일한
역사에서 누가 “대표 인물”로 선택되는지는 서사 구성의 편향을 드러낸다.

### 성장 수치의 해석

+90% 성장이라는 수치는 맥락이 필요하다. 이것이 절대적 개발자 수 증가인지, 상대적
비율 변화인지, 특정 도메인에 집중된 것인지에 따라 함의가 달라진다. AI/ML 인프라와
게임 개발 분야의 성장이 전체 수치를 끌어올리는 구조라면, C++의 “르네상스”는
도메인 특화된 현상일 수 있다.

## 인사이트

### 언어의 수명은 생태계가 결정한다

C++가 “사멸”하지 않은 이유는 언어 자체의 우수성보다 그것이 깔려 있는 생태계
때문이다. 게임 엔진, 금융 시스템, 물리 시뮬레이션, AI 가속기 드라이버 — 이 모든
곳에서 수십 년간 C++로 작성된 코드가 돌아가고 있다. 새로운 언어가 등장하더라도
이 레거시를 재작성하는 비용은 전환을 억제한다. 기술의 수명은 기술 자체의 품질이
아닌 기존 코드와 지식의 누적, 즉 경로 의존성에 의해 결정된다.

### 메모리 안전성 문제는 기술이 아닌 거버넌스 문제다

Rust가 컴파일러 수준에서 해결한 메모리 안전성 문제를 C++는 40년간 해결하지 못했다.
이것이 기술적 불가능 때문이 아님은 C++26의 경계 검사 기능이 증명한다. 문제는
527명의 위원회가 어떤 트레이드오프를 어떤 속도로 수용하느냐의 거버넌스 문제였다.
하위 호환성을 보존하면서 언어를 안전하게 만드는 것은 기술적 어려움과 함께 수백만
명의 기존 코드를 어떻게 대할 것인가의 사회적 결정이다.

### 다큐멘터리가 포착한 것과 놓친 것

Bjarne Stroustrup, Alexander Stepanov, Brian Kernighan 같은 인물들의 인터뷰는
언어의 지적 계보를 가시화한다. 기술은 개인이 만들고, 그 개인들의 판단과 취향이
수백만 명의 작업 방식을 형성한다. 그러나 다큐멘터리가 놓치기 쉬운 것은 익명의
기여자들이다. 표준 위원회 바깥에서 오픈소스 라이브러리를 만들고, 버그를 리포트하고,
강의를 작성한 수많은 이름 없는 개발자들의 누적이 C++ 생태계를 유지해왔다.
역사는 영웅 서사로 압축되지만, 실제 기술의 생존은 집단적 유지보수의 산물이다.
jdw64[^jdw64_cpp]는 C++의 모순을 날카롭게 포착한다. “C++로 만들어진 게임이나
프로그램은 성능이 보장되어 좋지만, C++를 직접 써야 한다고 하면 눈물이 난다.” 사용하기
좋은 결과물과 쓰기 고통스러운 과정 사이의 이 간극이 C++의 정체성을 한 문장으로
요약한다. GodelNumbering[^GodelNumbering]은 반론을 제시한다. “비트 단위까지
정밀한 정신 모델을 원하는 시스템화 성향의 사람들에게는 C++를 능가하는 언어가 없다.”
같은 언어에 대해 극단적으로 다른 경험을 갖는 사람들이 공존한다는 사실이 C++의
가장 이상한 특징이다.
