# 3DCellForge: AI 기반 3D 세포 시각화 스튜디오

<https://github.com/huangserva/3DCellForge>

## 소개

3DCellForge는 브라우저에서 생물학적 세포 모델을 3차원으로 탐색할 수 있는
인터랙티브 웹 애플리케이션이다.
React와 Three.js를 기반으로 구축됐으며, WebGL 실시간 렌더링으로 세포 구조를
시각화하고 조작하는 인터페이스를 제공한다.
핵심 기능은 현미경 참조 이미지에서 AI로 3D 모델을 생성하는 파이프라인이다.

## 명세

### 기술 스택

React, Vite, Three.js, React Three Fiber, Drei, Framer Motion으로
구성된 JavaScript 프론트엔드다.
선택적으로 Node.js 백엔드를 추가할 수 있으며, API 키를 서버 사이드에 보관해
프론트엔드 노출을 방지한다.

### 3D 생성 파이프라인

두 가지 이미지-to-3D 변환 경로를 지원한다.

| 프로바이더  | 유형        | 특징                                         |
| ----------- | ----------- | -------------------------------------------- |
| Tripo API   | 클라우드    | 기본 3D 생성 서비스                          |
| Hunyuan3D   | 로컬        | 백업 생성 수단, 오프라인 지원                |

브라우저 사이드 이미지 릴리프(relief) 처리와 PNG 폴백도 지원하며,
로컬 GLB 파일을 직접 임포트할 수도 있다.

### 주요 기능

- React Three Fiber 기반 인터랙티브 3D 세포 뷰어
- 드래그 회전, 스크롤 줌 오비탈 컨트롤
- 일반 모드와 3D 증명 모드(proof mode) 전환
- 세포소기관 상세 카드, 현미경 참조 패널
- 다중 세포 비교 도구, 메모 기능, 갤러리 액션
- 스크린샷 캡처 및 GLB 파일 내보내기
- 오프라인용 캐시 데모 모델, Khronos glTF 참조 모델

## 분석

### 현미경 이미지-to-3D 파이프라인의 기술적 맥락

전통적인 3D 세포 모델링은 공초점 현미경(confocal microscopy)이나 전자 현미경으로
촬영한 Z-스택(Z-stack) 이미지 시리즈를 쌓아서 3D를 재구성하는 방식이었다.
이 과정은 전문 소프트웨어(Imaris, Amira, Fiji/ImageJ)와 높은 전문성을 요구한다.
3DCellForge가 채택한 AI 이미지-to-3D 접근은 단일 2D 참조 이미지에서
즉시 3D 모델을 생성한다.
정확도보다 접근성과 탐색 속도를 우선시하는 설계 선택이다.

### 클라우드+로컬 이중 프로바이더 전략

Tripo API(클라우드)와 Hunyuan3D(로컬)를 동시에 지원하는 것은 흥미로운
아키텍처 패턴이다.
클라우드 서비스가 다운됐거나 네트워크가 없을 때도 로컬 모델로 기능을 유지한다.
이것은 AI 도구에서 “온라인 우선, 로컬 폴백” 패턴의 구현이다.
Hunyuan3D는 Tencent가 오픈소스로 공개한 3D 생성 모델로, 로컬 GPU에서 실행할 수 있다.

### React Three Fiber 생태계 위에서의 과학 시각화

React Three Fiber는 Three.js를 React 컴포넌트 모델로 추상화한 라이브러리다.
Drei는 그 위에서 카메라 컨트롤, 조명, 환경 맵 같은 3D 씬 구성 요소를
재사용 가능한 훅으로 제공한다.
이 스택이 과학 시각화에 채택된다는 것은 WebGL 기반 3D가 전문 소프트웨어 영역
밖으로 나오고 있음을 보여준다.
glTF/GLB는 Khronos Group이 정의한 3D 에셋 표준으로, 이 선택은 상호운용성을
확보한다.

## 비평

### 강점: 접근성 중심의 과학 시각화

설치 없이 브라우저만으로 3D 세포 모델을 탐색할 수 있다는 점은 교육 맥락에서
강력하다.
대학 생물학 수업이나 과학 커뮤니케이션에서 전문 소프트웨어 라이선스 없이
고품질 시각화를 제공할 수 있다.
MIT 라이선스 오픈소스로 공개한 것은 이 접근성을 극대화하는 선택이다.

### 강점: GLB 내보내기로 열린 에코시스템

생성된 3D 모델을 GLB로 내보낼 수 있으면 Blender, Unity, Unreal Engine,
혹은 다른 과학 시각화 파이프라인에서 활용할 수 있다.
특정 플랫폼에 가두지 않고 표준 포맷으로 데이터 이동성을 보장한다.

### 약점: AI 생성 3D의 과학적 정확도 문제

현미경 이미지 한 장에서 AI로 3D를 생성하는 것은 탐색적 시각화에는 적합하지만,
실제 세포 구조의 정확한 재현이라고 보기 어렵다.
AI가 “그럴듯한” 3D 모델을 만들지만 실제 세포소기관의 위치, 크기 비율, 형태가
정확하다는 보장이 없다.
과학 연구 목적으로 쓰이려면 이 한계를 명확히 표시해야 한다.

### 약점: Tripo API 의존성의 지속 가능성

주요 생성 경로가 클라우드 API에 의존한다는 것은 서비스 종료 또는 가격 정책
변경 시 기능이 제한될 수 있다는 위험이다.
Hunyuan3D 로컬 폴백이 있지만, 로컬 GPU 없이는 실행이 어려울 수 있다.
장기적 지속성을 위한 자급자족 경로가 더 명확했으면 한다.

## 인사이트

### AI 이미지-to-3D가 과학 시각화에 여는 새로운 가능성

과학 시각화의 역사는 계산 비용과 전문성 장벽을 낮추는 방향으로 움직여 왔다.
1990년대 분자 동역학 시뮬레이션은 슈퍼컴퓨터가 필요했고, 2000년대에는
워크스테이션, 2010년대에는 노트북에서 가능해졌다.
이제 브라우저에서 AI로 3D 세포 모델을 생성하는 시대가 됐다.

3DCellForge가 채택한 이미지-to-3D 접근은 과학적 정확도보다 직관적 이해를
우선시한다.
이것은 새로운 트레이드오프다.
전통적 공초점 현미경 Z-스택 재구성은 정확하지만 느리고 비싸다.
AI 기반 단일 이미지 3D 생성은 빠르고 저렴하지만 해석이 필요하다.
두 접근은 경쟁이 아니라 용도가 다르다.
교육과 과학 커뮤니케이션에는 후자가, 연구에는 전자가 적합하다.

장기적으로 이 격차는 좁혀질 것이다.
AI 모델이 더 많은 현미경 이미지로 훈련될수록, 생성된 3D 모델의 생물학적
정확도가 높아질 것이다.
Alphafold가 단백질 구조 예측의 정확도를 실험과 경쟁하는 수준으로 끌어올린
것처럼, AI 기반 세포 3D 재구성도 유사한 경로를 걸을 수 있다.

### 브라우저가 과학 소프트웨어의 새 플랫폼이 되는 이유

Imaris, Amira 같은 전문 3D 세포 시각화 소프트웨어는 수천 달러의 라이선스
비용이 든다.
대부분의 대학 연구실도 전체 팀원이 접근하기 어렵고, 개발도상국 연구자에게는
더욱 높은 장벽이다.
브라우저 기반 도구는 이 장벽을 제거한다.

WebGL과 WebGPU, React Three Fiber 같은 추상화 계층의 성숙으로 브라우저에서
GPU 가속 3D 렌더링이 일상적인 개발 작업이 됐다.
과거에는 OpenGL을 직접 다루거나 전문 3D 엔진을 배워야 했던 기능이
NPM 패키지 몇 개로 구현 가능해졌다.
이것은 생물정보학, 화학, 물리학 연구자가 소프트웨어 엔지니어 없이도
자신의 데이터를 3D로 시각화하는 도구를 만들 수 있다는 의미다.

이 민주화의 부작용은 도구의 과학적 검증 부담이 사용자에게 옮겨간다는 것이다.
전문 소프트웨어는 수년간의 과학적 검증을 거쳐 신뢰를 쌓지만,
오픈소스 웹 도구는 사용자가 직접 결과의 타당성을 평가해야 한다.
이것은 과학 도구 민주화가 풀어야 할 핵심 과제다.

### 이중 AI 프로바이더 아키텍처와 오픈소스 AI의 현실

3DCellForge가 Tripo API(클라우드)와 Hunyuan3D(로컬)를 동시에 지원하는 것은
현재 AI 도구 생태계의 구조적 현실을 반영한다.
고품질 AI 모델은 클라우드 API 형태로 제공되는 경우가 많고, 이는 비용과
가용성에 대한 외부 의존성을 만든다.
동시에 Hunyuan3D 같은 오픈소스 모델의 등장으로 로컬 실행이 가능해졌다.

이 이중 구조는 “클라우드 AI의 성능”과 “로컬 AI의 자율성” 사이의 긴장을
보여준다.
Tripo API는 더 나은 결과를 줄 수 있지만 API 키와 비용이 필요하다.
Hunyuan3D는 자급자족이 가능하지만 로컬 GPU와 설치 과정이 필요하다.
이 트레이드오프는 3DCellForge만의 문제가 아니라, AI 기반 도구를 만드는
모든 개발자가 직면하는 구조적 선택이다.

오픈소스 AI 모델의 성장이 이 긴장을 해소하는 방향으로 작동한다.
LLaMA, Mistral, Hunyuan3D 같은 모델이 클라우드 API와 경쟁하는 수준으로
발전하면, 개발자는 외부 의존성 없이 고품질 AI 기능을 임베딩할 수 있다.
3DCellForge의 로컬 폴백 전략은 이 방향을 선제적으로 반영한 설계 선택이다.
