# OpenLogi — Rust로 만든 Logitech Options+ 대체 오픈소스

https://github.com/AprilNEA/OpenLogi

## 소개

OpenLogi는 Logitech Options+를 대체하는 프라이버시 중심의 오픈소스 마우스
컨트롤 소프트웨어다. Rust로 작성되었으며, 계정 등록, 클라우드 연동, 텔레메트리
없이 마우스 버튼 재매핑, DPI 설정, SmartShift 제어를 로컬에서 완전히 처리한다.
HID++(Logitech의 고수준 장치 프로토콜)를 직접 구현해 Logi Bolt 수신기,
블루투스, 유선 연결을 모두 지원한다.

## 주요 기능

- **버튼 재매핑**: 39가지 내장 액션과 커스텀 키보드 단축키 지정 가능
- **DPI 제어**: DPI 프리셋 관리 및 프로필 전환
- **SmartShift 설정**: 휠 모드, 감도, 영구 래칫(ratchet) 설정
- **앱별 프로필**: 활성 애플리케이션에 따라 자동으로 설정 전환
- **GUI + CLI 이중 인터페이스**: 시각적 마우스 다이어그램이 있는 데스크톱 앱과
  헤드리스 진단 도구를 모두 제공
- **설정 파일**: TOML 포맷, 로컬 저장, 클라우드 동기화 없음

## 아키텍처

두 개의 바이너리로 구성된다.

**OpenLogi GUI**는 GPUI 프레임워크 기반 데스크톱 앱으로, 마우스 시각 다이어그램,
액션 피커, 설정 패널을 포함한다. 6개 언어를 지원한다.

**OpenLogi CLI**는 헤드리스 도구로 장치 목록 조회와 진단에 쓰인다.

## 플랫폼 지원 현황

macOS는 완전 지원이고, Linux와 Windows는 개발 중이다. v0.5.3 기준 4,200개 이상의
GitHub 스타를 보유하고 있으며, 18회 릴리스를 진행했다. “활발히 개발 중이며 아직
안정적이지 않다”고 스스로 밝히고 있다.

라이선스는 Apache License 2.0 또는 MIT 이중 라이선스이며, 로고와 아이콘 등
브랜드 자산은 별도 소유권을 유지한다.

## 분석

### Logitech Options+의 문제가 만든 수요

Logitech의 공식 소프트웨어는 오랫동안 “집으로 전화 거는(phone home)” Electron
기반 앱이라는 비판을 받아왔다. 2026년에는 만료된 개발자 인증서로 인해 macOS에서
G HUB와 Logi Options+가 일시 작동 불능 상태가 되면서, 수천 명의 사용자가 스크롤
방향과 버튼 매핑 기능을 잃는 사건이 발생했다. 클라우드 의존적인 설계가 단일
장애점이 된 것이다.

OpenLogi는 이 사건이 보여주는 근본적인 문제 — 주변기기 설정 소프트웨어가 불필요하게
클라우드와 결합되어 있다 — 에 정면으로 대응한다. 마우스 설정은 본질적으로 로컬
시스템 소프트웨어여야 하며, 클라우드나 계정 없이도 완전히 동작해야 한다는 관점이다.

### Rust + GPUI 조합

Rust로 macOS 네이티브 앱을 작성한다는 점이 흥미롭다. GPUI는 Zed 에디터 팀이
만든 GPU 가속 UI 프레임워크로, 고성능 데스크톱 앱을 Rust로 구현할 때 쓰인다.
Electron의 대척점에 있는 선택이다 — 번들 크기, 메모리 사용량, 네이티브 느낌
모두에서 근본적으로 다른 접근이다.

## 비평

### 강점

HID++ 프로토콜을 직접 구현해 퍼스트파티 소프트웨어 없이 장치를 제어한다는 점은
기술적으로 의미 있는 작업이다. TOML 기반 로컬 설정은 버전 관리가 가능하고 이식성이
높다. “로컬 퍼스트(local-first)” 원칙을 주변기기 소프트웨어에 적용한 선도적 사례다.

### 한계

macOS 전용 현황은 Linux와 Windows 사용자에게 당장 선택지가 아니다. “아직 안정적이지
않다”는 공식 경고는 업무용 마우스에 도입하기 전 신중함을 요구한다. Logitech이
HID++ 프로토콜을 변경하거나 암호화를 강화하면 유지보수 부담이 급격히 늘어날 수
있다는 취약성도 있다.

## 인사이트

### 컴패니언 앱은 참여 깔때기가 되었다

하드웨어 제조사들의 컴패니언 소프트웨어는 원래 장치 기능을 확장하기 위해 존재했다.
그러나 점차 사용자 데이터 수집, 클라우드 계정 유도, 프리미엄 기능 유료화를 위한
“참여 깔때기(engagement funnel)”로 변질되었다. 마우스를 쓰는 데 계정이 필요하고,
앱이 백그라운드에서 서버와 통신한다.

이 변질이 OpenLogi 같은 오픈소스 대안을 낳는다. 사용자들이 하드웨어를 샀는데
소프트웨어에 종속된다는 느낌을 받기 시작할 때, 대안 생태계가 형성된다.

### Rust는 시스템 레벨 도구의 새 기본 언어가 되고 있다

HID 프로토콜 구현, 커널 레벨 통신, 네이티브 데스크톱 앱 — 10년 전이라면 C나 C++의
영역이었던 작업들을 Rust가 대체하고 있다. 메모리 안전성과 성능을 동시에 제공하는
Rust의 특성은 정확히 이런 저수준 시스템 도구에서 빛난다.

OpenLogi의 Rust 선택은 순전히 이념적인 것이 아니다. HID++ 패킷을 직접 파싱하고
장치와 통신하는 코드에서 메모리 버그는 실제 위험이다. Rust는 컴파일 타임에 이
위험을 제거한다.

### 단일 장애점으로서의 인증서 만료

2026년 Logitech 인증서 만료 사건은 소프트웨어 공급망의 취약성을 잘 보여준다.
코드 서명 인증서가 만료되면 macOS의 Gatekeeper가 앱 실행을 차단하고, 사용자는
마우스 기능을 잃는다. 하드웨어는 완전히 정상인데, 소프트웨어 인프라 문제로 사용
불능이 된다.

이는 더 넓은 패턴이다 — 클라우드 의존적인 설계는 오프라인에서도 작동해야 할
장치들을 온라인 의존적으로 만든다. 로컬 퍼스트 소프트웨어는 이 위험에 대한
아키텍처적 대응이다.
