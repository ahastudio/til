# GitHub Copilot, 사용량 기반 과금으로 전환

원문: <https://github.blog/news-insights/company-news/github-copilot-is-moving-to-usage-based-billing/>

## 요약

GitHub은 2026년 4월 27일, 모든 Copilot 플랜이 2026년 6월 1일부터 사용량 기반 과금
(usage-based billing)으로 전환된다고 발표했다. 기존에는 “프리미엄 요청(premium request)”
단위로 소비량을 측정했으나, 새 모델은 입력 토큰, 출력 토큰, 캐시 토큰을 합산한 실제
토큰 소비량을 기준으로 요금을 부과한다. 공개된 API 요율과 동일한 기준이 적용된다.

새 과금 단위는 “GitHub AI Credits”다. 기존 플랜의 월정액은 변경되지 않지만, 월정액이
곧 해당 월에 사용할 수 있는 AI Credits 한도가 된다. 개인 플랜 기준으로 Copilot Pro는
월 $10($10 크레딧), Copilot Pro+는 월 $39($39 크레딧)다. 비즈니스 플랜은 각각
Copilot Business $19/사용자, Copilot Enterprise $39/사용자이며, 6월~8월 한시적으로
Business는 $30, Enterprise는 $70의 추가 크레딧이 제공된다.

GitHub이 이번 전환의 이유로 밝힌 것은 “지속 가능성”이다. 현행 구조에서는 짧은 채팅
질문 하나와 수 시간짜리 자율 코딩 세션이 같은 비용을 소비한다. 즉, 헤비 유저가
라이트 유저를 사실상 보조하는 구조였다. 토큰 기반 과금은 실제 소비에 비례하는 요금을
부과해 이 불균형을 해소하는 것이 목표다.

코드 자동완성(code completions)과 Next Edit Suggestions는 크레딧을 소비하지 않고 계속
무료로 제공된다. 반면 저비용 모델로의 폴백(fallback) 경험은 더 이상 제공되지 않는다.
Copilot 코드 리뷰는 AI Credits와 GitHub Actions 분(minutes)을 동시에 소비한다.
5월 초에는 전환 전 예상 비용을 미리 확인할 수 있는 미리보기 청구 화면이 공개된다.

## 분석

### 정액제에서 종량제로의 구조적 전환

이번 변화는 단순한 가격 조정이 아니라 과금 모델의 패러다임 전환이다. 정액제(flat-rate)는
예측 가능성을 제공하지만 플랫폼 입장에서 헤비 유저의 비용을 내재화하는 구조다.
GitHub은 AI 에이전트 기능이 확산되면서 사용량 분산이 극단적으로 커졌고, 그 결과 단위
비용 예측이 불가능해졌다고 판단한 것으로 보인다. 종량제는 플랫폼 리스크를 사용자에게
이전한다.

HN 댓글에서는 이 전환을 “enshittification”의 교과서적 사례로 규정하는 시각이 강했다.[^hn1]
저렴한 정액제로 사용자를 끌어들여 의존성을 형성한 뒤 가격을 인상하는 고전적 패턴이라는
것이다. GitHub 측의 “지속 가능성” 명분이 진심이라 해도, 시장에서 이미 구독 기반 종속이
충분히 형성된 시점에 전환이 이루어진다는 타이밍 자체가 의혹을 증폭시킨다.

### 토큰 기반 과금과 API 요율의 연동

공개된 API 요율과 동일한 기준을 적용한다는 선언은 중요한 의미를 갖는다. 이는 Copilot이
더 이상 “구독 소프트웨어”가 아니라 “IDE에 내장된 API 소비자”임을 공식화한 것이다.
사용자 입장에서 Copilot 크레딧과 직접 API 호출의 비용 비교가 가능해지고, 이는 곧
가격 경쟁력 검증이 투명해진다는 뜻이기도 하다.

그 투명성은 즉시 역효과로 돌아왔다. HN에서 여러 사용자가 모델별 크레딧 승수를 직접
계산한 결과, Claude Sonnet은 기존 1x에서 9x로, Opus 계열은 3x에서 27x로 사실상
인상된다는 수치가 공유되었다.[^hn2] 이를 두고 “GitHub tax”라는 표현이 등장했다.
직접 API를 호출하는 것보다 Copilot을 통해 소비하면 더 비싸지는 역전 현상이 발생한다는
우려다. “API 요율과 동일하다”는 GitHub의 주장은, 승수를 적용한 실효 요율 앞에서
사실상 공허해졌다.

## 비평

### 강점: 투명성과 예측 가능성의 명분

GitHub의 발표는 “지속 가능성”이라는 명분을 내세웠고, 예상 비용 미리보기 도구를 제공하는
점은 실용적인 배려다. 5월 미리보기 기간 동안 사용 패턴을 확인하고 계획을 조정할 수
있다는 점에서, 6월 전환 전에 충분한 유예 시간을 두었다고 볼 수 있다.

### 약점: 라이트 유저에게 불리한 크레딧 만료 구조

월정액이 곧 크레딧 한도인 구조에서, 크레딧이 이월(rollover)되지 않으면 라이트 유저는
구독료를 내고도 크레딧을 다 쓰지 못하는 손해를 본다. 경쟁사에 직접 지불하는 것보다
Copilot 구독이 비효율적이 되는 상황이 생길 수 있다. GitHub이 크레딧 이월 정책을
명확히 밝히지 않은 것은 발표의 약점이다.

HN 댓글에서는 이미 대안 탐색이 시작되었다. OpenRouter(5.5% 수수료 있음), 직접 API
접속, 9B 파라미터 수준의 로컬 모델로 전환을 검토한다는 개발자들이 다수 등장했다.[^hn3]
“Copilot 수준의 출력은 로컬 모델로도 충분하다”는 주장도 있었다. 가격 압박이 개발자로
하여금 Copilot의 편의성을 실제로 얼마나 가치 있게 여기는지 직접 계산하게 만들고
있는 것이다.

### 약점: 엔터프라이즈 잠금(lock-in)의 비대칭성

커뮤니티 반응에서 주목할 점은 기업 고객과 개인 개발자 사이의 반응 차이다. 기업은
Microsoft 통합, 컴플라이언스, IT 거버넌스 때문에 쉽게 이탈하지 못한다. HN 댓글에서도
“엔터프라이즈는 이미 Microsoft와 깊이 묶여 있어 가격이 올라도 떠나기 어렵다”는
현실적 관측이 이어졌다.[^hn4] 반면 개인 개발자는 OpenRouter, 직접 API, 로컬 모델 등으로
즉시 전환 가능하다. 이 비대칭이 장기적으로 Copilot의 사용자 구성을 기업 중심으로
재편할 위험이 있다.

## 인사이트

### AI 보조금 시대의 종말과 실제 비용의 가시화

지난 2~3년간 AI 서비스들은 사용자 기반을 구축하기 위해 원가 이하의 보조 가격을
유지해왔다. GitHub Copilot, ChatGPT Plus, 각종 AI 툴들이 경쟁적으로 저렴한 정액제를
내걸었다. 이 구조는 투자자 자본과 전략적 시장 점유율 확보 의지가 뒷받침될 때만
지속 가능하다. 사용량이 폭발적으로 늘고 모델 호출 비용이 내려오지 않는 상황이 이어지자,
각 플랫폼은 실제 원가를 가격에 반영하지 않으면 안 되는 지점에 도달했다.
HN에서 한 댓글은 이를 “AI 보조금 시대의 끝”이라고 표현했다.[^hn5]
GitHub의 이번 전환은 단독 이벤트가 아니다. Anthropic, OpenAI 등 다른 제공자들의
가격 합리화와 동시에 진행되고 있으며, 이는 AI 서비스 전반이 성장기에서 수익화기로
이행하고 있다는 신호다. 사용자 입장에서 이 변화는 “AI 도구를 공짜에 가깝게 쓰던”
경험이 끝나고, 실제로 얼마나 사용하는지에 따라 비용이 발생하는 새로운 기준선으로
이동함을 의미한다.

### 종량제는 단순한 과금 모델이 아니라 사용 행동을 바꾼다

정액제와 종량제의 차이는 숫자가 아니라 심리다. 정액제 환경에서 개발자는 AI 도구를
“무한히 사용 가능한 자원”처럼 다룬다. 요청을 반복하고, 긴 컨텍스트를 쌓고, 에이전트를
오래 돌려도 추가 비용이 없다. 종량제 전환 이후에는 토큰 소비가 비용으로 직결되기
때문에, 개발자는 요청 방식을 최적화하기 시작한다. 짧고 명확한 프롬프트, 필요한
컨텍스트만 포함하는 선택적 첨부, 에이전트 실행 시간 단축이 곧 비용 절감으로 연결된다.
이런 최적화 압력은 장기적으로 AI 도구 활용 숙련도를 높이는 방향으로 작용할 수 있다.
하지만 동시에, 비용에 민감한 개발자가 AI 도구 사용 자체를 위축시킬 위험도 있다.
“이 요청을 하면 얼마가 나갈까?”라는 생각이 개입하는 순간 사용자의 자연스러운 탐색이
줄어든다. GitHub이 코드 자동완성을 크레딧 소비 대상에서 제외한 것은 이 위축을
방지하기 위한 의식적인 선택으로 읽힌다.

### Copilot의 정체성 변화: 구독 소프트웨어에서 AI 인프라로

Copilot이 처음 출시되었을 때 그것은 “AI가 탑재된 IDE 플러그인”이었다. 월 $10짜리
구독이었고, 사용자는 그것을 GitHub Actions, npm 패키지와 비슷한 도구로 인식했다.
토큰 기반 과금으로 전환되는 지금, Copilot은 점점 “IDE에 내장된 API 엔드포인트”에
가까워진다. GitHub AI Credits는 AWS 크레딧이나 GCP 크레딧과 같은 클라우드 인프라
과금 언어를 사용한다. 이 언어 선택은 의식적이다.
Microsoft가 Azure를 통해 OpenAI API를 제공하고, GitHub이 Copilot 크레딧으로 그것을
소비하는 구조는 Copilot을 Microsoft AI 인프라 생태계의 소비 레이어로 자리매김시킨다.
개인 개발자에게는 “AI 도구 구독”처럼 보이지만, 기업 IT 관리자에게는 “클라우드 리소스
예산 관리”의 문제가 된다. 이 이중적 포지셔닝이 Copilot의 다음 단계 성장 전략이다.
엔터프라이즈 고객에게 Copilot은 Azure 청구서의 한 항목이 되고, 그 순간 이탈 장벽은
기술적 선택의 문제가 아니라 조달 계약과 컴플라이언스의 문제가 된다.
