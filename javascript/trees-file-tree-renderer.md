# Trees — 파일 트리 렌더링 라이브러리

<https://github.com/pierrecomputer/pierre>

## 소개

Trees는 VS Code 파일 탐색기와 같은 파일·디렉터리 구조를 렌더링하는 오픈소스 라이브러리다. “높은 커스터마이징 자유도와 성능 중심 설계”를 지향하며, React 래퍼와 바닐라 JavaScript 클래스 구현을 모두 제공한다.

패키지명은 `@pierre/trees`이고, React 통합은 `@pierre/trees/react` 진입점으로 제공된다. 서버 사이드 렌더링(SSR)과 하이드레이션도 지원한다.

## 주요 기능

**가상화(Virtualization)**: 수만 개 항목도 현재 화면에 보이는 행만 마운트해 빠르게 렌더링한다. 대형 레포지터리 탐색에서도 성능 저하 없이 동작한다.

**Git 통합**: 파일 수정·추가·삭제·미추적 상태를 뱃지로 표시한다. 코드 편집기 수준의 파일 상태 표시를 UI 컴포넌트로 바로 가져올 수 있다.

**접근성**: WCAG 2.1 준수, 키보드 내비게이션, ARIA 지원을 기본 제공한다.

**드래그 앤 드롭**: 파일과 폴더 이동을 지원하며, 호버 시 폴더가 자동 확장된다.

**검색 및 필터**: `hide-non-matches`, `collapse-non-matches`, `expand-matches` 세 가지 필터링 모드를 제공한다.

**커스터마이징**: CSS 변수, Shiki 테마, 밀도(density) 컨트롤로 외형을 조정할 수 있다. 단일 자식 폴더 체인을 하나로 접는 tree flattening, 커스텀 컨텍스트 메뉴, 세 가지 아이콘 세트(minimal, standard, complete)도 포함한다.

## 아키텍처

Trees는 “경로 우선(path-first) 모델”을 채택한다. 트리 항목의 기본 식별자로 표준 경로 문자열을 사용하며, React·바닐라 JS·SSR 구현 모두에 일관된 경로 기반 API를 제공한다. 선택(selection), 포커스, 검색, 이름 바꾸기, 드래그 앤 드롭 등 모든 조작이 경로를 중심으로 설계된다.

SSR 최적화를 위해 서버 측에서 트리 데이터를 미리 준비하는 기능을 제공해, 초기 렌더링 성능과 SEO를 함께 챙긴다.

## 분석

UI 라이브러리 생태계에서 파일 트리는 수요가 지속적으로 존재하지만 제대로 된 독립 라이브러리가 드물었다. 대부분의 구현이 특정 편집기나 IDE에 종속되어 있거나, 성능과 접근성을 동시에 만족시키지 못했다.

Trees는 VSCode 스타일 탐색기를 범용 컴포넌트로 추출한다는 명확한 포지셔닝을 가진다. Git 상태 통합과 WCAG 2.1 준수는 실제 개발 도구 구축에 필요한 기능을 기본 제공한다는 점에서 경쟁 우위가 있다.

경로 우선 설계는 파일 시스템의 실제 구조를 API 계약으로 삼아 다양한 렌더링 환경에서 일관성을 유지하는 영리한 선택이다. 그러나 현재 베타 단계로, 장기적인 유지보수와 생태계 성숙도는 아직 검증되지 않았다.

## 비평

### 강점: 틈새 시장의 명확한 해결

파일 트리 렌더링은 코드 에디터, 파일 관리 앱, 문서 편집기 등 다양한 웹 앱에서 반복 구현되는 UI 패턴이다. 성능(가상화), 접근성(WCAG), Git 통합을 한 번에 제공하는 독립 라이브러리는 실질적인 개발 생산성을 높인다.

### 약점: 베타 상태와 생태계 불확실성

GitHub 저장소명이 `pierre`이고 패키지명이 `@pierre/trees`인 것에서 알 수 있듯, 개인 프로젝트 규모의 라이브러리다. 프로덕션 환경에서의 안정성, 장기 지원, 커뮤니티 규모는 아직 미지수다. 비슷한 포지셔닝이었던 라이브러리들이 메인테이너 부재로 폐기된 사례가 많다.

## 인사이트

### 범용 UI 컴포넌트의 분화 가속

UI 라이브러리 시장은 “모든 것을 포함한 컴포넌트 라이브러리”에서 “특정 패턴에 최적화된 독립 라이브러리”로 점진적으로 이동하고 있다. 테이블은 TanStack Table, 가상 스크롤은 Virtua, 파일 트리는 Trees처럼, 특정 UX 패턴의 복잡성이 충분히 높을 때 전문 라이브러리가 등장한다.

이 분화는 번들 크기와 관심사 분리 측면에서 이점이 있지만, 동시에 의존성 관리의 복잡성을 높인다. AI 코딩 에이전트가 라이브러리 선택을 자동화하는 시대에, 전문화된 소규모 라이브러리의 발견 가능성과 신뢰성이 새로운 과제가 된다.

### 개발 도구 UI의 소비자화

파일 트리, Git 상태 표시, 터미널 에뮬레이터 같은 개발자 도구 고유의 UI 요소들이 범용 웹 앱에서도 사용되는 추세가 가속되고 있다. 클라우드 IDE, AI 코딩 어시스턴트, 파일 관리 SaaS의 성장이 이 수요를 만들어 낸다. Trees 같은 라이브러리의 출현은 이 수요의 구체적 증거다.
