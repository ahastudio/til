# JetBrains Mono와 Pretendard를 합친 한글 코딩 폰트, Jetendard

<https://github.com/kuskhan/jetendard>

## 소개

Jetendard는 JetBrainsMono Nerd Font Mono의 라틴 글리프와
Pretendard의 한글 글리프를 하나의 폰트로 재구성한 빌드 프로젝트다.
목표는 "균형 잡힌 한글 고정폭 렌더링"이다.
터미널이나 코드 에디터에서 영문과 한글이 섞일 때 자주 발생하는
자간 불균형과 정렬 문제를 해결하려는 시도다.
이를 위해 Pretendard 글리프를 1.15배 확대해
한글 문자 주변 여백을 줄이고, 한글 한 글자를 라틴 문자 정확히
두 칸 너비에 맞춰 배치한다.

이 프로젝트는 완성된 폰트 파일을 배포하는 대신,
소스에서 직접 빌드하는 재현 가능한(reproducible) 빌드 스크립트 형태로 제공된다.

## 특징

**이중 언어 지원**: 라틴 문자는 JetBrainsMono, 한글 및 CJK 문자는
Pretendard에서 가져온다.

**프로그래밍 리거처**: JetBrainsMono Nerd Font에서 물려받은
코딩 리거처를 그대로 사용할 수 있다.

**16종 패밀리**: Thin부터 ExtraBold까지 여덟 가지 굵기를
각각 정자체와 이탤릭체로 제공해 총 16개 변형을 만든다.

**정밀한 글리프 배치**: 한글 문자를 라틴 고정폭 두 칸에
정확히 맞춰 정렬 문제를 없앤다.

**이탤릭 처리 방식**: 이탤릭 변형에서는 라틴 문자만 JetBrainsMono의
진짜 이탤릭체를 사용하고, 한글은 정자체 Pretendard를 그대로 쓴다.
Pretendard 자체에 정식 이탤릭 변형이 없기 때문이다.

**범위 한정**: 기반 폰트로는 `JetBrainsMonoNerdFontMono`만 사용하며
Black 굵기는 포함하지 않는다.
Nerd Font 아이콘은 기반 폰트에 이미 패치되어 있어
별도의 2차 패치 과정이 필요 없다.

## 사용법

빌드에는 `uv`가 필요하다.

```bash
uv sync --all-groups
make download
make run
make test
```

`make run`을 실행하면 16개 변형이 모두 생성되며,
결과물은 `fonts/ttf/`, `fonts/otf/`, `fonts/webfont/` 디렉터리에 저장된다.

CLI 도구는 다음과 같은 옵션으로 빌드 범위를 조정할 수 있다.

- `--all`: 16종 전체 매트릭스 빌드
- `--weights`: 특정 굵기만 선택
- `--styles`: 정자체, 이탤릭, 또는 둘 다 선택
- `--korean-scale`: 한글 확대 비율 조정 (기본값 1.15)

예를 들어 `uv run jetendard --weights Regular Bold --styles normal italic`
명령으로 Regular와 Bold 굵기의 정자체·이탤릭체만 골라 빌드할 수 있다.

라이선스는 SIL Open Font License 1.1을 따른다.
JetBrains Mono, Nerd Fonts, Pretendard, Yeomil Mono 등
상위 프로젝트의 저작권 고지도 함께 확인해야 한다.

## 분석

Jetendard가 해결하려는 문제는 오래된 것이다.
영문 고정폭 폰트에 한글 대체 폰트(fallback)를 지정하는 방식은
이미 널리 쓰이고 있지만, 폰트 렌더링 엔진이나 환경에 따라
대체 폰트 설정 자체가 불가능하거나 일관되게 적용되지 않는 경우가 있다.
Jetendard는 대체 설정에 의존하지 않고, 애초에 하나의 폰트 파일
안에 두 언어의 글리프를 결합해 이 문제를 원천적으로 없앤다.

기술적으로 흥미로운 지점은 "1.15배 확대"라는 구체적인 수치다.
한글은 원래 라틴 문자보다 획이 복잡하고 문자 자체가 크게 디자인되는
경향이 있어, 단순히 폭만 맞추면 시각적으로 작아 보이거나
자간이 어색해지는 문제가 생긴다.
이 프로젝트는 폭 정렬과 시각적 크기 균형이라는
두 가지 다른 문제를 별도로 다루고 있다는 점에서
단순한 폰트 병합 이상의 튜닝 작업을 수행한 것으로 보인다.

빌드 방식 선택도 눈에 띈다.
완성된 폰트 파일을 그대로 배포하지 않고 `uv`와 `make` 기반의
재현 가능한 빌드 파이프라인으로 제공한다.
이는 사용자가 원하는 굵기와 스타일만 선택적으로 빌드할 수 있게 하고,
동시에 상위 폰트들의 라이선스 조건을 지키기 쉽게 만드는
실용적인 선택이다.

## 비평

이탤릭 처리 방식은 실용적인 타협이지만 완전하지는 않다.
라틴 문자만 기울어지고 한글은 그대로 남기 때문에,
영문과 한글이 한 줄에 섞인 이탤릭 텍스트에서는
시각적 리듬이 부분적으로 어긋날 수 있다.
Pretendard에 정식 이탤릭이 없다는 상위 제약을 고려하면
불가피한 선택이지만, 사용자가 이탤릭을 많이 쓰는 환경이라면
체감되는 한계다.

빌드 전용 배포 방식은 재현성과 유연성 면에서 장점이 있지만,
당장 폰트 파일 하나만 내려받아 쓰고 싶은 사용자에게는
진입 장벽이 된다.
`uv`, `make`, 빌드 스크립트 실행이라는 절차를 거쳐야 하므로
일반 사용자보다는 개발 도구에 익숙한 사용자를 우선 대상으로 삼은
프로젝트로 보인다.
릴리스 페이지에 미리 빌드된 폰트 파일을 함께 제공한다면
접근성이 크게 개선될 것이다.

[GeekNews 댓글](https://news.hada.io/topic?id=31174)에서는 구름 산스 코드를
대체 폰트로 지정하는 기존 방식이 이미 정렬과 가독성 면에서 훌륭하다는
의견도 나왔다.
다만 폰트 대체 설정 자체가 불가능한 환경이 존재한다는 점에서
Jetendard의 접근이 여전히 유효하다는 데는 공감이 모였다.
도구를 현대화하는 인터페이스로 개선하자는 제안에
제작자 본인이 TypeScript 포팅과 실시간 미리보기 UI 아이디어로
화답한 점도 눈에 띈다.

## 인사이트

### 폰트 병합은 결국 미세 조정의 싸움이다

두 폰트를 하나로 합치는 작업 자체는 폰트 편집 도구로 어렵지 않게
할 수 있다.
정말 어려운 부분은 서로 다른 문자 체계가 만들어내는
시각적 불일치를 얼마나 세밀하게 조정하느냐다.
Jetendard가 굳이 확대 비율을 옵션으로 노출하고
`--korean-scale` 같은 조정 지점을 CLI에 남겨둔 것은,
이 값이 폰트마다, 사용자 환경마다 최적값이 다를 수 있다는
사실을 인정하는 설계다.
완벽한 기본값 하나를 찾기보다 조정 가능성을 열어두는 쪽을 택한 셈이다.

### 대체 폰트 설정이 안 되는 환경은 생각보다 많다

터미널 에뮬레이터, 코드 에디터, 웹 폰트 렌더링 등
각 환경마다 폰트 대체 규칙을 지정하는 방식과 지원 수준이 다르다.
특히 웹 환경이나 임베디드 환경처럼 폰트 대체 체인을
세밀하게 제어하기 어려운 곳에서는, 하나의 폰트 파일 안에
모든 문자를 담는 접근이 오히려 더 안정적인 해결책이 된다.
Jetendard, 그리고 앞서 언급된 구름 산스 코드 같은 프로젝트들이
같은 문제를 서로 다른 층위에서 풀고 있다는 점은,
한글 고정폭 렌더링 문제가 아직 표준화된 해법 없이
각자 다른 방식으로 계속 재발명되고 있다는 뜻이기도 하다.

### 오픈소스 폰트 생태계의 연쇄적 재조합

Jetendard는 JetBrains Mono, Nerd Fonts, Pretendard, Yeomil Mono라는
서로 다른 기원의 오픈소스 폰트 프로젝트를 조합해 만들어졌다.
이런 재조합이 가능한 이유는 관련 폰트들이 모두
SIL Open Font License 같은 관대한 라이선스로 배포되기 때문이다.
개별 폰트 제작자가 모든 언어와 용도를 커버할 필요 없이,
각자 잘하는 영역에 집중하고 그 결과물을 자유롭게 재조합할 수 있는
생태계가 이런 틈새 도구들의 지속적인 등장을 뒷받침하고 있다.
