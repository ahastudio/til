# WebVM

> WebVM is a server-less virtual Linux environment running fully client-side in
> HTML5/WebAssembly.

<https://webvm.io/>

<https://github.com/leaningtech/webvm>

[WebVM: server-less x86 virtual machines in the browser | by Alessandro Pignotti | leaningtech | Feb, 2022 | Medium | leaningtech](https://medium.com/leaningtech/40a60170b361)

## 소개

WebVM은 Leaning Technologies가 개발한 서버리스 Linux 가상 환경으로, HTML5/WebAssembly를 사용해 브라우저 클라이언트 사이드에서 완전히 실행된다.
수정되지 않은 Debian 배포판을 Linux ABI 호환 방식으로 실행하며, 서버 없이 브라우저만으로 전체 Linux 환경을 구동할 수 있다.
Apache License 2.0으로 공개되어 있으며, 개인 탐구 용도는 자유롭게 사용 가능하나 상업적·조직적 사용에는 CheerpX 라이선스 조건이 적용된다.

## 아키텍처

| 컴포넌트  | 역할                                              |
| --------- | ------------------------------------------------- |
| CheerpX   | x86→WebAssembly JIT 컴파일러 및 가상화 엔진       |
| xterm.js  | 브라우저 내 터미널 에뮬레이터                     |
| Tailscale | VPN 네트워킹(프라이빗 네트워크 및 인터넷 접속)    |
| lwIP      | Cheerp으로 컴파일된 TCP/IP 스택                   |

핵심 동작 원리는 x86 명령어를 Wasm으로 JIT 컴파일하는 CheerpX 엔진이다.
가상 블록 기반 파일 시스템과 Linux 시스템 콜 에뮬레이터를 조합해 표준 Linux 바이너리를 수정 없이 실행한다.
실행은 완전한 클라이언트 사이드 샌드박스 안에서 이루어진다.

## 배포 및 커스터마이징

GitHub Pages에 직접 배포하거나, Dockerfile을 통해 디스크 이미지를 커스터마이징할 수 있다.
`.ext2` 포맷의 커스텀 디스크 이미지를 지원하며, Python REPL 등 특정 환경 설정도 지원한다.
Claude AI 연동도 지원한다. API 키를 WebVM 인터페이스에 입력하면 로컬 브라우저에만 저장되며 서버 측 접근은 없다.

## 분析

### JIT 컴파일의 의미

WebVM의 핵심 기술 도전은 x86 명령어를 런타임에 Wasm으로 변환하는 JIT 컴파일러다.
x86과 Wasm은 아키텍처적으로 근본적으로 다르다. x86은 레지스터 기반의 CISC 명령어 집합이고, Wasm은 스택 기반 가상 머신이다.
이 간극을 실시간으로 메우면서 실용적인 성능을 내는 것이 CheerpX의 기술적 핵심이다.
JIT 컴파일은 AOT(사전 컴파일)와 달리 실행 중인 코드의 패턴을 분석해 최적화할 수 있어, 임의의 x86 바이너리를 처음 실행할 때도 점진적으로 성능이 향상된다.

### 네트워킹 문제의 해결

브라우저 환경에서 Linux 네트워킹을 구현하는 것은 까다로운 문제다.
브라우저는 TCP/IP 소켓을 직접 노출하지 않고, WebSocket과 WebRTC만 허용한다.
WebVM은 이 제약을 두 가지 방식으로 극복한다. 하나는 lwIP 스택을 Wasm으로 컴파일해 사용자 공간에서 TCP/IP를 에뮬레이션하는 것이고, 다른 하나는 Tailscale WireGuard 터널을 통해 실제 네트워크 접속을 가능하게 하는 것이다.
이 조합으로 브라우저 안에서 실행되는 Linux 프로세스가 실제 네트워크 통신을 할 수 있다.

### WebAssembly의 새로운 활용 지평

WebVM은 WebAssembly가 단순한 성능 최적화 도구가 아니라 완전한 실행 환경을 브라우저에 구현하는 데 쓰일 수 있음을 보여준다.
“WebAssembly로 무엇을 실행할 수 있는가”의 답이 특정 언어나 라이브러리에서 “임의의 Linux 바이너리”로 확장된다.
이것은 플랫폼으로서의 웹이 가진 가능성의 한계를 다시 설정한다.

## 비평

### 성능의 현실

x86→Wasm JIT 컴파일 레이어가 있는 만큼, 네이티브 실행이나 Docker 컨테이너 대비 성능 오버헤드가 존재한다.
CPU 집약적인 작업이나 I/O 중심 서버 소프트웨어에서 이 오버헤드가 얼마나 되는지에 대한 투명한 벤치마크가 공개적으로 많지 않다.
실험적 탐구에는 충분하지만, 프로덕션 워크로드 대체로 쓰기엔 성능 특성을 직접 측정해봐야 한다.

### 라이선스의 복잡성

Apache 2.0이라는 오픈소스 라이선스 뒤에 CheerpX 상업 라이선스가 있다는 점은 채택을 고민하는 팀에게 불확실성을 준다.
개인 탐구와 상업적 사용의 경계가 명확히 문서화되지 않았다.
핵심 기술인 CheerpX가 독점 소프트웨어라는 사실은, WebVM이 완전히 투명하게 작동하는 오픈소스 스택이 아님을 의미한다.

## 인사이트

### 브라우저가 운영체제가 되는 순간

WebVM은 “브라우저 안에서 Linux를 실행한다”는 기술 데모이지만, 그 함의는 훨씬 넓다.
운영체제의 역할은 하드웨어 추상화와 애플리케이션 실행 환경 제공이다.
웹 브라우저가 x86 Linux를 실행할 수 있다면, 브라우저 자체가 하나의 운영체제 계층이 된다.
사용자가 OS에 종속될 이유가 사라지는 미래의 단초다.

ChromeOS가 이 방향의 선례다. “클라우드와 브라우저만 있으면 된다”는 전제가 Chromebook을 가능하게 했다.
WebVM은 그 전제를 한 단계 강화한다. 클라우드 서버도 없이 브라우저 탭 하나에서 Debian Linux가 돌아간다.
이것은 오프라인에서도 기능하는 웹 운영체제의 기반이 될 수 있다.

클라이언트 사이드 실행의 중요한 부산물은 프라이버시다.
서버가 없다는 것은 사용자의 코드, 데이터, 실행 패턴이 서버에 기록되지 않는다는 의미다.
Claude API 키가 “로컬에만 저장”되는 설계는 이 원칙의 실천이다.
컴퓨팅의 서버 집중화에 대한 대안 모델을 브라우저가 제공할 수 있다는 가능성이다.

### 격리된 코드 실행 환경의 새로운 패러다임

WebVM은 신뢰되지 않는 코드를 안전하게 실행하는 문제에 대한 웹 기반 해법이기도 하다.
온라인 코딩 교육 플랫폼(Replit, CodeSandbox)이 서버에서 학습자 코드를 실행하는 것은 보안, 비용, 확장성 모두에서 도전이다.
WebVM은 이 실행을 클라이언트로 이동시킨다. 학습자의 코드가 학습자 자신의 브라우저에서 실행된다.
서버 비용이 없고, 악의적 코드가 서버 인프라를 위협하지 않으며, 수백만 명이 동시에 사용해도 서버 부하가 없다.

이 원리는 CTF(Capture the Flag) 보안 챌린지, 인터랙티브 문서, 소프트웨어 데모에도 적용된다.
“Try it in your browser” 버튼이 실제 격리된 Linux 환경을 제공할 수 있다면, 설치 없이 소프트웨어를 경험하는 방식이 바뀐다.
Docker Hub가 이미지를 배포하듯, 브라우저 탭이 완전한 Linux 환경을 즉시 제공하는 미래가 WebVM이 가리키는 방향이다.
