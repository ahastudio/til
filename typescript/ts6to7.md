# ts6to7

<https://github.com/zi-gae/ts6to7>

## 소개

`ts6to7`은 TypeScript 5 또는 6 프로젝트를 TypeScript 7(네이티브
Go 재작성 버전, tsgo)로 자동 마이그레이션하는 도구다.
TypeScript 5와 6에서 지원 종료(deprecated)로 표시된 옵션을
모두 제거하고, 일부 기본값을 TS7 방식으로 전환한다.

## 주요 기능

**설정 파일 갱신**: `tsconfig*.json` 파일의 주석과 서식을
유지한 채로 다시 작성한다.
지원 종료된 컴파일러 옵션을 TS7 대응 옵션으로 변환하고,
모듈 해석 방식(`node` → `NodeNext` 또는 `Bundler`) 변경을
처리한다.
import 관련 옵션들을 `verbatimModuleSyntax` 하나로 통합하고,
`ignoreDeprecations` 같은 임시 우회 옵션을 제거한다.
`baseUrl`을 `paths` 설정으로 접어 넣는(fold) 작업도
자동화한다.
`strict` 기본값이 `true`로 바뀌는 등 TS7의 주요 breaking
change도 함께 반영한다.

**패키지 관리**: `package.json`의 TypeScript 의존성을
`^7.0.0`으로 갱신하고, `ts-node`·`ts-jest`·`ts-patch` 등
호환되지 않는 도구를 표시해 알려준다.

**모노레포 지원**: 여러 패키지에 걸친 프로젝트에서도
일괄 마이그레이션이 가능하다.

**안전장치**: 자동화할 수 없는 마이그레이션 항목은 수동
검토 체크리스트로 출력한다.
ES5 런타임 요구 사항, 모듈 구문 엄격성 문제, 앰비언트
타입 해석 변경 등은 도구가 자동으로 처리하지 못하며
사용자가 직접 손봐야 한다.

## CLI

```bash
npx ts6to7 --dry            # 변경 사항 미리보기
npx ts6to7                  # 현재 디렉터리에 적용
npx ts6to7 packages/app     # 특정 디렉터리만 대상으로 적용
```

## 라이선스 및 상태

MIT 라이선스이며, 버전 0.2.0으로 활발히 개발 중이다.
`tsconfig` 변환이 잘못된 경우의 버그 리포트를 받고 있으며,
아직 정식 릴리스는 나오지 않은 초기 단계 도구다.

## 분석

TypeScript 7이 컴파일러를 Go로 재작성하면서 컴파일 속도가
크게 개선됐다는 점은 이미 널리 알려져 있지만, 실제 마이그레이션
과정에서는 수년간 쌓인 지원 종료 옵션과 구식 모듈 설정을
정리하는 작업이 만만치 않다.
`ts6to7`은 이 정리 작업 중 기계적으로 처리 가능한 부분(옵션
이름 변경, 폴더 구조 통합)과 사람이 판단해야 하는 부분(런타임
타깃, 모듈 시스템 엄격성)을 명확히 구분해, 자동화 범위를
현실적으로 설정했다는 점이 특징이다.
주석과 서식을 보존하며 `tsconfig.json`을 재작성하는 기능은
실무에서 흔히 겪는 "코드모드가 파일을 망가뜨린다"는 우려를
줄여 준다.

## 비평

버전 0.2.0에 커밋 6개, 아직 정식 릴리스가 없는 초기 단계
프로젝트라는 점은 프로덕션 코드베이스에 바로 적용하기에는
이르다는 신호다.
자동화하지 못하는 항목(런타임 타깃, 앰비언트 타입 해석)이
프로젝트마다 규모가 다를 수 있어, `--dry` 실행 결과가
프로젝트에 따라 "거의 자동화됨"과 "수동 작업이 태반"
사이를 크게 오갈 가능성이 있다.
TS7 자체가 아직 이제 막 출시된 버전인 만큼, 마이그레이션
도구의 성숙도 역시 TS7 생태계 전반의 성숙도에 발맞춰
따라갈 필요가 있다.

## 인사이트

메이저 버전 전환기마다 등장하는 마이그레이션 도구들은
대개 "완전 자동화"를 표방하다 예외 케이스에서 신뢰를
잃는 경우가 많다.
`ts6to7`이 자동화 불가 항목을 체크리스트로 명시적으로
분리해 낸 접근은, 완벽한 자동화보다 정직한 범위 설정이
도구의 신뢰도를 높이는 데 더 효과적일 수 있음을 보여준다.
