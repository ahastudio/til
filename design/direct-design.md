# Direct Design

AI 코딩 도구(Cursor, Claude Code 등)를 사용해
동작하는 제품 위에서 직접 디자인하는 방법입니다.
디자인-개발 핸드오프를 거치지 않고,
제품을 아는 사람이 코드에서 바로 만듭니다.

## 배경

기존 프로세스의 병목은 디자인이 아니라
핸드오프입니다. 디자이너의 의도와 맥락은
스펙 문서로 완전히 전달할 수 없고,
핸드오프 과정에서 정보가 손실됩니다.

Direct Design은 전달 과정 자체를 없앱니다.
만드는 사람이 곧 판단하는 사람이므로
정보 손실이 발생하지 않습니다.

## 기존 프로세스와의 차이

| 기존 프로세스           | Direct Design          |
|-------------------------|------------------------|
| 디자인 도구에서 목업    | 코드에서 실제 제품     |
| 스펙 작성 후 핸드오프   | 핸드오프 없음          |
| 엔지니어가 해석해 구현  | 본인이 직접 구현       |
| 피드백 루프 수 주       | 피드백 루프 수 분      |

## 워크플로우

1. AI 코딩 도구를 엽니다.
2. 무엇을 해야 하는지, 어떤 느낌이어야 하는지,
   누구를 위한 것인지 설명합니다.
3. 동작하는 결과물을 보면서 실시간으로
   판단합니다.
4. 어색한 부분을 코드에서 바로 수정합니다.

## 특징

- **피드백 루프 압축.**
  목업이 아닌 동작하는 제품으로 판단하므로
  매 결정의 정확도가 올라갑니다.
- **제품 감각의 직접 실행.**
  도구 숙련도가 아니라 제품 판단력이
  결과물을 결정합니다.
  "Taste now has a compiler."
- **역할 변화.**
  디자이너가 시각물을 만드는 사람에서
  제품 판단을 직접 실행하는 사람이 됩니다.

## 바이브 코딩과의 구분

도구는 같지만 전제가 다릅니다.

- 바이브 코딩은 뭘 만들지 모르면서 만듭니다.
- Direct Design은 뭘 만들지 정확히 알면서
  번역 과정을 없앱니다.
  제품 감각, 사용자 이해, 엣지 케이스 파악 등
  깊은 제품 이해가 전제됩니다.
