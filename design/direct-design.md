# Direct Design

## 정의

AI 코딩 도구(Cursor, Claude Code 등)를 사용해
동작하는 제품 위에서 직접 디자인하는 방법입니다.
디자인-개발 핸드오프 없이,
제품을 아는 사람이 코드에서 바로 만듭니다.

## 해결하는 문제

기존 프로세스에서는 디자이너가 목업을 만들고,
스펙을 작성하고, 엔지니어에게 전달합니다.
이 핸드오프 과정에서 의도와 맥락이 손실됩니다.

Direct Design은 전달 과정 자체를 제거합니다.
만드는 사람이 곧 판단하는 사람이므로
정보 손실이 발생하지 않습니다.

## 기존 프로세스와의 비교

| 항목          | 기존 프로세스          | Direct Design          |
|---------------|------------------------|------------------------|
| 작업 환경     | 디자인 도구에서 목업   | 코드에서 실제 제품     |
| 전달 방식     | 스펙 작성 후 핸드오프  | 핸드오프 없음          |
| 구현 주체     | 엔지니어가 해석해 구현 | 본인이 직접 구현       |
| 피드백 주기   | 수 주                  | 수 분                  |

## 진행 순서

1. AI 코딩 도구를 엽니다.
2. 무엇을 만들지, 어떤 느낌이어야 하는지,
   누구를 위한 것인지 설명합니다.
3. 동작하는 결과물을 보면서 판단합니다.
4. 수정이 필요한 부분을 코드에서 바로 고칩니다.
5. 3-4를 반복합니다.

## 전제 조건

- 제품 감각: 무엇을 왜 만들어야 하는지 알아야 합니다.
- 사용자 이해: 대상 사용자와 사용 맥락을 파악하고 있어야 합니다.
- 엣지 케이스 인식: 예외 상황을 미리 고려할 수 있어야 합니다.

이 전제가 없으면 바이브 코딩과 구분되지 않습니다.

## 바이브 코딩과의 차이

사용하는 도구는 같지만 전제가 다릅니다.

| 항목        | 바이브 코딩                  | Direct Design                |
|-------------|------------------------------|------------------------------|
| 전제        | 뭘 만들지 모르면서 시작      | 뭘 만들지 정확히 알고 시작   |
| 판단 주체   | AI에게 위임                  | 본인이 판단                  |
| 제품 이해   | 불필요                       | 필수                         |
