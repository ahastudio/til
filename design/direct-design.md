# Direct Design

AI 코딩 도구(Cursor, Claude Code 등)를 사용해
동작하는 제품 위에서 직접 디자인하는 방법.
디자인-개발 핸드오프 없이,
제품을 아는 사람이 코드에서 바로 만든다.

## 기존 프로세스와의 차이

| 기존 프로세스           | Direct Design          |
|-------------------------|------------------------|
| 디자인 도구에서 목업    | 코드에서 실제 제품     |
| 스펙 작성 후 핸드오프   | 핸드오프 없음          |
| 엔지니어가 해석해 구현  | 본인이 직접 구현       |
| 피드백 루프 수 주       | 피드백 루프 수 분      |
| 비전의 80%로 출시       | 비전에 가깝게 출시     |

## 워크플로우

1. AI 코딩 도구를 연다.
2. 무엇을 해야 하는지, 어떤 느낌이어야 하는지,
   누구를 위한 것인지를 설명한다.
   (어떻게 보여야 하는지가 아니라)
3. 동작하는 결과물을 보면서 실시간으로 판단한다.
4. 어색한 부분을 코드에서 바로 수정한다.

## 바이브 코딩과의 구분

- 바이브 코딩: 뭘 만들지 모르면서 만든다.
  AI에게 프롬프트를 던지고 잘 되길 바란다.
- Direct Design: 뭘 만들지 정확히 알면서
  번역 과정을 없앤다.
  제품 감각, 사용자 이해, 엣지 케이스 파악 등
  깊은 제품 이해가 전제된다.

> "Taste now has a compiler."
