# Direct Design

AI 코딩 도구로 동작하는 제품 위에서 직접 디자인하는 방법입니다. 핸드오프 없이,
제품을 아는 사람이 코드에서 바로 만듭니다.

## 해결하는 문제

핸드오프 과정에서 의도와 맥락이 손실됩니다. 만드는 사람이 곧 판단하는 사람이면
이 문제가 사라집니다.

## 비교

| 항목        | 기존                   | Direct Design      |
| ----------- | ---------------------- | ------------------ |
| 작업 환경   | 디자인 도구에서 목업   | 코드에서 실제 제품 |
| 전달 방식   | 스펙 작성 후 핸드오프  | 핸드오프 없음      |
| 구현 주체   | 엔지니어가 해석해 구현 | 본인이 직접 구현   |
| 피드백 주기 | 수 주                  | 수 분              |

## 진행 순서

1. AI 코딩 도구에서 무엇을, 왜, 누구를 위해 만드는지 설명합니다.
2. 동작하는 결과물을 보고 판단합니다.
3. 코드에서 바로 수정합니다.
4. 2-3을 반복합니다.

## 전제 조건

- 제품 감각 — 무엇을 왜 만들어야 하는지
- 사용자 이해 — 대상과 사용 맥락
- 엣지 케이스 인식 — 예외 상황 고려

이 전제가 없으면 바이브 코딩과 구분되지 않습니다.

## vs 바이브 코딩

도구는 같지만 전제가 다릅니다. 바이브 코딩은 뭘 만들지 모르면서 AI에게 위임하고,
Direct Design은 뭘 만들지 알면서 본인이 판단합니다.

> "Taste now has a compiler."
