# 평균 사용자를 위해 디자인하지 마라

- 원문: <https://www.uxtigers.com/post/p50-vs-p95>

## 요약

P50(중앙값) 사용자가 아닌 P95(상위 5% 극단 사용자)를 고려한 UX 설계의
중요성을 다룬 글이다. 평균에 최적화된 설계가 놓치는 사용자 경험의 비용을
분석한다.

## 핵심 개념

- **P50 설계**: 가장 일반적인 케이스에 최적화. 평균은 잘 동작하지만
  극단적 상황에서 무너진다.
- **P95 설계**: 99명 중 95명을 수용하는 견고한 설계.
  극단 케이스를 처리하면 평균 케이스는 자동으로 처리된다.

## 인사이트

### 1. 평균값은 존재하지 않는 사용자를 위한 설계다

키가 175cm인 사람이 통계적으로 가장 많다고 해도,
정확히 175cm인 사람은 드물다. 평균에 최적화하면
모두에게 어느 정도 불편한 설계가 된다.

사용자 분포의 양 끝을 이해하는 것이
실제 사용 경험을 개선하는 출발점이다.

### 2. 극단 케이스는 결함이 아니라 설계 정보다

오류 상태, 빈 상태, 과부하 상태에서 무너지는 UI는
P50만 테스트한 결과다. P95 사용자가 마주하는 상황—
목록이 10,000개, 텍스트가 3줄 넘침, 네트워크가 느림—이
실제 프로덕션 환경이다.

극단 케이스를 시나리오로 등록하고 설계 단계에서 처리하면
사후 패치 비용이 줄어든다.

### 3. 소프트웨어 설계에도 같은 원칙이 적용된다

API 설계, 데이터 모델, 성능 설계 모두 P50 기준으로 짜면
예외 상황에서 취약하다. 99번째 요청, 가장 긴 입력,
동시 접속 폭증 시나리오를 미리 설계에 반영해야 한다.

UX 원칙이 시스템 설계 원칙과 일치한다.
견고함(Robustness)은 극단값을 처리하는 능력이다.
