# getdesign.md — AI 코딩 에이전트를 위한 DESIGN.md 컬렉션 사이트

<https://getdesign.md>

HN 토론: <https://news.ycombinator.com/item?id=47719485> (11점, 6개 댓글)

## 소개

getdesign.md는 유명 기업과 브랜드의 디자인 시스템을 DESIGN.md 포맷으로 정리한
큐레이션 컬렉션 사이트다.
Anthropic의 Stitch가 DESIGN.md 포맷을 대중화한 이후, 전 세계 프로젝트에서 생성된
DESIGN.md 파일들을 한곳에 모아 공개한다.
현재 70개 이상의 브랜드를 수록하고 있으며, 각 파일은 해당 브랜드의 색상 팔레트·타이포그래피·
UI 컴포넌트·미적 원칙을 문서화한다.

핵심 가치 제안은 명확하다.
프로젝트 루트에 DESIGN.md 파일 하나를 복사하면, AI 코딩 에이전트가 해당 브랜드 스타일에
맞는 UI를 자동으로 생성한다.

## 컬렉션 현황

분야별 수록 브랜드 예시:

- **테크 플랫폼**: Figma, Notion, Linear, Supabase, Vercel, Stripe
- **AI/LLM 기업**: Claude, Cohere, Mistral AI, xAI
- **소비자 브랜드**: Apple, Spotify, Nike, Starbucks, Airbnb
- **자동차**: BMW, Tesla, Ferrari, Lamborghini
- **기타**: Uber, Shopify, Raycast, Sentry, Mastercard, IBM

## 사용법

1. 원하는 브랜드의 DESIGN.md 파일을 사이트에서 다운로드한다.
2. 프로젝트 루트 디렉터리에 복사한다.
3. AI 코딩 에이전트(Cursor, Claude Code 등)에게 UI 생성을 요청한다.
4. 에이전트가 DESIGN.md를 참조해 브랜드 일관성 있는 UI를 생성한다.

## 분석

### DESIGN.md 포맷 확산의 구조적 배경

DESIGN.md가 부상한 것은 AI 코딩 에이전트의 컨텍스트 소비 방식과 관련이 있다.
시각적 디자인 파일(Figma)이나 CSS 변수는 에이전트가 직접 해석하기 어렵다.
반면 자연어로 작성된 마크다운 파일은 에이전트의 컨텍스트 윈도우에 그대로 들어간다.
Stitch가 이 패턴을 워크플로의 핵심으로 삼으면서, “AI에게 디자인 의도를 전달하는 포맷”으로서
DESIGN.md의 가치가 확인됐다.

getdesign.md는 이 워크플로를 위한 공유 라이브러리 역할을 한다.
개발자 개인이 각 브랜드의 디자인 시스템을 직접 연구하고 문서화할 필요 없이,
커뮤니티가 구축한 컬렉션을 즉시 활용할 수 있다.

### AI 생성 UI의 균질화 위험

70개 브랜드를 수록한다는 것은 선택지를 넓혀주지만, 동시에 수렴을 가속할 수 있다.
Apple, Notion, Linear 같은 인기 브랜드의 DESIGN.md가 광범위하게 사용되면,
AI가 생성하는 UI가 특정 몇 가지 스타일로 수렴하는 현상이 발생할 수 있다.
다양한 브랜드 스타일을 제공하지만, 실제로 자주 사용되는 것은 소수의 “미니멀한 테크 스타일”에
집중될 가능성이 높다.

## 비평

### 강점

별도 도구 없이 파일 하나를 복사하는 것만으로 즉시 활용 가능하다는 마찰 최소화가 강점이다.
기존 디자인 토큰이나 스타일 가이드를 AI 친화적인 포맷으로 재구성했다는 점도 실용적이다.
공식 디자인 가이드라인 접근이 어렵거나 복잡한 개인 개발자에게 특히 유용하다.

### 약점

수록된 DESIGN.md의 정확성을 보장하는 메커니즘이 없다.
제3자가 특정 브랜드의 디자인을 문서화한 것이므로, 불완전하거나 오래된 정보가
포함될 수 있다.
Ferrari나 Lamborghini 같은 고급 브랜드의 스타일을 AI 생성 UI에 복제하는 것의
법적 지위—상표권, 트레이드 드레스(trade dress) 침해 여부—는 불명확하다.

기여 모델도 투명하지 않다.
hbcondo714는 새로운 디자인을 추가하려면 유료 요청 폼을 거쳐야 하고,
CONTRIBUTING.md에도 기여 방법이 명시되어 있지 않다고 지적했다.[^hbcondo714]
커뮤니티 주도 컬렉션을 표방하면서도 기여 경로를 막아둔 것은 일관성 부족이다.

## 인사이트

### 디자인 시스템의 텍스트화가 만드는 새로운 계층

전통적인 디자인 시스템은 Figma 파일, Storybook 컴포넌트, CSS 변수 등 인간 디자이너와
개발자를 위한 포맷으로 존재했다.
DESIGN.md는 이 정보를 AI 에이전트가 소비하기 쉬운 자연어 포맷으로 변환한다.
이 전환은 단순한 포맷 변경이 아니다.
디자인 지식의 “소비자”가 인간에서 AI로 이동하는 구조적 변화를 반영한다.

그 결과 디자인 시스템의 가치 사슬이 재편된다.
“Figma를 잘 다루는 디자이너” 대신 “AI가 참조하기 좋은 디자인 문서를 잘 작성하는 사람”이
더 높은 레버리지를 갖게 된다.
getdesign.md는 이 전환에서 등장하는 새로운 자원 유형—AI용 디자인 참조 문서—의
공유 인프라 역할을 자임한다.

Finnoid는 DESIGN.md가 시각적 요소를 넘어 브랜드 톤, 카피 스타일,
아이콘·일러스트레이션 가이드까지 확장될 수 있다고 제안했다.[^Finnoid]
이 관점은 DESIGN.md의 잠재력이 UI 생성에 국한되지 않음을 보여준다.
브랜드 전체의 창작 일관성을 하나의 파일에 담을 수 있다면,
getdesign.md 같은 컬렉션은 “UI 레퍼런스”에서 “AI 창작의 공통 컨텍스트 플랫폼”으로
진화할 수 있다.

### 브랜드 모방의 의미론적 모호성

Ferrari나 Lamborghini의 DESIGN.md를 사용해 AI가 만든 UI는 시각적으로 해당 브랜드와
유사하다.
이것이 영감인지, 모방인지, 침해인지의 경계가 명확하지 않다.
전통적인 시각 디자인에서 “영감을 받은” 작업에는 창작자의 해석이 개입하지만,
AI가 DESIGN.md를 직접 구현하는 경우 그 해석의 여지는 최소화된다.
브랜드 소유자 입장에서 이것은 상표권 보호의 새로운 도전이다.

동시에, 이 사이트는 “디자인 자본의 민주화”이기도 하다.
전통적으로 대기업의 전유물이었던 정교한 디자인 시스템이 개인 개발자에게 개방된다.
Apple이나 Stripe 수준의 UI를 한 명의 개발자가 빠르게 만들 수 있다면,
디자인 자본이 더 이상 대형 기업의 경쟁 우위로 작동하지 않는 시대가 예견된다.
이 변화의 수혜자는 소규모 팀이고, 피해자는 전통적인 UI/UX 에이전시다.
