# Modern Geek News - 긱뉴스를 모던하게 바꾸는 커스텀 CSS

원문: <https://github.com/Su-Yong/modern-geek-news>

## 요약

Modern Geek News는 GeekNews(긱뉴스) 웹사이트의 인터페이스를
현대적인 디자인으로 변환하는 커스텀 CSS 프로젝트다.
"Modern Hacker News"와 "Modern Wikipedia" 프로젝트에서 영감을 받아
기존 사이트의 기능을 유지하면서 시각적 경험을 개선하는 것을 목표로 한다.

이 프로젝트는 다크 모드와 라이트 모드를 지원하며,
시스템 테마를 자동으로 감지해 적절한 모드를 적용한다.
코드 블록의 신택스 하이라이팅 색상도 사용자가 커스터마이즈할 수 있다.
100% 순수 CSS로 구현되어 있고 JavaScript는 사용하지 않는다.

설치 방법은 Stylus 브라우저 확장 프로그램(Chrome/Firefox)을 설치한 뒤
UserStyles.world(<https://userstyles.world/style/27566>)에서
스타일을 적용하는 방식이다.
Apache 2.0 라이선스로 공개되어 있고
GitHub에서 5개의 스타와 1개의 포크를 기록하고 있다.

제작자는 AI 도움 없이 CSS 연습 목적으로 직접 작성했다고 밝혔다.
My Page, Weekly, Blog 섹션은 아직 리디자인이 제한적이며
향후 업데이트가 예정되어 있다.

## 분석

### 사용자 스타일시트라는 비침습적 접근

Modern Geek News는 Stylus 확장 프로그램이 브라우저에서
특정 사이트에 사용자 정의 CSS를 주입하는 메커니즘을 활용한다.
원본 사이트의 코드를 변경하지 않으면서
시각적 경험만 교체하는 비침습적 방식이다.

### 순수 CSS만으로 구현한 테마 시스템

JavaScript 없이 다크/라이트 모드 전환을 구현하기 위해
`prefers-color-scheme` 미디어 쿼리와 CSS 커스텀 프로퍼티를 적극 활용한다.
JavaScript 의존성을 제거하면 확장 프로그램의 권한 요구도 최소화할 수 있다.

### "Modern" 시리즈의 계보

Modern Hacker News, Modern Wikipedia 등의 계보를 잇는 프로젝트다.
"기능적으로 우수하지만 디자인이 구시대적인 사이트에 현대적 스타일을 입힌다"는
커뮤니티 주도 디자인 개선 운동의 일환이다.

## 비평

### 사용자 스타일시트의 구조적 취약성

원본 사이트의 HTML 구조와 CSS 클래스에 의존하므로
GeekNews가 마크업을 변경할 때마다 스타일이 깨질 수 있다.
실제로 커뮤니티에서 좋아요/검색 아이콘이 깨지는 문제가 보고되었고,
제작자가 GeekNews CSS 업데이트를 추적해 수동으로 수정해야 했다.

### 미완성 영역의 문제

My Page, Weekly, Blog 섹션의 리디자인이 미완성이라는 점은
사이트 전체를 커버하는 사용자 스타일시트의 난이도를 보여준다.
모든 페이지와 상태를 파악하고 대응해야 하므로
개인 프로젝트로서의 유지 부담이 크다.

### 진입 장벽

Stylus 확장 프로그램 설치가 전제 조건이라는 점은
일반 사용자에게 진입 장벽이 될 수 있다.
GeekNews 댓글에서 jhk0530이 Chrome 확장 프로그램 변환을 제안한 이유다.
다만 확장 프로그램화하면 "100% CSS"라는 정체성과 충돌한다.

## 인사이트

### 사용자 스타일시트 - 웹의 잊혀진 민주주의

웹의 초기 설계에서 사용자 스타일시트는
"콘텐츠 소비자도 표현에 대한 권한을 가져야 한다"는
철학의 구현이었다.
CSS의 캐스케이드 자체가 작성자 스타일과 사용자 스타일 사이의
우선순위 협상을 위해 설계된 것이다.

이 철학은 웹이 플랫폼화되면서 거의 사라졌다.
브라우저들은 사용자 스타일시트 기본 지원을 축소했고,
사용자는 사이트가 제공하는 디자인을 그대로 소비하는 존재가 되었다.
Stylus 같은 확장 프로그램은 이 사라진 권한을
브라우저 확장 생태계를 통해 복원하는 도구다.

Modern Geek News는 이 복원의 구체적 사례다.
사용자가 자주 방문하는 사이트의 시각적 경험을
직접 재설계할 수 있다는 것은 웹의 원래 약속에 가깝다.
단순한 테마 변경이 아니라 "누가 인터페이스를 결정하는가"에 대한 실천적 답변이다.

### "AI 비사용 선언"이 갖는 의미

제작자가 "AI 도움 없이" 작성했다고 명시한 것은
AI 코드 생성 도구가 보편화된 환경에서 점점 더 의미 있는 선언이 되고 있다.
"직접 작성했다"는 것은 결과물이 아니라 학습 과정 자체에 가치를 둔다는 표명이다.

CSS는 특히 AI 생성과 수동 작성의 차이가 큰 영역이다.
캐스케이드, 특이성(specificity), 상속 규칙은
코드를 읽는 것만으로는 체득하기 어렵고
직접 예상과 다른 결과를 마주하며 디버깅하는 과정에서 학습된다.
사용자 스타일시트 제작은 기존 사이트의 복잡한 CSS와
충돌하고 협상하는 경험을 제공하므로
CSS의 핵심 메커니즘을 이해하는 데 특히 효과적인 학습 방법이다.

이 관점에서 Modern Geek News의 가치는
완성도 높은 디자인 결과물보다
제작 과정에서 축적된 CSS 역량에 있다.
특이성 전쟁을 해결하고, 미디어 쿼리로 테마를 분기하고,
다른 사람의 마크업 위에 자신의 디자인 의도를 구현하는 경험은
프론트엔드 엔지니어의 기초 체력에 해당한다.

### 기생 프로젝트의 구조적 딜레마

사용자 스타일시트 프로젝트는 본질적으로 "기생적" 구조를 갖는다.
대상 사이트의 HTML과 CSS에 의존하지만
해당 사이트의 변경에 대한 통제권은 전혀 없다.
GeekNews가 클래스명 하나를 바꾸거나 레이아웃을 조정하면
Modern Geek News의 스타일은 즉시 깨질 수 있다.

이 구조적 취약성은 커뮤니티 피드백에서 이미 현실화되었다.
대상 사이트가 활발히 개발 중일수록
사용자 스타일시트의 유지보수 부담은 기하급수적으로 증가한다.
이 딜레마는 브라우저 확장, 유저스크립트 등
모든 서드파티 커스터마이제이션 프로젝트가 공유하는 문제다.

장기적 지속 가능성을 확보하려면
대상 사이트가 공식 테마 API나 안정적인 CSS 클래스 계약을
제공하는 것이 이상적이다.
하지만 현실에서 이를 제공하는 사이트는 드물다.
역설적으로 사용자 스타일시트 문화가 활성화되면
사이트 운영자도 의미론적 마크업을 더 신경 쓰게 되어
웹 표준 준수가 촉진되는 선순환이 가능하다.
