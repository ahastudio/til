# CodeGraph

<https://github.com/colbymchenry/codegraph>

## 소개

CodeGraph는 AI 코딩 에이전트(Claude Code, Cursor, Codex 등)를 위한 코드 지식 그래프 도구다.
코드베이스를 미리 인덱싱해 심볼 관계, 호출 그래프, 프레임워크 라우팅을 SQLite에 저장하고,
에이전트가 파일을 반복 탐색하지 않고 한 번의 도구 호출로 필요한 컨텍스트를 조회하게 한다.

로컬 전용으로 동작하며 외부 API나 API 키가 필요 없다.
파일 감시자(FSEvents/inotify/ReadDirectoryChangesW)로 변경 사항을 자동 동기화한다.

## 성능 벤치마크

7개 실제 코드베이스 대상 중앙값 기준:

| 지표           | 개선율  |
| -------------- | ------- |
| 비용 절감      | 35%     |
| 토큰 감소      | 57%     |
| 실행 속도 향상 | 46%     |
| 도구 호출 감소 | 71%     |

대형 저장소에서 효과가 두드러진다.
VS Code(TypeScript, ~10k 파일) 기준으로는 비용 26% 절감, 토큰 78% 감소, 속도 52% 향상, 도구 호출 85% 감소를 기록했다.

## 주요 기능

**스마트 컨텍스트 빌드**: `codegraph_context` 한 번 호출로 진입점, 연관 심볼, 코드 스니펫을 반환한다.

**전체 텍스트 검색**: FTS5 기반으로 심볼명을 코드베이스 전체에서 즉시 검색한다.

**영향 분석**: `codegraph_impact`, `codegraph_callers`, `codegraph_callees`로 변경 전 영향 범위를 추적한다.

**크로스 언어 브릿지**: Swift↔Objective-C, React Native TurboModule, Expo Modules, Fabric 컴포넌트 등 정적 파싱이 놓치는 흐름을 처리한다.

**프레임워크 라우팅 인식**: Django, Flask, FastAPI, Express, NestJS, Laravel, Rails 등에서 URL 패턴을 핸들러에 연결한다.

## 지원 언어 및 통합

20개 이상의 언어(TypeScript, Python, Go, Rust, Java, C#, PHP, Ruby, Swift, Kotlin, Dart 등)와
Svelte, Vue, Liquid, Lua 전용 지원을 제공한다.

MCP 서버를 통해 Claude Code, Cursor, Codex CLI, Gemini CLI, opencode 등에 통합된다.

## 설치

```bash
# macOS/Linux
curl -fsSL https://raw.githubusercontent.com/colbymchenry/codegraph/main/install.sh | sh

# npm 대안
npx @colbymchenry/codegraph
```

설치 후 `codegraph init -i`로 프로젝트를 초기화하면 에이전트 설정이 자동으로 구성된다.

## 분석

### 에이전트 비용 구조에 대한 구조적 접근

AI 코딩 에이전트의 비용 대부분은 컨텍스트 수집 과정에서 발생한다.
에이전트가 코드베이스를 이해하려면 파일을 열고, 심볼을 찾고, 관련 파일을 다시 여는 과정을 반복한다.
이 “탐색 오버헤드”는 실제 추론 비용보다 클 수 있다.
CodeGraph는 이 탐색 단계를 사전 인덱싱으로 대체해 비용 구조 자체를 바꾼다.
[Dirac](dirac.md)이 컨텍스트 큐레이션으로 토큰을 줄여 비용 64.8%를 절감한 것과 같은 문제 — “모델이 받는 컨텍스트의 정밀도가 비용을 결정한다” — 를 인덱싱 측면에서 공략한 셈이다.

### MCP를 활용한 에이전트 도구 통합

CodeGraph가 MCP 서버 형태로 배포된다는 점은 중요한 설계 결정이다.
에이전트마다 별도 플러그인을 만드는 대신 MCP 표준을 통해 단일 인터페이스로 여러 에이전트를 지원한다.
이는 도구 생태계가 에이전트 독립적으로 발전할 수 있는 패턴이다.

### 정확도 대비 환경 가정

[GeekNews 댓글](https://news.hada.io/topic?id=29873)에서 제기되었듯,
그래프 기반 도구의 실질적 적용 범위가 의문이다.
동적 언어, 메타프로그래밍, 런타임 의존성이 많은 코드베이스에서는
정적 분석 기반 그래프가 실제 관계를 충분히 표현하지 못할 수 있다.

## 비평

### 강점: 에이전트 워크플로우에 최적화된 설계

파일 탐색 → 심볼 검색 → 영향 분석이라는 에이전트의 실제 작업 흐름에 맞춰 API가 설계되어 있다.
단순히 코드 검색 도구를 MCP로 감싼 것이 아니라,
에이전트가 실제로 필요한 정보를 최소 호출로 제공하는 방식에 집중했다.

### 약점: 동적 언어에서의 한계

Python, Ruby 같은 동적 언어에서 런타임 동작이 많은 코드베이스는
정적 심볼 그래프만으로 전체 관계를 표현하기 어렵다.
벤치마크가 TypeScript 위주라는 점도 주목할 필요가 있다.

### 벤치마크 신뢰성

성능 수치가 인상적이나, 자체 보고된 벤치마크다.
실제 효과는 코드베이스 특성, 에이전트 설정, 작업 유형에 크게 의존할 것이다.

## 인사이트

### 에이전트 비용의 숨겨진 구성 요소

AI 코딩 에이전트를 도입한 팀이 가장 먼저 마주하는 것은 토큰 비용이다.
그런데 이 비용의 상당 부분은 추론 자체가 아니라 컨텍스트 수집에서 온다.
에이전트가 `grep`, `read`, `find`를 반복하며 코드베이스를 이해하는 과정이 비용의 주요 원천이다.
CodeGraph 같은 도구는 이 탐색 비용을 사전 인덱싱 비용으로 전환한다.
한 번 인덱싱하고 매번 조회하는 구조다.
이것은 데이터베이스 인덱스와 정확히 같은 트레이드오프다.
에이전트 비용 최적화의 다음 전선이 여기에 있다.

### 코드 지식 그래프는 새로운 개발 인프라가 될 수 있다

데이터베이스, 버전 관리, CI/CD처럼 코드 지식 그래프가 개발 인프라의 표준 요소가 될 가능성이 있다.
에이전트가 코드를 이해하는 방식이 파일 기반 탐색에서 그래프 기반 질의로 전환된다면,
이 그래프를 유지하고 최신 상태로 관리하는 것이 팀의 기본 운영 과제가 된다.
지금은 옵션 도구지만, 에이전트 사용이 심화될수록 필수 인프라로 격상될 수 있다.
[AI가 읽을 수 있는 코드베이스](ai-readable-codebase.md)가 강조하는 “에이전트를 위한 코드 구조화”는 그래프 인덱싱과 같은 방향의 노력이다.

### MCP 표준화가 만드는 에이전트 도구 시장

CodeGraph가 단일 에이전트 플러그인이 아니라 MCP 서버로 출시된 것은 시장 전략적으로도 영리하다.
MCP를 지원하는 에이전트라면 어디서든 작동한다는 것은,
개별 에이전트의 성패와 무관하게 도구의 가치가 유지된다는 의미다.
MCP가 에이전트 도구 생태계의 표준으로 자리잡으면,
도구 제작자는 에이전트 특화 통합 대신 MCP 준수에 집중할 인센티브를 갖는다.
이것은 에이전트 시장을 OS처럼 추상화하는 움직임의 일환이다.
반면 [Dirac](dirac.md)이 MCP를 의도적으로 거부하고 네이티브 도구 호출만 쓰는 결정은, 표준화가 토큰 효율과 충돌할 때 어떤 트레이드오프가 생기는지를 드러낸다.
