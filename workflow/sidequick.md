# SideQuick

<https://www.sidequick.co/>

## 소개

SideQuick은 사이드 프로젝트를 끝까지 완주하도록 돕는 데스크톱 애플리케이션이다.
개발자가 134개의 GitHub 저장소를 방치한 경험에서 착안해 만든 도구로,
퀘스트(Quest) 기반 작업 분해와 스트릭(Streak) 추적을 결합해 일관성 있는 작업 흐름을 지원한다.

macOS(ARM64 및 Intel), Windows, Linux를 지원하며 무료로 제공된다.
계정이나 구독이 필요 없고, 모든 데이터는 로컬에만 저장된다.

## 주요 기능

**퀘스트 시스템**: 프로젝트를 순서대로 잠금 해제되는 단계적 작업 트리로 구성한다.
작업을 직접 추가하거나 AI가 생성할 수 있으며, 이전 단계를 완료해야 다음 단계로 진행할 수 있다.

**스트릭 추적**: 매일 일관된 참여를 유도하는 연속 달성 카운터와 마일스톤을 제공한다.
소셜 경쟁이 아닌 개인의 진행 상황에만 집중하도록 설계됐다.

**재진입 요약(Re-entry Summary)**: 오랜만에 프로젝트로 돌아올 때 AI가 지난 진행 상황과
다음 할 일을 요약해 제공한다. 맥락을 되찾는 데 드는 인지적 마찰을 줄이는 것이 목적이다.

**포커스 모드**: 작업 중 불필요한 인터페이스 요소를 숨겨 집중력을 높인다.

**XP 및 레벨 시스템**: 경험치와 레벨을 정직한 진행 지표로 사용한다.

## AI 통합

AI 기능을 사용하려면 Anthropic 또는 OpenAI API 키를 직접 제공해야 한다.
Ollama, Groq, LM Studio 같은 OpenAI 호환 엔드포인트도 지원한다.
AI를 사용하지 않아도 핵심 기능은 모두 동작한다.

## 분석

SideQuick은 프로젝트 완수 실패의 근본 원인을 “재진입 마찰”로 진단한다.
어디까지 했는지 기억하고 다시 맥락을 잡는 데 드는 비용이
프로젝트를 열지 않게 만드는 주요 원인이라는 것이다.
퀘스트 잠금 순서와 AI 재진입 요약이 이 문제를 직접 해결하는 두 가지 핵심 메커니즘이다.

게임화(gamification) 요소는 기존 생산성 도구에서 이미 흔하지만,
SideQuick의 차별점은 이를 순수 개인 지표로만 사용한다는 점이다.
외부 압박 없이 내재적 동기를 유지하는 방향을 택한 설계 결정이다.

## 비평

퀘스트 순서 잠금은 탐색적이고 비선형적인 작업보다 단계가 명확한 프로젝트에 더 적합하다.
소프트웨어 개발처럼 초기 계획이 자주 변경되는 경우, 잠금 순서를 재조정하는 작업 자체가 마찰을 만들 수 있다.

AI 재진입 요약은 유용하지만, API 키를 직접 관리해야 하는 설정이
비기술적 사용자에게는 진입 장벽이 될 수 있다.
“계정 불필요”와 “AI 기능 사용”이 상충하는 지점이다.

[GeekNews 댓글](https://news.hada.io/topic?id=29685)에서 가장 많이 등장한 반응은
“134개 저장소를 방치한 사람이 이건 완성했다”는 아이러니였다.
이는 도구 자체의 설득력을 증명하는 동시에, 사이드 프로젝트 완수 문제가
얼마나 보편적인 공감대를 형성하는지를 보여준다.

## 인사이트

### 도구 제작자의 역설

SideQuick 제작자는 134개의 미완성 프로젝트에서 영감을 받아 새로운 프로젝트를 시작했다.
이는 생산성 도구 생태계의 고질적인 역설을 드러낸다.
생산성 문제를 해결하려는 행위 자체가 그 문제의 또 다른 표현이 될 수 있다는 것이다.
개발자는 문제를 만나면 도구로 해결하려는 성향이 강하기 때문에,
진짜 문제가 도구 부재가 아닌 습관과 동기의 문제일 때도 도구부터 만들게 된다.

그러나 이 경우에는 도구를 실제로 완성했다는 사실이 역설을 해소한다.
SideQuick은 이 패턴 자체를 인식하고 이를 투명하게 제품 이야기로 만든 점에서 영리하다.
자신의 실패를 제품의 동기로 삼아 사용자와 즉각적인 공감대를 형성하는 마케팅 전략이기도 하다.
“134개”라는 구체적인 숫자가 공감을 강화하는 동시에 문제의 심각성을 전달한다.

### 재진입 마찰이라는 과소평가된 문제

많은 생산성 연구는 작업 시작의 어려움에 집중하지만,
재개(resumption)의 어려움은 상대적으로 덜 조명된다.
비정기적으로 작업하는 사이드 프로젝트에서는 재진입 비용이 지속성의 가장 큰 장벽이다.
어디까지 했는지, 왜 그 방식으로 설계했는지를 파악하는 데 30분이 걸린다면
짧은 시간 창에서 프로젝트를 열 동기가 사라진다.

AI 재진입 요약은 이 문제를 처음 제품 수준에서 해결하려는 시도다.
코드 편집기의 AI 어시스턴트가 세션 간 맥락을 기억하는 방향으로 발전하는 트렌드와도 일치한다.
앞으로 개발 환경 자체가 “세션 연속성” 기능을 기본으로 제공하는 방향으로 진화할 가능성이 높다.
SideQuick은 이 흐름의 초기 형태를 독립적인 생산성 도구로 구현했다.

### 로컬 우선 설계의 신뢰 자본

계정 없이, 서버 전송 없이, 구독 없이 무료로 제공하는 SideQuick의 모델은
개인 생산성 도구 시장에서 점점 중요해지는 신뢰 경제의 반영이다.
Notion, 각종 SaaS 생산성 도구에 데이터를 맡기는 것에 피로감을 느끼는 사용자들이
로컬 우선 도구로 이동하는 흐름이 가속화되고 있다.

무료 + 로컬 모델은 장기적인 지속 가능성 문제를 내포한다.
스폰서십, 유료 기능 추가, 또는 프리미엄 전환이 불가피한 시점이 오는데,
이 전환 자체가 초기 사용자의 신뢰를 시험하는 순간이 된다.
초기 무료 약속을 지키면서도 지속 가능한 비즈니스 모델을 찾는 것이
이런 도구들의 공통 과제다.
