# serve-sim: Apple 시뮬레이터 원격 호스팅 도구

<https://github.com/EvanBacon/serve-sim>

## 소개

`serve-sim`은 iOS 시뮬레이터를 브라우저와 AI 에이전트가 원격으로 접근할 수
있도록 호스팅하는 CLI 도구다.
Expo 핵심 기여자인 Evan Bacon이 만든 이 도구는 스스로를 “Apple 시뮬레이터의
`npx serve`”라고 표방한다.
로컬, LAN, 터널링을 통한 원격 공유를 지원하며, Claude Desktop, Cursor, Codex
같은 AI 에이전트 도구와의 통합을 주요 사용 시나리오로 제시한다.

## 명세

### 아키텍처

세 컴포넌트로 구성된다.

1. **Swift 헬퍼 바이너리** (`serve-sim-bin`): `simctl io`를 통해 시뮬레이터
   프레임버퍼를 캡처
2. **MJPEG 스트림 서버**: WebSocket 제어 채널을 포함한 영상 스트림 전송
3. **React 프리뷰 UI**: 기본 포트 `localhost:3200`으로 제공

헬퍼는 디바이스 단위 데몬으로 실행되며, 개별 앱 실행보다 긴 수명을 갖는다.
이 덕분에 앱 재시작 없이 카메라 소스를 핫스왑(hot-swap)할 수 있다.

### 주요 기능

- 60 FPS MJPEG 영상 스트리밍
- 핀치-투-줌, 스와이프 등 제스처 지원
- Apple Watch, iPad, iOS 디바이스 호환
- MCP 도구를 위한 시뮬레이터 로그 브라우저 포워딩
- 영상·이미지 드래그 앤 드롭 미디어 핸들링
- `CMD+SHIFT+H` 홈 버튼 등 키보드 단축키 포워딩
- 카메라 인젝션(플레이스홀더, 웹캠, 파일 소스)

### 카메라 인젝션

DYLD 라이브러리 인젝션으로 AVFoundation을 스위즐링(swizzle)해서 앱에
합성 카메라 피드를 제공한다.
지원 소스는 애니메이션 플레이스홀더, 로컬 이미지·영상 파일, 내장·외장 웹캠이다.

### 통합 방식

| 환경             | 방법                                   |
| ---------------- | -------------------------------------- |
| Claude Code      | `.claude/launch.json` 설정             |
| Expo             | Metro 설정에 미들웨어 통합             |
| 일반 서버        | Connect 스타일 미들웨어로 기존 인프라에 연결 |

### 요구 사항

- macOS + Xcode 커맨드라인 도구 (`xcrun simctl`)
- Node.js 18 이상

## 분석

### AI 에이전트 + 시뮬레이터 통합의 의미

`serve-sim`이 Claude Desktop, Cursor, Codex와의 통합을 전면에 내세우는 것은
AI 에이전트가 실제 UI를 “보고” 상호작용하는 테스트 패러다임의 등장을 반영한다.
기존 자동화 테스트(XCUITest, Detox)는 접근성 트리나 셀렉터를 통해 UI를 조작했다.
`serve-sim`은 시각 스트림과 제스처 채널을 제공함으로써, AI 에이전트가 사람처럼
화면을 보고 손가락으로 조작하는 방식을 가능하게 한다.

### DYLD 인젝션과 AVFoundation 스위즐링

카메라 인젝션에 DYLD 라이브러리 인젝션과 메서드 스위즐링을 쓰는 것은
macOS/iOS 개발에서 매우 낮은 수준의 기법이다.
`DYLD_INSERT_LIBRARIES` 환경 변수로 공유 라이브러리를 프로세스에 강제 삽입하고,
Objective-C 런타임의 메서드 교체(swizzle)를 통해 AVFoundation의 카메라
인터페이스를 가로챈다.
이 기법은 시뮬레이터 환경에서는 동작하지만, 실제 디바이스에서는 코드 서명과
SIP(System Integrity Protection)로 차단된다.
테스트 전용 도구임을 명확히 하는 설계다.

### “npx serve” 비유의 전략적 의미

`npx serve`는 정적 파일을 한 줄 명령어로 공유하는 도구로, 개발자 사이에서
즉각적인 이해를 유도하는 레퍼런스다.
“시뮬레이터의 npx serve”라는 표현은 복잡한 기술 설명 없이 도구의 목적과
편의성을 전달한다.
단 하나의 비유로 대상 독자(JS/React Native 개발자), 사용 목적(공유·접근),
편의성(한 줄 실행)을 동시에 전달하는 효율적인 포지셔닝이다.

## 비평

### 강점: 계층화된 통합 전략

Claude Desktop, Expo, 일반 Connect 미들웨어를 동시에 지원하는 것은 채택
진입 장벽을 낮추는 영리한 설계다.
Expo 프로젝트에서는 Metro 설정 한 줄로 통합되고,
이미 Connect 기반 서버가 있다면 미들웨어로 얹을 수 있다.
특정 도구 스택에 종속되지 않으면서도 각 환경에 자연스럽게 녹아드는 구조다.

### 강점: 데몬 아키텍처로 핫스왑 실현

헬퍼 바이너리를 앱 수명이 아닌 디바이스 단위 데몬으로 분리한 것은 실용적인
설계 결정이다.
카메라 소스를 바꿀 때마다 앱을 재시작하는 마찰이 없어진다.
테스트 자동화 시나리오에서 다양한 카메라 입력을 연속으로 시험할 때 유용하다.

### 약점: 시뮬레이터 전용 한계

DYLD 인젝션과 `simctl` 의존성은 실제 디바이스에서는 작동하지 않는다.
실제 디바이스의 카메라 동작, 네트워크 조건, 하드웨어 가속은 시뮬레이터와
다르며, 이 도구로 커버할 수 없는 테스트 영역이 존재한다.
“시뮬레이터의 npx serve”라는 표현이 이 한계를 자연스럽게 숨기고 있다.

### 약점: 보안 경계 불명확

원격 터널링으로 시뮬레이터를 외부에 노출할 때의 보안 고려사항이 문서에 없다.
시뮬레이터 안에는 개발용 자격증명, 테스트 계정, 민감한 디버그 데이터가 있을 수 있다.
기업 환경에서 CI 파이프라인에 통합할 때 어떤 네트워크 격리가 필요한지에 대한
안내가 필요하다.

## 인사이트

### 시각 채널을 가진 AI 에이전트가 바꾸는 테스트의 정의

기존 UI 자동화는 DOM이나 접근성 트리 같은 구조화된 표현에 의존했다.
Selenium, XCUITest, Playwright 모두 “어떤 버튼이 어디 있는지”를 구조적으로
기술하고 그것을 찾아 클릭한다.
`serve-sim`이 MJPEG 스트림을 AI 에이전트에게 제공한다는 것은 전혀 다른 접근이다.
AI는 픽셀을 보고, 무엇이 보이는지 판단하고, 그에 따라 행동한다.

이 접근의 함의는 크다.
구조화된 자동화는 UI가 변경될 때마다 셀렉터가 깨진다.
비전 기반 자동화는 레이아웃이 바뀌어도 “로그인 버튼”을 찾을 수 있다.
테스트가 UI 구현 세부사항에 덜 결합되고, 사용자 경험에 더 가깝게 정의된다.
이것은 테스트 작성 방식의 패러다임 전환이다.

그러나 이 접근은 새로운 불확실성을 도입한다.
픽셀 기반 판단은 비결정론적이다.
같은 화면도 렌더링 조건에 따라 다르게 보일 수 있고,
AI의 “이것이 버튼이다”라는 판단이 항상 옳지 않을 수 있다.
자동화 테스트의 핵심 가치인 재현성과 신뢰도를 어떻게 확보할 것인지가
이 패러다임의 핵심 과제다.

### DYLD 인젝션이 드러내는 동적 링킹의 힘과 위험

`serve-sim`의 카메라 인젝션은 `DYLD_INSERT_LIBRARIES`를 사용해 실행 중인
프로세스에 코드를 삽입한다.
이것은 macOS 고유의 강력한 기능이지만, 동시에 역사적으로 악성 소프트웨어가
즐겨 쓰던 기법이기도 하다.
Apple이 SIP와 강화된 코드 서명 정책으로 이 기법을 점점 제한하는 이유가 여기 있다.

`serve-sim`이 시뮬레이터에서만 동작하도록 설계된 것은 이 긴장 관계를 인식한
결과다.
시뮬레이터는 SIP 보호 밖에 있고, 개발용 신뢰 컨텍스트 안에 있다.
이 경계를 명확히 함으로써 강력한 기능을 안전하게 노출한다.
도구 설계에서 “어디까지 허용하고 어디서 막는가”의 경계를 의도적으로 그은 것이다.

소프트웨어 역사에서 이런 기법은 디버거, 프로파일러, 테스트 프레임워크가
반복적으로 사용해 왔다.
`LD_PRELOAD`(Linux), `DYLD_INSERT_LIBRARIES`(macOS)는 동적 링커가 제공하는
합법적인 확장점이다.
이 기법을 알고 활용하는 개발자와 모르는 개발자 사이에는 문제 해결 역량에
큰 차이가 생긴다.
`serve-sim`은 이 기법의 실용적 적용 사례로서 교육적 가치도 있다.

### “npx serve” 계보가 보여주는 개발 도구의 민주화 패턴

`npx serve`(정적 파일), `ngrok`(로컬 터널링), `localtunnel`(공개 URL),
그리고 이제 `serve-sim`(시뮬레이터)까지, 개발 도구의 역사는 “전문가만 할 수
있던 것을 한 줄 명령어로”라는 방향으로 움직여 왔다.
이 패턴의 공통점은 복잡한 인프라 설정을 추상화해서, 본래 문제(앱 개발)에
집중할 수 있게 한다는 것이다.

`serve-sim`의 맥락에서 이것은 특히 의미 있다.
원격 디바이스 팜(device farm) 서비스는 기업 계약과 높은 비용을 요구한다.
`serve-sim`은 이미 가진 맥북과 시뮬레이터로 동일한 원격 접근 경험을 제공한다.
스타트업이나 개인 개발자가 CI 파이프라인에 AI 기반 시각 테스트를 통합하는
비용 장벽을 낮춘다.

이 민주화 패턴은 결국 생태계를 확장한다.
비싼 도구에만 접근할 수 있었던 기능이 누구나 쓸 수 있게 되면,
그 기능의 활용 방법이 다양해지고 새로운 응용이 등장한다.
`ngrok`이 webhook 개발을 대중화했듯, `serve-sim`이 AI 에이전트 기반
모바일 테스트를 대중화할 수 있는 포지션에 있다.
