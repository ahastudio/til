# Turbo Vision 2.0

<https://github.com/magiblot/tvision>

## 소개

Turbo Vision 2.0은 1990년대 Borland가 만든 텍스트 UI 프레임워크를
현대 환경에서 동작하도록 포팅한 오픈소스 프로젝트다.
2018년 말부터 개발을 시작해 2020년 5월 오픈소스로 공개됐다.

슬로건은 “같은 코드로 Linux, Windows, DOS 모두에서 동작”이다.
원본 1990년대 Turbo Vision 소스 코드와 하위 호환성을 유지하면서
UTF-8 유니코드, 24비트 트루 컬러, 마우스 입력, 시스템 클립보드 같은
현대적 기능을 추가했다.

이 프레임워크로 만들어진 프로젝트로는 Turbo(텍스트 에디터),
tvterm(터미널 에뮬레이터), TMBASIC(콘솔 프로그래밍 언어)이 있다.

## 기능

### 위젯 시스템

- 크기 조절·겹침 가능한 윈도우
- 풀다운 메뉴
- 다이얼로그 박스, 버튼, 스크롤바, 입력 박스, 체크박스, 라디오 버튼
- 중간 마우스 버튼 지원 및 향상된 스크롤바 조작

### 문자·색상 지원

- UTF-8 전체 지원: 더블와이드 문자(CJK), 조합 문자, 이모지
- 24비트 RGB 트루 컬러
- 256색 xterm 팔레트
- 터미널 기본 색상 및 Bold, Italic, Underline 같은 스타일 어트리뷰트
- 제한된 터미널을 위한 자동 색상 양자화

### 플랫폼 지원

| 플랫폼   | 구현 방식                              | 비고                                   |
| -------- | -------------------------------------- | -------------------------------------- |
| Linux    | Ncurses, GPM 마우스                    | SGR, Kitty 프로토콜, far2l 확장 지원   |
| Windows  | Win32 Console API                      | UTF-8 자동 모드                        |
| DOS      | Borland C++ 하위 호환                  | 유니코드 미지원                        |

최대 32,767행×열의 대형 화면도 지원한다.

## 분석

### “포팅”이 아니라 “현대화”다

단순한 플랫폼 포팅이라면 원본 기능을 그대로 옮기는 것으로 충분하다.
Turbo Vision 2.0은 여기서 더 나아가 24비트 컬러, UTF-8, 마우스 지원,
클립보드 통합 같은 기능을 추가했다.
이것은 원본 설계의 정신을 유지하면서 30년 이상의 기술 진보를 통합하는 작업이다.

`#ifdef`로 가득 찬 플랫폼별 분기 없이 동일한 코드가
Linux, Windows, DOS에서 작동하는 설계는
“포팅”보다 “추상화 레이어 구축”에 가깝다.

### TUI 르네상스의 맥락

2020년대는 TUI(Terminal UI) 르네상스다.
Neovim, Lazygit, Helix, Ghostty, WezTerm 같은 TUI 도구들이
개발자 커뮤니티에서 다시 인기를 얻고 있다.
SSH 환경, 서버 관리, 저대역폭 상황, 리소스 제약 환경에서
TUI의 실용성은 여전히 유효하다.

Turbo Vision 2.0은 이 르네상스에서 30년 전 설계가
여전히 경쟁력 있음을 보여주는 사례다.
위젯 시스템, 이벤트 루프, 레이아웃 관리의 핵심 아이디어는
시간을 초월한다.

### 하위 호환성이 가져오는 복리 효과

원본 1990년대 소스 코드와 호환성을 유지한다는 것은
30년간 Turbo Vision으로 만들어진 모든 레거시 소프트웨어가
현대 플랫폼에서 다시 살아날 수 있다는 의미다.
실제로 이를 위해 포팅하는 프로젝트들이 존재한다.

이것은 오픈소스 프로젝트가 “과거를 위한 다리”와
“새 프로젝트를 위한 기반” 두 가지 역할을 동시에 한다는 점에서
특이한 포지션이다.

## 비평

### 강점: 검증된 설계 위에 현대성을 더한다

1990년대 Turbo Vision은 수백만 개의 DOS 프로그램에서 사용된
전투 검증된 TUI 프레임워크다.
이 유산 위에 현대적 기능을 추가하는 것은
바퀴를 재발명하지 않고 기존의 것을 개선하는 접근이다.

GeekNews 댓글에서 macOS에서 .NET 래퍼를 만든 개발자가
“꽤 마법 같다”고 표현한 것은
이 프레임워크가 실제로 사용성이 높음을 보여준다.

### 강점: 크로스 플랫폼 추상화가 깔끔하다

플랫폼별 `#ifdef` 없이 동일한 코드로 여러 플랫폼에서 동작하는 것은
현대 TUI 라이브러리들이 종종 실패하는 지점이다.
Terminal.Gui 같은 .NET TUI 라이브러리와 비교해
Turbo Vision이 플랫폼 추상화에서 낫다는 평가도 있다.

### 약점: C++ 기반으로 진입 장벽이 높다

현대 개발자들은 Rust(Ratatui), Go(tview), Python(Textual)으로
TUI를 만드는 경향이 있다.
C++ 기반 프레임워크는 이 생태계에서 접근성이 낮다.
바인딩이 있어야 다른 언어에서 사용할 수 있지만,
공식 바인딩 생태계가 풍부하지 않다.

### 약점: 문서와 학습 자료가 부족하다

1990년대 Borland 시절 문서를 기반으로 한다는 것은
현대적인 시작 예제, 튜토리얼, API 문서가 부족할 수 있음을 의미한다.
소규모 커뮤니티가 이 공백을 메우기 어렵다.

## 인사이트

### 30년 된 UI 철학이 AI 터미널 시대에 다시 유효해진다

1990년대 TUI 프레임워크가 2020년대에 부활하는 것은
단순한 레트로 취향이 아니다.
AI 코딩 에이전트, SSH 원격 개발, 컨테이너 기반 환경이 확산되면서
브라우저 없이 터미널만으로 완결되는 개발 환경의 수요가 다시 증가하고 있다.

Claude Code 같은 AI 코딩 도구가 터미널에서 동작하는 것이 디폴트가 된 것은
이 트렌드의 단적인 예다.
TUI의 강점—가볍고, SSH로 동작하고, 렌더링 비용이 낮고, 스크린 리더와 호환된다—이
AI 에이전트 시대에 다시 주목받는다.
Turbo Vision 2.0은 이 수요를 위한 견고한 기반을 제공한다.

### “원본 설계를 죽이지 않는 포팅”이 오픈소스 유산 보존의 모델이 된다

방대한 레거시 소프트웨어 생태계가 DOS/Windows 3.1 시대의 TUI 프레임워크에
의존하고 있다. 이것들이 현대 플랫폼에서 동작하지 않으면 버려진다.

Turbo Vision 2.0이 보여주는 것은 원본 API를 유지하면서
플랫폼 지원을 확장하는 것이 가능하다는 것이다.
이것은 오픈소스 생태계에서 중요한 보존 모델이다.
DOSBox가 DOS 소프트웨어를 에뮬레이션으로 보존한다면,
Turbo Vision 2.0은 네이티브 동작으로 보존한다.

두 전략의 차이는 의미가 있다.
에뮬레이션은 원본 동작을 그대로 재현하지만 성능과 통합에 한계가 있다.
네이티브 포팅은 현대 OS와 직접 통합되어 새로운 기능을 추가할 수 있다.
Turbo Vision 2.0의 접근이 레거시 코드베이스 현대화의 더 좋은 모델일 수 있다.

### 텍스트 기반 UI의 접근성은 그래픽 UI가 영원히 대체하지 못한다

그래픽 UI가 지배하는 세상에서 TUI가 살아남는 이유는 접근성이다.
스크린 리더, 저대역폭 연결, 리소스 제약 환경, 자동화 파이프라인에서
텍스트 기반 인터페이스는 그래픽 UI가 따라오지 못하는 영역을 가진다.

Turbo Vision 2.0의 24비트 컬러와 UTF-8 지원은
“텍스트 기반이지만 현대적인 표현력”을 모두 갖추겠다는 의지다.
이모지와 CJK 문자를 올바르게 처리하는 TUI 프레임워크는
아직도 드물다.
이 조합이 Turbo Vision 2.0을 단순한 레거시 포팅이 아니라
현대적 TUI 개발의 실제 선택지로 만든다.
