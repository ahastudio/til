# winclipshot - Windows 터미널에서 Ctrl+V로 스크린샷 붙여넣기

원문: <https://github.com/Higangssh>

## 요약

winclipshot은 Windows 클립보드의 스크린샷 이미지를
터미널에서 `Ctrl+V`로 붙여넣을 수 있게 해주는 유틸리티다.
Go로 작성되었으며, 이벤트 기반 아키텍처로 동작한다.

동작 방식:

1. 클립보드에 이미지가 들어오면 현재 포커스된 앱을 확인
2. **터미널**(Windows Terminal, PowerShell, cmd, Alacritty, WezTerm):
   이미지를 PNG로 저장하고 클립보드 내용을 파일 경로로 교체
3. **그 외 앱**(Slack, Chrome, Word 등): 이미지를 그대로 유지

Claude Code 등 터미널 기반 AI 도구에서 스크린샷을 참조할 때
파일 경로를 수동으로 입력하는 번거로움을 제거하는 것이 주요 목적이다.
CPU 사용률 ~0%, 메모리 ~15MB로 가볍게 상주한다.

## 분석

### Claude Code 스크린샷 불안정성의 우회

Claude Code는 `Ctrl+V`로 스크린샷 붙여넣기를 지원하지만 간헐적으로 실패한다.
winclipshot은 이미지 대신 파일 경로를 클립보드에 넣어
이 불안정성을 우회하는 접근이다.

### 포커스 앱 기반 동작 분기

포커스된 앱에 따라 동작을 분기하는 설계가 핵심이다.
글로벌 핫키 없이 클립보드 변경 이벤트만 감지하므로
다른 앱의 정상적인 이미지 붙여넣기를 방해하지 않는다.
60초 pending 상태를 지원해 앱 간 전환 워크플로우도 처리한다.

### 커스터마이즈 플래그

`-out`, `-terminals`, `-pending`, `-v` 플래그를 제공하여
저장 경로, 대상 터미널 목록, 대기 시간 등을 조절할 수 있다.

## 비평

### Windows 전용이라는 제약

macOS에서는 `pbpaste`와 파일 경로 전달이 비교적 쉽고,
Linux에서는 `xclip`/`wl-clipboard`로 유사한 워크플로우를 구성할 수 있다.
Windows의 터미널 클립보드 처리가 유독 불편하기에 존재 의의가 있는 도구다.

### 클립보드 교체로 인한 혼란 가능성

클립보드 내용을 파일 경로로 교체하는 동작은
사용자가 이미지 자체를 붙여넣으려 했을 때 혼란을 줄 수 있다.
포커스 앱 감지가 실패하는 엣지 케이스(가상 데스크톱, 원격 데스크톱 등)에서
의도치 않은 동작이 발생할 가능성이 있다.

## 인사이트

### AI 도구의 "마지막 1마일" 문제가 만드는 새 유틸리티 카테고리

Claude Code 같은 터미널 AI 도구는 텍스트 기반이지만 시각 정보를 처리할 수 있다.
그러나 "화면의 스크린샷을 터미널 AI에게 전달하는" 물리적 인터페이스는
OS-터미널-AI 사이의 간극에 놓여 있다.
winclipshot은 이 간극을 메우는 "글루 유틸리티"이며,
AI 도구가 보편화될수록 이런 틈새 도구의 수요가 급증할 것이다.

### 클립보드를 "인텔리전트 라우터"로 활용하는 패턴의 범용성

winclipshot의 핵심 아이디어는
"클립보드 내용을 목적지 앱에 맞게 변환한다"는 것이다.
이 패턴은 스크린샷에 국한되지 않는다.
코드 스니펫의 포맷 변환, URL의 단축/확장, 텍스트의 언어 자동 감지 등
클립보드를 "인텔리전트 라우터"로 활용하는 다양한 응용이 가능하다.

### "자기 가려움 긁기"의 효과

winclipshot은 개발자 한 명이 자신의 Claude Code 사용 경험에서
불편함을 발견하고 직접 해결한 프로젝트다.
작은 범위, 명확한 문제, 즉각적인 해결이라는 삼박자가 맞는
전형적인 성공적 마이크로 오픈소스 패턴이다.
