# apfel: Mac 내장 AI를 터미널에서 쓰게 해주는 CLI 도구

<https://apfel.franzai.com>

<https://github.com/Arthur-Ficial/apfel>

## 요약

apfel은 macOS 26(Tahoe) Apple Silicon Mac에 내장된
Apple Foundation Model(AFM, ~3B 파라미터)을
터미널에서 직접 사용할 수 있게 해주는 오픈소스 CLI 도구다.

API 키 없이, 비용 없이, 완전 온디바이스로 동작한다.
Apple Neural Engine과 GPU에서 실행되며 데이터가 기기를 떠나지 않는다.

세 가지 사용 모드:

1. **CLI**: stdin/stdout 파이프 친화적 명령어 (`apfel "질문"`)
2. **OpenAI 호환 서버**: localhost:11434에서 HTTP 엔드포인트 제공.
   스트리밍, 도구 호출, JSON 응답 형식 지원.
3. **대화형 채팅**: 멀티턴 대화와 자동 컨텍스트 관리.

기술 사양:

- 혼합 2/4비트 양자화, 컨텍스트 윈도우 4,096 토큰
- 영어, 한국어 포함 9개 언어 지원
- MCP(Model Context Protocol) 지원 (v0.7.1+)
- Swift 6.3 기반, FoundationModels.framework 활용
- `brew install Arthur-Ficial/tap/apfel`로 설치

## 분석

apfel의 의미는 도구 자체보다 Apple이 FoundationModels.framework를
공개 API로 노출했다는 사실에 있다.
지금까지 Apple의 온디바이스 AI는 Siri와 시스템 기능에 갇혀 있었는데,
서드파티가 이를 범용 LLM처럼 쓸 수 있는 길이 열린 것이다.

OpenAI 호환 서버 모드는 전략적 선택이다.
기존 OpenAI SDK로 작성된 코드를 수정 없이 로컬 모델로 전환할 수 있다.
이는 ollama가 개척한 패턴이고, apfel은 이를 Apple 하드웨어 레벨로 내린 셈이다.

[GeekNews 댓글](https://news.hada.io/topic?id=28178)에서
할루시네이션이 심하고 성능이 약하다는 평가가 있었다.
3B 모델의 한계는 명확하다.
다만 한국어 프롬프트 조정으로 응답 안정성이 개선되었다는 보고도 있어,
프롬프트 엔지니어링 여지는 있다.

## 비평

4,096 토큰 컨텍스트는 실용적 한계가 크다.
코드 리뷰, 문서 요약 같은 개발자 워크플로에는 부족하고,
간단한 질의응답이나 커밋 메시지 생성 정도가 현실적 용도다.

3B 파라미터 모델을 "AI"라고 부르는 것은 마케팅적 과장에 가깝다.
ollama로 7B~70B 모델을 돌리는 것과 비교하면 성능 차이가 크다.
다만 Neural Engine 활용으로 전력 소비가 적다는 점은 차별화 요소다.

보안 측면에서 localhost에 HTTP 서버를 여는 것은 주의가 필요하다.
로컬 네트워크 내 다른 기기에서 접근 가능할 수 있고,
GeekNews 댓글에서도 이 점이 지적되었다.

## 인사이트

apfel은 "AI 접근성의 마지막 마일"을 보여주는 사례다.
클라우드 API → 로컬 ollama → OS 내장 모델로 이어지는 경로에서
마지막 단계는 사용자가 모델의 존재조차 의식하지 않는 것이다.
Apple이 OS에 모델을 내장하고 프레임워크를 공개한 것은 이 방향의 첫 걸음이다.

"무료"의 의미가 달라지고 있다.
클라우드 AI의 무료 티어는 데이터를 대가로 받지만,
온디바이스 AI의 무료는 이미 구매한 하드웨어의 유휴 자원 활용이다.
비용 구조가 근본적으로 다르고,
이는 프라이버시 민감한 용도에서 결정적 차이를 만든다.

Apple의 전략은 AI 경쟁에서 "가장 똑똑한 모델"이 아니라
"가장 가까운 모델"로 포지셔닝하는 것이다.
3B 모델로 GPT-4를 이길 수는 없지만,
네트워크 지연 0ms, 비용 0원, 프라이버시 100%라는 조합은
특정 사용 시나리오에서 대형 모델을 이긴다.
자동완성, 문법 교정, 간단한 분류 같은 작업이 그 영역이다.
