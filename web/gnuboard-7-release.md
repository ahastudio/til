# Gnuboard 7 정식 출시

<https://sir.kr/g7>

Show GN: [그누보드7 정식 출시: Laravel + React 기반 오픈소스 CMS | GeekNews](https://news.hada.io/topic?id=31257)

## 소개

Gnuboard 7은 한국의 대표적인 웹 게시판 CMS인 Gnuboard를 완전히
새로 작성한 버전으로, 2026년 7월 1일 정식 출시됐다. Laravel 12와
React 19를 기반으로 하며, 핵심 변화는 JSON 기반 레이아웃 엔진의
도입이다. 이를 통해 UI를 코드가 아니라 데이터로 정의할 수 있게
되어, 모듈과 플러그인이 프런트엔드 번들을 다시 빌드하지 않고도
인터페이스를 확장할 수 있다.

주요 구성 요소로는 WYSIWYG 레이아웃 에디터, 모듈·플러그인·템플릿
3단 확장 아키텍처, 세밀한 권한 제어, 멀티채널 알림, 내장 SEO
기능이 있다. 개발 과정에서는 문서를 `AGENTS.md` 형태로 저장소에
포함시켜 AI 코딩 도구와의 호환성을 우선적으로 고려했다고 밝혔다.
이 글을 쓰는 시점 기준 GitHub 저장소는 스타 36개, 포크 17개로
아직 초기 채택 단계이며, Gnuboard 5에서 넘어오기 위한 마이그레이션
도구는 아직 준비되지 않았다.

## 분석

### AI 친화적 문서화를 처음부터 설계에 포함시킨 선택

`AGENTS.md`를 프로젝트 초기 단계부터 문서 형식으로 채택한 것은,
Gnuboard 같은 오래된 국내 CMS 생태계에서도 AI 코딩 도구를 염두에
둔 설계가 표준 관행으로 자리잡고 있음을 보여준다. JSON 기반
레이아웃 엔진 역시 AI 에이전트가 UI 구조를 코드 생성 없이 데이터
조작만으로 변경할 수 있게 한다는 점에서, 이 결정과 방향이
맞물려 있다고 볼 수 있다.

## 비평

### 마이그레이션 도구 부재가 실질적 채택을 늦출 수 있다

Gnuboard는 오랜 기간 국내 소규모 웹사이트와 커뮤니티에서 널리
쓰여 온 만큼, 기존 Gnuboard 5 사용자 다수가 실사용 데이터를
안전하게 이전할 방법 없이는 7로 넘어가기 어렵다. 완전히 새로
작성된 아키텍처(Laravel 12 + React 19)라는 점을 고려하면
마이그레이션은 단순한 스키마 변환을 넘어서는 작업일 가능성이
크며, 이 도구가 언제 제공될지에 대한 로드맵이 없다는 점은 기존
사용자 기반의 전환 속도에 대한 불확실성을 남긴다.

## 인사이트

### 오래 이어진 이름의 지속성 자체가 신뢰 자산이다

GeekNews 댓글에서 tsboard 사용자는 Gnuboard가 하나의 이름으로
오래 이어져 온 점에 대한 고마움을 표했다. 국내 오픈소스 프로젝트
생태계에서는 유지보수 주체가 바뀌거나 프로젝트가 중단되는 경우가
드물지 않은데, Gnuboard처럼 수십 년 가까이 하나의 브랜드로
지속되며 대규모 재작성까지 이뤄낸 사례는 국내 커뮤니티 소프트웨어의
장기 생존 가능성을 보여주는 참고 사례로 볼 만하다.
