# GPT-5.6 모델 선택 기준: Sol·Terra·Luna 비교

원문: <https://www.onlydecks.app/deck/gpt-5-6>

## 요약

OpenAI의 GPT-5.6은 단일 모델이 아니라 세 가지 변형(Sol, Terra, Luna)으로
제공된다.
각 변형은 성능과 비용 사이에서 다른 지점을 노린다.

공식 가격 기준으로 정리하면 다음과 같다.

| 모델        | 용도               | 입력 단가($/MTok) | 출력 단가($/MTok) |
| ----------- | ------------------ | ----------------- | ----------------- |
| GPT-5.6 Sol | 복잡한 전문 작업   | $5.00             | $30.00            |
| GPT-5.6 Terra | 균형형 범용 작업 | $2.50             | $15.00            |
| GPT-5.6 Luna | 비용 최적화 작업  | $1.00             | $6.00             |

세 모델 모두 컨텍스트 윈도우 105만 토큰, 최대 출력 128K 토큰을 지원하며
텍스트·이미지 입력, 함수 호출, 웹 검색, 컴퓨터 사용(computer use)이
가능하다.

Max와 Ultra는 별도 모델이 아니라 추론 깊이 설정(inference setting)이다.
추론 연산량을 더 많이 쓰는 설정으로, 동일 기반 모델에서 응답 품질을
높이는 대신 지연(latency)과 비용이 증가한다.

## 분석

### Sol이 항상 정답이 아닌 이유

Sol은 GPT-5.6의 최상위 변형이지만 모든 작업에 Sol이 최선이 아니다.
Sol과 Luna의 가격 차이는 입력 기준 5배, 출력 기준 5배다.
대규모 배치 처리나 반복적인 분류 작업처럼 복잡한 추론이 필요 없는
작업에 Sol을 쓰는 것은 비용을 낭비한다.

작업의 복잡도를 기준으로 모델을 선택하는 원칙이 중요하다.
코드 생성, 멀티스텝 추론, 복잡한 분석처럼 Sol의 추론 능력이 실제로
차이를 만드는 작업에는 Sol을 쓰고, 텍스트 분류, 간단한 요약, 형식
변환 같은 작업에는 Luna나 Terra로 충분하다.

### Terra가 존재하는 이유

Sol과 Luna 사이의 Terra는 얼핏 어중간해 보이지만, 실제로는 중요한
역할을 한다.
“이 작업이 Luna로 충분한지 Sol이 필요한지 모르겠다”는 상황에서
Terra가 안전한 중간 지점이 된다.
Luna 대비 2.5배 비싸지만, Sol 대비 절반 가격이다.
범용 챗봇, 일반 콘텐츠 생성, API 기반 사용자 대면 앱에서 Terra가
기본 선택지가 되는 이유다.

또한 성능 민감도가 다른 두 가지 작업을 처리하는 파이프라인에서
Terra가 파이프라인의 기본값이 되고, 특정 단계에만 Sol을 사용하는
방식으로 비용을 최적화할 수 있다.

### Max/Ultra 설정이 별도 모델이 아닌 이유

Max와 Ultra가 별도 모델이 아니라 추론 설정이라는 것은 가격 구조에서
중요한 함의가 있다.
별도 모델이라면 선택의 기준이 명확하지만, 추론 설정이라면 기반 모델
선택과 추론 깊이 선택을 독립적으로 해야 한다.
예를 들어 “Luna + Max”와 “Sol + Standard” 중 무엇이 나은지는 작업
유형에 따라 달라진다.

추론 설정이 깊을수록 비용과 지연이 증가하는 구조는, 응답 품질이
중요한 배치 작업(지연 허용)과 실시간 대화(지연 민감)에서 다른
전략이 필요함을 의미한다.

## 비평

### 세 모델의 실제 품질 차이가 어느 작업에서 얼마나 나는지가 핵심이다

가격 차이는 명확하지만, 실제 작업에서 품질 차이가 얼마나 나는지는
벤치마크만으로 판단하기 어렵다.
공식 벤치마크는 모델이 잘 보이는 작업에서 측정된 수치이므로,
실제 프로덕션 사용 패턴과 다를 수 있다.

특히 창의적 글쓰기, 코드 리뷰, 도메인 특화 질문 같은 작업에서
Sol과 Terra의 실제 차이가 가격 차이를 정당화하는지는 직접 비교
테스트 없이 판단하기 어렵다.
모델 선택 기준으로 벤치마크를 참고하되, 실제 작업 샘플로 직접
검증하는 과정이 필수다.

### 동일 컨텍스트 윈도우가 주는 착시

세 모델 모두 105만 토큰의 컨텍스트 윈도우를 제공한다는 사실은,
긴 문서 처리나 대용량 코드베이스 분석이 필요한 경우 컨텍스트 길이가
모델 선택 기준이 되지 않음을 의미한다.
이 경우 가격이 유일한 차별 기준이 되므로, Luna를 먼저 시도하고
품질이 부족할 때만 상위 모델로 올라가는 방식이 합리적이다.

## 인사이트

### 모델 계층화는 AI 서비스 비용 구조의 새로운 표준이 되고 있다

OpenAI의 Sol/Terra/Luna 구조는 Claude의 Opus/Sonnet/Haiku,
Google의 Ultra/Pro/Flash 구조와 동일한 패턴이다.
AI 서비스 제공자들이 단일 프리미엄 모델 대신 성능/비용 트레이드오프를
명시적으로 제시하는 계층화 구조로 수렴하고 있다.

이 패턴은 사용자에게 선택의 복잡도를 전가하는 측면이 있지만,
동시에 대규모 배포에서 비용 최적화 여지를 열어준다.
모델 선택이 제품 설계의 핵심 결정이 되면서, “어떤 AI를 쓰는가”에서
“어떤 계층의 AI를 어떤 작업에 쓰는가”로 질문이 정교해지는 추세다.

### 비용 최적화 전략으로서의 라우팅

작업 유형에 따라 모델을 자동으로 라우팅하는 시스템이 점점 중요해진다.
단순 작업은 Luna로, 복잡한 작업은 Sol로 자동 분기하는 라우터 레이어가
AI 앱 아키텍처의 표준 컴포넌트가 될 가능성이 있다.
이미 LiteLLM, OpenRouter 같은 도구들이 이 라우팅 기능을 제공하며,
여러 모델 제공자를 통합해 비용을 최적화하는 전략이 현실화되고 있다.
GPT-5.6의 세 모델 구조는 이 라우팅 전략을 적용하기 좋은 구조다.
