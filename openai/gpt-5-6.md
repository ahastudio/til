# OpenAI, GPT-5.6 공식 출시

<https://openai.com/index/gpt-5-6/>

HN 토론: <https://news.ycombinator.com/item?id=48849066>

## 요약

OpenAI가 2026년 7월 9일 GPT-5.6 계열을 정식 출시했다.
6월 26일 프리뷰 공개 이후 약 2주 만의 정식 출시다.
이번 계열은 세 가지 등급으로 나뉜다.
플래그십 모델 Sol, 균형형 모델 Terra, 비용 최적화 모델 Luna다.

Sol은 코딩, 지식 노동, 사이버보안, 과학 분야에서 이전 세대 대비
더 적은 토큰과 더 낮은 비용으로 최고 수준의 성능을 낸다고 밝혔다.
새로운 "울트라" 모드는 복잡한 작업을 위해 여러 에이전트를 병렬로
조율한다.
이미지 처리는 원본 해상도를 유지한 채 리사이즈 없이 처리하도록
개선됐다.

디자인과 컴퓨터 사용 능력이 강화됐고, 취약점 탐지 능력도
향상됐다.
다만 자율 공격보다는 방어 목적에 초점을 맞췄다고 설명한다.
생명과학과 화학 벤치마크에서도 진전이 있었다.

가격은 Sol이 입력 100만 토큰당 5달러, 출력 100만 토큰당 30달러다.
Terra는 각각 2.5달러와 15달러, Luna는 1달러와 6달러로 책정됐다.
생물학적 위험과 사이버보안 위험 모두 광범위한 레드팀 검증을 거쳐
"치명적(critical)" 임계값 아래에 있다고 밝혔다.

## 분석

세 등급 구조는 GPT-5 계열부터 이어진 가격·성능 분리 전략을
그대로 유지한다.
플래그십 모델의 토큰 효율을 강조한 점이 눈에 띈다.
같은 성능을 더 적은 토큰으로 낸다는 것은 곧 실사용 비용을
낮춘다는 의미이며, 에이전트가 반복적으로 모델을 호출하는
워크플로에서 특히 중요한 지표다.

"울트라" 모드의 다중 에이전트 병렬 조율 기능은 단일 모델 호출을
넘어 에이전트 오케스트레이션 자체를 제품 계층에 포함시키려는
방향을 보여준다.
이는 Claude Code의 서브에이전트나 병렬 워크트리 패턴과 유사한
문제의식을 공유한다.

## 비평

발표 자료는 벤치마크 수치와 가격을 상세히 제시하지만, 정작
"울트라" 모드가 내부적으로 에이전트를 어떻게 조율하는지에 대한
구체적인 메커니즘은 공개하지 않는다.
HN 스레드에서도 프롬프트 가이드 변경을 둘러싸고 논쟁이 일었다.
spathi_fwiffo는 두 단어짜리 지시("간결하게")를 38단어짜리
대체 문구로 바꾸라는 가이드가 오히려 사용자 편의를 떨어뜨리는
역설이라고 꼬집었다.[^spathi_fwiffo]
tshaddox는 더 나은 모델이라면 애초에 장황함의 수준을 사용자가
일일이 지정하지 않아도 알아서 판단해야 한다고 반박했다.[^tshaddox]
이는 모델의 원시 성능만큼이나 사용자 쪽의 프롬프트 엔지니어링
부담이 여전히 크다는 뜻이다.

Codex와 Claude Code의 비교도 스레드에서 갈렸다.
nilkn은 Codex가 단순함과 비용, 안정성 면에서 몇 달째 Claude
Code보다 우위였는데도 주목을 덜 받았을 뿐이라고 주장했다.[^nilkn]
반면 hk__2는 일상 작업에서 Opus·Fable 계열이 더 간결하고
정확한 코드를 꾸준히 만들어낸다며 반대 경험을 전했다.[^hk__2]
벤치마크 우위가 실제 작업 전반의 우위로 이어지지는 않는다는
뜻이며, 벤치마크 선택 편향에 대한 우려로 이어진다.

## 인사이트

이번 출시에서 가장 주목할 부분은 성능 수치보다 토큰 효율과
가격 구조다.
모델 성능이 상향 평준화되는 국면에서는, 같은 작업을 더 적은
토큰으로 처리하는 능력이 차별화 요소로 부상한다.
에이전트가 장시간 자율적으로 작업을 반복하는 시대에는 토큰
비용이 곧 운영 비용이기 때문이다.
"울트라" 모드가 실제로 어떤 조율 전략을 쓰는지는 커뮤니티의
후속 검증을 지켜봐야 알 수 있을 것이다.
