# goose - Block이 만든 오픈소스 로컬 실행 자율 AI 개발 에이전트

> 원문: <https://block.github.io/goose/>

## 요약

goose는 Square/Cash App의 모회사인 Block이 만든 오픈소스 로컬 AI 개발 에이전트다.
디버깅부터 배포까지 복잡한 엔지니어링 작업을 자율적으로 처리하며, 사용자 기기에서
로컬로 실행되어 데이터 제어권을 보장한다. Apache 2.0 라이선스이며 Rust와
TypeScript로 구현되었다.

20개 이상의 LLM 프로바이더를 지원하며 Claude 모델에서 최적 성능을 보인다. 데스크톱
앱과 CLI 인터페이스를 모두 제공하며, MCP 서버 연동으로 브라우저 자동화, 웹 스크래핑
등 기능을 확장할 수 있다.

## 분석

goose의 주요 특징:

- **20+ LLM 프로바이더 지원**: OpenAI, Anthropic, Google 등 다양한 모델 선택 가능
- **듀얼 인터페이스**: 데스크톱 앱 + CLI, 상황에 따라 선택
- **Lead/Worker 모델 전환**: 복잡한 작업을 여러 모델이 분업 처리
- **Planning Mode**: 실행 전 계획 수립으로 오류 예방
- **MCP 통합**: 외부 서비스와 연결해 기능 확장
- **`.goosehints` 파일**: 자율 실행 범위를 제어하는 접근 제어 메커니즘

실제 활용 사례로는 가짜 데이터 생성, Google Script 작성, 다국어 리소스 관리,
CLI 명령어 개발, Google Docs 확장 등이 공유되었다.

GeekNews에 따르면 Jack Dorsey(Twitter 창업자, 현 Block CEO)의 "꽤 좋아졌다"는
언급이 이번 재조명의 계기였으며, 프로젝트 자체는 약 1년 전에 처음 출시되었다.

## 비평

goose가 기존 AI 코딩 도구(Claude Code, Cursor 등)와 차별화되는 지점은 "로컬 실행"과
"오픈소스"의 조합이다. 그러나 대부분의 AI 에이전트가 어차피 클라우드 LLM API를
호출한다면, "로컬 실행"이 의미하는 것은 추론 엔진이 아니라 오케스트레이션 레이어만
로컬이라는 점이다. 진정한 프라이버시를 원한다면 LLM 자체도 로컬이어야 한다.

Jack Dorsey의 언급이 재조명의 계기라는 점은 기술적 완성도보다 커뮤니티 마케팅이
오픈소스 도구의 가시성에 더 중요할 수 있음을 보여준다.

## 인사이트

**자율 AI 에이전트 시장에서 오픈소스의 위치.** Claude Code, Devin, Cursor 같은
상용 도구들이 시장을 선점한 상황에서 goose 같은 오픈소스 대안의 역할은 특정 틈새
시장(프라이버시, 커스터마이징, 비용)에 집중된다. 범용 에이전트 시장에서 오픈소스가
상용 도구를 따라잡으려면 생태계 구축이 핵심이며, goose의 "스킬과 레시피" 커뮤니티가
그 전략이다.

**Block이 AI 에이전트를 만드는 이유.** 결제 인프라 회사가 AI 개발 에이전트를 만드는
것은 표면적으로 어울리지 않는다. 하지만 Block의 핵심 서비스를 구축하는 내부 개발자
생산성을 높이고, 그 도구를 오픈소스화해 개발자 커뮤니티에서 브랜드를 구축하는
전략으로 읽을 수 있다. 개발자 도구는 기업의 개발자 채용과 브랜드에도 영향을 미친다.

**`.goosehints`가 보여주는 자율 에이전트의 신뢰 문제.** 접근 제어 파일이 필요하다는
것은 에이전트를 완전히 신뢰하지 못한다는 의미다. 자율 AI 에이전트가 광범위하게
사용되려면 기술적 능력과 더불어 "얼마나 통제 가능한가"라는 신뢰의 문제가 핵심이 된다.
이는 에이전트 설계의 핵심 트레이드오프다.
