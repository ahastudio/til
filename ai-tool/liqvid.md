# Liqvid

<https://github.com/liqvidjs/liqvid>

## 소개

Liqvid는 React로 인터랙티브 비디오(interactive video)를 제작하는 오픈소스 라이브러리다.
“텍스트, 이미지, 오디오, 비디오의 경계가 유동적이 되는” 새로운 미디어를 표방하며,
단순히 재생되는 영상이 아니라 시청자가 직접 상호작용할 수 있는 영상을 만든다.

2018년부터 개발됐으며 MIT 라이선스 하에 공개됐다.
GitHub 스타 810개로 소규모 커뮤니티를 형성하고 있고,
2026년 4월까지 꾸준히 유지보수되고 있다.

KaTeX 수식, React Three Fiber(3D), CodeMirror(코드 편집기), THREE.js 등
웹 생태계 라이브러리와 직접 통합할 수 있어,
수학 강의, 코딩 튜토리얼, 3D 인터랙티브 설명 같은 기술 교육 콘텐츠에 특히 유용하다.

## 아키텍처

Liqvid의 핵심은 세 클래스의 협력이다.

**Playback** 클래스는 재생 제어 로직을 담당한다.
`HTMLMediaElement` 인터페이스를 모사하며 재생, 볼륨, 속도 조절을 처리한다.

**Script** 클래스는 Playback 위에서 타임라인을 마커(marker)로 구조화한다.
마커는 `[이름, 지속시간]` 쌍으로 정의하며 내부적으로 `[이름, 시작, 끝]` 타임스탬프로 변환된다.
이 분리 덕분에 순서를 바꿔도 타이밍을 직접 재계산할 필요가 없다.

**Player** 컴포넌트는 콘텐츠를 감싸는 React 컴포넌트로,
Script 초기화를 통해 Playback을 자동으로 생성한다.

### React 컴포넌트와 재생 시간의 동기화

Liqvid는 DOM 조작 대신 CSS로 컴포넌트의 시간 동기화를 처리한다:

| 데이터 어트리뷰트    | 동작                                              |
| -------------------- | ------------------------------------------------- |
| `data-from-first`    | 지정 마커 이후부터 요소를 표시                    |
| `data-from-last`     | 지정 마커 임계값 이전에는 요소를 숨김             |
| `data-during`        | 현재 마커 이름이 접두사 패턴과 일치할 때만 표시   |

`opacity: 0; pointer-events: none`으로 구현해 DOM 리플로우 없이
레이아웃 스래싱(layout thrashing)을 방지한다.

### 녹화 워크플로우

개발자는 스크립트에 타임라인 세그먼트를 정의한 뒤,
`RecordingControl` 컴포넌트를 통해 브라우저에서 직접 오디오를 녹음한다.
E/W 키로 마커를 수동으로 이동하며 녹음하면 지속 시간이 자동으로 채워진다.
브라우저가 생성한 WebM 오디오는 시크(seek) 메타데이터가 없으므로
CLI로 변환한다:

```bash
liqvid audio convert audio.webm
```

## 패키지 구성

| 패키지             | 역할                                               |
| ------------------ | -------------------------------------------------- |
| `liqvid`           | 메인 패키지 (Playback, Script, Player)             |
| `@liqvid/keymap`   | 키보드 단축키 맵                                   |
| `@liqvid/playback` | Playback 클래스                                    |
| `@liqvid/utils`    | 애니메이션 헬퍼 함수                               |
| `@liqvid/katex`    | KaTeX 수식 통합                                    |
| `@liqvid/react-three` | React Three Fiber 3D 통합                       |
| `@liqvid/renderer` | CLI audio, build, render, thumbs 명령              |
| `@liqvid/magic`    | 리소스 매크로 문법                                 |

개발 중인 패키지로는 캡션 편집기(`captioning`), GSAP 통합, i18n,
Web Components 기반 새 Player(`player`), MathJax/XyJax 통합이 있다.

## 분석

### “인터랙티브 비디오”라는 포지셔닝이 독특하다

Remotion, HyperFrames, Revideo가 모두 “HTML/React로 비디오를 만든다”는
공통 포지셔닝을 갖는 것과 달리, Liqvid는 “인터랙티브 비디오”를 내세운다.
이 차이는 의미가 있다.

다른 도구들이 최종 결과물로 MP4를 목표로 하는 반면,
Liqvid는 브라우저에서 재생되는 인터랙티브 웹 앱이 목표다.
클릭, 코드 입력, 3D 객체 조작이 영상 재생 중에 가능하다.
이것은 YouTube 영상과 코드펜(CodePen) 데모를 합쳐놓은 개념에 가깝다.

### 마커 시스템이 핵심 추상화다

Liqvid의 가장 강력한 아이디어는 타임라인을 절대 시간(timestamp) 대신
이름 있는 세그먼트(marker)로 표현한다는 것이다.

`[name, duration]`으로 정의하면, 순서를 바꾸거나 구간을 추가·삭제할 때
뒤따르는 모든 시간을 손으로 다시 계산할 필요가 없다.
마커 이름으로 `script.parseStart(“intro/”)` 같이 접근하므로
타임라인 리팩토링이 코드 리팩토링만큼 자연스럽다.

이것은 CSS 클래스명 기반 스타일링이 인라인 스타일보다 유지보수하기 쉬운 것과
같은 원리다. 세부 구현(타임스탬프)을 숨기고 의미(마커 이름)로 접근하게 한다.

### 교육 콘텐츠 시장의 특수 요구를 겨냥한다

KaTeX, MathJax, XyJax(수학 다이어그램), CodeMirror 통합은
수학·코딩 교육 콘텐츠 제작자를 명확한 타겟으로 설정하고 있음을 보여준다.
YouTube 강의 영상으로는 불가능한 것—수식이 실제로 편집 가능하고,
코드가 실제로 실행되며, 3D 다이어그램을 직접 회전할 수 있는—을
Liqvid가 제공하는 맥락이다.

## 비평

### 강점: 브라우저 재생의 풍부한 표현력

MP4로 내보낸 영상은 YouTube에서 재생될 수 있지만,
인터랙티브성은 잃는다.
Liqvid는 브라우저 재생을 1차 타겟으로 설정해
웹의 모든 인터랙티브 기술을 영상에 내재화한다.
3D 장면 회전, 코드 편집, 수식 입력이 영상 안에서 가능하다는 것은
다른 비디오 제작 도구로는 대체하기 어렵다.

### 강점: CSS 기반 동기화가 성능을 지킨다

`data-from-first` 같은 데이터 어트리뷰트로 컴포넌트의 가시성을
CSS로 제어하는 것은 React 상태로 DOM을 조작하는 것보다
레이아웃 스래싱 없이 부드러운 재생을 보장한다.
이 선택은 60fps 재생 중에도 React 렌더링 비용을 최소화한다.

### 약점: 작은 생태계가 신뢰 문제를 만든다

GitHub 스타 810개는 주요 경쟁 도구들(Remotion 44k, HyperFrames 12k)과
비교하면 매우 작은 수치다.
이것은 커뮤니티 지원, 문서 품질, 버그 수정 속도 모두에 영향을 준다.
개인 유지보수자 의존도가 높을 가능성이 크다.

### 약점: 배포가 복잡하다

인터랙티브 비디오를 배포하려면 단순히 MP4 파일을 올리는 것으로는 부족하고,
웹 앱을 호스팅해야 한다.
이것은 YouTube, Vimeo 같은 기존 비디오 배포 채널을 사용할 수 없다는 의미다.
콘텐츠 제작자가 서버 관리나 웹 배포 지식을 별도로 갖춰야 한다.

## 인사이트

### 비디오와 앱의 경계를 지우는 것이 Liqvid의 진짜 목표다

Liqvid의 슬로건 “텍스트, 이미지, 오디오, 비디오의 경계가 유동적이 된다”는
단순한 마케팅 문구가 아니라 설계 철학이다.

전통적인 비디오는 선형(linear)이다. 시간이 흐르고, 시청자는 앉아서 본다.
웹 앱은 비선형이다. 클릭하고, 입력하고, 탐색한다.
Liqvid는 두 패러다임을 합치려 한다. 시간이 흐르면서 콘텐츠가 진행되지만,
그 안에서 시청자가 상호작용할 수 있다.

이 발상은 오래됐다. DVD의 “챕터”와 “메뉴”도 비선형 접근이었고,
HTML5 video의 `<track>` 엘리먼트도 인터랙티브 자막을 제공했다.
그러나 Liqvid는 React 컴포넌트를 타임라인 위에 올림으로써
이 아이디어를 프로그래밍 가능한 수준으로 끌어올린다.
앞으로 이 방향이 더 발전하면, “비디오”와 “인터랙티브 앱”이라는 카테고리
구분 자체가 무의미해질 수 있다.

### 마커 기반 타임라인은 소프트웨어 개발 방법론을 영상 편집에 이식한다

Liqvid의 마커 시스템이 흥미로운 것은 단순히 편리해서가 아니다.
이것은 “시간 값의 추상화”라는 소프트웨어 엔지니어링의 근본 원리를
영상 타임라인에 적용한 것이다.

전통적인 비디오 편집 소프트웨어에서 타임라인 편집은 절대 시간을 다룬다.
After Effects, Premiere에서 레이어를 앞뒤로 옮기면
겹침을 수동으로 조정해야 한다.
마커 이름으로 타임라인을 표현하면, 세그먼트를 추가하거나 순서를 바꿔도
이름으로 참조하는 코드는 전혀 수정할 필요가 없다.

이것은 하드코딩된 픽셀 값 대신 CSS 변수를 쓰는 것과 정확히 같은 원리다.
“디자인 토큰”이 UI 일관성을 유지하듯,
“마커 이름”이 타임라인 일관성을 유지한다.
Liqvid는 이 개념을 비디오 편집 도메인에 이식함으로써,
소프트웨어 엔지니어가 비디오 타임라인을 코드처럼 리팩토링할 수 있게 한다.

### “재생 가능한 웹 앱”은 기술 교육의 다음 형식이 될 수 있다

현재 기술 교육 콘텐츠는 두 형식으로 나뉜다.
정적 글(블로그, 문서)과 수동적 영상(YouTube, Udemy)이다.
둘 다 한계가 있다. 글은 흐름을 보여주기 어렵고, 영상은 직접 해보기가 어렵다.

Liqvid가 제안하는 “재생 가능한 웹 앱”은 이 둘의 장점을 합친다.
코딩 튜토리얼에서 코드가 실제로 실행되고,
수학 강의에서 수식을 직접 편집해보며,
3D 설명에서 모델을 돌려볼 수 있다.
이것은 학습자가 “보는” 것이 아니라 “함께 하는” 경험이다.

KaTeX, CodeMirror, THREE.js 통합이 이 비전을 구체화한다.
Khan Academy가 인터랙티브 수학 교육으로 만든 학습 경험을,
모든 교육 콘텐츠 제작자가 React 컴포넌트로 구현할 수 있게 하겠다는 것이다.
생태계가 성숙한다면, 기술 교육 플랫폼의 콘텐츠 형식이 MP4에서
“재생 가능한 웹 앱”으로 이동하는 것을 촉진할 수 있다.
