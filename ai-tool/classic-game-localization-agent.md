# Show GN: 레트로 게임 한글화 에이전트 스킬

<https://github.com/mcpads/create-retro-game-kr-patch>

## 소개

`create-retro-game-kr-patch`는 세가 새턴, 슈퍼패미컴, 게임기어,
메가드라이브, 드림캐스트 등 레트로 게임을 한글로 패치하는
엔드투엔드 에이전트 스킬이다.
제작자 mcpads는 Claude Opus 4.6을 활용해 여러 플랫폼의 고전
게임에 한글 패치를 만들며 정리한 방법론을 GeekNews에 공유했다.

## 아키텍처

핵심 철학은 사람이 쓰는 전용 툴(롬 에디터, 폰트 편집기 등)에
의존하는 대신, 에이전트가 ROM·파일시스템·폰트·텍스트 엔진을
직접 해석하게 만드는 것이다.
정적 오류 탐지를 문서화·체크섬·테스트로 구현해 사람의 개입을
최소화한다.
비용이 큰 구현에 들어가기 전에 개념 증명(proof-of-concept)
실험을 먼저 수행해, 되돌리기 힘든 대규모 롤백을 피한다.

제작자의 GitHub 프로필에는 이 스킬 외에도 관련 인프라가
공개돼 있다.
`emucap`은 에뮬레이터 모니터링과 사람 개입 루프(human-in-the-loop)
적응을 결합한 Rust 기반 도구로, 패치 디버깅용 인프라로 쓰인다.
마도 모노가타리 시리즈, 뿌요뿌요 시리즈 등 실제 게임별 패치
저장소도 여러 개 공개돼 있으며, 세가 새턴판 마도 모노가타리(1998),
드림캐스트판 뿌요뿌요~ん(1999) 패치가 대표적이다.

## 분석

이 프로젝트가 흥미로운 지점은 "번역 자체"가 아니라 "번역
파이프라인을 에이전트가 스스로 구축하게 만드는 방법"에 있다.
레트로 게임 롬 해킹은 전통적으로 디스어셈블리, 폰트 테이블 분석,
텍스트 포인터 재배치 같은 고도로 전문화된 수작업 지식을
요구해 왔다.
제작자는 이 전문 지식을 사람이 직접 습득하는 대신, 에이전트가
ROM 구조를 스스로 탐색하고 해석하도록 위임하는 쪽을 택했다.

정적 오류 탐지와 체크섬 검증을 강조한 점도 눈여겨볼 만하다.
롬 패치는 한 바이트만 어긋나도 게임이 부팅조차 되지 않는
작업이라, 사람이 매번 실행 검증을 하기보다 결정론적 검증
장치를 마련해 에이전트가 스스로 오류를 걸러내게 하는 접근이
합리적이다.

## 비평

GeekNews 페이지와 GitHub 저장소 모두 구체적인 실패 사례나
실패율 데이터를 제시하지 않는다.
"개념 증명을 먼저 한다"는 원칙은 합리적이지만, 실제로 어떤
유형의 실패가 얼마나 자주 발생했는지, 에이전트가 어떤 종류의
오해석을 반복했는지에 대한 정량적 근거는 부족하다.
방법론이 소수의 성공 사례(마도 모노가타리, 뿌요뿌요 시리즈)에서
귀납된 것이라, 다른 플랫폼이나 다른 텍스트 인코딩 방식에도
일반화되는지는 검증되지 않았다.

## 인사이트

이 프로젝트는 "AI 에이전트에게 전용 도구를 만들어 주는 대신
사람의 도구를 그대로 해석하게 하라"는 접근을 구체적인
도메인에서 실증한 사례다.
레트로 게임 롬 해킹처럼 지식이 파편화되고 문서화가 부실한
영역일수록, 전용 도구를 새로 만드는 비용보다 에이전트가
기존 도구와 자료를 스스로 해석하는 비용이 더 낮을 수 있다는
가능성을 보여준다.
동시에 체크섬과 정적 검증 같은 결정론적 안전장치를 함께
설계해야, 되돌리기 어려운 실패를 피하면서 이 접근을 실용적으로
만들 수 있다는 교훈도 함께 담고 있다.
