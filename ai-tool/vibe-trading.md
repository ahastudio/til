# Vibe-Trading — 자연어로 구동하는 AI 멀티에이전트 트레이딩 플랫폼

<https://github.com/HKUDS/Vibe-Trading>

## 소개

Vibe-Trading은 홍콩대학교 데이터과학 연구소(HKUDS)에서 개발한 오픈소스 AI 멀티에이전트 금융 워크스페이스다. 자연어 입력만으로 트레이딩 전략 코드를 자동 생성·백테스트·내보낼 수 있으며, A주, 홍콩, 미국, 한국, 일본 주식부터 암호화폐, 선물, 외환까지 글로벌 시장을 아우른다. MIT 라이선스로 공개되었으며 연구 목적 전용임을 명시하고 있다.

HKUDS의 에이전트 생태계(ClawTeam, NanoBot, CLI-Anything, OpenSpace) 중 금융 분석에 특화된 구성 요소로, 자기진화 스킬과 영구 메모리를 핵심으로 삼는다.

## 주요 기능

### 전략 개발 및 백테스팅

7가지 백테스팅 엔진이 내장되어 있으며 몬테카를로(Monte Carlo) 시뮬레이션, 부트스트랩(Bootstrap) 신뢰구간, Walk-Forward 분석을 통한 통계적 검증을 지원한다. 완성된 전략은 TradingView Pine Script, TDX, MetaTrader 5 형식으로 내보낼 수 있다.

### 멀티에이전트 스웜

72개의 전문화된 금융 스킬이 7개 도메인으로 분류되어 있고, 29개의 사전 구성 에이전트 팀 프리셋을 즉시 사용할 수 있다. 에이전트는 경험을 통해 스스로 새 스킬을 생성·수정하는 자기진화 기능을 갖추고 있으며, FTS5 전문 검색과 5단계 컨텍스트 압축으로 장기 리서치 세션을 지원한다.

### 데이터 소스

yfinance, OKX, AKShare를 통해 대부분의 데이터를 API 키 없이 무료로 사용할 수 있다. Tushare(A주), Futu 등 유료 데이터도 선택적으로 연결 가능하며 자동 폴백 메커니즘이 내장되어 있다.

### MCP 통합

17개의 MCP(Model Context Protocol) 도구를 제공하여 Claude Desktop, Cursor 등 MCP 호환 클라이언트와 직접 통합할 수 있다.

## 아키텍처

백엔드는 Python FastAPI 서버로 실행되며 ReAct 에이전트가 도구 호출과 스킬 실행을 담당한다. 프론트엔드는 React 19 웹 UI로 실시간 에이전트 상태 시각화와 챗 인터페이스를 제공한다. 데이터 계층은 모듈형 로더 레지스트리로 소스별 자동 폴백 체인을 처리한다.

## 설치 및 사용법

```bash
pip install vibe-trading-ai
vibe-trading init              # .env 설정
vibe-trading                   # CLI 실행
vibe-trading serve --port 8899 # 웹 UI 실행
```

Docker로도 구동 가능하며, MCP 통합은 `npx clawhub@latest install vibe-trading` 명령으로 설치한다.

전략 실행 예시:

```bash
vibe-trading run -p "Backtest 20/50-day moving average crossover on AAPL"
vibe-trading --swarm-run investment_committee '{"topic":"Is TSLA a buy?"}'
```

## 분석

Vibe-Trading이 차별화하는 지점은 단순한 LLM 래퍼가 아니라는 점이다. 백테스팅 엔진 7개, 통계 검증 3종, 에이전트 스킬 72개를 실제로 구현했으며, 특히 자기진화 스킬 메커니즘과 세션 간 메모리가 에이전트 연구의 산물임을 알 수 있다. HKUDS가 학술 연구 맥락에서 개발한 만큼 시스템 구조 자체가 논문 검증 환경에서 다져진 것으로 보인다.

그러나 GeekNews 댓글(https://news.hada.io/topic?id=29055)에서 제기된 “실제 성과가 있는지 궁금하다”는 의문은 핵심을 찌른다. MIT 라이선스에 “연구 목적 전용”을 명시한 것이 법적 면책이기도 하지만, 동시에 프로젝트 자체가 아직 실전 수익성을 검증받지 못했다는 사실을 인정하는 표현이기도 하다. 또한 “대부분 수수료에 죽는다”는 지적처럼, 백테스팅 기반 전략이 슬리피지(slippage)와 거래 수수료를 실제 환경 수준으로 반영하지 않으면 실전에서 역전되는 사례가 흔하다.

프로젝트의 기술적 포부(멀티마켓 동시 지원, MCP 도구, 자기진화 에이전트)는 시스템 복잡도를 높이는 방향이다. 좁고 깊은 전문화 대신 넓은 커버리지를 택한 이 설계는 연구 시스템에는 적합하지만 실용 배포 환경에서는 유지보수 부담이 커진다.

## 비평

### 강점

자연어 인터페이스로 전략 백테스팅 진입장벽을 낮춘 점은 실질적인 기여다. Pine Script나 MetaTrader MQL5를 모르는 연구자도 LLM의 코드 생성 능력을 빌려 전략 아이디어를 빠르게 프로토타입할 수 있다. MCP 지원을 통해 Claude Desktop이나 Cursor와 직접 통합되는 것도 현재 AI 에코시스템 트렌드에 잘 맞는 설계다.

### 약점

연구 목적 전용이라는 선언이 프로젝트의 정체성을 모호하게 만든다. 실전에서 쓸 수 없는 트레이딩 시스템이라면, 학습 도구 또는 전략 프로토타이핑 도구로 포지셔닝을 명확히 했을 때 사용자가 기대치를 적절하게 조정할 수 있다. 현재는 “트레이딩 플랫폼”이라는 이름과 “연구 전용”이라는 제약이 충돌한다.

또한 지원 시장이 지나치게 넓다. A주부터 암호화폐, 외환, 선물을 동시에 지원하면 각 자산군의 특수성(예: A주의 T+1 결제, 암호화폐의 24시간 거래)을 제대로 모델링하기 어렵다. 깊이 있는 시장 지원보다 폭넓은 커버리지를 우선한 결과다.

## 인사이트

### LLM 기반 퀀트 도구의 실질적 한계는 모델이 아니라 데이터 품질이다

자연어로 전략을 기술할 수 있게 되면서 트레이딩 전략 생성 비용이 급감했다. 하지만 전략 아이디어의 가치는 생성 비용이 아니라 실전 수익성으로 결정된다. LLM이 “RSI 역추세 전략을 BTC-USDT에 6개월 백테스팅하라”는 명령을 이해하고 코드를 생성하는 것은 이미 해결된 문제다. 진짜 병목은 백테스팅 데이터의 품질, 슬리피지와 수수료 모델의 현실성, 그리고 과적합(overfitting) 방지다.

무료 데이터(yfinance, AKShare)에 의존하는 시스템은 생존편향(survivorship bias)이나 조정되지 않은 가격 데이터를 그대로 백테스팅하는 오류를 범하기 쉽다. LLM이 코드를 완벽하게 생성하더라도, 그 코드가 오염된 데이터 위에서 돌아가면 결과는 현실을 반영하지 못한다. Vibe-Trading의 “연구 목적 전용” 주석은 이 문제를 우회하는 방법이기도 하다.

### 에이전트 스웜 아키텍처가 퀀트 워크플로에 적합한 이유

전통적인 퀀트 리서치 프로세스는 자연스럽게 다중 역할 구조를 갖는다. 데이터 수집, 팩터 발굴, 전략 설계, 리스크 분석, 포트폴리오 최적화가 각각 독립적인 전문성을 요구한다. 멀티에이전트 스웜이 “investment_committee”나 “quant_strategy_desk” 같은 프리셋으로 이 역할 분담을 모사한다는 발상은 단순히 기술적으로 흥미로운 것을 넘어, 실제 리서치 조직 구조를 소프트웨어로 구현하려는 시도다.

자기진화 스킬 메커니즘이 실제로 작동한다면 장기적으로 더 흥미롭다. 에이전트가 특정 시장 환경에서 유효했던 전략 패턴을 스킬로 저장하고 재사용할 수 있다면, 이는 기존 퀀트 라이브러리의 정적인 팩터 목록을 넘어서는 동적 지식 축적이 된다. 다만 이 기능이 실제로 유용한 스킬을 생성하는지, 아니면 노이즈를 축적하는지 검증된 사례가 아직 없다.

### “연구 목적 전용” 딱지가 AI 금융 도구의 표준 관행이 될 것이다

증권 관련 소프트웨어에는 금융규제 책임이 따른다. 투자 조언으로 해석될 수 있는 출력을 생성하는 AI 시스템은 미국 SEC, 한국 금융위원회 등 각국 규제기관의 레이더에 걸릴 수 있다. “연구 목적 전용” 선언은 법적 면책 장치이자 실질적인 규제 리스크 회피 수단이다.

이 패턴은 앞으로 더 많은 AI 금융 도구에서 반복될 것이다. 기술은 충분히 발전했지만 규제 환경은 아직 따라오지 못했고, 그 간극에서 프로젝트들은 “연구”, “교육”, “시뮬레이션” 같은 수식어로 자신을 포지셔닝한다. 실질적인 사용자가 이 딱지를 무시하고 실전에 사용하는 것을 막을 방법도 없다는 점에서, 책임 소재가 불명확한 회색지대가 형성된다.
