# OpenAI Codex가 컨텍스트 크기를 372k에서 272k로 줄였다

<https://github.com/openai/codex/pull/33972>

HN 토론: <https://news.ycombinator.com/item?id=48965850> (337점, 156개 댓글)

## 소개

OpenAI의 오픈소스 CLI 에이전트 Codex가 최대 컨텍스트 크기를 372,000 토큰에서 272,000 토큰으로 줄이는 변경을 적용했다.
GitHub PR 33972에 머지된 이 변경은 짧은 설명과 함께 배포됐고,
충분한 사전 공지나 설명 없이 이뤄진 것에 대해 커뮤니티 반발을 불렀다.

이 PR의 내용은 GitHub에서 직접 접근이 제한돼 있어,
HN 토론과 커뮤니티 반응이 실제 변경 내용을 파악하는 주요 경로가 됐다.

## 분석

### 컨텍스트 크기가 AI 코딩 도구의 실질적 경쟁 변수다

tekacs는 이 변경에 직접적인 불만을 표명했다.[^tekacs]
372k에서 272k 토큰으로의 감소가 그의 작업 흐름에 현실적 영향을 준다는 것이다.
Claude의 200k 컨텍스트를 선호하는 이유 중 하나로 긴 컨텍스트 지원을 꼽은 그는
OpenAI 모델이 더 나은 컨텍스트를 제공할 것이라고 기대했기 때문에 이 결정이 충격이었다고 밝혔다.

이 반응이 보여주는 것은 컨텍스트 크기가 AI 코딩 도구 선택에서 실질적 기준이 됐다는 점이다.
단순한 기능 비교가 아니라, 대규모 코드베이스에서 실제로 얼마나 많은 코드를 한 번에 다룰 수 있는가가
도구 선택의 핵심 변수가 됐다.

### 1M 컨텍스트를 기준점으로 보는 시각이 등장했다

damsta는 단계적 감소보다 1M 토큰 최솟값이 필요하다는 관점을 밝혔다.[^damsta]
현재 수준의 컨텍스트 창 크기는 여전히 충분하지 않고,
방향 자체가 확장이 아닌 축소로 갔다는 것이 우려된다는 입장이다.

이 시각은 컨텍스트 크기를 “부족한 것을 늘려야 한다”는 프레임이 아니라
“현재 가능한 최대치가 기준”이 되는 사용자 집단이 생겼음을 보여준다.
대형 모노레포, 복잡한 레거시 코드베이스 작업자에게 컨텍스트는 이미 제한 요소다.

### 커뮤니케이션 부재가 신뢰 문제로 이어진다

dannyw는 Twitter에서 별도 설명이 있었다고 언급했다.[^dannyw]
PR 설명이 짧거나 없었고, 공식 발표가 Twitter로 이동한 상황에서
GitHub를 주요 정보 채널로 사용하는 개발자들은 맥락을 잃었다.

오픈소스 도구에서 성능 관련 회귀(regression)는 특히 민감하다.
“왜 줄였는가”에 대한 답이 PR에 없으면,
커뮤니티는 비용 절감인지, 안정성 문제인지, 내부 정책 변경인지 추측에 의존해야 한다.
추측은 불신으로 이어진다.

## 비평

### 오픈소스로 공개했지만 의사결정은 불투명하다

OpenAI Codex가 오픈소스로 공개됐다는 것은 코드를 누구나 볼 수 있다는 의미다.
그러나 “왜 이 결정을 내렸는가”는 여전히 불투명하다.
PR에 충분한 설명이 없고, 변경 이유가 Twitter로 이동한 상황은
오픈소스의 형식을 갖추되 오픈소스의 정신인 공개적 의사결정 과정이 없는 패턴이다.

클로즈드 소스였다면 “기업 결정”으로 받아들여질 수 있지만,
오픈소스 저장소의 PR에서 이뤄지는 변경은 커뮤니티 피드백을 수용하는 맥락이 있다.
그 맥락 없이 일방적으로 적용된 변경은 오히려 불만을 키운다.

### 컨텍스트 감소의 기술적 이유가 명확하지 않다

372k에서 272k로의 감소가 어떤 기술적 이유에서인지 공개된 정보가 없다.
모델 성능 품질 문제인지, 비용 최적화인지, API 한도 변경인지에 따라
사용자 대응 전략이 달라진다.
이 정보를 PR이나 릴리스 노트에 담지 않은 것은
오픈소스 도구 유지관리 관행으로 좋지 않은 선례다.

## 인사이트

### 컨텍스트 크기 경쟁이 AI 코딩 도구 시장을 가를 수 있다

AI 코딩 도구 시장에서 컨텍스트 창 크기는 마케팅 숫자에서 실질 경쟁 변수로 전환되고 있다.
tekacs가 Anthropic을 선호하는 이유 중 하나로 긴 컨텍스트를 든 것은
커뮤니티 안에 이미 컨텍스트 크기로 도구를 선택하는 세그먼트가 존재함을 보여준다.

이 경쟁에서 회귀는 단순한 기능 저하가 아니라 시장 점유율 이탈로 이어질 수 있다.
AI 코딩 도구 전환 비용이 낮기 때문에,
컨텍스트처럼 수치로 비교 가능한 항목에서 열세가 되면
사용자는 다른 도구로 이동한다.

### 오픈소스 AI 도구의 거버넌스 문제가 수면 위로 오르고 있다

OpenAI Codex는 오픈소스로 공개됐지만,
핵심 결정은 여전히 OpenAI 내부에서 내려지고 커뮤니티 참여 없이 적용된다.
이것은 오픈소스의 형식과 클로즈드 소스의 의사결정 구조가 공존하는 하이브리드 모델이다.

이 구조에서 커뮤니티의 기대치 관리가 중요해진다.
“누가 결정을 내리는가”, “어떤 과정으로 PR이 머지되는가”,
“커뮤니티 피드백이 실제로 반영되는가”에 대한 명확한 거버넌스가 없으면
오픈소스 공개가 오히려 신뢰 비용을 높인다.
