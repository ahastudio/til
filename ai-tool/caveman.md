# Caveman - 원시인 말투로 LLM 출력 토큰 절약하기

<https://github.com/JuliusBrussee/caveman>

## 요약

Caveman은 Claude Code/Codex의 응답을 “원시인 말투”로 간소화하여 출력 토큰을
약 65% 줄이는 클로드 스킬이다. 불필요한 인사말, 부연 설명, 권고 표현을 제거하고
기술 용어와 코드 블록은 정확히 보존한다. Lite/Full/Ultra 세 단계 강도를 지원하며,
메모리 파일(CLAUDE.md 등)도 압축하여 입력 토큰까지 절감한다.

## 분석

작동 방식은 간단하다. “I'd be happy to help”, “The reason is because” 같은 장황한
표현을 제거하고 필수 정보만 남긴다. 벤치마크에서는 React 리렌더링 설명 87% 절감
(1,180 → 159토큰), PostgreSQL 연결 풀 설명 84% 절감이라는 극적인 수치를 보였다.

Caveman Compress 기능은 별도로, Python 파이프라인이 CLAUDE.md 같은 메모리 파일을
재작성하여 입력 토큰을 약 45% 줄인다. 원본은 CLAUDE.original.md로 백업된다.

설치는 `npx skills add JuliusBrussee/caveman`으로 간단하다. 이후 `/caveman`
명령어나 “토큰 줄여줘” 같은 자연어로 활성화한다.

## 비평

GeekNews 댓글에서 지적된 비판은 핵심을 찌른다. 저자 스스로도 “~75%”는 예비
테스트 수치이며 정식 벤치마크를 준비 중이라 인정했다. 몇 가지 구조적 문제가 있다.

첫째, 입력 토큰이 출력 토큰보다 실제 비용에서 더 큰 비중을 차지하는 경우가 많다.
Caveman의 주된 효과(출력 토큰 절감)가 실제 비용 절감과 직결되지 않을 수 있다.

둘째, 모델이 스타일 제약(원시인 말투 유지)에 처리 용량을 소비하면 추론 품질이
낮아질 가능성이 있다. 간결함과 정확도의 트레이드오프가 벤치마크에 반영되지
않았다.

## 인사이트

**토큰 절감의 역설**: 가장 비싼 토큰은 LLM이 스스로 생성하는 자기 설명용 토큰이다.
“도움을 드리게 되어 영광입니다”류의 표현은 사용자에게는 맥락을 채우는 듯 느껴지지만
실제로는 정보 밀도가 0에 가깝다. Caveman은 이 구조를 명시적으로 겨냥한다. LLM이
사회적 동물처럼 행동하도록 훈련된 부작용을 역설계하는 도구다.

**메타 압축의 탄생**: CLAUDE.md를 Caveman으로 압축한다는 발상은 흥미롭다. 메모리
파일이 장황해질수록 매 요청마다 입력 토큰이 늘어나는 누적 비용 문제를 해결하는
방향이다. 이는 “에이전트에게 주는 지침도 압축 가능한 코드”라는 관점의 전환이다.

**비인간 언어 아이디어**: 댓글에서 제안된 “모델이 내부적으로 비인간 언어로 사고하고
소형 모델이 번역층을 담당”하는 구조는, 현재 LLM의 자연어 출력이 인간을 위한
인터페이스이지 모델의 실제 처리 단위가 아니라는 점을 파고든다. 장기적으로
에이전트 간 통신에서는 인간 친화적 언어가 오버헤드가 될 수 있다.

**모방의 빠른 확산**: “npx skills add”로 설치하는 구조는 Claude Code 스킬
생태계의 네트워크 효과를 잘 활용한다. 개인이 만든 토큰 절약 해킹이 수천 명에게
즉시 배포될 수 있는 인프라가 이미 존재한다. 도구의 품질보다 배포 편의성이
채택률을 결정하는 시대다.
