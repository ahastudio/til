# wigolo

<https://github.com/KnockOutEZ/wigolo>

## 소개

AI 에이전트를 위한 로컬 퍼스트 웹 인텔리전스 MCP 서버다.
검색, 패치, 크롤링, 추출, 캐시, 유사 문서 탐색, 리서치, 자율 수집 루프를 하나의 서피스로 제공한다.
API 키 없이, 클라우드 없이, 쿼리당 비용 없이 실행된다.

Claude Code, Cursor, Codex, Gemini CLI, Windsurf, Zed, Antigravity를 비롯해
LangChain, CrewAI, LlamaIndex, Vercel AI SDK, n8n, 임의의 MCP 클라이언트, REST 엔드포인트를 지원한다.

```bash
npx wigolo init                                        # 로컬 엔진 설정
npx wigolo init --agents=claude-code,cursor            # 설정 + 에이전트 연결
```

Node.js 20+, macOS/Linux/Windows 지원, 약 1.5GB 디스크 필요 (브라우저 엔진 + 온디바이스 모델).

## 분석

### 로컬 퍼스트 웹 인텔리전스의 의미

웹 검색을 제공하는 MCP 서버 대부분은 외부 API를 호출한다.
Brave Search API, Serper API, Tavily API 같은 서비스를 거치면
비용이 발생하고, 데이터가 제3자를 통과하며, 네트워크 의존성이 생긴다.

wigolo는 온디바이스 모델과 로컬 브라우저 엔진으로 이 파이프라인을 로컬에서 처리한다.
`~/.wigolo/`에만 데이터가 남는다는 설계는 프라이버시 민감 환경에서 중요한 차별점이다.

### 에이전트 플랫폼 독립성

12개 이상의 에이전트 플랫폼과 통합된다는 것은
MCP 표준화가 실제로 다중 플랫폼 도구 생태계를 만들고 있다는 증거다.
wigolo 같은 도구가 “Claude Code용 플러그인”이 아니라
“MCP를 지원하는 모든 에이전트용 도구”로 포지셔닝할 수 있게 됐다.

## 비평

### 1.5GB는 진입 장벽이다

브라우저 엔진과 온디바이스 모델 때문에 초기 설치가 1.5GB다.
빠른 실험이나 가벼운 사용을 원하는 개발자에게 이 진입 비용은 높다.
클라우드 API 기반 대안과의 비교에서 “로컬 퍼스트” 이점이 이 비용을 정당화하는지는
사용 패턴에 따라 다르다.

## 인사이트

### 로컬 AI 인프라가 개발자 도구의 새 레이어가 된다

2024년까지 대부분의 AI 도구는 클라우드 API를 감싸는 래퍼였다.
wigolo처럼 온디바이스 모델을 활용하는 로컬 퍼스트 AI 인프라가 등장하면서
클라우드 의존 없이 실질적인 AI 기능을 제공하는 레이어가 형성되고 있다.
이 방향이 확산되면 프라이버시, 비용, 응답 속도 모두에서 다른 트레이드오프가 생긴다.
