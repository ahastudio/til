# Understand-Anything

<https://github.com/Lum1104/Understand-Anything>

## 소개

Understand-Anything은 코드베이스와 지식베이스를 인터랙티브 지식 그래프로 변환하는 멀티플랫폼 플러그인이다.
파일을 직접 읽는 대신 그래프 형태로 탐색하고, 함수와 클래스를 노드로 시각화하며, 비즈니스 도메인 관점의 흐름을 별도 레이어로 제공한다.
Claude Code, Cursor, VS Code, Copilot, Codex, Gemini CLI 등 10개 AI 코딩 플랫폼을 지원한다.
GitHub Stars 12.9k, TypeScript 86.7%, MIT 라이선스.

## 아키텍처

6개 에이전트로 구성된 파이프라인이 순차적으로 코드베이스를 분석한다.

| 에이전트            | 역할                               |
| ------------------- | ---------------------------------- |
| Project Scanner     | 언어 및 프레임워크 감지            |
| File Analyzer       | 함수, 클래스, 임포트 추출          |
| Architecture Analyzer | 아키텍처 레이어 식별             |
| Tour Builder        | 의존성 기반 학습 순서 생성         |
| Graph Reviewer      | 무결성 검증                        |
| Domain Analyzer     | 비즈니스 프로세스 추출             |

파일 분석은 5개 동시 실행, 20~30개 파일 배치로 처리된다.
증분 업데이트를 지원하므로 전체 재분석 없이 변경된 파일만 갱신된다.

## 주요 기능

**비주얼 탐색**: 파일, 함수, 클래스 간 의존 관계를 노드 그래프로 시각화한다.
노드를 클릭하면 상세 정보를 확인할 수 있고, 퍼지(fuzzy) 및 시맨틱 검색을 지원한다.

**도메인 뷰**: 코드 구조를 실제 비즈니스 프로세스로 번역한다.
도메인, 흐름, 절차적 단계를 수평 그래프 레이아웃으로 표현한다.

**지식베이스 분석**: Karpathy 패턴의 LLM 위키를 처리하여 위키링크와 관계를 추출하고 커뮤니티 클러스터링이 적용된 포스 다이렉티드 그래프를 생성한다.

**변경 영향 분석**: `/understand-diff` 커맨드로 커밋 전에 변경 사항의 파급 효과를 시각화한다.

**팀 공유**: 그래프 JSON을 리포지토리에 커밋하여 팀이 재분석 없이 공유할 수 있다.
포스트 커밋 훅으로 그래프를 자동 갱신한다.

## CLI

```bash
/understand              # 분석 파이프라인 실행
/understand-dashboard    # 인터랙티브 탐색기 열기
/understand-chat         # 코드베이스 Q&A
/understand-diff         # 변경 영향 분석
/understand-domain       # 비즈니스 지식 추출
/understand-knowledge <path>  # 위키 분석
```

## 설치

**Claude Code**: 플러그인 마켓플레이스에서 네이티브 설치.

**크로스 플랫폼**: Codex, OpenCode, Gemini CLI 등을 위한 원라인 bash/PowerShell 인스톨러 제공.

**Cursor/VS Code**: 플러그인 설정 파일을 통한 자동 감지.

## 분석

### 코드 이해 문제를 시각화로 접근하는 발상

소프트웨어 엔지니어가 낯선 코드베이스를 이해하는 데 쓰는 시간은 전체 개발 시간의 상당 부분을 차지한다.
기존 접근은 파일을 순차적으로 읽거나, IDE의 "go to definition"을 반복하거나, 동료에게 묻는 방식이었다.
Understand-Anything은 이 문제를 그래프 시각화로 접근한다.
의존 관계가 시각적으로 드러나면 코드의 흐름을 파악하는 데 걸리는 시간이 줄어든다는 가설이다.

특히 두 레이어를 분리한 것이 흥미롭다.
구조 그래프(코드 아키텍처)와 도메인 그래프(비즈니스 프로세스)를 별도로 제공한다.
개발자, PM, 신입 팀원이 같은 시스템을 다른 추상화 수준에서 탐색할 수 있다.
이것은 "코드를 이해하는 사람"의 정의를 확장한다.

### 팀 협업 도구로서의 가능성

그래프 JSON을 리포지토리에 커밋하는 접근은 코드베이스 이해를 팀의 공유 자산으로 만든다.
신규 팀원 온보딩 시간을 줄이고, 팀 전체가 같은 아키텍처 멘탈 모델을 공유할 수 있다.
변경 영향 분석 기능은 리뷰어가 PR의 파급 효과를 빠르게 파악하는 데 활용될 수 있다.

## 비평

### 대규모 코드베이스에서의 실용성 한계

[GeekNews 댓글](https://news.hada.io/topic?id=29178)에서 laeyoung은 큰 코드베이스에서 초기 테스트 시 그래프 정확도 문제를 보고했다.
rlaaudgjs5638은 성능 한계와 UI 불안정성(클릭, 드래그 시)을 언급했다.
오픈소스 프로젝트로서 "시도 자체는 가치 있다"고 평가했지만, 프로덕션 코드베이스에 바로 적용하기에는 성숙도가 부족할 수 있다.

AI 에이전트가 생성하는 그래프의 정확도는 코드베이스의 복잡성, 언어, 패턴에 따라 크게 달라진다.
마이크로서비스 아키텍처처럼 서비스 간 의존성이 런타임에 결정되는 경우, 정적 분석 기반 그래프는 실제 관계를 충분히 포착하지 못할 수 있다.

### 유지보수 비용 문제

자동 생성 그래프가 코드 변경에 얼마나 빠르게 동기화되느냐가 실용성의 핵심이다.
포스트 커밋 훅으로 증분 업데이트를 지원한다고 하지만, 대규모 리팩토링 시의 그래프 갱신 비용과 안정성은 별도로 검증이 필요하다.
그래프가 오래되거나 부정확하면 오히려 잘못된 이해를 고착화할 위험이 있다.

## 인사이트

### 코드 이해의 비용을 시장이 과소평가해온 방식

코드베이스 이해 비용은 소프트웨어 개발 전체 비용에서 체계적으로 과소평가된다.
버그 수정이나 기능 추가에 걸리는 시간의 상당 부분은 실제 구현이 아니라 기존 코드를 이해하는 데 소비되지만, 이것은 추적되거나 최적화 대상으로 인식되지 않는다.
Understand-Anything 같은 도구가 12.9k Stars를 빠르게 얻은 것은 이 수요가 실재한다는 증거다.

코드 이해 도구의 발전 경로는 흥미롭다. IDE 검색, 정적 분석, 그래프 시각화, AI 기반 Q&A로 이어지는 흐름이 있고, Understand-Anything은 이 흐름의 현재 지점에 있다.
각 단계는 이전 단계가 제공하지 못하는 무언가를 제공하고, 동시에 새로운 한계를 드러낸다.

코드를 그래프로 보는 것의 가치는 특히 신규 팀원과 팀 간 지식 전달에서 두드러질 것이다.
시각적 표현은 언어적 설명보다 구조를 빠르게 전달하고, 질문이 어디에 집중돼야 하는지를 드러낸다.
"이 서비스는 저것에 의존한다"는 말 대신 그래프를 보여주는 것이 더 효율적인 경우가 많다.

### AI 에이전트 파이프라인을 코드 분석에 적용하는 패턴

6개 에이전트로 구성된 파이프라인 설계는 AI 에이전트 오케스트레이션의 교과서적 예다.
분석 작업을 단일 에이전트에 맡기지 않고 역할을 분리한 설계는 각 에이전트의 전문화와 오류 격리를 가능하게 한다.
특히 Graph Reviewer 에이전트가 독립적으로 무결성을 검증하는 구조는 AI 출력의 신뢰성 문제를 파이프라인 내부에서 다루려는 시도다.

이 패턴은 코드 분석을 넘어 다양한 복잡한 분석 작업에 적용될 수 있다.
단일 LLM 호출로 처리하기 어려운 대규모 분석을 여러 전문화된 에이전트로 분해하고, 각 단계의 결과를 검증하는 구조는 AI 시스템 신뢰성의 핵심 설계 패턴이 되어가고 있다.
