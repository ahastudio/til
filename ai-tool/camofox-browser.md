# Camofox Browser

<https://github.com/jo-inc/camofox-browser>

## 소개

Camofox Browser는 AI 에이전트를 위해 설계된 스텔스 헤드리스 브라우저 서버다.
Firefox 기반의 Camoufox 엔진을 REST API로 래핑하여 Cloudflare와 각종 봇 탐지를 우회한다.
C++ 레벨에서 브라우저 지문(fingerprint)을 위조하는 방식으로 탐지를 피하며,
접근성 스냅샷(accessibility snapshot)을 통해 원본 HTML 대비 토큰을 90%까지 절약한다.

jo-inc가 개발했으며, MIT 라이선스로 배포된다.

## 아키텍처

```text
브라우저 인스턴스 (Camoufox)
└─ 사용자 세션 (BrowserContext)
   ├─ 탭 그룹 (sessionKey별)
   └─ 탭들
```

세션은 30분 비활성 후 자동 만료되고, 브라우저는 5분 유휴 후 종료된다.
탭 한도에 도달하면 가장 오래된 탭이 자동 재활용된다.
유휴 상태에서 메모리 사용량은 약 40MB다.

## 주요 기능

- **안티-탐지**: WebGL, AudioContext, 화면 기하학 등 브라우저 지문을 C++ 수준에서 위조
- **토큰 효율**: 접근성 스냅샷이 원본 HTML보다 ~90% 작음
- **안정적 참조**: 요소를 `e1`, `e2` 등으로 참조해 안정적인 인터랙션 지원
- **세션 격리**: 쿠키·저장소 관리, Netscape 형식 쿠키 파일 가져오기
- **프록시 지원**: 프록시 및 GeoIP 라우팅
- **YouTube 자막**: `yt-dlp` 활용 자막 추출
- **검색 매크로**: `@google_search`, `@amazon_search` 등 14개 내장 매크로

## API 엔드포인트

| 메서드 | 엔드포인트                 | 기능              |
| ------ | -------------------------- | ----------------- |
| POST   | `/tabs`                    | 탭 생성           |
| GET    | `/tabs/:id/snapshot`       | 접근성 스냅샷     |
| POST   | `/tabs/:id/click`          | 요소 클릭         |
| POST   | `/tabs/:id/type`           | 텍스트 입력       |
| POST   | `/tabs/:id/navigate`       | URL 또는 매크로   |
| POST   | `/youtube/transcript`      | YouTube 자막 추출 |

## 설치 및 실행

```bash
# 로컬 실행
git clone https://github.com/jo-inc/camofox-browser
npm install && npm start  # 포트 9377

# Docker
make up  # 아키텍처 자동 감지 및 바이너리 다운로드

# npm 패키지
npm install @askjo/camofox-browser
```

## 분석

Camofox Browser는 AI 에이전트가 웹을 탐색할 때 겪는 두 가지 문제를 동시에 해결한다.
하나는 봇 탐지 우회이고, 다른 하나는 토큰 효율이다.

기존 Playwright나 Puppeteer는 봇 탐지 시스템에 쉽게 걸린다.
Camoufox는 Firefox 소스를 직접 수정해 C++ 레벨에서 지문을 위조하므로,
JavaScript 레벨에서 패치하는 방식보다 훨씬 탐지가 어렵다.

토큰 효율 측면에서 접근성 스냅샷은 중요한 혁신이다.
LLM에게 전체 HTML을 넘기는 대신 구조화된 트리 형태의 스냅샷을 전달하면,
같은 정보를 훨씬 적은 토큰으로 표현할 수 있다.
Claude Code나 MCP 기반 에이전트가 브라우저를 다룰 때 이 방식이 표준이 되어가고 있다.

## 비평

봇 탐지 우회 기능은 명백히 양날의 검이다.
정당한 자동화 테스트나 AI 에이전트 개발에 유용하지만,
악용 가능성도 분명히 존재한다.
MIT 라이선스로 배포되는 만큼 사용 책임은 전적으로 개발자에게 있다.

VNC 웹 UI를 통한 인터랙티브 로그인 지원은 실용적인 기능이다.
OAuth나 2FA가 필요한 사이트에서 초기 인증을 사람이 직접 하고,
이후 세션을 에이전트가 이어받는 패턴을 지원한다.

## 인사이트

### 에이전트를 위한 웹 인프라의 재설계

기존 웹 자동화 도구(Selenium, Playwright)는 사람이 작성한 스크립트를 실행하기 위해 만들어졌다.
Camofox Browser는 설계 철학이 다르다. 처음부터 LLM 에이전트를 1차 소비자로 가정한다.
접근성 스냅샷, 안정적 요소 참조, 검색 매크로 같은 기능들이 모두 이 전제에서 나온다.
앞으로 AI 에이전트가 웹을 사용하는 방식이 일반화되면,
웹 표준 자체도 에이전트 친화적인 방향으로 진화할 가능성이 있다.
WAI-ARIA 접근성 트리는 원래 장애인을 위해 설계됐지만, AI 에이전트의 웹 이해를 위한 인프라가 되고 있다.

### 토큰 절약이 에이전트 경제학을 바꾼다

접근성 스냅샷으로 토큰을 90% 절약한다는 것은 단순한 효율 개선이 아니다.
이는 에이전트가 처리할 수 있는 웹 페이지의 규모를 10배 늘린다는 의미다.
컨텍스트 창 안에 더 많은 페이지 내용을 담을 수 있고, 비용도 그만큼 줄어든다.
AI 에이전트가 실용화되는 데 가장 큰 장벽 중 하나가 비용이라는 점에서,
이런 효율화 기술은 에이전트 보급의 속도를 결정하는 인프라 레이어가 된다.

### 지문 위조 기술의 군비 경쟁

Camoufox가 C++ 레벨에서 지문을 위조한다고 해도, 봇 탐지 시스템도 계속 발전한다.
행동 패턴, 타이밍 분석, 네트워크 특성 등 새로운 탐지 벡터가 계속 등장한다.
이는 본질적으로 군비 경쟁(arms race)이다.
오픈소스 우회 도구가 공개되면 탐지 시스템이 업데이트되고,
다시 새로운 우회 기법이 개발되는 사이클이 반복된다.
장기적으로 이 경쟁의 승자는 불분명하지만, 단기적으로는 도구를 사용하는 쪽이 유리하다.
