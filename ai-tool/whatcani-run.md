# whatcani.run — 내 하드웨어에서 돌릴 수 있는 AI 모델 찾기

원문: <https://whatcani.run/>

## 요약

whatcani.run은 로컬 하드웨어에서 AI 모델을 실행할 때의 실제 성능 데이터를
크라우드소싱으로 수집하여 보여주는 벤치마크 플랫폼이다.
이론적 스펙이 아닌 실사용자의 실측 데이터를 기반으로 한다.

표준화된 테스트 파라미터(입력 4,096 토큰, 출력 1,024 토큰)로
디코드 속도(tok/s), 프리필 속도(tok/s), TTFT(Time To First Token),
최대 메모리 사용량, 복합 점수(0~1)를 측정한다.
186명의 사용자가 4,429회 시행에서 22,658,944개 토큰을 생성한
결과가 축적되어 있다.

주로 Apple Silicon(M1~M4) 기기를 지원하며,
llama.cpp(GGUF 포맷)와 MLX(Apple ML 프레임워크) 런타임을 다룬다.
모델은 성능에 따라 "Runs great", "Runs well", "Runs ok"로 분류된다.
`bunx whatcanirun` 명령으로 CLI에서 바로 벤치마크를 실행할 수 있다.

현재 상위권 모델은 Liquid AI의 LFM2.5 시리즈로,
Q4_0 양자화 시 447.3 tok/s의 디코드 속도를 기록했다.
그 외 Qwen, Llama 3.2, Gemma 4 등 오픈소스 모델이 포함되어 있다.

## 분석

### 크라우드소싱 벤치마크의 가치

모델 성능은 이론적 파라미터 수나 벤치마크 점수만으로 판단하기 어렵다.
같은 모델이라도 양자화 방식, 런타임, 하드웨어 구성에 따라
실제 추론 속도가 크게 달라진다.
whatcani.run은 이 변수들의 조합별 실측 데이터를 모아
"내 기기에서 이 모델이 얼마나 빠른가"라는 질문에 직접 답한다.

### Apple Silicon 중심 설계

현재 지원 하드웨어가 M1~M4와 A18 Pro에 집중되어 있다.
이는 Apple Silicon의 통합 메모리(Unified Memory) 아키텍처가
LLM 로컬 추론에 특히 유리하기 때문이다.
GPU와 CPU가 메모리를 공유하므로 VRAM 병목 없이
대형 모델을 로드할 수 있다.
MLX 프레임워크가 이 아키텍처에 최적화된 추론을 제공한다.

### 복합 점수 설계

디코드 속도, 프리필 속도, TTFT, 메모리 사용량을
0~1 사이의 복합 점수로 정규화한다.
단일 지표가 아니라 여러 차원을 종합하여 평가하므로
"빠르지만 메모리를 과도하게 쓰는 모델"과
"느리지만 가벼운 모델"을 균형 있게 비교할 수 있다.

## 비평

### 하드웨어 범위의 제한

Apple Silicon에 집중된 것은 강점이자 약점이다.
NVIDIA GPU, AMD CPU/GPU, Qualcomm Snapdragon X 등
다른 로컬 추론 플랫폼 사용자에게는 유용하지 않다.
특히 리눅스 서버의 NVIDIA GPU에서 로컬 모델을 운영하는
사용자층이 상당함에도 이를 다루지 않는다.

### 벤치마크 조건의 단일성

입력 4,096 토큰, 출력 1,024 토큰이라는 고정된 조건은
모든 사용 시나리오를 대표하지 못한다.
짧은 대화형 응답(100토큰 미만)에서의 TTFT가 중요한 경우나,
긴 문서 요약(8K+ 토큰 입력)에서의 프리필 성능이 중요한 경우를
구분하지 않는다.
사용 패턴별 성능 프로파일을 제공하면 더 유용할 것이다.

### 모델 품질 지표의 부재

속도와 메모리만 측정하고 모델의 출력 품질은 다루지 않는다.
빠르게 동작하지만 답변 품질이 낮은 모델이 상위에 오를 수 있다.
현재 상위권인 LFM2.5-VL-450M은 450M 파라미터 모델로,
속도는 빠르지만 복잡한 추론 능력은 제한적이다.
속도-품질 트레이드오프를 함께 보여주는 것이 이상적이다.

## 인사이트

### 로컬 추론의 민주화와 "충분히 좋은" 모델의 부상

whatcani.run이 등장한 맥락은 로컬 LLM 추론이
전문가의 영역에서 일반 사용자의 영역으로 이동하고 있다는 것이다.
2023년만 해도 로컬에서 LLM을 돌리려면 GGUF 포맷 변환,
양자화 레벨 선택, 런타임 컴파일 등 상당한 기술 지식이 필요했다.
이제는 "내 맥북에서 뭐가 잘 돌아가나?"라는 질문에
데이터 기반으로 답하는 서비스가 나올 만큼 저변이 넓어졌다.

이 변화의 핵심은 "충분히 좋은(good enough)" 모델의 부상이다.
GPT-4급 성능이 필요 없는 사용 사례가 대부분이며,
로컬에서 빠르게 동작하는 작은 모델이
API 호출의 지연 시간과 비용을 대체할 수 있다.
whatcani.run의 상위 모델이 350M~4B 파라미터 범위인 것은
이 "충분히 좋은" 지점이 생각보다 작은 모델에 있음을 시사한다.

이러한 흐름은 클라우드 AI API 시장에도 영향을 미친다.
간단한 텍스트 처리, 분류, 요약 등의 작업이
로컬 모델로 이동하면 API 제공업체의 수익 구조가 변할 수 있다.
프리미엄 추론(복잡한 코딩, 긴 맥락 분석)에 대한 수요만 남고,
일상적인 추론은 엣지로 이동하는 양극화가 진행될 수 있다.

### 벤치마크의 사회적 구성 — 무엇을 측정하느냐가 무엇을 만드느냐

whatcani.run이 측정하는 지표(속도, 메모리)는
로컬 추론의 "좋음"을 특정 방식으로 정의한다.
이 정의는 모델 개발의 방향에 영향을 미친다.
벤치마크에서 높은 점수를 받는 모델이 주목받으므로,
모델 개발자는 해당 벤치마크에 최적화하는 인센티브를 갖는다.

이것은 구들하트의 법칙(Goodhart's Law)의 전형적인 사례가 될 수 있다.
"측정 지표가 목표가 되면 좋은 지표가 되기를 멈춘다."
속도만 최적화하면 정확도가 희생되고,
메모리만 최적화하면 컨텍스트 길이가 줄어든다.
whatcani.run이 품질 지표 없이 속도 중심으로 운영되면,
"빠르지만 멍청한" 모델이 양산되는 역효과가 나타날 수 있다.

반면, 이 벤치마크가 기여하는 긍정적 측면도 있다.
"작은 모델도 충분히 유용하다"는 증거를 축적함으로써
무조건 큰 모델을 만드는 경쟁(스케일링 법칙 맹신)에
균형추를 제공한다.
효율성이 하나의 경쟁 차원으로 자리 잡으면
같은 성능을 더 적은 자원으로 달성하는 연구가 촉진된다.

### Apple의 조용한 AI 플랫폼 전략

whatcani.run이 Apple Silicon에 집중하는 현상은
Apple의 AI 전략을 간접적으로 드러낸다.
Apple은 자체 AI 모델을 대대적으로 발표하는 대신,
MLX 프레임워크와 통합 메모리 아키텍처를 통해
"AI가 가장 잘 돌아가는 하드웨어"로 포지셔닝하고 있다.

이는 1990년대 "Intel Inside" 전략의 AI 버전이다.
Intel은 CPU 성능을 마케팅하여 PC 생태계를 장악했고,
Apple은 이제 "Apple Silicon Inside"로
로컬 AI 추론의 기본 플랫폼이 되려 한다.
whatcani.run 같은 서드파티 벤치마크가 이 서사를 강화한다.
"M4 Pro에서 이 모델이 이만큼 빠르다"는 데이터가 쌓일수록
Apple Silicon은 로컬 AI의 사실상 표준 하드웨어로 자리 잡는다.

그러나 이 전략에는 리스크가 있다.
NVIDIA는 RTX 시리즈에 로컬 추론 최적화를 강화하고 있고,
Qualcomm은 Snapdragon X Elite에서 NPU 성능을 내세운다.
Apple의 통합 메모리 우위가 영원하지 않을 수 있으며,
경쟁 플랫폼의 벤치마크 데이터가 축적되면
whatcani.run의 Apple 중심성도 바뀔 수 있다.
