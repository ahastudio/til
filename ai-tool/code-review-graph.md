# code-review-graph

<https://github.com/tirth8205/code-review-graph>

## 소개

AI 코딩 도구가 코드 리뷰 시 코드베이스 전체를 반복해서 읽으면서 토큰을 과소비하는 문제를 해결하는 도구다.
Tree-sitter로 코드 구조 맵을 만들고 변경 사항을 증분(incremental)으로 추적해
AI 어시스턴트에게 MCP를 통해 필요한 컨텍스트만 정확히 제공한다.

6개 실제 레포지토리에서 38배~528배의 토큰 절감 효과를 벤치마크로 측정했다.
Python 3.10+ 환경에서 `pip install code-review-graph`로 설치하고
`code-review-graph install` 명령 하나로 지원 플랫폼을 자동 감지해 설정한다.

## 주요 기능

지원 플랫폼은 Claude Code, Codex, Cursor, Windsurf, Zed, Gemini CLI, GitHub Copilot,
Kiro, Qoder, Antigravity, Continue, OpenCode, CodeBuddy 등이다.

`code-review-graph install` 실행 시 설치된 AI 코딩 도구를 자동으로 감지하고
각 도구의 MCP 설정 파일을 자동으로 수정한다.
플랫폼별 훅과 스킬을 설치하고 규칙 파일에 그래프 인식 지침을 삽입한다.

```bash
pip install code-review-graph
code-review-graph install   # 지원 플랫폼 자동 감지 및 설정
code-review-graph build     # 코드베이스 파싱
```

## 분석

### 토큰 절감 이면의 구조

AI 코딩 도구가 코드 리뷰 시 전체 파일을 읽는 것은 구조적 이유가 있다.
변경된 파일만 보면 그 파일이 의존하는 함수, 호출하는 모듈, 공유하는 타입을 알 수 없다.
code-review-graph는 Tree-sitter로 심볼 의존성 그래프를 미리 구축해
변경 파일에서 실제 연결된 코드만 컨텍스트로 전달한다.

38배~528배 절감이라는 범위 차이가 중요하다.
528배는 대형 모노레포에서 발생하는 극단적 케이스일 것이다.
일반적인 중간 규모 프로젝트에서는 더 낮은 절감율이 나올 수 있다.

### MCP를 활용한 도구 독립적 접근

MCP 서버로 구현해 특정 AI 도구에 종속되지 않는 것이 핵심 설계 선택이다.
AI 코딩 도구 생태계가 빠르게 변하는 상황에서
프로토콜 수준 통합은 신규 도구 지원을 플러그인 추가로 처리할 수 있게 한다.

## 비평

### 증분 추적이 대규모 리팩토링에서 어떻게 작동하는가

파일 하나 수정이나 함수 추가의 증분 추적은 명확하다.
대규모 리팩토링, 인터페이스 변경, 디렉토리 재구성 같은 경우에
그래프 업데이트가 얼마나 정확하게 이뤄지는지는 실제 사용으로 확인해야 한다.
정확하지 않은 그래프는 필요한 컨텍스트를 누락시켜 리뷰 품질을 낮춘다.

## 인사이트

### 컨텍스트 품질이 토큰 수보다 중요하다

적은 토큰으로 좋은 리뷰를 하려면 어떤 코드를 컨텍스트로 넣는가가 핵심이다.
전체 파일을 넣는 것보다 관련 심볼 의존성을 정확히 추출해서 넣는 것이
리뷰 품질과 비용을 동시에 개선한다.
code-review-graph가 이 문제를 정적 분석으로 해결하는 접근은
AI 코딩 도구의 컨텍스트 관리 문제에 대한 좋은 방향이다.
