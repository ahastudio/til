# LLM Router by Cafe24

<https://llm-router.cafe24.com/>

GN 토론: <https://news.hada.io/topic?id=30813>

## 소개

LLM Router는 Cafe24가 운영하는 통합 LLM 인프라 서비스다.
단일 OpenAI 호환 API 엔드포인트로 Claude, Gemini, Llama, DeepSeek, Mistral 등
100개 이상의 모델을 호출할 수 있다.
프롬프트 분석을 통한 자동 모델 선택, 프로바이더 장애 시 자동 전환, 의미 기반 캐시를
하나의 서비스로 묶는다.

원화(₩) 기반 종량제로 과금하며, 월 기본요금이 없고 실패한 요청은 과금하지 않는
ZCI(Zero Completion Insurance) 정책을 적용한다.
가입 시 무료 크레딧을 제공한다.

## 주요 기능

### Auto Router

프롬프트를 분석해 코딩, 추론, 번역, 창작 등 작업 유형을 자동 판별하고 최적 모델을 선택한다.
예를 들어 “React 무한 스크롤 코드 만들어줘” 같은 요청은 코딩 작업으로 분류되어
`claude-sonnet-4-6` 등 코딩에 강한 모델로 라우팅된다.

### Provider Routing

비용 최소화, 응답 속도, 처리량, 균형 모드 중 우선순위 전략을 선택할 수 있다.
동일한 API 호출이라도 전략에 따라 사용하는 프로바이더와 모델이 달라진다.

### Auto Fallback

장애나 타임아웃 발생 시 사전 정의된 대체 경로로 즉시 전환된다.
호출 코드 변경 없이 안정성이 확보된다.

### Managed Cache (Semantic Cache)

의미적으로 유사한 쿼리가 반복될 경우 LLM 호출을 건너뛰고 캐시된 응답을 반환한다.
`SEMANTIC HIT 0 tokens · 8ms` 형태로 응답이 반환되어, 표준 레이턴시 대비 수십 배 빠른
응답과 토큰 비용 절감을 동시에 달성한다.

### BYOK (Bring Your Own Key)

OpenAI, Anthropic, Google 등 기존 API 키를 직접 등록해 사용한다.
비용은 원래 프로바이더에게 지불하고, Cafe24의 라우팅 인프라만 활용하는 구조다.

### PII Mask

로그 저장 전, 그리고 모델 전송 전에 민감 정보를 자동으로 마스킹한다.

### 기타

| 기능               | 설명                                             |
| ------------------ | ------------------------------------------------ |
| Realtime Dashboard | 요청, 비용, 토큰 추이를 실시간 모니터링          |
| Playground         | 코드 없이 브라우저에서 모델 비교 테스트          |
| Preset             | 설정 저장 및 코드 배포 없는 라우팅 변경          |

## 지원 모델

Claude, Gemini, Qwen, Llama, DeepSeek, GLM, Gemma, Seed, Nemotron, Mistral,
MiMo, Kimi, Hunyuan, Phi, Step, MiniMax, Ling, MythoMax 등 100개 이상의 모델을 포함한다.

## 분석

### OpenAI 호환 단일 엔드포인트 전략의 의미

“OpenAI 호환”을 전면에 내세운 것은 기술 선택이 아니라 마케팅 전략이다.
OpenAI API는 사실상 LLM 호출의 업계 표준 인터페이스가 됐다.
LangChain, LlamaIndex, Vercel AI SDK를 비롯한 대부분의 AI 개발 툴체인이
OpenAI 포맷을 기본으로 설계되어 있다.
LLM Router가 이 포맷을 그대로 지원하면, 기존 코드베이스에서 API 키와 베이스 URL만 바꿔도
서비스를 즉시 사용할 수 있다.

이 전략은 OpenRouter, LiteLLM 같은 경쟁 서비스가 이미 채택한 경로다.
차별화 포인트는 인터페이스가 아니라 라우팅 품질, 신뢰성, 가격이어야 한다.
LLM Router는 여기에 더해 원화 과금, 한국 기업 운영, ISMS 인증, 세금계산서 발행을
한국 기업 고객을 위한 차별점으로 내세운다.

### Auto Router의 작동 원리가 공개되지 않는다

“프롬프트를 분석해 최적 모델 선택”이라는 설명은 핵심 질문에 답하지 않는다.
분류 기준은 무엇인가.
규칙 기반인가, 별도 분류 모델을 사용하는가, 아니면 LLM 자체로 판단하는가.
분류 오류율은 얼마인가.
사용자가 분류 결과를 확인하거나 재지정할 수 있는가.

이 정보 없이는 Auto Router가 실제로 “최적” 모델을 선택하는지 검증할 방법이 없다.
“React 무한 스크롤 코드”가 `claude-sonnet-4-6`으로 라우팅됐다는 예시는
결과를 보여줄 뿐이고, 그 선택이 왜 최적인지 설명하지 않는다.
서비스가 어떤 판단 기준으로 모델을 선택하는지 불투명한 상태에서는,
사용자는 라우팅 결과를 신뢰할 이유가 없다.

### Cafe24가 이 서비스를 만든 이유

Cafe24는 국내 쇼핑몰 솔루션과 호스팅 시장의 지배적 사업자다.
LLM Router는 이 맥락에서 읽어야 한다.

Cafe24의 주 고객은 소규모 이커머스 사업자다.
AI 챗봇, 상품 설명 자동 생성, 고객 응대 자동화 등의 수요가 이 고객군에서 급증하고 있다.
LLM Router는 Cafe24 생태계 안에서 AI 기능을 쉽게 붙일 수 있는 인프라이기도 하다.
“쇼핑몰 자동화”를 사용 사례로 명시한 것은 이 배경을 확인해준다.

동시에 원화 과금, 세금계산서, ISMS 인증은 한국 기업 시장 전반을 겨냥하는 신호다.
OpenRouter나 AWS Bedrock은 달러 과금이고 한국어 지원이 제한적이다.
이 공백을 Cafe24가 채우려는 포지셔닝으로 읽을 수 있다.

### Semantic Cache는 비결정적 서비스에 결정적 레이어를 얹는다

LLM의 핵심 속성 중 하나는 동일한 프롬프트에도 다른 응답이 나올 수 있다는 것이다.
Semantic Cache는 “의미적으로 유사한” 쿼리에 캐시된 응답을 반환한다.
이 설계는 비용과 속도를 개선하지만, 동시에 응답의 다양성과 최신성을 희생한다.

캐시 적중 판단 임계값이 공개되지 않는다.
“유사하다”는 기준이 느슨하면 다른 의미의 질문에 잘못된 답이 반환된다.
기준이 엄격하면 캐시 효율이 낮아진다.
이 트레이드오프는 서비스 문서에 언급되지 않는다.

## 비평

### 라우팅 품질의 검증 불가능성이 서비스의 핵심 결함이다

LLM Router의 핵심 가치 제안은 “최적 모델 자동 선택”이다.
그런데 이 선택이 최적인지 사용자가 확인할 방법이 없다.
대시보드는 어떤 모델로 라우팅됐는지 보여주지만,
“이 요청에 다른 모델을 썼다면 어땠을까”를 비교할 기준이 제공되지 않는다.

벤치마크 없는 라우터는 블랙박스다.
코딩 작업에 모델 A 대신 모델 B가 선택됐을 때 품질 차이가 얼마나 나는지,
라우팅이 틀렸을 때 얼마나 자주 그 오류가 발생하는지,
사용자가 이 정보 없이 “자동”을 믿기로 결정해야 한다.
실제로 Auto Router가 최선의 선택을 하고 있는지는 서비스 운영자만 알고 있다.

### BYOK와 Managed 과금의 혼재가 투명성을 떨어뜨린다

BYOK 모드에서는 API 비용이 원래 프로바이더에 직접 지불된다.
Managed 모드에서는 LLM Router에 토큰 기반으로 지불한다.
두 모드가 공존하는 구조는 실제 비용 비교를 복잡하게 만든다.

BYOK를 쓰면 라우팅 인프라 비용을 별도로 지불하는가, 아닌가가 불명확하다.
Managed 모드의 토큰 가격이 각 프로바이더 직접 가격의 마크업이라면 얼마인가가 공개되지 않는다.
“원화 기반”이라는 점은 편의성이지만, 환율 적용 방식과 마진 구조가 투명하게 공개되어야
진짜 비용 최적화 서비스로 신뢰받을 수 있다.
GeekNews에서 nokdu는 “Cafe24는 어디서 수수료를 챙길려나요”라고 물었고,[^nokdu]
helio는 서비스 전체를 “K-오픈라우터”라고 꼬집었다.[^helio]
두 반응 모두 같은 지점을 찌른다 — OpenRouter와 기능적으로 다르지 않은데 수익 구조가 불명확하다면,
이 서비스를 쓸 이유가 무엇인가.

### ZCI 정책은 실질적 보호가 아니라 심리적 보호에 가깝다

“실패한 요청은 과금하지 않는다”는 ZCI는 매력적으로 들린다.
그러나 실패 요청은 이미 어떤 프로바이더도 과금하지 않는 것이 일반적이다.
완료되지 않은 API 호출에 대한 청구는 업계 표준이 아니다.

ZCI가 진짜 보호가 되려면 “부분 완료” 시나리오를 다루어야 한다.
스트리밍 응답이 도중에 끊겼을 때, 타임아웃 직전에 일부 토큰이 소비됐을 때,
Auto Fallback이 첫 번째 프로바이더 호출 후 실패해 두 번째로 전환됐을 때의 과금 방식이
구체적으로 설명되지 않는다.
“실패한 요청”의 정의가 명확하지 않으면 ZCI는 마케팅 언어에 머문다.

## 인사이트

### LLM 라우터 시장은 수렴이 아니라 분기하고 있다

범용 LLM 라우터 시장에는 이미 OpenRouter, LiteLLM, AWS Bedrock, Azure OpenAI Service가 있다.
LLM Router by Cafe24의 등장은 이 시장이 글로벌 단일 서비스로 수렴하지 않고
지역별 특화 서비스로 분기하고 있음을 보여준다.

일본에서는 Sakura Internet이 유사한 포지셔닝을 시도했다.
유럽에서는 GDPR 준수와 데이터 주권을 내세운 로컬 AI 인프라 서비스들이 생겨났다.
한국에서 Cafe24가 원화 과금, ISMS 인증, 세금계산서를 내세우는 것은 같은 흐름이다.

이 분기의 핵심 동인은 규제와 신뢰다.
LLM 호출에 포함된 데이터가 어느 나라 서버를 거치는지, 그 데이터가 모델 훈련에 쓰이는지는
기업 고객에게 점점 중요해지는 질문이다.
글로벌 서비스가 이 질문에 명확히 답하지 못할 때, 로컬 서비스의 공간이 생긴다.

### 라우터 레이어의 부상은 LLM 상품화의 신호다

LLM Router 같은 서비스가 가능해지려면, 다양한 LLM이 서로 교환 가능할 만큼 품질이 균질화되어야 한다.
모델 간 차이가 극명하면 자동 라우팅은 의미가 없다.
특정 작업에 특정 모델만 쓸 수 있다면 라우팅할 것이 없기 때문이다.

라우터 서비스의 성장은 역설적으로 LLM 자체가 상품(commodity)화되고 있다는 신호다.
AWS가 컴퓨팅 리소스를 상품화했을 때 등장한 것이 클라우드 비용 최적화 레이어(CloudHealth, Spot.io)였다.
LLM 라우터는 이와 구조적으로 동일하다.
모델이 상품이 되면, 그 위에 라우팅·캐싱·모니터링을 얹는 레이어가 새로운 가치를 창출한다.

이 흐름이 계속된다면, LLM 프로바이더는 점점 가격 경쟁에 빨려 들어가고
수익성은 라우터 레이어로 이동할 것이다.
이것은 AWS와 인프라 최적화 SaaS의 관계가 LLM 시장에서 재연되는 패턴이다.

### Semantic Cache는 LLM의 비결정성을 조용히 제거한다

Semantic Cache가 광범위하게 적용될 경우, LLM의 비결정성은 사용자 경험에서 사라진다.
“같은 질문에 매번 다른 답”이라는 LLM의 특성이 캐시로 가려지면,
사용자는 LLM이 아니라 사실상 캐시된 데이터베이스와 상호작용하게 된다.

이 변화는 일부 유스케이스에서는 긍정적이다.
FAQ 응답, 표준화된 데이터 추출, 반복적 코드 생성에서는 일관성이 다양성보다 중요하다.
그러나 창의적 작업, 최신 정보가 중요한 질문, 개인화된 맥락이 필요한 대화에서는
캐시된 응답이 오히려 품질을 낮춘다.

캐시 적중률이 높아질수록 서비스의 경제성은 좋아지지만,
그 서비스가 LLM을 사용하는 이유가 있는지 다시 묻게 된다.
“의미적으로 유사한 질문”에 항상 같은 답을 준다면, 그것은 LLM인가 검색 엔진인가.
Semantic Cache가 해결하려는 비용 문제는 궁극적으로 LLM이 필요한 영역을 축소한다.

### 한국 시장의 AI 인프라 의존성이 새로운 리스크를 만든다

한국 기업들이 원화 과금, 한국어 지원, 로컬 규정 준수를 이유로 LLM Router 같은 서비스에 의존하게 되면,
그 서비스가 단종되거나 품질이 저하될 때의 전환 비용이 문제가 된다.

OpenAI 호환 API는 마이그레이션을 쉽게 만드는 것처럼 보인다.
그러나 Preset 기능으로 라우팅 설정을 저장하고, BYOK로 키를 등록하고,
Semantic Cache에 의존한 아키텍처를 구축하면 실제 전환 비용은 상당하다.
특히 캐시된 응답에 의존하는 시스템을 다른 라우터로 이전하면
응답 품질이 갑자기 달라지는 경험을 하게 된다.

로컬 AI 인프라 서비스는 규제와 신뢰 문제를 해결하지만,
동시에 벤더 락인(vendor lock-in)의 새로운 레이어를 만든다.
Cafe24가 쇼핑몰 호스팅으로 한국 이커머스의 인프라가 된 것처럼,
LLM Router가 한국 AI 개발의 인프라 레이어가 된다면
그 의존성의 리스크도 함께 커진다.
