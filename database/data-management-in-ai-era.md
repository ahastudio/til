# AI 시대의 데이터 관리

원문: [DATA MANAGEMENT IN THE AGE OF AI - by William Inmon](https://williaminmon.substack.com/p/data-management-in-the-age-of-ai)

GN 토론: <https://news.hada.io/topic?id=31681>

## 요약

데이터 웨어하우스(Data Warehouse) 개념을 창시한 Bill Inmon이
컴퓨터 산업 초기부터 생성형 AI 시대까지 데이터 관리가 어떻게 진화했는지를 정리했다.

초기에는 개발자가 데이터 설계를 부수적인 작업으로 처리했다.
거래 처리 시스템이 등장하면서 데이터베이스 설계자가 필요해졌고,
다중 데이터베이스 환경이 되면서 DBA(데이터베이스 관리자)가 생겼다.
데이터 무결성 문제가 기업 전략 문제로 격상되면서 CIO가 탄생했고,
정형/비정형 데이터 통합 요구가 CDO(최고데이터책임자)를 만들었다.

생성형 AI 환경에서 데이터 관리자의 역할은 근본적으로 달라진다.
구조화된 데이터 환경에서는 데이터베이스를 직접 제어했지만,
LLM 기반 시스템에서는 “어떤 텍스트가 LLM에 입력되는가”를 간접적으로 통제한다.

저자는 이를 “puppet manipulation”에 비유한다.
데이터 관리자가 무대 뒤에서 입력 텍스트를 선별하고 정제함으로써
LLM의 행동을 간접적으로 조율한다.
온톨로지와 분류 체계가 전통적 데이터 모델을 대체하며,
ELDM(엔터프라이즈 논리 데이터 모델)이 의미론적 일관성을 유지하는 도구가 된다.

## 분석

### 데이터 관리자 역할이 “직접 제어”에서 “입력 큐레이션”으로 이동한다

전통적 데이터 관리는 스키마 설계, 인덱스 최적화, 쿼리 튜닝처럼 구조를 직접 다루는 일이었다.
LLM 시스템에서는 모델의 내부 처리에 개입할 수 없다.
개입 가능한 지점은 “모델에 무엇을 보여주는가”뿐이다.

이 전환은 데이터 관리의 레버가 바뀐다는 의미다.
스키마보다 분류 체계가, 쿼리보다 텍스트 선별이, 인덱스보다 컨텍스트 조립이 중요해진다.
Inmon이 “간접 통제”라고 부르는 이 방식은 사실 RAG 파이프라인 설계와 정확히 같은 문제다.

### CDO 역할이 AI 시대에 재정의된다

CDO는 정형/비정형 데이터 통합 시대의 산물이다.
AI 시대에 CDO의 역할은 더 복잡해진다.
어떤 텍스트 데이터가 LLM 입력으로 적합한가, 어떤 온톨로지가 조직의 지식을 일관되게 표현하는가,
LLM이 생성한 콘텐츠의 품질을 어떻게 평가하는가 — 이것들이 CDO의 새 책임이 된다.

Inmon의 분석이 예리한 지점은, AI 도입이 데이터 관리를 단순화하는 것이 아니라 더 복잡하게 만든다는 점이다.
자동화가 늘수록 그 자동화를 제어하는 데이터의 품질이 더 중요해진다.

## 비평

### “puppet manipulation” 비유는 통제 환상을 줄 수 있다

Inmon이 제시하는 간접 통제 모델은 실제보다 통제 가능성을 낙관적으로 본다.
LLM의 출력은 입력 텍스트만으로 결정되지 않는다.
사전 훈련 데이터, RLHF, 모델 아키텍처, 추론 파라미터가 복합적으로 작용한다.

데이터 관리자가 “어떤 텍스트를 넣는가”를 정교하게 통제해도,
LLM이 그 텍스트를 어떻게 처리하는가는 블랙박스로 남는다.
“입력을 통제하면 출력을 통제할 수 있다”는 가정은
LLM의 비결정론적 특성 앞에서 쉽게 무너진다.

### 온톨로지 기반 접근은 구축 비용이 높다

저자는 온톨로지와 분류 체계를 전통 데이터 모델의 대안으로 제시한다.
그러나 기업 온톨로지 구축은 수년이 걸리는 작업이고,
도메인 전문가와 지식 엔지니어가 집중적으로 협업해야 한다.

전통적 DB 스키마도 구축 비용이 높았지만, 적어도 구조화된 쿼리로 검증이 가능했다.
온톨로지 기반 접근이 LLM 출력 품질을 얼마나 실질적으로 향상시키는지는
대규모 실증 데이터 없이 판단하기 어렵다.

## 인사이트

### 데이터 품질 문제는 AI 도입으로 해결되지 않고 증폭된다

많은 조직이 AI를 도입하면 데이터 관리 부담이 줄어들 것으로 기대한다.
Inmon의 분석은 반대를 시사한다.
“AI는 신뢰할 수 있는 데이터에서만 신뢰할 수 있는 분석을 만든다.”

AI가 의사결정에 더 깊이 개입할수록, 잘못된 입력 데이터의 파급 효과는 커진다.
스프레드시트 오류는 보고서 하나를 망치지만, LLM이 잘못된 지식 베이스를 기반으로 자율적 결정을 내리면
그 오류가 시스템 전체로 확산된다.
데이터 관리가 부수적 업무에서 전략적 기능이 된 역사가 AI 시대에 또 한 번 반복된다.

### 역할 이름이 바뀌어도 근본 문제는 같다

DBA에서 CDO로, 다시 AI 시대의 데이터 관리자로.
역할 이름이 바뀔 때마다 “신뢰할 수 있는 데이터를 어떻게 만들고 유지하는가”라는 근본 문제는 변하지 않는다.
새 기술이 새 데이터 문제를 만들고, 그 문제를 해결하기 위한 새 역할이 생기는 순환이다.

AI 시대에 데이터 관리자가 직면하는 문제 — LLM 입력의 품질 보증, 온톨로지 유지, 의미론적 일관성 — 는
이전 시대의 스키마 설계, 정합성 검증, 마스터 데이터 관리와 구조적으로 같다.
새 이름으로 불리는 오래된 문제다.
