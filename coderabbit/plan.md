# CodeRabbit Plan

<https://www.coderabbit.ai/plan>

<https://docs.coderabbit.ai/overview/coderabbit-plan>

아이디어, 이슈, PRD, 디자인을 프로젝트에 맞는 상세한 코딩 플랜으로 변환하는
기능. 코드베이스에 대한 깊은 이해를 바탕으로 플랜을 생성하며, 완성된 플랜은
코딩 에이전트에게 바로 넘길 수 있음.

## 핵심 가치

"프롬프트를 처음부터 제대로 만드는 것"에 초점. 개발자와 AI 에이전트 사이의
반복 작업을 줄여줌. 코딩 에이전트가 자체적으로 생성하는 플랜은 보통 소수의
파일만 참조하지만, CodeRabbit Plan은 지속적 코드 분석과 광범위한 지식 베이스를
기반으로 올바른 파일을 참조하고, 기존 패턴을 따르며, 기존 코드와 자연스럽게
통합됨.

## 작동 방식

### 1. 플랜 생성

- **웹 인터페이스**: app.coderabbit.ai/plan에서 요구사항을 입력하고 저장소를
  선택해 플랜 생성.
- **이슈 트래커**: GitHub, GitLab, Jira, Linear 이슈에 `@coderabbitai plan`
  코멘트로 플랜 생성. 자동 플래닝(Auto-planning) 설정도 가능.

### 2. 플랜 개선

팀원들이 채팅 인터페이스를 통해 세부 사항을 조정하고 설계 결정에 대해 논의.
웹 앱 또는 이슈 코멘트에서 플랜을 편집할 수 있어 엔지니어와 PM이 함께 소유권을
가짐. 완전한 버전 히스토리 제공.

### 3. 에이전트 핸드오프

완성된 플랜은 코드베이스 컨텍스트가 포함된 구조화된 프롬프트로 내보내짐.
Claude Code, Cursor, Gemini, VS Code, Windsurf 등 어떤 코딩 에이전트에든
재해석 없이 바로 전달 가능.

## 지원 플랫폼

GitHub Issues, GitLab, Jira, Linear.

## 특징

- 코드베이스 아키텍처, 패턴, 기존 코드를 분석해 프로젝트 구조에 맞는 플랜 생성.
- 관련 이슈, 디자인 문서, 프로젝트 학습 내용을 활용한 컨텍스트 기반 플래닝.
- 조직 내 누구나 플랜을 확인하고 논의하고 재플래닝을 요청할 수 있음.
- 레이블, 프로젝트, 담당자 기준으로 설정 가능한 규칙 세트.
- 코드는 명시적으로 공유하지 않는 한 CodeRabbit 인프라 외부로 나가지 않음.
- 특정 에이전트에 종속되지 않으며 플랜은 완전히 이동 가능.
