# Huashu Design — HTML-native 디자인 스킬

원문: <https://github.com/alchaincyf/huashu-design>

## 소개

Huashu Design(花叔 Design)은 Claude Code를 비롯한 AI 에이전트에서 쓸 수 있는
HTML-native 디자인 스킬(skill)이다.
“한 문장 입력 → 납품 가능한 디자인”을 표방하며,
고보존도 인터랙티브 프로토타입, 슬라이드 덱, 모션 애니메이션, 인포그래픽을
단일 HTML 파일로 생성한다.

설치는 `npx skills add alchaincyf/huashu-design` 한 줄이며,
Claude Code·Cursor·Codex·OpenClaw·Hermes 등 마크다운 스킬을 지원하는
모든 에이전트에서 동작한다(Agent-Agnostic).
4일 만에 GitHub Star 5,000개를 넘겼고 Fork가 790여 개에 달한다.

저자는 독립 개발자 '花生(huasheng)'으로,
AppStore 유료 순위 1위 앱 제작·AI 책 출판·30만 팔로워 테크 채널을 운영하는
중국의 AI Native Coder다.
이 스킬은 Anthropic이 발표한 Claude Design의 시스템 프롬프트와 브랜드 자산
프로토콜에서 아이디어를 얻어 에이전트 스킬 형태로 증류(distill)한 것이다.

## 주요 기능

| 기능                  | 결과물                                                    | 소요 시간  |
| --------------------- | --------------------------------------------------------- | ---------- |
| 인터랙티브 프로토타입 | 단일 HTML · iPhone 15 Pro 베젤 · 클릭 가능 · Playwright 검증 | 10–15분    |
| 슬라이드 덱           | HTML deck + 편집 가능한 PPTX(텍스트 프레임 보존)         | 15–25분    |
| 모션 디자인           | MP4(25/60fps) + GIF(palette 최적화) + BGM                | 8–12분     |
| 디자인 변형           | 3개 이상 병렬 비교 · Tweaks 실시간 파라미터              | 10분       |
| 인포그래픽/시각화     | 인쇄 품질 타이포그래피 · PDF/PNG/SVG 내보내기            | 10분       |
| 디자인 방향 어드바이저 | 5유파 × 20 디자인 철학 → 3방향 추천 + Demo 병렬 생성     | 5분        |
| 5차원 전문가 리뷰     | 레이더 차트 + Keep/Fix/Quick Wins 수정 목록              | 3분        |

## 핵심 메커니즘

### 핵심 자산 프로토콜(Core Asset Protocol)

브랜드가 특정될 때 강제로 실행되는 5단계 하드 플로우다.

1. **질문** — 로고·제품 이미지·UI 스크린샷·색상·폰트를 항목별로 한 번에 요청
2. **공식 채널 검색** — 브랜드 공식 페이지, press kit, App Store 스크린샷 탐색
3. **다운로드** — SVG/PNG 직접 취득 → HTML 파싱 인라인 SVG 추출 → SNS 아바타 순서로 세 가지 대안
4. **검증 + 추출** — “5회 검색 → 10개 후보 → 2개 선별 · 각 8점 이상”의 철칙
5. **`brand-spec.md` 고정** — CSS 변수로 선언하여 모든 HTML이 `var(--brand-*)` 참조

이 프로토콜의 핵심 철학은 **“자산 > 규범”**: 색상표보다 로고와 제품 사진이
브랜드 인식에 훨씬 더 크게 기여한다는 것이다.

### 디자인 방향 어드바이저(Fallback 모드)

요청이 모호할 때 진입하는 대안 흐름이다.
5개 유파(Pentagram 정보 건축, Field.io 운동 시학, Kenya Hara 동방 미니멀,
Sagmeister 실험 선구 등)와 20종 디자인 철학 라이브러리에서
**서로 다른 유파에 속하는** 3개 방향을 추천하고,
24개의 사전 제작 쇼케이스(8 시나리오 × 3 스타일)를 참고로 제시한다.
세 방향의 Demo를 병렬로 생성해 사용자가 선택하게 한다.

### Junior Designer 워크플로

기본 작업 모드다.
착수 전 가정·플레이스홀더·추론 주석을 HTML에 먼저 써넣고 사용자에게 일찍
보여준다. “착수 → 플레이스홀더 show → 실제 콘텐츠 채우기 → 변형 → Tweaks”
단계마다 중간 결과를 공개해 오해를 조기에 수정한다.

### 반(反) AI slop 규칙

보라색 그라디언트, 이모지 아이콘, 왼쪽 보더 accent 카드, SVG 얼굴 그림,
Inter 폰트 디스플레이 사용 같은 “AI 공통 최대공약수” 시각 패턴을 금지한다.
대신 `text-wrap: pretty`, CSS Grid, OKLCH 색공간, 선별된 serif display 폰트를
적극 사용한다.

### 사실 검증 우선 원칙(#0)

에이전트가 특정 제품·기술·발표 상태에 대해 단언하기 전에 반드시 `WebSearch`로
검증한다. “기억에서 단언하지 말라”는 규칙을 `SKILL.md` 최상위에 배치해
모든 다른 플로우보다 높은 우선순위를 부여했다.

## 분석

### HTML을 최종 출력물로 사용하는 전략

대부분의 디자인 스킬이 이미지나 Figma 파일 생성을 목표로 삼는 반면,
huashu-design은 HTML 단일 파일을 **완성 결과물**로 삼는다.
HTML은 에이전트가 코드로 직접 작성할 수 있고, 브라우저에서 즉시 확인 가능하며,
Playwright로 자동 검증할 수 있고, MP4/GIF로 변환도 된다.
이 선택은 에이전트-GUI 인터페이스를 아예 없애는 동시에,
브라우저라는 범용 뷰어를 활용하는 실용적 타협이다.

### 스킬 설계 철학의 성숙도

`SKILL.md`에는 단순한 태스크 기술이 아니라, 에이전트가 오작동하는 구체적 실패
사례(“대疆 Pocket 4는 아직 출시 안 됐다”고 단언했다가 이미 발매된 제품임이
밝혀진 2026-04-20 사건)와 그 교훈이 내장되어 있다.
“A/B 테스트에서 v2 안정성 분산이 v1 대비 5배 낮다”는 실험 데이터도 포함된다.
이는 단순 프롬프트 엔지니어링을 넘어, 반복 실패에서 귀납한 **스킬 엔지니어링**에
가깝다.

### 오픈소스와 상업적 통제의 이중 구조

개인 사용은 완전 자유이지만 기업 상업용 사용에는 명시적 허가가 필요하다.
이 라이선스 구조는 기여자 커뮤니티를 통한 스타 확산을 허용하면서도,
기업 의존성이 생기는 시점에서 수익화 채널을 확보하는 전형적인
“개방형 코어(Open Core)” 전략이다.

### `npx skills` 생태계

`npx skills add <repo>` 명령은 GitHub 저장소를 에이전트 스킬로 설치하는
`skills.sh` 인프라를 전제로 한다.
이 생태계가 성숙하면, npm 생태계처럼 AI 에이전트용 스킬 패키지 매니저가
형성될 가능성이 있다.

## 비평

### 강점

5단계 브랜드 자산 프로토콜은 “기억에서 색상을 추측하지 말라”는 원칙을
체계적 플로우로 구현한 점에서 탁월하다.
많은 AI 디자인 도구가 프롬프트에 의존하는 반면, 이 스킬은 실패 시나리오와
대안 경로(fallback)까지 명시해 실무 안정성을 높였다.
Junior Designer 워크플로는 “이른 공개, 늦은 완성” 원칙을 에이전트 동작에
직접 삽입한 드문 사례다.
반 AI slop 규칙 목록은 “AI 특유의 미적 최대공약수”를 의식적으로 회피하려는
시도로, 결과물 품질에 실질적으로 기여한다.

### 약점

HTML 단일 파일 접근은 Framer Motion 수준의 복잡 애니메이션, 3D, 물리 시뮬레이션,
파티클 시스템을 지원하지 못한다.
PPTX 내보내기는 “진짜 텍스트 프레임”을 자랑하지만, 레이어 단위 Figma 편집이나
Keynote 텍스트 위치 변경은 불가능하다.
완전히 새로운 브랜드를 처음부터 디자인할 경우 품질이 60–65점으로 떨어진다고
저자 스스로 인정한다.
`SKILL.md`가 중국어로 작성되어 있어, 영어권 에이전트가 내부 규칙을 완전히
따르는지 여부가 불확실하다(영어 태스크를 처리할 수는 있다고 밝히고 있으나,
규칙 자체가 중국어로 인코딩된 에이전트 지시문이다).

### 아직 열린 질문

`skills.sh` 인프라의 지속성과 표준화 여부가 스킬 생태계 전체의 미래를 결정한다.
또한 Playwright를 이용한 클릭 테스트가 에이전트 실행 환경에서 항상 작동한다는
보장이 얼마나 견고한지는 사용 환경에 따라 다를 수 있다.

## 인사이트

### 프롬프트 엔지니어링에서 스킬 엔지니어링으로

huashu-design의 `SKILL.md`는 단순 지시문(instruction)이 아니다.
실패 사례, 원인 분석, 개선된 규칙, A/B 테스트 데이터가 한 문서 안에 누적된
**공학 문서**다.
“Pocket 4 발매 여부를 기억으로 단언했다가 반환 작업 2시간”이라는 구체적 비용
계산은 규칙에 동기를 부여한다.
이 패턴은 소프트웨어 엔지니어링에서 postmortem 문화가 코드베이스가 아닌
에이전트 행동 규칙으로 이식된 것이다.
AI 에이전트를 “쓰는” 것과 “키우는” 것의 차이가 여기서 생긴다.
실패를 기록하고 규칙으로 변환하는 루프를 구축하지 않으면,
에이전트는 매번 같은 실수를 반복하는 값비싼 도구에 머문다.
반면 이 루프가 자리를 잡으면, 스킬은 사용할수록 안정성이 올라가는 복리 자산이 된다.

### HTML이 설계 매체(medium)가 된다는 것의 의미

모든 UI 도구는 “무엇을 표현 단위로 삼는가”라는 질문에 대한 답이다.
Figma는 레이어와 컴포넌트, After Effects는 타임라인과 키프레임,
HTML은 DOM과 CSS 계산 스타일(computed style)이다.
huashu-design이 `html2pptx.js`에서 “DOM의 computedStyle을 PowerPoint 객체로
번역한다”고 설명할 때, 이는 두 설계 매체 사이의 의미론적 가교(semantic bridge)다.
에이전트가 HTML을 생성하고, 그 HTML에서 PPTX·MP4·PDF를 파생시키는 파이프라인은
“HTML이 단일 진실의 원천(single source of truth)”임을 선언한다.
이 구조는 장기적으로 설계 결과물의 버전 관리, diff, 자동화 테스트를
텍스트 파일 수준에서 가능하게 만든다.
Figma나 PowerPoint가 바이너리 파일로 남아 있는 한, 이 장점은 에이전트 시대에
점점 더 두드러질 것이다.

### “GUI의 소멸”이 새로운 UI 패러다임을 요구한다

저자는 “Claude Design은 더 좋은 그래픽 도구, huashu-design은 그래픽 도구라는
층 자체를 없애는 것”이라고 규정한다.
그러나 이것은 단순히 인터페이스 레이어 하나를 제거하는 일이 아니다.
GUI가 없어지면 사용자는 의도를 자연어로 표현해야 하고,
에이전트는 그 자연어에서 구체적 픽셀 배치로 이어지는 경로를
스스로 판단해야 한다.
이 갭을 메우는 것이 스킬 문서 안의 “20종 디자인 철학 라이브러리”다.
사용자가 “좋아 보이는 것”을 말로 설명하지 못할 때,
에이전트가 대신 선택지를 제시하고 선택을 받는 구조는
새로운 종류의 인터랙션 패턴을 탄생시킨다.
이것은 GUI의 버튼을 대화 턴으로 교체한 것이고,
결과적으로 사용성(usability)의 정의 자체를 다시 쓰는 실험이다.
어떤 사람에게는 타이핑이 클릭보다 빠를 수 있고,
또 어떤 사람에게는 정반대다.
두 세계는 당분간 공존하겠지만, 에이전트 중심 작업 흐름이 성숙할수록
“열기·드래그·닫기”의 GUI 관용구가 이미 늙은 언어처럼 느껴지는 날이 올 것이다.
