# Skills for Real Engineers

<https://github.com/mattpocock/skills>

## 소개

Matt Pocock의 “Skills for Real Engineers”는 Claude Code와 다른 코딩 에이전트를 위한 슬래시 커맨드 모음이다.
“바이브 코딩”이 아닌 실제 엔지니어링 원칙에 기반한 AI 보조 개발을 목표로 한다.
작고 조합 가능하며 적응 가능한 도구들로 AI 에이전트의 네 가지 공통 실패 모드를 해결한다.
GitHub에서 66.1k 스타, 5.7k 포크를 기록하고 있다.

## 설치 및 설정

```bash
npx skills@latest add mattpocock/skills
```

설치 후 `/setup-matt-pocock-skills`를 실행해 세 가지를 구성한다.

- 이슈 트래커 선택 (GitHub, Linear, 로컬 파일)
- 트리아지 레이블 어휘 정의
- 문서 저장 위치 지정

## 해결하는 네 가지 실패 모드

**1. 에이전트-개발자 간 정렬 실패 (Misalignment)**
`/grill-me`와 `/grill-with-docs`가 구현 시작 전 상세한 질의응답 세션을 진행해
공유 이해를 구축한다.

**2. 장황한 출력 (Verbosity)**
`CONTEXT.md`로 도메인 특화 용어 사전을 만들어 토큰 낭비를 줄이면서
코드 일관성과 탐색성을 높인다.

**3. 동작하지 않는 코드 (Non-functional code)**
`/tdd`가 red-green-refactor 사이클을 구현하고,
`/diagnose`가 구조화된 디버깅 방법론을 제공한다.

**4. 아키텍처 퇴화 (Code entropy)**
`/to-prd`와 `/zoom-out`이 모듈 경계와 시스템 비전을 복원한다.
`/improve-codebase-architecture`가 도메인 언어와 아키텍처 결정 기록에 기반해
복잡한 코드베이스 리팩토링을 안내한다.

## 스킬 목록

**엔지니어링 스킬 (10개)**: TDD, 트리아지, PRD 생성, 진단, 아키텍처 개선 등.

**생산성 스킬 (3개)**:
- `caveman`: 75% 토큰 절감 모드로 압축 커뮤니케이션 실행
- `grill-me`: 구조화된 질의응답으로 요구사항 명료화
- 커스텀 스킬 생성 도구

**기타 스킬 (4개)**: git 안전장치, 테스트 마이그레이션, 연습 스캐폴딩, pre-commit 설정.

## 분석

### 공유 언어(Shared Language)가 핵심 철학이다

Matt Pocock이 강조하는 것은 도구보다 언어다.
`CONTEXT.md`의 도메인 특화 어휘 예시로 “materialization cascade”를 든다.
에이전트와 개발자가 이 용어의 의미를 공유하면,
프롬프트에서 해당 개념을 매번 설명할 필요가 없다.
토큰 절감은 부수 효과이고, 실제 목표는 의사소통 정밀도의 향상이다.

이 접근은 도메인 주도 설계(DDD)의 유비쿼터스 언어(Ubiquitous Language) 개념을
에이전트-개발자 협업에 적용한 것이다.
팀 내 공유 언어가 팀 역량을 높이듯,
에이전트와의 공유 언어가 에이전트 협업 역량을 높인다.

### 고전 엔지니어링 원칙의 에이전트 시대 재적용

TDD, DDD, 익스트림 프로그래밍(XP)을 슬래시 커맨드로 구체화하는 것이 이 저장소의 핵심 아이디어다.
에이전트가 일으키는 새로운 문제들—정렬 실패, 코드 엔트로피—이
실제로는 오래된 소프트웨어 엔지니어링 문제의 가속된 형태임을 인식한다.

에이전트 시대의 “바이브 코딩”이 요구사항 없이 빠르게 코드를 생성하는 것이라면,
이 스킬들은 그 반대 방향을 추구한다.
빠른 생성 이전에 느린 이해와 정렬을 요구한다.

## 비평

### 강점: 작고 조합 가능한 설계

모노리틱 자동화 프레임워크 대신 작은 슬래시 커맨드들을 조합하는 방식은
사용자가 자신의 워크플로에 맞게 선택하고 커스터마이징할 수 있게 한다.
`npx skills@latest add mattpocock/skills` 한 줄로 시작하는 낮은 진입 장벽도 긍정적이다.

### 약점: Claude Code 생태계 의존성

이 스킬들은 Claude Code 에코시스템에 최적화되어 있다.
다른 에이전트 환경(GitHub Copilot, Cursor 등)에서의 호환성이나 이식성은
명시적으로 다루지 않는다.
Claude Code 사용자 기반이 좁아지거나 스킬 포맷이 변경되면 유지보수 부담이 생긴다.

또한 스킬 효과는 에이전트의 능력에 크게 의존한다.
`/tdd`가 제대로 동작하려면 에이전트가 테스트 작성과 red-green-refactor 사이클을
충분히 이해해야 한다. 스킬 자체가 그 역량을 만들지는 않는다.

## 인사이트

### 프롬프트 엔지니어링의 다음 단계는 워크플로 엔지니어링이다

초기 AI 보조 개발은 “더 좋은 프롬프트”를 쓰는 데 초점을 맞췄다.
Matt Pocock의 스킬들은 그 다음 단계를 제안한다.
개별 프롬프트를 최적화하는 것이 아니라, 에이전트와의 협업 워크플로 전체를 설계하는 것이다.

/grill-me가 요구사항을 명료화하고, /tdd가 구현 루프를 관리하고,
/zoom-out이 아키텍처 일관성을 유지하는 구조는
에이전트를 단발성 질의응답 도구가 아니라 협업 파트너로 다루는 방식이다.
이 전환은 에이전트 생산성을 측정하는 단위를 코드 생성 속도에서
의사결정 품질과 아키텍처 건강으로 바꾼다.

### 도메인 언어 투자의 복리 효과

`CONTEXT.md`를 만드는 초기 투자는 부담스럽게 보일 수 있다.
그러나 이것은 한 번 만들면 모든 에이전트 상호작용에서 재사용되는 자산이다.
도메인 언어가 정교해질수록 에이전트와의 소통 정밀도가 높아지고,
고품질 컨텍스트가 고품질 출력을 만드는 복리 효과가 생긴다.

이 관점에서 `CONTEXT.md`는 팀의 도메인 지식을 외부화하는 도구이기도 하다.
새 팀원(인간이든 에이전트든)이 합류할 때 온보딩 비용을 낮추는 역할을 한다.
암묵적 지식을 명시적으로 만드는 습관이 에이전트 협업의 질을 결정한다.
