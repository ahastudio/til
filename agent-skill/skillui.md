# SkillUI — 웹사이트에서 디자인 시스템을 역공학하는 CLI

<https://skillui.vercel.app/>

<https://github.com/amaancoderx/skillui>

## 소개

SkillUI는 웹사이트, GitHub 저장소, 로컬 프로젝트를 정적 분석해 디자인 시스템을
역공학하는 커맨드라인 도구다. AI, API 키, 클라우드 서비스 없이 완전히 로컬에서
동작한다. 분석 결과를 `.skill` ZIP 파일로 패키징해 Claude Code가 해당 디자인
시스템과 일치하는 UI를 자동으로 생성할 수 있도록 컨텍스트를 제공한다.

## 주요 기능

추출하는 디자인 데이터의 범위가 넓다.

- **색상 팔레트**: 사이트 전체에서 사용된 색상 추출
- **타이포그래피**: 서체, 크기, 행간 등 텍스트 속성
- **간격 시스템**: 패딩, 마진, gap 값 체계
- **border-radius**: 모서리 둥글기 스펙
- **CSS 애니메이션**: keyframe 정의 및 애니메이션 라이브러리 탐지
- **레이아웃 구조**: Flex/Grid 컨테이너 패턴
- **DOM 컴포넌트 패턴**: HTML 핑거프린트 기반 컴포넌트 분류
- **인터랙티브 상태**: hover/focus의 before-after 스타일 비교
- **스크롤 여정 스크린샷**: 비주얼 가이드용 이미지 시퀀스

## CLI

`npm install -g skillui` 로 전역 설치하며 Node.js 18 이상이 필요하다.

```bash
skillui --url https://yoursite.com       # 웹사이트 크롤링
skillui --dir ./my-app                   # 로컬 프로젝트 스캔
skillui --repo <github-url>              # 저장소 클론 후 분석
skillui --mode ultra                     # Playwright 기반 시각적 추출
```

`--mode ultra`를 사용하면 Playwright로 브라우저를 실제 실행해 스크롤 여정
스크린샷 등 동적 시각 데이터까지 추출한다. 기본 모드는 정적 분석만 수행한다.

## 출력물 구조

결과는 `.skill` 확장자의 ZIP 파일로 패키징된다. 내부 구조:

```text
output.skill
├── DESIGN.md          # 색상, 타이포그래피, 간격 문서
├── ANIMATIONS.md      # 애니메이션 사양
├── LAYOUT.md          # 레이아웃 패턴
├── design-tokens.json # JSON 형식 디자인 토큰
├── screenshots/       # 스크롤 여정 이미지 시퀀스
└── fonts/             # 번들된 Google Fonts
```

Claude Code에 이 파일을 전달하면, AI가 추출된 색상, 타이포그래피, 컴포넌트
패턴을 바탕으로 동일한 디자인 언어를 따르는 UI를 생성할 수 있다.

## 기술 스택

TypeScript(97.7%), CSS, JavaScript로 구성되며, 시각적 추출 모드에서는
Playwright를 사용한다. MIT 라이선스로 공개되어 있다.

## 분석

### “디자인 시스템 역공학”이 해결하는 문제

기존에 AI로 UI를 생성하는 워크플로의 가장 큰 한계는 컨텍스트 부족이었다.
“우리 서비스와 비슷한 스타일로 버튼 컴포넌트를 만들어줘”라는 요청에서 AI는
그 서비스의 정확한 색상값, 폰트, 간격 체계를 모른다. 결과물이 늘 “그럴듯하지만
우리답지 않은” 이유다.

SkillUI는 이 간격을 `.skill` 파일로 메운다. 실제 운영 중인 사이트의 디자인
시스템을 기계가 읽을 수 있는 형태로 추출해, AI가 정확한 사양을 기반으로
UI를 생성할 수 있게 한다. “추측 생성”에서 “사양 기반 생성”으로의 전환이다.

### 정적 분석의 한계와 ultra 모드의 의미

기본 모드의 정적 분석(HTML/CSS 파싱)은 실제 렌더링 결과를 반영하지 못한다.
CSS-in-JS, Tailwind의 JIT, 런타임에 주입되는 스타일, JavaScript로 토글되는
클래스들은 정적 분석으로 잡기 어렵다. `--mode ultra`가 Playwright로 브라우저를
실제 실행하는 이유다. 동적 스타일을 캡처하고 스크롤 여정에 따른 상태 변화도
기록한다.

이 설계는 정직하다. 완전한 해결책이 아니라 두 가지 모드를 분리해, 정적
분석의 빠른 결과와 동적 분석의 정확한 결과 중 사용자가 선택하게 한다.

### Claude Code와의 통합 패턴

`.skill` ZIP에 Markdown 문서와 JSON 토큰을 함께 패키징하는 것은 의도적인
선택이다. Claude Code가 소화하기 좋은 형태 — 구조화된 문서와 기계 읽기
가능한 JSON — 를 모두 제공한다. `DESIGN.md`는 Claude가 맥락을 이해하는 데,
`design-tokens.json`은 정확한 값을 참조하는 데 쓰인다.

## 비평

### 강점

API 키나 외부 서비스 없이 로컬에서 완전히 동작한다는 점은 실용적이다.
기업 환경에서 내부 디자인 시스템을 분석할 때 데이터가 외부로 나가지 않는다.
`--repo` 옵션으로 GitHub 저장소를 직접 분석할 수 있는 것도 편리하다.
오픈소스(MIT) 공개로 커스터마이징이 가능하다.

### 한계와 우려

정적 분석 기반이라 현대적인 프론트엔드(React, Vue, Svelte의 CSS-in-JS,
CSS Modules, styled-components)에서 추출 정확도가 낮아질 수 있다. 실제 운영
사이트는 번들된 CSS가 난독화되어 있거나, 디자인 토큰이 런타임에 JavaScript로
생성되는 경우가 많다.

타인의 사이트를 크롤링해 디자인 시스템을 추출하는 행위 자체는 저작권, 이용약관,
윤리적 측면에서 그레이 영역이다. 도구가 이 경계를 명시적으로 다루지 않는 점은
주의가 필요하다.

스크린샷이 포함된 `.skill` 파일의 크기가 얼마나 될지도 미지수다. Claude Code
컨텍스트에 로드할 때 토큰 비용이 상당할 수 있다.

## 인사이트

### 디자인 시스템의 코드화가 AI 시대의 핵심 인터페이스가 된다

SkillUI가 제안하는 것은 단순한 편의 도구가 아니다. 디자인 시스템을 AI가
읽을 수 있는 형태로 구조화하는 것이 AI 시대의 핵심 인터페이스 문제임을
암시한다. 인간 개발자는 Figma 파일, 디자인 가이드라인 문서, 스타일가이드
사이트를 눈으로 보고 감으로 재현한다. AI는 그 “감”을 학습하기 어렵다. 명확한
사양이 필요하다.

`.skill` 파일 포맷이 만약 표준화된다면 — Figma가 디자인 툴의 표준 파일 형식이
된 것처럼 — AI 기반 UI 생성의 인터페이스 레이어로 자리잡을 수 있다. 디자인
시스템의 “컨텍스트 압축”이라는 문제를 해결하는 포맷 전쟁이 이미 시작되었을 수 있다.

### 역공학이 합법적인 지식 이전 수단이 되는 경계

소프트웨어 개발에서 역공학은 오랫동안 법적, 윤리적 긴장 관계에 있었다.
바이너리를 디컴파일하거나 프로토콜을 분석하는 것은 제조사의 이익과 충돌한다.
그러나 공개적으로 서비스 중인 웹사이트의 CSS를 파싱하는 것은 다르다 — 브라우저가
이미 하는 일을 자동화할 뿐이다.

SkillUI가 건드리는 경계는 더 미묘하다. 경쟁사 서비스의 디자인 시스템을 추출해
자신의 제품에 적용하는 것은 기술적으로 가능하지만, 이것이 정당한 영감인지
부당한 도용인지는 컨텍스트에 따라 달라진다. AI가 이 과정을 자동화하고 가속화할수록,
“관찰과 영감”과 “복제”의 경계는 더 흐려진다. 도구 자체보다 사용 방식에 대한
커뮤니티 규범이 필요한 시점이다.

### 컨텍스트 엔지니어링으로서의 디자인 추출

Claude Code에 `.skill` 파일을 제공하는 것은 본질적으로 컨텍스트 엔지니어링이다.
AI의 출력 품질은 입력 컨텍스트의 품질에 정비례한다. 막연한 “회사 스타일로
만들어줘”보다, 정확한 색상 팔레트와 컴포넌트 패턴이 담긴 `.skill` 파일을
전달하면 결과물이 극적으로 개선된다.

이 아이디어는 더 넓게 적용된다. 코드베이스 분석, 데이터베이스 스키마 추출,
API 명세 구조화 — 이 모든 것이 동일한 패턴이다: 복잡한 현실 세계의 정보를
AI가 효과적으로 소화할 수 있는 구조로 압축하는 것. SkillUI는 이 패턴을
디자인 도메인에 적용한 특수화된 사례다. 도메인 특화 컨텍스트 엔지니어링 도구들의
시대가 열리고 있다.
