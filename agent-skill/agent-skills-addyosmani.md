# Agent Skills - Addy Osmani의 프로덕션급 AI 코딩 에이전트 스킬 모음

<https://github.com/addyosmani/agent-skills>

## 요약

Google Cloud AI 디렉터 Addy Osmani가 공개한 오픈소스 프로젝트로, AI 코딩 에이전트가
스펙·테스트·보안 리뷰를 건너뛰는 문제를 해결하기 위해 19개의 엔지니어링 스킬을
제공한다. 개발 사이클 6단계(정의 → 계획 → 구축 → 검증 → 검토 → 배포)에 걸쳐
7개의 슬래시 명령어(`/spec`, `/plan`, `/build`, `/test`, `/review`,
`/code-simplify`, `/ship`)로 자동화된 워크플로우를 구성한다.

## 분석

설계 철학의 핵심은 “검증은 협상 불가”다. 모든 단계가 실제 증거(테스트 통과, 빌드
출력, 런타임 데이터)로 완료 기준을 정의한다. 에이전트가 “완료했다”고 선언하는 것이
아니라 관찰 가능한 결과물로 완료를 증명하게 만든다.

Google 엔지니어링 원칙들이 스킬에 직접 내장되어 있다.

- **Hyrum's Law**: 모든 공개 인터페이스는 결국 의존된다 — 변경 시 영향 범위를 검토하라
- **Beyoncé Rule**: “좋으면 테스트를 달았어야지” — 검증 없이 배포하지 말라
- **Chesterton's Fence**: 이유를 모르면 제거하지 말라

각 스킬에는 “나중에 테스트 추가하겠다” 같은 흔한 합리화와 반박 논거를 담은
안티-합리화 표도 포함된다. 에이전트의 지름길 유혹을 명시적으로 차단하는 구조다.

Claude Code뿐 아니라 Cursor, Gemini CLI, Windsurf, GitHub Copilot 등 다양한
플랫폼을 지원하며, 모든 스킬은 마크다운 파일로 시스템 프롬프트에 주입된다.

## 비평

GeekNews의 xguru는 핵심을 짚었다. “마크다운 파일이라 전부 도입할 필요는 없고,
토큰 소모량을 고려해 필요한 것만 선별해야 한다.” 19개 스킬을 전부 활성화하면
컨텍스트 창을 상당히 소비하고 에이전트의 반응 속도와 유연성이 떨어질 수 있다.

또한 댓글에서는 “자신만의 하네스를 만들어가는 것이 효율적”이라는 의견이 지배적이었다.
남이 만든 스킬 모음을 그대로 쓰기보다는, 이 저장소를 출발점으로 팀 혹은 개인의
맥락에 맞게 커스터마이징하는 전략이 더 현실적이다.

Addy Osmani가 Chrome 팀 리드에서 Google Cloud AI 디렉터로 이직한 직후 발표된
프로젝트라는 점도 맥락으로 읽힌다. 이 프로젝트는 기술적 기여이자 새 역할에서의
포지셔닝이기도 하다.

## 인사이트

**에이전트의 지름길 본능**: AI 코딩 에이전트가 테스트, 스펙, 보안 리뷰를 건너뛰는
이유는 훈련 데이터에 “일단 동작하는 코드”가 과잉 대표되기 때문이다. 이 스킬 모음이
해결하려는 문제는 에이전트의 기술적 한계가 아니라 인센티브 구조의 문제다. 명시적으로
“건너뛰지 말라”는 규칙을 주입하지 않으면 에이전트는 최단 경로를 택한다.

**코딩 문화의 외재화**: Hyrum's Law, Beyoncé Rule, Chesterton's Fence 같은
구글의 엔지니어링 문화 원칙이 에이전트의 행동 지침으로 변환된다는 점은 의미심장하다.
수십 년에 걸쳐 조직 내에서 구전되던 엔지니어링 지혜가 마크다운 파일 몇 줄로
누구나 에이전트에 주입할 수 있는 형태가 됐다. 문화가 코드가 되는 전환이다.

**스킬 생태계의 파편화 위험**: Claude Code, Cursor, Gemini CLI 각각을 위한 스킬
형식이 다르면서도 이 프로젝트는 통합 지원을 주장한다. 실제로는 플랫폼마다 스킬이
동작하는 방식이 달라 “한 번 작성, 어디서나 실행”이 허상일 가능성이 높다. 에이전트
스킬 표준화가 생태계의 다음 과제로 부상하고 있다.

**시니어 엔지니어 지식의 증류**: 이 프로젝트의 가장 큰 가치는 기술 스택이 아니라
시니어 엔지니어들이 코드 리뷰에서 반복적으로 지적하는 패턴들을 명시화했다는 점이다.
“나중에 테스트 추가하겠다”는 말이 왜 거짓말인지를 에이전트에게 가르치는 것은,
주니어 개발자에게 같은 내용을 가르치는 것과 구조적으로 동일한 문제다.
