# Natural Korean Skills

<https://github.com/bear2u/natural-korean-skills>

## 소개

`natural-korean-skills`는 한국어 문서 작성과 교정을 위한 에이전트 스킬(Agent Skills) 모음이다.
일본어 스타일 교정 프로젝트인 `coji/natural-japanese`의 설계 철학을 한국어 형태소 구조에 맞게 재구성한 것으로,
bear2u가 제작했다.

저장소의 핵심 철학은 “검출은 기계, 판단은 사람 또는 AI”다.
정적 린터가 검토 후보를 찾아내되, 실제 수정 여부는 문서 목적과 맥락에 따라 판단하는 구조다.
규칙 위반을 곧바로 오류로 취급하지 않고 검토 항목으로 분류한다는 점이 기존 맞춤법 교정기와 근본적으로 다르다.

워크플로는 다섯 단계로 구성된다.
“설계 → 작성/수정 → 정적 탐지 → 문맥 판단 → 재검사”의 순환 구조이며,
각 단계가 독립된 관심사를 담당한다.

## 스킬 구성

저장소는 8개의 스킬을 제공한다.

**교정 스킬:**

- `natural-korean`: 기존 문서의 자연스러움·명확성·문체 일관성을 개선한다.
  독자, 목적, 장르를 먼저 확인한 뒤 사실·수치·고유명사·인용문을 보존하면서 교정한다.

**범용 작성 스킬:**

- `write-natural-korean`: 블로그, 칼럼, 실용 가이드를 기획하고 작성한다.
  주제 확장에 앞서 독자 기대와 핵심 메시지를 먼저 설계하며,
  검증되지 않은 통계나 인용은 생성하지 않는다.

**분야별 작성 스킬:**

- `write-korean-business`: 의사결정 메모, 기획서, 보고서를 작성한다.
  결정권자가 첫 화면에서 무엇을 승인해야 하는지 파악할 수 있도록 구성한다.

- `write-korean-technical`: 개발자 가이드, API 문서, 런북을 작성한다.
  버전, 전제 조건, 명령어, 예상 결과, 실패 분기를 구조화하여 재현 가능성을 보장한다.

- `write-korean-blog`: 독자의 질문과 핵심 답변을 먼저 확정한 뒤 작성한다.
  검색 키워드보다 자연스러운 한국어 표현을 우선한다.

- `write-korean-essay`: 에세이, 회고, 산문을 작성한다.
  실제 장면과 기억을 중심으로 구성하며, 사용자의 어휘와 리듬을 보존한다.

- `write-korean-public`: 공고, 안내문, 보도자료를 작성한다.
  시민이 자신과 관련 있는 내용인지 즉시 파악하고 “언제, 어디서, 무엇을” 찾을 수 있도록 구성한다.

- `write-korean-meetings-slides`: 회의록, 액션 로그, 슬라이드를 작성한다.
  발언·결정·미결 항목을 구분하고, 슬라이드 논리가 제목만으로 따라갈 수 있도록 설계한다.

## 문체 검사 규칙

린터는 K001~K006 여섯 개 항목을 검사하며, 장르(일반·업무·기술·블로그)에 따라 임계값이 달리 적용된다.

| 코드 | 항목                        | 목적                         |
| ---- | --------------------------- | ---------------------------- |
| K001 | 상투 표현과 번역투 후보     | 어색한 표현 탐지             |
| K002 | 동일 문장 종결 반복         | 단조로운 리듬 탐지           |
| K003 | 문체 혼용(합니다/한다/해요) | 문체 일관성 확보             |
| K004 | 추상어·범용 동사 과밀       | 모호함 감소                  |
| K005 | 지시 표현·접속사 반복       | 흐름 개선                    |
| K006 | 문장·문단 길이 균질성       | 가독성 향상                  |

블로그 구조 검사는 A001~A004로 별도 구성된다.
상위 제목 누락·중복(A001), 소제목 반복(A002), 미완성 표지 잔존(A003), 빈 섹션(A004)을 확인한다.

## 보조 도구

Python 3.11+ 환경에서 `uv`로 실행하는 명령줄 도구를 포함한다.

| 도구              | 역할                                     |
| ----------------- | ---------------------------------------- |
| `lint.py`         | 문체 검토 후보와 통계 출력               |
| `outline.py`      | 제목과 문단 첫 문장 추출                 |
| `terms.py`        | 영문 약어·고유명사 초출 확인             |
| `article_check.py`| 블로그 구조와 미완성 표지 검사           |
| `calibrate.py`    | 문서 유형별 규칙 정확도 비교             |

마크다운 코드 블록, 표, URL, 인용 블록은 통계 왜곡을 막기 위해 검사에서 제외된다.

## 분석

### 에이전트 스킬의 새로운 역할 정의

이 저장소는 AI 에이전트에게 “무엇을 쓸 것인가”보다 “어떻게 쓸 것인가”를 가르치는 시도다.
기존 프롬프트 엔지니어링이 출력 품질에 집중했다면,
이 접근은 글쓰기 과정 자체를 단계별로 분해하여 각 단계의 책임을 명확히 한다.

에이전트 스킬을 문체 교정에 특화하는 방식은 크게 두 가지 의미를 가진다.
첫째, 범용 LLM(Large Language Model)이 언어별 문체 감각을 충분히 내재화하고 있다는 가정에 도전한다.
둘째, 스킬이라는 모듈 단위가 반복 가능하고 검증 가능한 행동 집합으로 기능함을 보여준다.

`natural-japanese`를 원류로 삼는다는 사실은 이 구조가 언어 독립적인 패턴임을 시사한다.
한국어판이 일본어판의 철학을 이어받으면서도 형태소 분석 체계를 별도로 구현한다는 점에서,
언어별 문체 교정 에이전트 시리즈의 가능성이 열려 있다.

### 정적 탐지와 동적 판단의 분리

K001~K006 규칙 체계의 핵심은 탐지와 교정을 의도적으로 분리한다는 점이다.
린터는 후보를 찾지만 수정 여부를 결정하지 않는다.
이 분리는 단순한 설계 선택이 아니라 언어의 본성에서 비롯된다.

한국어 문체는 동일한 표현이 어떤 맥락에서는 상투어가 되고 어떤 맥락에서는 적절한 형식이 된다.
“검토 결과에 따라”라는 표현이 행정 공문에서는 자연스럽지만 개인 블로그에서는 번역투로 느껴진다.
규칙만으로 이 판단을 자동화하기 어렵기 때문에, 저장소는 판단을 사람 또는 AI에게 명시적으로 위임한다.

장르별 임계값 분기(일반·업무·기술·블로그)는 이 판단을 부분적으로 자동화한 시도다.
같은 규칙이라도 맥락에 따라 허용 범위를 달리 적용함으로써
과도한 경고와 과소한 경고 사이의 균형을 찾는다.

### 검증 체계의 설계

`dev/` 디렉터리의 fixture 회귀 검사는 스킬의 신뢰성을 유지하기 위한 구조적 장치다.
자연스러운 업무 문서에서 불필요한 탐지가 억제되는지,
상투적 문서에서 K001~K005가 실제로 검출되는지를 자동으로 확인한다.

이 검증 방식은 소프트웨어 테스트의 회귀 방지 전략을 글쓰기 스킬에 적용한 것이다.
스킬이 업데이트될 때 이전에 통과하던 케이스가 무너지지 않도록 보호하며,
`calibrate.py`는 인간 문서와 AI 문서 집단 간의 규칙 정확도 차이를 측정한다.
이는 스킬의 초기 임계값이 MVP 수준임을 솔직하게 인정하면서도
코퍼스 기반 보정의 방향을 제시한다.

## 비평

### 스킬 분리의 경계가 실제 작업에서 모호해진다

8개 스킬의 경계는 문서 유형을 기준으로 나뉘지만,
실제 작업에서 하나의 문서가 여러 스킬의 영역에 걸치는 경우가 빈번하다.
기술 블로그는 `write-korean-blog`와 `write-korean-technical`의 중간에 위치하고,
의사결정 메모가 포함된 회의록은 `write-korean-business`와 `write-korean-meetings-slides`가 충돌한다.

사용자는 어느 스킬을 선택해야 하는지 판단해야 하며,
그 판단이 틀렸을 때 어떤 결과가 나오는지에 대한 안내가 없다.
스킬 선택의 오류를 교정하는 메타 스킬이 없다는 점은 구조적 공백이다.
분류 경계를 명확히 하거나, 복합 문서 유형을 처리하는 전략을 문서화할 필요가 있다.

### 기준 코퍼스의 부재가 규칙의 정당성을 약화한다

K001의 “상투 표현 후보” 목록이 어떤 기준으로 선정되었는지는 공개되어 있지 않다.
K006의 “문장 길이 균질성” 임계값이 왜 특정 수치로 설정되었는지도 설명이 없다.
저장소 자체가 MVP 임계값을 인정하지만, 그 임계값이 어떤 데이터에서 나왔는지는 불투명하다.

언어 교정 도구에서 규칙의 정당성은 데이터에서 나온다.
코퍼스 없이 설정된 임계값은 작성자의 직관이며, 직관은 편향될 수 있다.
“자연스러운 한국어”의 기준이 특정 문체, 세대, 도메인에 편향되어 있을 가능성을
저장소는 인정하지 않는다.

### 에이전트 런타임 의존성이 검증되지 않는다

스킬은 Codex에 설치되어 에이전트 환경에서 실행됨을 전제한다.
그러나 스킬이 실제로 에이전트의 행동을 어떻게 바꾸는지,
에이전트가 스킬의 5단계 워크플로를 실제로 따르는지에 대한 검증은 없다.

`evals/` 디렉터리의 평가 케이스가 이 검증을 일부 담당하지만,
에이전트가 스킬을 얼마나 충실하게 따르는지를 측정하는 정량적 지표는 공개되어 있지 않다.
린터의 Python 도구는 독립적으로 검증 가능하지만,
스킬 자체는 에이전트의 해석에 전적으로 의존한다는 점에서 블랙박스로 남는다.

## 인사이트

### 문체 스킬은 언어 교육의 새로운 형태다

`natural-korean-skills`가 만드는 것은 교정 도구가 아니라 교정 방법론이다.
스킬 파일은 에이전트에게 절차를 가르치고, 린터는 에이전트의 판단을 보조한다.
이 구조는 전통적인 맞춤법 교정기와 근본적으로 다른 계층에서 작동한다.

교정기가 규칙을 직접 적용한다면, 스킬은 에이전트가 규칙을 적용하는 방식을 설계한다.
이 차이는 교육학의 “직접 교수법”과 “메타인지 훈련”의 차이와 유사하다.
스킬을 통해 에이전트는 특정 오류를 고치는 법을 배우는 것이 아니라
문서를 읽고 판단하는 방식을 배운다.

이 방향이 확장되면, 에이전트 스킬은 특정 전문가의 글쓰기 방식을 모델링하는
수단이 될 수 있다. 기자, 법조인, 의료진의 문서 작성 패턴을 스킬로 정의하고
에이전트가 그 맥락에서 작성하도록 유도하는 시도로 이어질 수 있다.

### 다국어 확장은 언어별 형태소 분석의 복잡도를 드러낸다

일본어판에서 한국어판으로의 이식은 단순한 번역이 아니다.
한국어는 일본어와 형태소 구조가 달라, 동일한 규칙이 다른 방식으로 구현되어야 한다.
K002(종결형 반복) 규칙은 한국어 어미 체계의 복잡성 때문에 일본어판보다 훨씬 세밀한 분기가 필요하다.

이 이식 작업은 “언어 독립적 설계”가 실제로는 불가능함을 보여준다.
범용 글쓰기 원칙은 언어를 가로질러 이전될 수 있지만,
구현은 언어 고유의 문법 체계에 깊이 의존한다.
자연어 처리 도구가 아무리 발전해도 언어별 전문화의 필요성이 사라지지 않는 이유가 여기 있다.

언어별 스킬 생태계가 형성된다면, 그 관리 비용은 도구의 정교함에 비례하여 증가한다.
`natural-korean-skills`는 그 비용을 가시화하는 첫 사례다.

### “판단을 위임하는 도구”는 책임 귀속 문제를 바꾼다

린터가 검토 후보를 제시하고 판단은 사람 또는 AI에게 남긴다는 설계는
도구의 책임 범위를 의도적으로 제한한다.
이는 정직한 설계이지만, 동시에 책임 귀속의 문제를 새로운 방식으로 제기한다.

에이전트가 린터의 K001 결과를 보고 수정하지 않기로 판단했을 때,
그 판단이 틀렸다면 누구의 책임인가.
스킬 작성자는 “판단은 사람 또는 AI에게 맡긴다”고 명시했기 때문에 책임에서 벗어난다.
에이전트는 스킬의 지시를 따랐기 때문에 스스로를 정당화할 수 있다.

이 책임의 공백은 에이전트 시대의 구조적 문제다.
도구가 판단을 위임할수록 오류의 원인을 찾기가 어려워진다.
`natural-korean-skills`가 이 문제를 해결하지는 않지만,
판단 위임을 명시적으로 설계에 포함시킨다는 점에서 투명성은 높다.
책임 귀속을 설계 문서에서 다루는 관행이 에이전트 도구 전반으로 확산될 필요가 있다.
